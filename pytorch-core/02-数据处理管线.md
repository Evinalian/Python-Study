## 1. 为什么需要 DataLoader

在训练循环中，我们需要高效地从数据集中取出小批量样本。如果手工完成，你需要处理：

- **批量组装**：将 N 个独立样本的 Tensor 堆叠为 shape `(N, ...)` 的批次
- **随机打乱**：每个 epoch 重新 shuffle，打破样本顺序对训练的影响
- **并行加载**：I/O 可能成为瓶颈（尤其大图片或从磁盘读文件时）
- **内存管理**：预取下一批数据，让 GPU 不用等待 CPU 搬运数据

PyTorch 的 `DataLoader` 将这些问题统一封装。其核心抽象分两层：

```
Dataset        — 定义"如何读取第 i 个样本"
DataLoader     — 定义"如何将样本组装成批次并高效供给训练循环"
```

这种分离让数据读取逻辑与批次组装逻辑解耦，你可以独立修改其中的任意部分。

### 1.1 DataLoader 的内部工作流程

```
1. DataLoader 根据 Sampler 生成一批索引 (indices)
2. 对这批索引，多进程并行调用 Dataset.__getitem__(idx)
3. 将返回的样本列表传给 collate_fn 组装成一个批次
4. 如果 pin_memory=True，将批次数据拷贝到锁页内存
5. 将批次 yield 给训练循环
```

理解这个流程有助于定位数据加载的性能瓶颈。

## 2. Dataset 基类

### 2.1 抽象接口

```python
from torch.utils.data import Dataset

class MyDataset(Dataset):
    def __init__(self, ...):
        # 在此处加载或记录数据路径
        pass

    def __len__(self):
        # 返回数据集总样本数
        return 0

    def __getitem__(self, idx):
        # 返回第 idx 个样本，通常是一个 (data, label) 元组
        return None
```

`DataLoader` 通过 `__len__` 知道有多少样本（用于计算 epoch 结束和进度条），通过 `__getitem__(idx)` 按索引取样本。这种设计把随机访问的复杂度藏在了 Dataset 内部，DataLoader 只负责编排。

### 2.2 TensorDataset：最简单的情况

当数据已经是 Tensor 时，不需要自定义 Dataset：

```python
from torch.utils.data import TensorDataset, DataLoader

# 假设有 1000 个样本，每个 10 维特征，标签是整数
X = torch.randn(1000, 10)
y = torch.randint(0, 3, (1000,))

dataset = TensorDataset(X, y)
dataloader = DataLoader(dataset, batch_size=32, shuffle=True)

# 使用
for batch_X, batch_y in dataloader:
    print(batch_X.shape)  # torch.Size([32, 10])
    print(batch_y.shape)  # torch.Size([32])
    break
```

`TensorDataset` 的工作原理：将传入的所有 Tensor 沿第 0 维对齐，`__getitem__(i)` 返回每个 Tensor 的第 i 个切片组成的元组。

**TensorDataset 支持多个输入**：

```python
# 多个特征 Tensor + 标签
X1 = torch.randn(1000, 10)   # 特征集 1
X2 = torch.randn(1000, 5)    # 特征集 2
y = torch.randint(0, 3, (1000,))

dataset = TensorDataset(X1, X2, y)
loader = DataLoader(dataset, batch_size=32, shuffle=True)

for batch_x1, batch_x2, batch_y in loader:
    print(batch_x1.shape, batch_x2.shape, batch_y.shape)
    # torch.Size([32, 10]) torch.Size([32, 5]) torch.Size([32])
    break
```

### 2.3 自定义 Dataset：从 CSV 文件加载

```python
import pandas as pd
from torch.utils.data import Dataset

class CSVDataset(Dataset):
    """从 CSV 文件读取特征和标签的 Dataset"""

    def __init__(self, csv_path, label_col="label"):
        """
        csv_path: CSV 文件路径
        label_col: 标签列的名称
        """
        df = pd.read_csv(csv_path)                 # 读取整个 CSV
        self.labels = df[label_col].values         # 提取标签列
        self.features = df.drop(columns=[label_col]).values.astype(np.float32)

    def __len__(self):
        return len(self.labels)                    # 返回样本数

    def __getitem__(self, idx):
        # 返回 (特征 Tensor, 标签 Tensor)
        x = torch.tensor(self.features[idx])       # shape (n_features,)
        y = torch.tensor(self.labels[idx])         # 标量
        return x, y
```

> 注意：`__getitem__` 在每次取样本时被调用，因此如果预处理很重（如读图片、做变换），应尽量在 `__init__` 中预加载或使用缓存。

**进阶版 CSVDataset —— 支持预处理**：

```python
class CSVDatasetV2(Dataset):
    """支持特征预处理的 CSV Dataset"""

    def __init__(self, csv_path, label_col="label",
                 feature_cols=None, normalize=True):
        df = pd.read_csv(csv_path)

        # 指定特征列（如果为 None，只用数值列）
        if feature_cols is None:
            feature_cols = df.select_dtypes(include=[np.number]).columns.tolist()
            feature_cols = [c for c in feature_cols if c != label_col]

        self.labels = df[label_col].values.astype(np.int64)
        features = df[feature_cols].values.astype(np.float32)

        # 可选：标准化
        if normalize:
            self.mean = features.mean(axis=0, keepdims=True)
            self.std = features.std(axis=0, keepdims=True) + 1e-8
            features = (features - self.mean) / self.std
        else:
            self.mean = None
            self.std = None

        self.features = features
        self.feature_cols = feature_cols

    def __len__(self):
        return len(self.labels)

    def __getitem__(self, idx):
        x = torch.tensor(self.features[idx], dtype=torch.float32)
        y = torch.tensor(self.labels[idx], dtype=torch.long)
        return x, y

    def inverse_transform(self, x_normalized):
        """将标准化后的特征反标准化"""
        if self.mean is None:
            return x_normalized
        return x_normalized * self.std + self.mean
```

### 2.4 从文件目录加载图像的自定义 Dataset

```python
import os
from PIL import Image
from torch.utils.data import Dataset

class ImageFolderDataset(Dataset):
    """从目录结构加载图像：root/class_A/xxx.jpg, root/class_B/xxx.jpg"""

    def __init__(self, root_dir, transform=None):
        """
        root_dir: 根目录，其子目录名为类别名
        transform: 图像预处理变换（torchvision.transforms.Compose）
        """
        self.samples = []          # 存储 (path, label) 对
        self.classes = sorted(os.listdir(root_dir))  # 类别名列表

        for label, class_name in enumerate(self.classes):
            class_dir = os.path.join(root_dir, class_name)
            if not os.path.isdir(class_dir):
                continue
            for fname in os.listdir(class_dir):
                if fname.lower().endswith(('.png', '.jpg', '.jpeg')):
                    self.samples.append((os.path.join(class_dir, fname), label))

        self.transform = transform

    def __len__(self):
        return len(self.samples)

    def __getitem__(self, idx):
        img_path, label = self.samples[idx]
        image = Image.open(img_path).convert("RGB")  # 用 PIL 读取

        if self.transform:
            image = self.transform(image)             # 应用变换

        return image, label
```

**进阶版 —— 支持缓存和错误处理**：

```python
class RobustImageDataset(Dataset):
    """更健壮的图像 Dataset：支持缓存、路径列表、错误处理"""

    def __init__(self, image_paths, labels, transform=None,
                 cache_images=False):
        """
        image_paths: 图像路径列表
        labels: 标签列表（与 image_paths 一一对应）
        """
        self.image_paths = image_paths
        self.labels = labels
        self.transform = transform
        self.cache_images = cache_images

        # 可选：预加载所有图像到内存
        if cache_images:
            self.cached = []
            for path in image_paths:
                try:
                    img = Image.open(path).convert("RGB")
                    self.cached.append(img)
                except Exception as e:
                    print(f"警告：无法加载 {path}: {e}")
                    self.cached.append(None)
        else:
            self.cached = None

    def __len__(self):
        return len(self.image_paths)

    def __getitem__(self, idx):
        if self.cached is not None:
            image = self.cached[idx]
            if image is None:
                # 损坏的图像，返回全黑图像
                image = Image.new("RGB", (224, 224), (0, 0, 0))
        else:
            try:
                image = Image.open(self.image_paths[idx]).convert("RGB")
            except Exception:
                image = Image.new("RGB", (224, 224), (0, 0, 0))

        label = self.labels[idx]

        if self.transform:
            image = self.transform(image)

        return image, label
```

### 2.5 文本数据 Dataset

```python
from torch.utils.data import Dataset
import torch

class TextClassificationDataset(Dataset):
    """文本分类 Dataset：支持词表映射和截断/padding"""

    def __init__(self, texts, labels, vocab, max_length=128):
        """
        texts: 文本列表
        labels: 标签列表
        vocab: dict，word -> id
        max_length: 最大序列长度
        """
        self.texts = texts
        self.labels = labels
        self.vocab = vocab
        self.max_length = max_length
        self.unk_id = vocab.get("<UNK>", 0)
        self.pad_id = vocab.get("<PAD>", 0)

    def __len__(self):
        return len(self.texts)

    def __getitem__(self, idx):
        text = self.texts[idx]
        label = self.labels[idx]

        # 文本 -> token IDs
        tokens = []
        for word in text.split():
            tokens.append(self.vocab.get(word, self.unk_id))

        # 截断或填充到 max_length
        if len(tokens) > self.max_length:
            tokens = tokens[:self.max_length]
        else:
            tokens = tokens + [self.pad_id] * (self.max_length - len(tokens))

        return torch.tensor(tokens, dtype=torch.long), torch.tensor(label, dtype=torch.long)


# 使用示例
texts = ["I love deep learning", "PyTorch is amazing", "Machine learning rocks"]
labels = [1, 1, 1]

# 构建简单词表
vocab = {"<PAD>": 0, "<UNK>": 1, "I": 2, "love": 3, "deep": 4,
         "learning": 5, "PyTorch": 6, "is": 7, "amazing": 8,
         "Machine": 9, "rocks": 10}
dataset = TextClassificationDataset(texts, labels, vocab, max_length=10)

for i in range(len(dataset)):
    tokens, label = dataset[i]
    print(f"样本 {i}: tokens={tokens}, label={label}")
```

### 2.6 多模态 Dataset（图像+文本）

```python
class MultiModalDataset(Dataset):
    """同时包含图像和文本的数据集"""

    def __init__(self, image_paths, texts, labels,
                 image_transform=None, tokenizer_fn=None):
        self.image_paths = image_paths
        self.texts = texts
        self.labels = labels
        self.image_transform = image_transform
        self.tokenizer_fn = tokenizer_fn  # 函数：text -> tensor

    def __len__(self):
        return len(self.labels)

    def __getitem__(self, idx):
        # 图像
        image = Image.open(self.image_paths[idx]).convert("RGB")
        if self.image_transform:
            image = self.image_transform(image)

        # 文本
        text = self.texts[idx]
        if self.tokenizer_fn:
            text_tokens = self.tokenizer_fn(text)
        else:
            text_tokens = text  # 返回原始文本

        label = torch.tensor(self.labels[idx])
        return (image, text_tokens), label
```

### 2.8 IterableDataset（流式数据）

当数据不能完全加载到内存，或数据是实时产生的（如日志流、网络数据），应使用 `IterableDataset`。

```python
from torch.utils.data import IterableDataset, DataLoader

class RandomDataStream(IterableDataset):
    """生成无限的随机数据流"""

    def __init__(self, num_features=10):
        self.num_features = num_features

    def __iter__(self):
        # 不断 yield 样本
        while True:
            x = torch.randn(self.num_features)
            y = torch.randint(0, 5, (1,)).item()
            yield x, y

# IterableDataset 不支持 shuffle（除非用特殊的 buffer shuffle）
stream = RandomDataStream(num_features=10)
loader = DataLoader(stream, batch_size=32)

for batch_x, batch_y in loader:
    print(batch_x.shape, batch_y.shape)
    break  # 只取第一批，否则无限循环
```

> `IterableDataset` 的多 worker 模式需要注意数据分片：每个 worker 会独立调用 `__iter__`，可能导致数据重复。需要通过 worker 信息 (`torch.utils.data.get_worker_info()`) 做分片处理。

## 3. DataLoader 核心参数

```python
from torch.utils.data import DataLoader

loader = DataLoader(
    dataset,                # Dataset 实例
    batch_size=64,          # 每批样本数
    shuffle=True,           # 每个 epoch 是否随机打乱
    num_workers=4,          # 加载数据的子进程数（0 = 主进程加载）
    pin_memory=True,        # 将数据锁页到 CPU 内存，加速 CPU -> GPU 传输
    drop_last=True,         # 丢弃最后不足 batch_size 的批次
    collate_fn=None,        # 自定义批次组装函数
)
```

下面详细解释每个参数。

### 3.1 `batch_size`

每批返回的样本数量。选择依据：
- 太小：GPU 利用率低，梯度噪声大，收敛慢
- 太大：显存溢出 (OOM)，泛化能力可能下降
- 实践经验：CV 任务 32-256，NLP 任务 16-128，取决于显存

**batch_size 对训练的影响 —— 对比实验**：

```python
import time
import torch
import torch.nn as nn
from torch.utils.data import TensorDataset, DataLoader

# 模拟数据
X = torch.randn(10000, 100)
y = torch.randint(0, 10, (10000,))
dataset = TensorDataset(X, y)

model = nn.Linear(100, 10).cuda()
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

for batch_size in [1, 16, 64, 256]:
    loader = DataLoader(dataset, batch_size=batch_size, shuffle=True)

    model.zero_grad()
    start = time.time()
    for batch_x, batch_y in loader:
        batch_x, batch_y = batch_x.cuda(), batch_y.cuda()
        output = model(batch_x)
        loss = criterion(output, batch_y)
        loss.backward()
        optimizer.step()
        model.zero_grad()

    elapsed = time.time() - start
    print(f"batch_size={batch_size:3d}: {elapsed:.3f}s")
```

典型输出（GPU）：
```
batch_size=  1: 2.345s  (GPU 利用率极低)
batch_size= 16: 0.421s
batch_size= 64: 0.187s  (GPU 利用率高)
batch_size=256: 0.156s  (继续提升但边际效益递减)
```

### 3.2 `shuffle`

- `True`：每个 epoch 开始时重新随机排列样本顺序，**训练时必须设为 True**，否则模型会记住样本顺序
- `False`：验证集/测试集保持原始顺序即可（不需要打乱梯度，也不依赖样本顺序）

**为什么不 shuffle 会导致问题**：

```python
# 假设数据按类别排序：[A类样本, A类样本, ..., B类样本, B类样本, ...]
# 不 shuffle 时，每个 batch 只包含一个类别的样本：
#   batch 1: 全是 A 类
#   batch 2: 全是 A 类
#   ...
#   batch N: 全是 B 类
# 梯度在每个 batch 只学到单一类别的模式，导致训练不稳定
```

### 3.3 `num_workers`

```python
# num_workers=0: 主进程加载（慢，但易调试）
# num_workers=4: 4 个子进程并行加载（快，推荐）

# 常见设置
import os
num_workers = min(os.cpu_count(), 8)  # 一般不超过 CPU 核心数
```

加载图像/大文件时，`num_workers` 能显著加速；纯内存数据（TensorDataset）设置 `num_workers=0` 即可。

**num_workers 对加载速度的影响 —— 对比实验**：

```python
import time
from torch.utils.data import Dataset, DataLoader

# 模拟 I/O 密集型 Dataset
class SlowDataset(Dataset):
    def __init__(self, n=10000):
        self.n = n
    def __len__(self):
        return self.n
    def __getitem__(self, idx):
        time.sleep(0.001)  # 模拟磁盘读取延迟
        return torch.randn(3, 224, 224), 0

dataset = SlowDataset(500)
for nw in [0, 2, 4, 8]:
    loader = DataLoader(dataset, batch_size=32, num_workers=nw)
    start = time.time()
    for batch_x, batch_y in loader:
        pass
    print(f"num_workers={nw}: {time.time()-start:.2f}s")
```

典型输出：
```
num_workers=0: 5.21s (串行)
num_workers=2: 2.68s
num_workers=4: 1.45s
num_workers=8: 0.89s
```

> Windows 下使用 `num_workers > 0` 可能因 multiprocessing 问题卡死。解决方案：将 DataLoader 创建语句放在 `if __name__ == "__main__":` 保护中，或设置 `num_workers=0`。

### 3.4 `pin_memory`

将 Tensor 复制到 CUDA 的"锁页内存"区域。当使用 GPU 训练时，`pin_memory=True` 可以**加速 Host to Device 的数据传输**（约 2-3 倍）。代价是占用更多 CPU 内存。

```python
# 典型搭配
loader = DataLoader(dataset, batch_size=64, num_workers=4, pin_memory=True)
for batch_x, batch_y in loader:
    batch_x = batch_x.to("cuda", non_blocking=True)  # non_blocking 配合 pin_memory
    batch_y = batch_y.to("cuda", non_blocking=True)
    # non_blocking=True 让 CPU 在传输数据的同时继续做其他事情
```

**pin_memory 的工作原理**：
- 普通内存（pageable memory）：OS 可能随时将数据交换到磁盘，GPU 无法直接 DMA 访问
- 锁页内存（pinned memory）：OS 保证数据驻留在物理内存中，GPU 可以直接 DMA 传输

### 3.5 `drop_last`

- `True`：丢弃最后一批不足 `batch_size` 的样本
- `False`：保留最后一批（可能小于 `batch_size`）

如果模型中有 `BatchNorm`（依赖批次统计量），建议 `drop_last=True`，否则最后一个小批次可能导致 BatchNorm 统计量不稳定。

```python
# drop_last 对 epoch 批次数量的影响
dataset = TensorDataset(torch.randn(100, 10), torch.randint(0, 5, (100,)))

loader_drop = DataLoader(dataset, batch_size=32, drop_last=True)
loader_keep = DataLoader(dataset, batch_size=32, drop_last=False)

print(f"drop_last=True: {len(loader_drop)} batches")   # 3 (32+32+32=96, 丢弃4)
print(f"drop_last=False: {len(loader_keep)} batches")  # 4 (32+32+32+4)
```

### 3.6 `prefetch_factor` 和 `persistent_workers`

```python
loader = DataLoader(
    dataset,
    batch_size=64,
    num_workers=4,
    prefetch_factor=4,       # 每个 worker 预取 4 批数据
    persistent_workers=True,  # worker 进程在 epoch 之间保持存活
)
```

- `prefetch_factor`：每个 worker 提前加载的批次数，默认 2。增大可减少 GPU 等待时间。
- `persistent_workers=True`：worker 进程不会被每个 epoch 销毁重建，节省进程启动开销。

### 3.7 `sampler` 参数

如果指定了 `sampler`，则 `shuffle` 必须为 `False`（sampler 已经决定了采样顺序）。

```python
from torch.utils.data import RandomSampler, SequentialSampler

# RandomSampler: 随机采样（等价于 shuffle=True）
sampler = RandomSampler(dataset)
loader = DataLoader(dataset, batch_size=32, sampler=sampler)

# SequentialSampler: 顺序采样（等价于 shuffle=False）
sampler = SequentialSampler(dataset)
loader = DataLoader(dataset, batch_size=32, sampler=sampler)
```

## 4. `collate_fn`：自定义批次组装

DataLoader 内部获取 `batch_size` 个样本后，调用 `collate_fn` 将它们组装成一个批次 Tensor。默认的 `collate_fn` 会将列表中的样本沿第 0 维堆叠：

```python
# 默认行为等价于
def default_collate(batch):
    # batch 是 list of (x, y) tuples
    xs, ys = zip(*batch)
    return torch.stack(xs, 0), torch.stack(ys, 0)
```

### 4.1 处理变长序列

当样本长度不一致时（如 NLP 中不同长度的句子），默认 `torch.stack` 会失败：

```python
# 变长数据：3 个样本，长度分别为 2, 4, 3
data = [
    (torch.tensor([1, 2]), 0),
    (torch.tensor([3, 4, 5, 6]), 1),
    (torch.tensor([7, 8, 9]), 0),
]

# 默认 collate_fn 会报错：stack expects each tensor to be equal size
```

```python
from torch.nn.utils.rnn import pad_sequence

def collate_variable_length(batch):
    """自定义 collate：将变长序列 padding 到批次内最大长度"""
    sequences, labels = zip(*batch)

    # pad_sequence: 将 list of tensors 填充到相同长度（补 0）
    # batch_first=True: 输出 shape (batch_size, max_len)
    padded = pad_sequence(sequences, batch_first=True, padding_value=0)

    # 记录每个样本的真实长度（给 RNN pack_padded_sequence 用）
    lengths = torch.tensor([len(seq) for seq in sequences])

    labels = torch.tensor(labels)
    return padded, lengths, labels

# 使用
loader = DataLoader(dataset, batch_size=16, collate_fn=collate_variable_length)
```

**处理变长序列的完整方案 —— 配合 RNN 的 pack_padded_sequence**：

```python
import torch.nn as nn
from torch.nn.utils.rnn import pack_padded_sequence, pad_packed_sequence

# collate_fn（同上）
def collate_fn(batch):
    sequences, labels = zip(*batch)
    lengths = torch.tensor([len(s) for s in sequences])
    padded = pad_sequence(sequences, batch_first=True)
    labels = torch.tensor(labels)
    return padded, lengths, labels

# 在模型中使用
class LSTMModel(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, num_classes):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.lstm = nn.LSTM(embed_dim, hidden_dim, batch_first=True)
        self.fc = nn.Linear(hidden_dim, num_classes)

    def forward(self, padded, lengths):
        embedded = self.embedding(padded)              # (batch, seq, embed)

        # pack: 告诉 LSTM 忽略 padding 部分
        packed = pack_padded_sequence(embedded, lengths.cpu(),
                                      batch_first=True, enforce_sorted=False)
        packed_output, (h_n, c_n) = self.lstm(packed)

        # 使用最后一个有效时间步的隐藏状态做分类
        # 或者将 packed_output 再 unpack 取每个序列的真实长度处
        output, _ = pad_packed_sequence(packed_output, batch_first=True)
        # 取每个序列的最后一个非 padding token
        last_outputs = output[torch.arange(len(lengths)), lengths - 1]
        logits = self.fc(last_outputs)
        return logits
```

### 5.3 `Normalize` 的必要性

神经网络对输入尺度敏感。如果不同特征的数据范围差距大（如特征1在 [0,1]，特征2在 [0,1000]），梯度更新的尺度会不一致，导致训练不稳定。标准化将每个特征/通道变为均值 0、标准差 1 的分布，缓解此问题。

ImageNet 的 `mean=[0.485, 0.456, 0.406]` 和 `std=[0.229, 0.224, 0.225]` 是预训练模型的默认值。**如果你用预训练权重做微调，必须使用这些值；如果从头训练，应该用你自己的数据集统计量。**

**计算自定义数据集的 mean/std**：

```python
def compute_mean_std(dataset):
    """计算数据集的逐通道均值和标准差"""
    loader = DataLoader(dataset, batch_size=64, shuffle=False, num_workers=4)

    mean = torch.zeros(3)
    std = torch.zeros(3)
    total_pixels = 0

    for images, _ in loader:
        # images: (batch, C, H, W)
        batch_pixels = images.size(0) * images.size(2) * images.size(3)
        total_pixels += batch_pixels
        mean += images.mean(dim=[0, 2, 3]) * images.size(0)
        std += images.std(dim=[0, 2, 3]) * images.size(0)

    mean /= len(dataset)
    std /= len(dataset)
    return mean.tolist(), std.tolist()
```

```python
class AddGaussianNoise:
    """自定义 Transform：添加高斯噪声"""

    def __init__(self, mean=0.0, std=0.1):
        self.mean = mean
        self.std = std

    def __call__(self, tensor):
        # tensor 是 (C, H, W) 的 Tensor (ToTensor 之后)
        noise = torch.randn(tensor.size()) * self.std + self.mean
        return tensor + noise

    def __repr__(self):
        return f"{self.__class__.__name__}(mean={self.mean}, std={self.std})"


class RandomErasingCustom:
    """自定义随机擦除"""

    def __init__(self, p=0.5, scale=(0.02, 0.1)):
        self.p = p
        self.scale = scale

    def __call__(self, tensor):
        if torch.rand(1).item() > self.p:
            return tensor

        _, h, w = tensor.shape
        area = h * w
        target_area = np.random.uniform(*self.scale) * area
        aspect_ratio = np.random.uniform(0.3, 3.3)

        erase_h = int(np.sqrt(target_area * aspect_ratio))
        erase_w = int(np.sqrt(target_area / aspect_ratio))

        if erase_h < h and erase_w < w:
            top = np.random.randint(0, h - erase_h)
            left = np.random.randint(0, w - erase_w)
            tensor[:, top:top+erase_h, left:left+erase_w] = 0.0

        return tensor


# 使用自定义 Transform
custom_transform = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.ToTensor(),
    AddGaussianNoise(mean=0.0, std=0.05),  # 自定义变换
    transforms.Normalize(mean=[0.5]*3, std=[0.5]*3),
])
```

## 6. 数据增强策略

数据增强通过在训练时对原始样本施加随机变换，生成更多"变体"。其核心作用是**提高模型对输入变化的鲁棒性**，抑制过拟合。

### 6.1 图像分类的标准增强管线

```python
# 训练时的增强管线
train_transform = transforms.Compose([
    transforms.RandomResizedCrop(224, scale=(0.8, 1.0)),  # 随机裁剪+缩放
    transforms.RandomHorizontalFlip(p=0.5),                # 随机水平翻转
    transforms.ColorJitter(brightness=0.2, contrast=0.2),  # 颜色抖动
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406],
                         std=[0.229, 0.224, 0.225]),
])

# 验证时的管线（不做增强，只需 resize + normalize）
val_transform = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406],
                         std=[0.229, 0.224, 0.225]),
])
```

### 6.2 为什么要区分训练/验证的 transform

训练时的随机变换（RandomCrop、Flip、ColorJitter）让模型在每次 epoch 看到略有不同的输入，等效于扩大了训练集。但验证和测试需要确定性结果：对同一张图片，每次前向传播必须得到相同输出，否则无法公平评估模型性能。

> 核心原则：训练用随机变换增强，验证只用确定性的 resize + normalize。

**语义分割（需要同步变换图像和 mask）**：

```python
import random

class SegmentationTransform:
    """对图像和 mask 同步做相同变换"""

    def __init__(self, size=256):
        self.size = size

    def __call__(self, image, mask):
        # 同步 Resize
        image = transforms.Resize((self.size, self.size))(image)
        mask = transforms.Resize((self.size, self.size), interpolation=transforms.InterpolationMode.NEAREST)(mask)

        # 同步随机水平翻转
        if random.random() > 0.5:
            image = transforms.RandomHorizontalFlip(p=1)(image)  # p=1 强制翻转
            mask = transforms.RandomHorizontalFlip(p=1)(mask)    # p=1 强制翻转

        # ToTensor
        image = transforms.ToTensor()(image)
        mask = transforms.ToTensor()(mask)
        # 或 mask = torch.from_numpy(np.array(mask)).long()

        image = transforms.Normalize(mean, std)(image)

        return image, mask
```

## 7. 数据采样器 (Sampler)

Sampler 决定了 DataLoader 按什么顺序从 Dataset 中取样本。PyTorch 提供了多种采样器。

### 7.1 内置采样器

```python
from torch.utils.data import (
    RandomSampler, SequentialSampler,
    SubsetRandomSampler, WeightedRandomSampler,
    BatchSampler
)

# 1. SequentialSampler: 按顺序采样 (0, 1, 2, ..., n-1)
sampler = SequentialSampler(dataset)

# 2. RandomSampler: 随机采样（有放回或无放回）
sampler = RandomSampler(dataset, replacement=False, num_samples=None)

# 适用于类别不平衡问题
class_counts = [500, 300, 200]  # 3 个类别的样本数
class_weights = 1.0 / torch.tensor(class_counts, dtype=torch.float)
sample_weights = class_weights[labels]  # 每个样本的权重

    weights=sample_weights,
    num_samples=len(sample_weights),
    replacement=True,
)
```

### 7.2 WeightedRandomSampler 处理类别不平衡

```python
import numpy as np
from collections import Counter

def create_balanced_sampler(labels):
    """为不平衡数据集创建加权采样器"""

    # 统计每个类别的样本数
    class_counts = Counter(labels)
    n_samples = len(labels)

    # 计算每个样本的权重 = 1 / 该样本所属类别的样本数
    class_weights = {cls: n_samples / count for cls, count in class_counts.items()}
    sample_weights = [class_weights[label] for label in labels]

    sampler = WeightedRandomSampler(
        weights=sample_weights,
        num_samples=n_samples,
        replacement=True
    )
    return sampler

# 使用
labels = [0, 0, 0, 1, 2]  # 极不平衡：0 类占多数
sampler = create_balanced_sampler(labels)
# 现在 DataLoader 采样时，各类别会被平衡采样
```

```python
import os
import torch
from torch.utils.data import Dataset, DataLoader
from torchvision import transforms
from PIL import Image

# ----- 模拟数据：在当前目录创建 fake_img/ 目录和一些 dummy 图像 -----

class CatDogDataset(Dataset):
    """猫狗分类的自定义 Dataset"""
    def __init__(self, root_dir, transform=None):
        self.samples = []
        # 假设目录结构: root_dir/cat/xxx.jpg, root_dir/dog/xxx.jpg
        for label, cls in enumerate(["cat", "dog"]):
            cls_dir = os.path.join(root_dir, cls)
            if os.path.isdir(cls_dir):
                for fname in os.listdir(cls_dir):
                    if fname.endswith(('.jpg', '.png')):
                        self.samples.append((os.path.join(cls_dir, fname), label))
        self.transform = transform

    def __len__(self):
        return len(self.samples)

    def __getitem__(self, idx):
        img_path, label = self.samples[idx]
        image = Image.open(img_path).convert("RGB")
        if self.transform:
            image = self.transform(image)
        return image, label
```

## 9. 性能优化建议

| 优化点 | 做法 | 适用场景 |
|--------|------|----------|
| `num_workers` | 设为 CPU 核心数（通常 4-8） | 从磁盘加载图片/文件 |
| `pin_memory` | 设为 True | 使用 GPU 训练 |
| `prefetch_factor` | 设为 2-4 | num_workers > 0 时预取更多批次 |
| `persistent_workers` | 设为 True | 避免每个 epoch 重启 worker 进程 |
| 预加载到内存 | 在 `__init__` 中把所有数据读入内存 | 小数据集（< 几 GB） |
| 使用 .to(non_blocking=True) | 配合 pin_memory，异步传输 | GPU 训练 |
| 序列化存储 | 使用 LMDB / HDF5 存储预处理的 Tensor | 大图像数据集 |
| DALI | NVIDIA DALI 库，GPU 端数据预处理 | 极大规模训练 |

```python
def create_dataloader(dataset, batch_size, is_train=True, num_workers=None):
    """创建配置良好的 DataLoader"""

    if num_workers is None:
        import os
        num_workers = min(os.cpu_count() or 1, 8)

    loader = DataLoader(
        dataset,
        batch_size=batch_size,
        shuffle=is_train,
        num_workers=num_workers,
        pin_memory=torch.cuda.is_available(),
        drop_last=is_train,                    # 训练时丢弃最后不完整 batch
        persistent_workers=num_workers > 0,    # 有 worker 时才设置
        prefetch_factor=2 if num_workers > 0 else None,
    )
    return loader

# 使用
train_loader = create_dataloader(train_dataset, batch_size=128, is_train=True)
val_loader = create_dataloader(val_dataset, batch_size=128, is_train=False)
```

## 10. 大规模数据集的加载策略

### 10.1 延迟加载（Lazy Loading）

对于超过内存容量的大数据集，不应在 `__init__` 中加载所有数据：

```python
class LargeImageDataset(Dataset):
    """只存储路径，在 __getitem__ 时动态读取"""

    def __init__(self, manifest_file, transform=None):
        self.samples = []
        with open(manifest_file, 'r') as f:
            for line in f:
                path, label = line.strip().split(',')
                self.samples.append((path, int(label)))
        self.transform = transform

    def __len__(self):
        return len(self.samples)

    def __getitem__(self, idx):
        path, label = self.samples[idx]
        image = Image.open(path).convert("RGB")  # 动态加载
        if self.transform:
            image = self.transform(image)
        return image, label
```

### 10.2 LMDB 存储

对于海量小文件（如 ImageNet），用 LMDB 替代文件系统可大幅提升 I/O：

```python
import lmdb
import pickle

class LMDBDataset(Dataset):
    """从 LMDB 数据库读取数据"""

    def __init__(self, lmdb_path, transform=None):
        self.lmdb_path = lmdb_path
        self.transform = transform

        # 打开 LMDB 环境
        self.env = lmdb.open(lmdb_path, readonly=True, lock=False,
                             readahead=False, meminit=False)
        with self.env.begin() as txn:
            self.length = txn.stat()['entries']

    def __len__(self):
        return self.length

    def __getitem__(self, idx):
        with self.env.begin() as txn:
            data = txn.get(str(idx).encode())
            image, label = pickle.loads(data)

        if self.transform:
            image = self.transform(image)

        return image, label
```

## 常见错误

### 错误 1: `__len__` 返回 0

```python
# 错误
class BadDataset(Dataset):
    def __init__(self):
        self.data = torch.randn(100, 10)

    def __getitem__(self, idx):
        return self.data[idx]

    # 忘记定义 __len__!

# DataLoader 报错: TypeError: object of type 'BadDataset' has no len()

# 正确
class GoodDataset(Dataset):
    def __init__(self):
        self.data = torch.randn(100, 10)

    def __len__(self):
        return len(self.data)          # 必须实现

    def __getitem__(self, idx):
        return self.data[idx]
```

### 错误 2: `__getitem__` 返回不一致类型

```python
# 错误
class InconsistentDataset(Dataset):
    def __getitem__(self, idx):
        if idx % 2 == 0:
            return torch.randn(10), 0      # 奇数返回 tuple
        else:
            return torch.randn(10)          # 偶数只返回 tensor

# DataLoader 的默认 collate_fn 试图 stack 不同结构时会出错

# 正确：始终返回相同结构
class ConsistentDataset(Dataset):
    def __getitem__(self, idx):
        return torch.randn(10), 0           # 始终返回 (tensor, label)
```

### 错误 3: 训练时忘记设 `shuffle=True`

```python
# 错误：每个 epoch 样本顺序完全相同
train_loader = DataLoader(train_dataset, batch_size=64, shuffle=False)
# 模型可能学会样本顺序而非内容本身，泛化能力差

# 正确
train_loader = DataLoader(train_dataset, batch_size=64, shuffle=True)
```

### 错误 4: `ToTensor()` 在 `Normalize` 之后调用

```python
# 错误：先 normalize 后 ToTensor，顺序颠倒！
transform = transforms.Compose([
    transforms.Normalize(mean=[0.5], std=[0.5]),  # 此时还是 PIL Image，报错！
    transforms.ToTensor(),
])
# TypeError: pic should be PIL Image or ndarray. Got <class 'torch.Tensor'>

# 正确
transform = transforms.Compose([
    transforms.ToTensor(),                          # 先转为 Tensor [0, 1]
    transforms.Normalize(mean=[0.5], std=[0.5]),    # 再标准化
])
```

### 错误 5: GPU Tensor 传给 DataLoader 做 shuffle

```python
# 错误
X_gpu = torch.randn(1000, 10).cuda()
y_gpu = torch.randint(0, 5, (1000,)).cuda()
dataset = TensorDataset(X_gpu, y_gpu)
# DataLoader 的默认 collate_fn 内部操作（stack）可能需要 CPU Tensor

# 正确：Dataset 中存 CPU Tensor，在训练循环中再移到 GPU
dataset = TensorDataset(X, y)   # X, y 在 CPU
loader = DataLoader(dataset, batch_size=64)
for bx, by in loader:
    bx, by = bx.to("cuda"), by.to("cuda")  # 训练时再移
```

### 错误 6: `num_workers > 0` 时 Dataset 使用了不可 pickle 的对象

```python
# 错误：在 __init__ 中打开文件句柄
class FileDataset(Dataset):
    def __init__(self, path):
        self.f = open(path, "r")      # 文件句柄不能被 pickle

# 正确：在 __getitem__ 中打开或使用预加载
class FileDataset(Dataset):
    def __init__(self, path):
        with open(path, "r") as f:
            self.data = f.readlines()  # 预加载到内存

    def __getitem__(self, idx):
        return self.data[idx]
```

## 本章小结

- **DataLoader** 封装了批量加载、shuffle、并行 I/O、内存预取的完整管线
- **Dataset** 只需实现 `__len__` 和 `__getitem__`，即可接入 DataLoader
- `TensorDataset` 适用于纯 Tensor 数据；自定义 Dataset 适用于文件读取、预处理等场景
- 可以为图像、文本、表格、多模态等各类数据构建 Dataset
- `collate_fn` 是解决变长数据、异质批次的关键扩展点，支持 padding、字典组装等
- `transforms.Compose` 组合图像预处理操作，`ToTensor` 必须在 `Normalize` 之前
- 训练使用随机增强，验证仅使用确定性变换 —— 这是标准做法
- 可以编写自定义 Transform 类，只需实现 `__call__` 方法
- Sampler 决定了数据采样顺序：RandomSampler、WeightedRandomSampler 等用于不同场景
- `num_workers` + `pin_memory` 是 GPU 训练的性能基石
- `persistent_workers` + `prefetch_factor` 进一步优化加载效率
- 大规模数据集使用延迟加载或 LMDB 等序列化存储
