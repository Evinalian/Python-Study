## 1. 训练循环：标准模板

### 1.1 伪代码

```
for epoch in range(num_epochs):
    # === 训练阶段 ===
    model.train()
    for batch in train_loader:
        optimizer.zero_grad()       # 清零上一步的梯度
        output = model(input)       # 前向传播
        loss = criterion(output, target)  # 计算损失
        loss.backward()             # 反向传播，计算梯度
        optimizer.step()            # 根据梯度更新参数
        record_loss(loss)

    # === 验证阶段 ===
    model.eval()
    with torch.no_grad():
        for batch in val_loader:
            output = model(input)
            val_loss = criterion(output, target)
            record_val_loss(val_loss)
```

### 1.2 完整代码实现

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader

def train_one_epoch(model, loader, criterion, optimizer, device):
    """执行一个 epoch 的训练，返回平均 loss"""
    model.train()
    total_loss = 0.0
    n_batches = 0

    for batch_x, batch_y in loader:
        batch_x = batch_x.to(device)
        batch_y = batch_y.to(device)

        optimizer.zero_grad()
        outputs = model(batch_x)
        loss = criterion(outputs, batch_y)
        loss.backward()
        optimizer.step()

        total_loss += loss.item()
        n_batches += 1

    return total_loss / n_batches

@torch.no_grad()
def evaluate(model, loader, criterion, device):
    """在验证/测试集上评估模型，返回平均 loss 和准确率（分类任务）"""
    model.eval()
    total_loss = 0.0
    correct = 0
    total = 0
    n_batches = 0

    for batch_x, batch_y in loader:
        batch_x = batch_x.to(device)
        batch_y = batch_y.to(device)

        outputs = model(batch_x)
        loss = criterion(outputs, batch_y)

        total_loss += loss.item()
        n_batches += 1

        # 计算准确率（仅分类任务）
        _, predicted = outputs.max(1)     # 取 logit 最大值的索引
        total += batch_y.size(0)
        correct += predicted.eq(batch_y).sum().item()

    avg_loss = total_loss / n_batches
    accuracy = 100.0 * correct / total
    return avg_loss, accuracy
```

### 1.3 每一步的详细解释

| 步骤 | 代码 | 为什么这样做 | 不这样做会发生什么 |
|------|------|-------------|-------------------|
| 1. 训练模式 | `model.train()` | 启用 Dropout（随机置零神经元）、启用 BatchNorm 的 batch 统计量更新 | Dropout 不工作（无正则化效果），BatchNorm 不更新 running stats（推理不稳定） |
| 2. 梯度清零 | `optimizer.zero_grad()` | `backward()` 的梯度是**累加**的（设计特性）。如果不清零，本次的梯度会叠加在上次的梯度上 | 梯度不断累加，相当于使用了错误的梯度方向，参数往随机方向更新 |
| 3. 前向传播 | `output = model(input)` | 数据流经网络各层，构建计算图。`output` 中的每个元素都记录了如何从参数计算得到（通过 `grad_fn`） | 没有前向传播就无法计算 loss |
| 4. 计算损失 | `loss = criterion(output, target)` | 将预测与真实标签比较，得到一个标量值。`backward()` 只能从标量开始反向传播 | 如果 loss 不是标量，backward 会报错 |
| 5. 反向传播 | `loss.backward()` | 沿计算图从 loss 反向计算所有 `requires_grad=True` 的参数的梯度，结果存到 `.grad` 属性 | 参数的 .grad 为 None，optimizer.step() 无法更新参数 |
| 6. 参数更新 | `optimizer.step()` | 对每个参数执行 `param = param - lr * param.grad`（SGD 的情况）。Adam 等优化器的更新公式更复杂，但概念相同 | 参数永远不会改变——模型不会学习 |
| 7. 记录 loss | `total_loss += loss.item()` | `.item()` 将单元素 Tensor 转为 Python float，释放计算图引用，避免内存累积 | 计算图持续占用内存，GPU 显存可能溢出 |

### 1.4 设备迁移的时机

```python
# 推荐做法：将模型在训练开始前一次性移到 GPU
model = MyModel().to(device)

# 数据在每批处理时移（Dataset 中保持 CPU Tensor）
for batch_x, batch_y in train_loader:
    batch_x = batch_x.to(device)
    batch_y = batch_y.to(device)
```

**为什么不把 Dataset 中的数据预先放到 GPU？**
- GPU 显存通常远小于内存（如 16GB vs 256GB）
- DataLoader 的 `num_workers` 多进程加载操作的是 CPU 内存
- 每批处理时搬移一次的开销很小（尤其配合 `pin_memory=True` + `non_blocking=True`）

### 1.5 梯度累积：模拟更大 batch_size

当显存不足以容纳目标 batch_size 时，可以用梯度累积来模拟：

```python
# 目标 batch_size = 128，但显存只够 32
# 解决方案：每 4 个小 batch 才做一次 optimizer.step()

accumulation_steps = 4  # 128 / 32 = 4

model.train()
optimizer.zero_grad()

for i, (batch_x, batch_y) in enumerate(train_loader):
    batch_x, batch_y = batch_x.to(device), batch_y.to(device)
    outputs = model(batch_x)
    loss = criterion(outputs, batch_y)

    # 除以累积步数，使得总梯度的量级与真实 batch_size 一致
    loss = loss / accumulation_steps
    loss.backward()  # 梯度累加（不清零）

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()

# 处理最后不足 accumulation_steps 的部分
if (i + 1) % accumulation_steps != 0:
    optimizer.step()
    optimizer.zero_grad()
```

> 梯度累积时需要注意：**BatchNorm 层在每次 forward 时使用当前 mini-batch 的统计量**，因此梯度累积不等价于真正的大 batch_size 下的 BatchNorm 行为。如果依赖 BatchNorm 统计量，梯度累积可能带来偏差。

## 2. 验证循环与 `model.eval()`

### 2.1 为什么需要 `model.eval()`

`model.eval()` 和 `model.train()` 改变的是某些特定层的行为，主要体现在两个组件上：

**Dropout**:
```
train 模式：随机将 p 比例的元素置 0，其余元素缩放 1/(1-p)
eval  模式：不做任何操作（恒等映射）
```

**BatchNorm**:
```
train 模式：用当前 batch 的均值和方差做归一化，同时更新全局 running_mean/running_var
eval  模式：用训练期间累积的 running_mean/running_var 做归一化（固定的）
```

如果在验证时忘记调用 `model.eval()`：
- Dropout 仍然随机丢弃神经元，验证结果不稳定且偏低
- BatchNorm 使用验证 batch 的统计量（而非全局统计量），每次验证结果不一致

**验证 Dropout 和 BatchNorm 在 train/eval 下的行为差异**：

```python
# 实验：验证 train 和 eval 的输出差异
import torch
import torch.nn as nn

model = nn.Sequential(
    nn.Linear(10, 20),
    nn.BatchNorm1d(20),
    nn.ReLU(),
    nn.Dropout(0.5),
    nn.Linear(20, 5),
)

x = torch.randn(8, 10)
# 重复运行多次 compare
model.train()
out_train_1 = model(x)
out_train_2 = model(x)

model.eval()
out_eval_1 = model(x)
out_eval_2 = model(x)

# train 模式：每次输出都不同（因为 Dropout 随机）
print(f"Train mode outputs equal: {(out_train_1 == out_train_2).all().item()}")  # False

# eval 模式：每次输出相同（Dropout 关闭，BN 使用 running stats）
print(f"Eval mode outputs equal: {(out_eval_1 == out_eval_2).all().item()}")   # True
```

### 2.2 `torch.no_grad()` 在验证时的双重作用

```python
@torch.no_grad()   # 装饰器或上下文管理器均可
def evaluate(model, loader, criterion, device):
    model.eval()
    for batch_x, batch_y in loader:
        outputs = model(batch_x)
        # ...
```

`torch.no_grad()` 在验证时的作用：
1. **省显存**：不构建计算图，中间激活值不会被保存（反向传播需要它们）
2. **加速**：省去计算图构建和梯度计算的开销（通常 10-20% 加速）
3. **防止意外**：避免在验证中意外调用 `backward()` 破坏模型状态

**有 `no_grad()` vs 无 `no_grad()` 的性能对比**：


## 3. 优化器

### 3.1 SGD + Momentum

```python
# 基础 SGD
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

# SGD + Momentum（推荐）
optimizer = torch.optim.SGD(model.parameters(), lr=0.01, momentum=0.9)

# SGD + Momentum + Nesterov（稍好于标准 Momentum）
optimizer = torch.optim.SGD(model.parameters(), lr=0.01,
                             momentum=0.9, nesterov=True)
```

Momentum 引入了"惯性"概念：参数更新不仅基于当前梯度，还累加了历史梯度的指数衰减平均：

```
v_t = momentum * v_{t-1} + grad_t
param = param - lr * v_t
```

好处：加速收敛，帮助跳出局部最优，在沟壑状（ravine）的损失曲面上减少震荡。

**SGD 的调参经验**：

- 学习率是最重要的超参数。CV 任务通常从 0.1 开始，NLP 从 0.01 开始
- Momentum 通常设为 0.9（几乎不需要调）
- 如果 loss 震荡剧烈，降低 lr；如果 loss 下降太慢，提高 lr
- SGD 通常需要配合学习率调度器（StepLR 或 CosineAnnealing）
- 使用 `weight_decay=1e-4` 到 `5e-4` 进行正则化

### 3.2 Adam / AdamW

```python
# Adam: 自适应学习率 + 动量
optimizer = torch.optim.Adam(model.parameters(), lr=0.001, betas=(0.9, 0.999))

# AdamW: Adam + 解耦权重衰减（推荐替代 Adam）
optimizer = torch.optim.AdamW(model.parameters(), lr=0.001, weight_decay=0.01)
```

Adam 维护两个统计量：
- 一阶矩（梯度均值）：类似 Momentum，提供方向平滑
- 二阶矩（梯度方差）：自适应调整每个参数的学习率

```python
# Adam 的更新公式（简化）
m_t = beta1 * m_{t-1} + (1 - beta1) * grad_t        # 一阶矩
v_t = beta2 * v_{t-1} + (1 - beta2) * grad_t^2       # 二阶矩
param = param - lr * m_t / (sqrt(v_t) + eps)          # 参数更新
```

**Adam vs AdamW 的数学本质**：

Adam 的 weight_decay 实现为在梯度上加 L2 惩罚项：

```
# Adam (旧式 L2 正则化)
grad = grad + weight_decay * param      # 梯度加了 weight_decay
# 问题：weight_decay 和自适应学习率耦合，正则化效果不均匀
```

AdamW 将 weight_decay 从自适应学习率中解耦：

```python
# AdamW 的 weight_decay 等价于
param = param - lr * weight_decay * param
# 直接在参数上衰减，与梯度更新独立
```

这是当前公认的最佳实践。

**Adam/AdamW 的调参经验**：

- lr: CV 任务通常 1e-4 到 1e-3；Transformer 通常 1e-4
- weight_decay: 通常 0.01 到 0.1（AdamW 用大值，Adam 用小值或 0）
- betas: 几乎总是用默认值 (0.9, 0.999)
- eps: 默认 1e-8，通常不需要改

### 3.3 优化器选择指南

| 场景 | 推荐 | 原因 |
|------|------|------|
| CV（CNN 训练） | SGD + Momentum (lr=0.1) | 经典配置，调参经验丰富，泛化性能好 |
| Transformer/BERT | AdamW (lr=1e-4) | 标准配置，warmup + decay |
| GAN / 强化学习 | Adam (lr=1e-4) | 对学习率鲁棒 |
| 快速实验/原型 | AdamW (lr=1e-3) | 默认参数通常 work，不需要精细调参 |
| 图像生成（扩散） | AdamW (lr=1e-4) | 稳定收敛 |
| 推荐系统 | Adam (lr=1e-3) | 稀疏梯度场景 |

### 3.4 优化器收敛曲线对比（概念性）

```
Loss
 |  SGD (无 momentum):  缓慢下降，震荡大
 |  SGD + Momentum:     较快下降，前期震荡
 |  Adam:               下降最快，前期收敛极快
 |  AdamW:              与 Adam 类似，但泛化更好
 |
 |  SGD 在大 batch 下 + 长训练，最终可能超越 Adam
 |  Adam 在前期最快，但最终精度可能不如精调 SGD
 +------------------------------------------------> Epochs
```

### 3.5 不同优化器的完整代码对比

```python
import torch.optim as optim

def create_optimizer(optimizer_name, model, lr, **kwargs):
    """根据名称创建优化器"""
    if optimizer_name == "sgd":
        return optim.SGD(model.parameters(), lr=lr,
                         momentum=kwargs.get("momentum", 0.9),
                         weight_decay=kwargs.get("weight_decay", 1e-4),
                         nesterov=kwargs.get("nesterov", True))
    elif optimizer_name == "adam":
        return optim.Adam(model.parameters(), lr=lr,
                          betas=kwargs.get("betas", (0.9, 0.999)),
                          weight_decay=kwargs.get("weight_decay", 0))
    elif optimizer_name == "adamw":
        return optim.AdamW(model.parameters(), lr=lr,
                           betas=kwargs.get("betas", (0.9, 0.999)),
                           weight_decay=kwargs.get("weight_decay", 0.01))
    elif optimizer_name == "rmsprop":
        return optim.RMSprop(model.parameters(), lr=lr,
                             weight_decay=kwargs.get("weight_decay", 0))
    else:
        raise ValueError(f"Unknown optimizer: {optimizer_name}")

# 使用
optimizer = create_optimizer("adamw", model, lr=1e-3, weight_decay=0.01)
```

## 4. 学习率调度器

### 4.1 常用调度器

```python
import torch.optim.lr_scheduler as lr_scheduler

optimizer = torch.optim.AdamW(model.parameters(), lr=0.001)

# StepLR: 每 step_size 个 epoch，lr 乘以 gamma
scheduler = lr_scheduler.StepLR(optimizer, step_size=30, gamma=0.1)
# epoch 0-29: lr=0.001, epoch 30-59: lr=0.0001, epoch 60-89: lr=0.00001

# MultiStepLR: 在指定的 epoch 处衰减
scheduler = lr_scheduler.MultiStepLR(optimizer, milestones=[30, 60, 90], gamma=0.1)

# ExponentialLR: 每 epoch 衰减 gamma
scheduler = lr_scheduler.ExponentialLR(optimizer, gamma=0.95)

# CosineAnnealingLR: 余弦曲线衰减到 eta_min
scheduler = lr_scheduler.CosineAnnealingLR(optimizer, T_max=100, eta_min=1e-6)
# lr 沿余弦曲线从 0.001 平滑衰减到 1e-6

# CosineAnnealingWarmRestarts: 带重启的余弦衰减
scheduler = lr_scheduler.CosineAnnealingWarmRestarts(optimizer, T_0=10, T_mult=2)
# 每 10 个 epoch 重启一次，每次重启周期乘 2

# ReduceLROnPlateau: 当指标停滞时降低 lr
scheduler = lr_scheduler.ReduceLROnPlateau(
    optimizer, mode='min', factor=0.5, patience=10
)
# 当 val_loss 连续 10 个 epoch 不下降时，lr *= 0.5

# OneCycleLR: 一个周期内先升后降
scheduler = lr_scheduler.OneCycleLR(
    optimizer, max_lr=0.01, steps_per_epoch=len(train_loader), epochs=100
)
```

### 4.3 Warmup 策略

训练初期模型参数是随机的，梯度方向不稳定。如果一开始就用目标学习率，可能导致训练初期震荡甚至发散。Warmup 让学习率在最初几个 epoch 从接近 0 线性增长到目标值，给优化器时间建立稳定的梯度统计量。

**手动实现线性 Warmup**：

```python
class WarmupScheduler:
    """线性 warmup：lr 从 warmup_start 线性增加到 optimizer 的初始 lr"""
    def __init__(self, optimizer, warmup_epochs, warmup_start=1e-6):
        self.optimizer = optimizer
        self.warmup_epochs = warmup_epochs
        self.warmup_start = warmup_start
        self.base_lrs = [group['lr'] for group in optimizer.param_groups]
        self.current_epoch = 0

    def step(self):
        self.current_epoch += 1
        if self.current_epoch <= self.warmup_epochs:
            # 线性增长
            progress = self.current_epoch / self.warmup_epochs
            for i, param_group in enumerate(self.optimizer.param_groups):
                param_group['lr'] = self.warmup_start + \
                                    (self.base_lrs[i] - self.warmup_start) * progress

# 使用方式：先用 WarmupScheduler.step()，再接主调度器
warmup = WarmupScheduler(optimizer, warmup_epochs=5)
main_scheduler = lr_scheduler.CosineAnnealingLR(optimizer, T_max=95)

for epoch in range(100):
    train_one_epoch(...)
    if epoch < 5:
        warmup.step()         # warmup 阶段
    else:
        main_scheduler.step()  # 主调度器
```

**使用 PyTorch 内置方式（更简洁）**：

更简洁的写法（PyTorch 1.11+ 支持 `SequentialLR`）：

```python
from torch.optim.lr_scheduler import LinearLR, CosineAnnealingLR, SequentialLR

optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3)

warmup_scheduler = LinearLR(
    optimizer, start_factor=1e-3, total_iters=5
)
main_scheduler = CosineAnnealingLR(optimizer, T_max=95)
scheduler = SequentialLR(
    optimizer,
    schedulers=[warmup_scheduler, main_scheduler],
    milestones=[5]
)

for epoch in range(100):
    train_one_epoch(...)
    scheduler.step()
```

**使用 LambdaLR 自定义任意 schedule**：

```python
# 自定义学习率函数: warmup 5 epoch + cosine decay 95 epoch
def lr_lambda(epoch):
    if epoch < 5:
        return (epoch + 1) / 5  # warmup: 0.2 -> 1.0
    else:
        # cosine decay from 1.0 to 0
        progress = (epoch - 5) / 95
        return 0.5 * (1 + torch.cos(torch.tensor(torch.pi * progress)).item())

scheduler = lr_scheduler.LambdaLR(optimizer, lr_lambda)
```

### 4.4 Cosine Annealing 的数学原理

```
CosineAnnealingLR 的公式:
    lr = eta_min + 0.5 * (eta_max - eta_min) * (1 + cos(pi * t / T_max))

其中:
    eta_max: 初始学习率
    eta_min: 最小学习率
    t: 当前 epoch (0 到 T_max-1)
    T_max: 半个余弦周期

特点:
    - 从 eta_max 开始，平滑下降，到 T_max 时降到 eta_min
    - 下降过程中速率变化：开始时慢，中间快，末尾又慢
    - 相比阶梯式下降（StepLR），余弦衰减更平滑，通常收敛更好
```

## 5. 梯度裁剪

当梯度范数过大时，参数更新步长过大，可能导致训练崩溃（loss 变为 NaN）。这在 RNN/LSTM 训练中尤为常见（梯度爆炸）。

```python
# 将整个模型参数的梯度范数限制在 max_norm 以内
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

# 在训练循环中的位置：loss.backward() 之后、optimizer.step() 之前
loss.backward()
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)  # 梯度裁剪
optimizer.step()
```

梯度裁剪的工作原理：如果梯度的 L2 范数超过 `max_norm`，则按比例缩放使范数等于 `max_norm`；如果未超过则不做处理。

```python
# 等价于：
total_norm = torch.sqrt(sum(p.grad.norm(2)**2 for p in model.parameters()))
if total_norm > max_norm:
    scale = max_norm / total_norm
    for p in model.parameters():
        p.grad *= scale
```

**按值裁剪（clip by value）**：

```python
# 将每个梯度元素限制在 [-clip_value, clip_value] 之间
torch.nn.utils.clip_grad_value_(model.parameters(), clip_value=1.0)
# 较少使用，clip_grad_norm_ 更常用
```

常用 `max_norm` 值：RNN 训练一般用 1.0 或 5.0；Transformer 训练通常用 1.0。

**梯度裁剪的完整训练步骤**：

```python
# 带梯度裁剪的完整训练 batch 处理
for batch_x, batch_y in train_loader:
    batch_x, batch_y = batch_x.to(device), batch_y.to(device)

    optimizer.zero_grad()
    outputs = model(batch_x)
    loss = criterion(outputs, batch_y)
    loss.backward()

    # 梯度裁剪（在 step 之前）
    torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

    optimizer.step()
```

## 6. 过拟合应对

当模型在训练集上表现很好但在验证集上表现差时，即为过拟合。以下是常用应对手段：

### 6.1 Dropout

```python
model = nn.Sequential(
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Dropout(p=0.5),    # 50% dropout 率
    nn.Linear(256, 10),
)
```

Dropout 在训练时随机"关闭"部分神经元，迫使网络学习冗余表示，从而提升泛化能力。常用 `p=0.2~0.5`。CNN 中通常在卷积层后用较小的 p（0.1-0.3），全连接层后用较大的 p（0.5）。

### 6.2 早停 (Early Stopping)

```python
class EarlyStopping:
    def __init__(self, patience=10, min_delta=0.0):
        """
        patience: 连续多少个 epoch 无改善后停止
        min_delta: 多小的改善算"有效"
        """
        self.patience = patience
        self.min_delta = min_delta
        self.counter = 0
        self.best_loss = float('inf')
        self.early_stop = False

    def __call__(self, val_loss):
        if val_loss < self.best_loss - self.min_delta:
            self.best_loss = val_loss
            self.counter = 0                  # 有改善，重置计数器
        else:
            self.counter += 1                 # 无改善，计数+1
            if self.counter >= self.patience:
                self.early_stop = True
        return self.early_stop

# 使用
early_stopping = EarlyStopping(patience=10)
for epoch in range(num_epochs):
    train_loss = train_one_epoch(...)
    val_loss, val_acc = evaluate(...)
    if early_stopping(val_loss):
        print(f"Early stopping at epoch {epoch}")
        break
```

**带模型保存的早停（推荐）**：

```python
class EarlyStoppingWithCheckpoint:
    """带最佳模型保存的早停"""
    def __init__(self, patience=10, min_delta=0.0, save_path="best_model.pth"):
        self.patience = patience
        self.min_delta = min_delta
        self.save_path = save_path
        self.counter = 0
        self.best_loss = float('inf')
        self.early_stop = False

    def __call__(self, val_loss, model):
        if val_loss < self.best_loss - self.min_delta:
            self.best_loss = val_loss
            self.counter = 0
            # 保存最佳模型
            torch.save(model.state_dict(), self.save_path)
            return False
        else:
            self.counter += 1
            if self.counter >= self.patience:
                self.early_stop = True
            return self.early_stop
```

### 6.3 权重衰减 (Weight Decay)

权重衰减等价于 L2 正则化：在损失函数中加入参数范数惩罚，推动参数向零收缩。

```python
# AdamW 中通过 weight_decay 参数实现（推荐）
optimizer = torch.optim.AdamW(model.parameters(), lr=0.001, weight_decay=0.01)

# SGD 中同样支持
optimizer = torch.optim.SGD(model.parameters(), lr=0.01, weight_decay=5e-4)
```

常用值：`1e-4 ~ 1e-2`。过大会导致欠拟合。

**Weight Decay vs L2 Regularization 的区别**：

```
L2 Regularization:
    loss_total = loss_orig + (lambda/2) * ||w||^2
    grad = grad_orig + lambda * w
    在 Adam 中，lambda * w 项也被自适应学习率缩放

Weight Decay (AdamW):
    loss_total = loss_orig  (不加 L2 项)
    param = param - lr * grad_orig  (Adam 步骤)
    param = param - lr * weight_decay * param  (直接衰减)
    权重衰减独立于梯度方向
```

### 6.5 标签平滑 (Label Smoothing)

将硬标签（one-hot）变为软标签，防止模型对预测过于自信：

```python
# 使用 CrossEntropyLoss 的 label_smoothing 参数（PyTorch 1.10+）
criterion = nn.CrossEntropyLoss(label_smoothing=0.1)

# 手动实现标签平滑
def label_smoothing_loss(logits, labels, smoothing=0.1):
    n_classes = logits.size(-1)
    # 平滑后的目标：正确类别 = (1 - smoothing + smoothing/n_classes)，其他 = smoothing/n_classes
    with torch.no_grad():
        smooth_labels = torch.full_like(logits, smoothing / n_classes)
        smooth_labels.scatter_(1, labels.unsqueeze(1), 1 - smoothing + smoothing / n_classes)

    log_probs = torch.log_softmax(logits, dim=1)
    loss = -(smooth_labels * log_probs).sum(dim=1).mean()
    return loss
```

| 2 | Dropout | 全连接层过拟合 | 有效，几乎没有代价 |

| 7 | 增加数据 | 如果可以获取更多数据 | 最有效但成本高 |

```python
def compute_accuracy(outputs, labels):
    """计算分类准确率"""
    _, preds = outputs.max(1)
    correct = preds.eq(labels).sum().item()
    return correct / labels.size(0)
```

### 7.2 多指标计算

```python
from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score
import numpy as np

def compute_metrics(outputs, labels, average="macro"):
    """计算多个分类指标"""
    _, preds = outputs.max(1)

    preds_np = preds.cpu().numpy()
    labels_np = labels.cpu().numpy()

    metrics = {
        "accuracy": accuracy_score(labels_np, preds_np),
        "precision": precision_score(labels_np, preds_np, average=average, zero_division=0),
        "recall": recall_score(labels_np, preds_np, average=average, zero_division=0),
        "f1": f1_score(labels_np, preds_np, average=average, zero_division=0),
    }
    return metrics

# 在 evaluate 函数中使用
@torch.no_grad()
def evaluate_with_metrics(model, loader, criterion, device):
    model.eval()
    all_preds = []
    all_labels = []
    total_loss = 0.0

    for batch_x, batch_y in loader:
        batch_x, batch_y = batch_x.to(device), batch_y.to(device)
        outputs = model(batch_x)
        loss = criterion(outputs, batch_y)
        total_loss += loss.item()

        all_preds.append(outputs.cpu())
        all_labels.append(batch_y.cpu())

    all_preds = torch.cat(all_preds)
    all_labels = torch.cat(all_labels)

    metrics = compute_metrics(all_preds, all_labels)
    metrics["loss"] = total_loss / len(loader)
    return metrics
```

### 7.3 纯 PyTorch 实现（不依赖 sklearn）

```python
def precision_recall_f1_pytorch(outputs, labels, num_classes, average="macro"):
    """纯 PyTorch 实现多分类指标"""
    _, preds = outputs.max(1)

    # 对每个类别计算 TP, FP, FN
    tp = torch.zeros(num_classes, dtype=torch.int64)
    fp = torch.zeros(num_classes, dtype=torch.int64)
    fn = torch.zeros(num_classes, dtype=torch.int64)

    for c in range(num_classes):
        tp[c] = ((preds == c) & (labels == c)).sum()
        fp[c] = ((preds == c) & (labels != c)).sum()
        fn[c] = ((preds != c) & (labels == c)).sum()

    # 每个类别的 precision, recall, f1
    precision_per_class = tp / (tp + fp).clamp(min=1)
    recall_per_class = tp / (tp + fn).clamp(min=1)
    f1_per_class = 2 * precision_per_class * recall_per_class / \
                   (precision_per_class + recall_per_class).clamp(min=1e-8)

    if average == "macro":
        return {
            "precision": precision_per_class.mean().item(),
            "recall": recall_per_class.mean().item(),
            "f1": f1_per_class.mean().item(),
        }
    elif average == "micro":
        total_tp = tp.sum()
        total_fp = fp.sum()
        total_fn = fn.sum()
        micro_precision = total_tp / (total_tp + total_fp).clamp(min=1).item()
        micro_recall = total_tp / (total_tp + total_fn).clamp(min=1).item()
        micro_f1 = 2 * micro_precision * micro_recall / (micro_precision + micro_recall + 1e-8)
        return {"precision": micro_precision, "recall": micro_recall, "f1": micro_f1}
```

## 9. 混合精度训练 (AMP) 简要引入

混合精度训练使用 float16 和 float32 混合计算，在保持模型精度的同时：
- 减少约 50% 显存占用
- 加速训练（尤其在支持 Tensor Cores 的 GPU 上，如 V100、A100、RTX 20/30/40 系列）

```python
# 使用 PyTorch 内置的 AMP (Automatic Mixed Precision)
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()  # 梯度缩放器，防止 float16 下溢

for batch_x, batch_y in train_loader:
    batch_x, batch_y = batch_x.to(device), batch_y.to(device)
    optimizer.zero_grad()

    # 前向传播使用 autocast: 自动选择 float16 或 float32
    with autocast():
        outputs = model(batch_x)
        loss = criterion(outputs, batch_y)

    # 反向传播使用 scaler: 缩放 loss 防止梯度下溢
    scaler.scale(loss).backward()
    scaler.unscale_(optimizer)          # 在裁剪前恢复梯度尺度

    # 梯度裁剪（在 unscale 之后）
    torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

    scaler.step(optimizer)              # 使用缩放的梯度更新参数
    scaler.update()                     # 更新缩放因子

# 检查 AMP 是否可用
print(f"AMP available: {torch.cuda.is_available()}")
# 在 CPU 上 autocast 退化为不做任何操作
```

**AMP 的关键组件**：

| 组件 | 作用 |
|------|------|
| `autocast()` | 自动决定哪些操作用 float16、哪些用 float32 |
| `GradScaler` | 动态缩放 loss 防止 float16 梯度过小导致下溢 |
| `scaler.scale(loss)` | 将 loss 乘以缩放因子 |
| `scaler.unscale_(optimizer)` | 在 step 前恢复梯度原始尺度 |
| `scaler.step(optimizer)` | 安全地更新参数 |
| `scaler.update()` | 动态调整缩放因子 |

### 10.2 自动保存最佳模型

```python
class ModelCheckpoint:
    """自动保存最佳模型"""
    def __init__(self, save_path="best_model.pth", mode="min"):
        self.save_path = save_path
        self.mode = mode
        self.best_metric = float("inf") if mode == "min" else float("-inf")

    def step(self, metric, model, optimizer=None, epoch=None):
        """如果 metric 更好，保存 checkpoint"""
        is_better = (self.mode == "min" and metric < self.best_metric) or \
                    (self.mode == "max" and metric > self.best_metric)

        if is_better:
            self.best_metric = metric
            checkpoint = {
                "model_state_dict": model.state_dict(),
                "metric": metric,
            }
            if optimizer is not None:
                checkpoint["optimizer_state_dict"] = optimizer.state_dict()
            if epoch is not None:
                checkpoint["epoch"] = epoch

            torch.save(checkpoint, self.save_path)
            print(f"Best model saved! metric={metric:.4f}")
            return True
        return False

# 使用
checkpoint = ModelCheckpoint("best_model.pth", mode="max")  # 准确率越高越好
for epoch in range(num_epochs):
    ...
    val_acc = evaluate(...)
    checkpoint.step(val_acc, model, optimizer, epoch)
```

```python
from tqdm import tqdm

def train(model, train_loader, val_loader, criterion, optimizer,
          scheduler, num_epochs, device, early_stopping=None):
    """完整的训练主循环"""
    history = {'train_loss': [], 'val_loss': [], 'val_acc': []}

    epoch_pbar = tqdm(range(num_epochs), desc="Epochs")
    for epoch in epoch_pbar:
        # ---- 训练 ----
        train_loss = train_one_epoch(model, train_loader, criterion, optimizer, device)

        # ---- 验证 ----
        val_loss, val_acc = evaluate(model, val_loader, criterion, device)

        # ---- 记录 ----
        history['train_loss'].append(train_loss)
        history['val_loss'].append(val_loss)
        history['val_acc'].append(val_acc)

        # ---- 调度器 ----
        scheduler.step()

        # ---- 显示 ----
        current_lr = optimizer.param_groups[0]['lr']
        epoch_pbar.set_postfix({
            'train_loss': f'{train_loss:.4f}',
            'val_loss': f'{val_loss:.4f}',
            'val_acc': f'{val_acc:.2f}%',
            'lr': f'{current_lr:.2e}',
        })

        # ---- 早停 ----
        if early_stopping and early_stopping(val_loss):
            print(f"Early stopping at epoch {epoch+1}")
            break

    return history
```

## 12. 训练中常见问题的排查 Checklist

### 12.1 Loss 不下降

- [ ] 学习率是否太小？（尝试增大 10 倍）
- [ ] 学习率是否太大？（尝试减小 10 倍）
- [ ] 优化器是否正确初始化？
- [ ] `optimizer.zero_grad()` 是否在每次迭代前调用？
- [ ] `optimizer.step()` 是否在 `loss.backward()` 之后调用？
- [ ] 数据加载是否正确？（打印一批数据检查 shape 和数值范围）
- [ ] 标签格式是否匹配损失函数要求？

### 12.2 Loss 变为 NaN

- [ ] 学习率是否太大？
- [ ] 是否存在除零操作？
- [ ] 数据中是否有 NaN？检查：`print(torch.isnan(data).any())`
- [ ] 是否使用了梯度裁剪？增加 `clip_grad_norm_`
- [ ] 是否正确初始化了参数？
- [ ] 损失函数是否可能输入了非法值（如 BCELoss 的 log(0)）？

### 12.3 训练集 loss 远低于验证集 loss（过拟合）

- [ ] 增加数据增强
- [ ] 增加 Dropout 率
- [ ] 增大 weight_decay
- [ ] 使用早停
- [ ] 减少模型参数量
- [ ] 收集更多训练数据

### 12.4 训练集 loss 和验证集 loss 都很高（欠拟合）

- [ ] 学习率是否太小？
- [ ] 模型容量是否不够？（增加层数或隐藏单元数）
- [ ] 数据集是否过于复杂？

### 12.5 训练不稳定（loss 剧烈震荡）

- [ ] batch_size 是否太小？（增大 batch_size）
- [ ] 学习率是否太大？
- [ ] 是否使用了 SGD？（考虑换 AdamW）
- [ ] 是否使用了 Warmup？

### 12.6 GPU 显存不足 (Out of Memory)

- [ ] 减小 batch_size
- [ ] 使用梯度累积
- [ ] 使用混合精度训练（AMP）
- [ ] 减小模型大小（减少层数/隐藏单元）
- [ ] 使用 `torch.cuda.empty_cache()` 清理缓存

### 12.7 训练速度慢

- [ ] `num_workers` 是否设置足够？（0 -> 4/8）
- [ ] `pin_memory` 是否设为 True？
- [ ] 是否在 `no_grad()` 下做验证？
- [ ] 是否使用了不必要的 `.item()` 调用？
- [ ] batch_size 是否填满了 GPU？

## 13. 代码实战：MNIST 分类完整训练流程

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms
from tqdm import tqdm

# ---- 1. 数据准备 ----
transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.1307,), (0.3081,))  # MNIST 的统计量
])

train_dataset = datasets.MNIST('./data', train=True, download=True, transform=transform)
test_dataset  = datasets.MNIST('./data', train=False, transform=transform)

train_loader = DataLoader(train_dataset, batch_size=128, shuffle=True, pin_memory=True)
test_loader  = DataLoader(test_dataset, batch_size=128, shuffle=False, pin_memory=True)

# ---- 2. 模型定义 ----
class MNISTCNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(1, 32, 3, padding=1)
        self.conv2 = nn.Conv2d(32, 64, 3, padding=1)
        self.pool = nn.MaxPool2d(2, 2)
        self.fc1 = nn.Linear(64 * 7 * 7, 128)
        self.fc2 = nn.Linear(128, 10)
        self.dropout = nn.Dropout(0.25)
        self.relu = nn.ReLU()

    def forward(self, x):
        x = self.pool(self.relu(self.conv1(x)))
        x = self.pool(self.relu(self.conv2(x)))
        x = x.view(x.size(0), -1)
        x = self.dropout(self.relu(self.fc1(x)))
        x = self.fc2(x)
        return x

# ---- 3. 训练配置 ----
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model = MNISTCNN().to(device)
criterion = nn.CrossEntropyLoss()
optimizer = optim.AdamW(model.parameters(), lr=0.001, weight_decay=0.01)
scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=10)

# ---- 4. 训练 ----
num_epochs = 10
for epoch in range(num_epochs):
    # 训练
    model.train()
    train_loss = 0.0
    pbar = tqdm(train_loader, desc=f"Epoch {epoch+1}/{num_epochs}")
    for data, target in pbar:
        data, target = data.to(device), target.to(device)
        optimizer.zero_grad()
        output = model(data)
        loss = criterion(output, target)
        loss.backward()
        optimizer.step()
        train_loss += loss.item()
        pbar.set_postfix({'loss': f'{loss.item():.4f}'})

    # 验证
    model.eval()
    correct = 0
    total = 0
    with torch.no_grad():
        for data, target in test_loader:
            data, target = data.to(device), target.to(device)
            output = model(data)
            _, predicted = output.max(1)
            total += target.size(0)
            correct += predicted.eq(target).sum().item()

    scheduler.step()
    avg_train_loss = train_loss / len(train_loader)
    acc = 100.0 * correct / total
    print(f"Epoch {epoch+1}: Train Loss={avg_train_loss:.4f}, Test Acc={acc:.2f}%")
```

## 14. 自定义 Trainer 类（封装训练逻辑）

将训练循环封装为一个可复用的 `Trainer` 类，支持所有常用功能。

```python
class Trainer:
    """通用训练器"""

    def __init__(self, model, train_loader, val_loader, criterion,
                 optimizer, scheduler=None, device="cpu",
                 grad_clip=None, early_stopping=None,
                 checkpoint_dir="./checkpoints",
                 use_amp=False):
        self.model = model
        self.train_loader = train_loader
        self.val_loader = val_loader
        self.criterion = criterion
        self.optimizer = optimizer
        self.scheduler = scheduler
        self.device = device
        self.grad_clip = grad_clip
        self.early_stopping = early_stopping
        self.checkpoint_dir = checkpoint_dir
        self.use_amp = use_amp and (device.type == "cuda")

        if self.use_amp:
            self.scaler = torch.cuda.amp.GradScaler()

        self.history = {"train_loss": [], "val_loss": [], "val_acc": []}
        self.best_val_acc = 0.0

    def train_epoch(self):
        self.model.train()
        total_loss = 0.0

        pbar = tqdm(self.train_loader, desc="Training", leave=False)
        for batch_x, batch_y in pbar:
            batch_x, batch_y = batch_x.to(self.device), batch_y.to(self.device)
            self.optimizer.zero_grad()

            if self.use_amp:
                with torch.cuda.amp.autocast():
                    outputs = self.model(batch_x)
                    loss = self.criterion(outputs, batch_y)
                self.scaler.scale(loss).backward()
                if self.grad_clip:
                    self.scaler.unscale_(self.optimizer)
                    torch.nn.utils.clip_grad_norm_(self.model.parameters(), self.grad_clip)
                self.scaler.step(self.optimizer)
                self.scaler.update()
            else:
                outputs = self.model(batch_x)
                loss = self.criterion(outputs, batch_y)
                loss.backward()
                if self.grad_clip:
                    torch.nn.utils.clip_grad_norm_(self.model.parameters(), self.grad_clip)
                self.optimizer.step()

            total_loss += loss.item()
            pbar.set_postfix({"loss": f"{loss.item():.4f}"})

        return total_loss / len(self.train_loader)

    @torch.no_grad()
    def validate(self):
        self.model.eval()
        total_loss = 0.0
        correct = 0
        total = 0

        for batch_x, batch_y in self.val_loader:
            batch_x, batch_y = batch_x.to(self.device), batch_y.to(self.device)
            outputs = self.model(batch_x)
            loss = self.criterion(outputs, batch_y)

            total_loss += loss.item()
            _, preds = outputs.max(1)
            total += batch_y.size(0)
            correct += preds.eq(batch_y).sum().item()

        avg_loss = total_loss / len(self.val_loader)
        acc = 100.0 * correct / total
        return avg_loss, acc

    def fit(self, num_epochs):
        for epoch in range(num_epochs):
            train_loss = self.train_epoch()
            val_loss, val_acc = self.validate()

            self.history["train_loss"].append(train_loss)
            self.history["val_loss"].append(val_loss)
            self.history["val_acc"].append(val_acc)

            if self.scheduler:
                if isinstance(self.scheduler, torch.optim.lr_scheduler.ReduceLROnPlateau):
                    self.scheduler.step(val_loss)
                else:
                    self.scheduler.step()

            current_lr = self.optimizer.param_groups[0]['lr']
            print(f"Epoch {epoch+1:3d} | "
                  f"Train Loss: {train_loss:.4f} | "
                  f"Val Loss: {val_loss:.4f} | "
                  f"Val Acc: {val_acc:.2f}% | "
                  f"LR: {current_lr:.2e}")

            # 保存最佳模型
            if val_acc > self.best_val_acc:
                self.best_val_acc = val_acc
                torch.save(self.model.state_dict(),
                          f"{self.checkpoint_dir}/best_model.pth")

            # 早停
            if self.early_stopping and self.early_stopping(val_loss):
                print(f"Early stopping at epoch {epoch+1}")
                break

        return self.history


# 使用 Trainer
trainer = Trainer(
    model=model,
    train_loader=train_loader,
    val_loader=val_loader,
    criterion=nn.CrossEntropyLoss(),
    optimizer=optim.AdamW(model.parameters(), lr=1e-3),
    scheduler=optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=10),
    device=device,
    grad_clip=1.0,
    use_amp=torch.cuda.is_available(),
    checkpoint_dir="./checkpoints",
)

history = trainer.fit(num_epochs=20)
print(f"Best val accuracy: {trainer.best_val_acc:.2f}%")
```

## 常见错误

### 错误 1: 忘记 `optimizer.zero_grad()`

```python
# 错误
for data, target in loader:
    output = model(data)
    loss = criterion(output, target)
    loss.backward()
    optimizer.step()   # 梯度不断累积，相当于用了错误梯度方向！
    # 缺少 optimizer.zero_grad()

# 正确
for data, target in loader:
    optimizer.zero_grad()   # 每次先清零
    output = model(data)
    loss = criterion(output, target)
    loss.backward()
    optimizer.step()
```

### 错误 2: 验证时忘记 `model.eval()` 和 `no_grad()`

```python
# 错误
def validate(model, loader):
    model.train()  # 错误！Dropout 仍然激活
    for data, target in loader:
        output = model(data)  # 构建了计算图，浪费显存

# 正确
@torch.no_grad()
def validate(model, loader):
    model.eval()
    for data, target in loader:
        output = model(data)
```

### 错误 3: `loss.backward()` 后没有 `optimizer.step()`

```python
# 错误
loss.backward()
# optimizer.step() 被遗忘 —— 参数永远不会更新！

# 正确
loss.backward()
optimizer.step()
```

### 错误 4: `scheduler.step()` 的位置错误

```python
# 错误：每批都 step
for batch in loader:
    ...
    loss.backward()
    optimizer.step()
    scheduler.step()  # 错误！应该每个 epoch 一次

# 正确
for epoch in range(num_epochs):
    for batch in loader:
        ...
        optimizer.step()
    scheduler.step()  # 每个 epoch 调用一次
```

### 错误 5: 在 `no_grad()` 中调用 `loss.backward()`

```python
# 错误
with torch.no_grad():
    output = model(data)
    loss = criterion(output, target)
    loss.backward()  # RuntimeError: 计算图已被禁用！

# 正确
output = model(data)          # 在 no_grad 外
loss = criterion(output, target)
loss.backward()
```

### 错误 6: 在验证循环中忘记 `with torch.no_grad()` 导致显存溢出

```python
# 错误：验证循环中构建完整计算图
model.eval()
for data, target in val_loader:
    output = model(data)        # 构建计算图
    loss = criterion(output, target)
    # 每个 batch 的计算图都保留在内存中，显存持续增长！

# 正确
model.eval()
with torch.no_grad():           # 必须加
    for data, target in val_loader:
        output = model(data)    # 不构建计算图
        loss = criterion(output, target)
```

## 本章小结

- 训练循环的核心 6 步：`model.train()` -> `zero_grad()` -> `forward` -> `loss` -> `backward()` -> `step()`
- **`model.train()`** 启用 Dropout 和 BatchNorm 训练行为；**`model.eval()`** 切换为推理行为
- **AdamW** 是当前最通用的优化器，配合 `weight_decay` 提供解耦的正则化
- SGD + Momentum 在 CV 任务中仍是强有力的选择（泛化性能好），但需要精心调参
- **学习率调度器**（CosineAnnealingLR, ReduceLROnPlateau）帮助收敛到更优解
- **Warmup** 在训练初期线性增加学习率，稳定早期训练
- 梯度裁剪是防止梯度爆炸的兜底手段，尤其在 RNN 训练中必要
- 过拟合应对按优先级：数据增强 > Dropout > Weight Decay > 早停 > 标签平滑 > 减小模型
- TensorBoard 是强大的实验可视化和日志工具，应尽早集成到训练流程中
- 混合精度训练 (AMP) 减少 50% 显存并加速，改动极小
- 模型检查点应保存 state_dict（而非整个模型），以便跨环境加载
- 验证时始终保持 `model.eval()` + `torch.no_grad()`
- 训练中遇到问题使用 Checklist 逐项排查
