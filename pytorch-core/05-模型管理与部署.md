## 5.1 模型保存与加载

### 5.1.1 核心概念：state_dict 是什么

在 PyTorch 中，每一个 `nn.Module` 子类都有一个 `state_dict()` 方法。它返回一个 **Python 有序字典（OrderedDict）**，键是层的名称，值是对应的参数张量（`nn.Parameter`）。

```python
import torch
import torch.nn as nn

class SimpleModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(10, 5)   # 权重 5x10 + 偏置 5 = 55 个参数
        self.fc2 = nn.Linear(5, 2)    # 权重 2x5 + 偏置 2 = 12 个参数

    def forward(self, x):
        x = torch.relu(self.fc1(x))
        x = self.fc2(x)
        return x

model = SimpleModel()

# 查看 state_dict 的键和形状
for name, param in model.state_dict().items():
    print(f"{name:20s}  shape={str(param.shape):15s}  dtype={param.dtype}")
```

输出示例：

```
fc1.weight            shape=torch.Size([5, 10])  dtype=torch.float32
fc1.bias              shape=torch.Size([5])       dtype=torch.float32
fc2.weight            shape=torch.Size([2, 5])    dtype=torch.float32
fc2.bias              shape=torch.Size([2])       dtype=torch.float32
```

几个关键事实：

1. **`state_dict` 只包含可学习参数**（`nn.Parameter`），不包含层的结构信息（层数、激活函数、连接方式等）。
2. **缓冲区（buffer）不在 `state_dict` 中**。比如 BatchNorm 的 `running_mean` 和 `running_var` 是 buffer，需要用 `model.state_dict()` 才能拿到？实际上 `state_dict()` **包含** buffer。我们用代码验证：

```python
# 验证 state_dict 是否包含 buffer
bn = nn.BatchNorm1d(3)
for name, buf in bn.state_dict().items():
    print(f"{name:20s}  shape={str(buf.shape)}")
# 输出：
# weight              shape=torch.Size([3])
# bias                shape=torch.Size([3])
# running_mean        shape=torch.Size([3])
# running_var         shape=torch.Size([3])
# num_batches_tracked shape=torch.Size([])
```

可以看到，`state_dict()` 确实包含了 `running_mean`、`running_var` 等 buffer。这是理解保存与加载的关键。

### 5.1.2 方式一：保存 state_dict（推荐）

这是 PyTorch 官方推荐的方式。

**保存：**

```python
# 保存模型的 state_dict
torch.save(model.state_dict(), "model_weights.pth")
```

**加载：**

```python
# 1. 先重新实例化模型（必须！因为 state_dict 不含结构信息）
model = SimpleModel()  # 模型结构必须与保存时完全一致

# 2. 加载权重
model.load_state_dict(torch.load("model_weights.pth"))

# 3. 切换到评估模式（如果用于推理）
model.eval()
```

**逐行解释：**

| 步骤 | 说明 |
|------|------|
| `torch.save(model.state_dict(), ...)` | 只保存参数字典，文件小。`model.state_dict()` 返回的是 Python dict，`torch.save` 将其序列化为 `.pth` 文件。 |
| `model = SimpleModel()` | 重新创建模型实例。**这一步是必须的**，因为 `.pth` 文件中只有权重数值，没有层的定义。 |
| `model.load_state_dict(...)` | 将加载的字典中的每个参数值复制到模型中对应层。键名必须完全匹配，否则会报错。 |
| `model.eval()` | 切换到评估模式，影响 Dropout 和 BatchNorm 的行为（见 5.3 节）。 |

**优点：**
- 文件小（只存参数，不存结构）
- 灵活：你可以修改模型结构后再加载（只要参数名匹配），例如加载预训练模型的前几层
- 跨版本兼容性更好

**缺点：**
- 加载时必须重新定义模型类，且结构必须与保存时一致

### 5.1.3 方式二：保存整个模型对象

```python
# 保存整个模型
torch.save(model, "model_full.pth")

# 加载整个模型
model = torch.load("model_full.pth")
model.eval()
```

**工作原理：**

`torch.save(model, ...)` 底层使用 Python 的 `pickle` 序列化整个模型对象，包括：
- 模型结构（类的 `__init__` 中定义的层）
- 参数值
- buffer 值

**陷阱：类定义必须存在且路径一致。**

```python
# 假设你在 my_project/models.py 中定义了 SimpleModel
# torch.save(model, "model_full.pth")

# 在另一个脚本中加载：
model = torch.load("model_full.pth")  # 可能报错！
# AttributeError: Can't get attribute 'SimpleModel' on <module '__main__'>
```

**原因：** pickle 序列化时记录了类的完整模块路径（如 `my_project.models.SimpleModel`）。如果加载时这个类不在相同的模块路径下，就会失败。

**解决方案：**

```python
# 方案 A：在加载脚本中导入或定义相同的类
from my_project.models import SimpleModel  # 确保类在命名空间中
model = torch.load("model_full.pth")

# 方案 B：使用 pickle_module 参数
import pickle
model = torch.load("model_full.pth", pickle_module=pickle)

# 方案 C（最稳妥）：使用 torch.serialization.add_safe_globals
torch.serialization.add_safe_globals([SimpleModel])
model = torch.load("model_full.pth", weights_only=False)
```

**对比总结：**

| 特性 | `state_dict` 方式 | 整个模型方式 |
|------|-------------------|-------------|
| 文件大小 | 小（仅参数） | 大（含结构） |
| 加载灵活性 | 高（可部分加载） | 低 |
| 跨版本兼容 | 好 | 差 |
| 使用便捷性 | 需重定义模型类 | 一行加载 |
| 安全性 | 高（`weights_only=True` 默认） | 低（pickle 风险） |
| 推荐程度 | **强烈推荐** | 不推荐 |

### 5.1.4 保存优化器状态（断点续训基础）

优化器也有 `state_dict()`，包含：
- 每个参数组的超参数（lr、weight_decay 等）
- 优化器内部状态（如 Adam 的动量 `exp_avg`、`exp_avg_sq`）

```python
import torch.optim as optim

model = SimpleModel()
optimizer = optim.Adam(model.parameters(), lr=0.001)

# 模拟几步训练
x = torch.randn(4, 10)
y = torch.randn(4, 2)
for _ in range(3):
    optimizer.zero_grad()
    loss = nn.MSELoss()(model(x), y)
    loss.backward()
    optimizer.step()

# 查看优化器的 state_dict 的键
print("Optimizer state_dict keys:")
for key in optimizer.state_dict().keys():
    print(f"  {key}")

# 保存模型 + 优化器
checkpoint = {
    "model_state_dict": model.state_dict(),
    "optimizer_state_dict": optimizer.state_dict(),
}
torch.save(checkpoint, "checkpoint.pth")
```

优化器的 `state_dict` 典型结构：

```
{
    'state': {
        0: {  # 参数 ID（对应 model.parameters() 中的索引）
            'exp_avg': tensor(...),   # Adam 一阶矩
            'exp_avg_sq': tensor(...), # Adam 二阶矩
            'step': tensor(0)          # 步数
        },
        1: {...},
        ...
    },
    'param_groups': [{
        'lr': 0.001,
        'betas': (0.9, 0.999),
        'eps': 1e-08,
        'weight_decay': 0,
        'amsgrad': False,
        'params': [0, 1, 2, 3]  # 参数 ID 列表
    }]
}
```

## 5.2 训练中断与恢复（Checkpoint）

### 5.2.1 完整的 Checkpoint 字典

一个生产级别的 checkpoint 通常包含：

```python
checkpoint = {
    "epoch": epoch,                        # 当前 epoch 编号
    "model_state_dict": model.state_dict(), # 模型参数
    "optimizer_state_dict": optimizer.state_dict(),  # 优化器状态
    "loss": loss.item(),                   # 最近的 loss 值（方便监控）
    "scheduler_state_dict": scheduler.state_dict(),  # 学习率调度器状态（可选）
    "best_accuracy": best_accuracy,        # 历史最佳准确率（可选）
    "args": args_dict,                     # 训练参数记录（可选，用于复现）
}
```

### 5.2.2 完整训练代码（含 Checkpoint 保存与恢复）

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset
import os

# ============================================================
# 1. 定义模型
# ============================================================
class Classifier(nn.Module):
    def __init__(self, input_dim=20, hidden_dim=64, num_classes=3):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dim, num_classes),
        )

    def forward(self, x):
        return self.net(x)

# ============================================================
# 2. 超参数与配置
# ============================================================
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
LEARNING_RATE = 0.001
NUM_EPOCHS = 50
CHECKPOINT_DIR = "./checkpoints"
CHECKPOINT_PATH = os.path.join(CHECKPOINT_DIR, "checkpoint.pth")
BEST_MODEL_PATH = os.path.join(CHECKPOINT_DIR, "best_model.pth")
RESUME_TRAINING = True  # 是否从 checkpoint 恢复

os.makedirs(CHECKPOINT_DIR, exist_ok=True)

# ============================================================
# 3. 准备数据（模拟数据用于演示）
# ============================================================
torch.manual_seed(42)
X_train = torch.randn(1000, 20)
y_train = torch.randint(0, 3, (1000,))
X_val = torch.randn(200, 20)
y_val = torch.randint(0, 3, (200,))

train_loader = DataLoader(
    TensorDataset(X_train, y_train), batch_size=32, shuffle=True
)
val_loader = DataLoader(
    TensorDataset(X_val, y_val), batch_size=32, shuffle=False
)

# ============================================================
# 4. 初始化模型、损失函数、优化器、调度器
# ============================================================
model = Classifier(input_dim=20, hidden_dim=64, num_classes=3).to(device)
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=LEARNING_RATE)
scheduler = optim.lr_scheduler.StepLR(optimizer, step_size=10, gamma=0.5)

# ============================================================
# 5. 从 checkpoint 恢复训练（如果存在）
# ============================================================
start_epoch = 0
best_val_acc = 0.0

if RESUME_TRAINING and os.path.exists(CHECKPOINT_PATH):
    print(f"[INFO] 发现 checkpoint: {CHECKPOINT_PATH}，正在恢复...")
    checkpoint = torch.load(CHECKPOINT_PATH, map_location=device)

    model.load_state_dict(checkpoint["model_state_dict"])
    optimizer.load_state_dict(checkpoint["optimizer_state_dict"])
    scheduler.load_state_dict(checkpoint["scheduler_state_dict"])
    start_epoch = checkpoint["epoch"] + 1  # 从下一个 epoch 开始
    best_val_acc = checkpoint.get("best_accuracy", 0.0)

    print(f"[INFO] 恢复成功！从 Epoch {start_epoch} 继续训练")
    print(f"[INFO] 历史最佳验证准确率: {best_val_acc:.4f}")
else:
    print("[INFO] 未找到 checkpoint，从头开始训练")

# ============================================================
# 6. 验证函数
# ============================================================
@torch.no_grad()
def evaluate(model, dataloader, criterion, device):
    model.eval()
    total_loss = 0.0
    correct = 0
    total = 0
    for inputs, labels in dataloader:
        inputs, labels = inputs.to(device), labels.to(device)
        outputs = model(inputs)
        loss = criterion(outputs, labels)
        total_loss += loss.item() * inputs.size(0)
        _, predicted = torch.max(outputs, 1)
        correct += (predicted == labels).sum().item()
        total += labels.size(0)
    return total_loss / total, correct / total

# ============================================================
# 7. 训练循环（含 checkpoint 保存）
# ============================================================
for epoch in range(start_epoch, NUM_EPOCHS):
    # ---- 训练阶段 ----
    model.train()
    train_loss = 0.0
    for inputs, labels in train_loader:
        inputs, labels = inputs.to(device), labels.to(device)

        optimizer.zero_grad()
        outputs = model(inputs)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()

        train_loss += loss.item() * inputs.size(0)

    train_loss /= len(train_loader.dataset)
    scheduler.step()  # 更新学习率

    # ---- 验证阶段 ----
    val_loss, val_acc = evaluate(model, val_loader, criterion, device)

    # ---- 打印进度 ----
    current_lr = optimizer.param_groups[0]["lr"]
    print(f"Epoch {epoch+1:3d}/{NUM_EPOCHS} | "
          f"Train Loss: {train_loss:.4f} | "
          f"Val Loss: {val_loss:.4f} | "
          f"Val Acc: {val_acc:.4f} | "
          f"LR: {current_lr:.6f}")

    # ---- 保存 checkpoint（每个 epoch） ----
    checkpoint = {
        "epoch": epoch,
        "model_state_dict": model.state_dict(),
        "optimizer_state_dict": optimizer.state_dict(),
        "scheduler_state_dict": scheduler.state_dict(),
        "loss": train_loss,
        "best_accuracy": best_val_acc,
    }
    torch.save(checkpoint, CHECKPOINT_PATH)
    # 注意：每个 epoch 覆盖同一个 checkpoint 文件。也可以保存最近 N 个。

    # ---- 保存最佳模型 ----
    if val_acc > best_val_acc:
        best_val_acc = val_acc
        torch.save(model.state_dict(), BEST_MODEL_PATH)
        print(f"  >>> 新的最佳模型！Val Acc: {best_val_acc:.4f}，已保存至 {BEST_MODEL_PATH}")

print(f"\n训练完成！最佳验证准确率: {best_val_acc:.4f}")
```

### 5.2.3 恢复训练的关键细节

**细节 1：`map_location` 参数**

```python
# 在 CPU 机器上加载 GPU 上保存的 checkpoint
checkpoint = torch.load("checkpoint.pth", map_location="cpu")

# 加载到指定 GPU
checkpoint = torch.load("checkpoint.pth", map_location="cuda:0")
```

如果 GPU 保存的 tensor 直接 `torch.load` 到 CPU 机器（没有 CUDA），会报错。`map_location` 解决了这个问题。

**细节 2：`optimizer.load_state_dict` 需要模型先在正确的 device 上**

```python
model = MyModel().to(device)  # 先移到 device
optimizer = optim.Adam(model.parameters())  # 再创建优化器
checkpoint = torch.load("checkpoint.pth", map_location=device)
model.load_state_dict(checkpoint["model_state_dict"])
optimizer.load_state_dict(checkpoint["optimizer_state_dict"])  # 此时才能正确加载
```

原因：优化器的 `state` 字典中存储的动量等张量，其 device 必须与模型参数的 device 一致。

**细节 3：Random Seed 的复现**

如果要在恢复训练后获得完全一致的结果，还需要保存和恢复随机种子：

```python
import random
import numpy as np

# 保存时：
checkpoint["python_seed"] = random.getstate()
checkpoint["numpy_seed"] = np.random.get_state()
checkpoint["torch_seed"] = torch.get_rng_state()
checkpoint["torch_cuda_seed"] = torch.cuda.get_rng_state()  # 如果使用 CUDA

# 恢复时：
random.setstate(checkpoint["python_seed"])
np.random.set_state(checkpoint["numpy_seed"])
torch.set_rng_state(checkpoint["torch_seed"])
torch.cuda.set_rng_state(checkpoint["torch_cuda_seed"])
```

### 5.2.4 生产级别的 Checkpoint 策略

```python
import glob

def save_checkpoint(state, filename, keep_best_n=3):
    """
    保存 checkpoint，并保留最近 keep_best_n 个文件。
    """
    torch.save(state, filename)

    # 清理旧 checkpoint
    checkpoint_files = sorted(glob.glob(os.path.join(CHECKPOINT_DIR, "checkpoint_epoch_*.pth")))
    if len(checkpoint_files) > keep_best_n:
        for old_file in checkpoint_files[:-keep_best_n]:
            os.remove(old_file)
            print(f"[INFO] 删除旧 checkpoint: {old_file}")

# 使用：
save_checkpoint(
    checkpoint,
    os.path.join(CHECKPOINT_DIR, f"checkpoint_epoch_{epoch+1:03d}.pth"),
    keep_best_n=3,
)
```

## 5.3 模型评估模式

### 5.3.1 `model.train()` vs `model.eval()`

这两个方法影响的是模型中**特定层**的行为，而不是所有层。

**受影响的层：**

| 层类型 | `model.train()` 行为 | `model.eval()` 行为 |
|--------|---------------------|---------------------|
| `nn.Dropout` | 随机将输入元素置零（概率为 p） | 不置零，直接通过（乘以 1-p 的缩放已在训练时由 PyTorch 处理） |
| `nn.BatchNorm1d/2d/3d` | 使用当前 batch 的均值/方差，同时更新 `running_mean` 和 `running_var` | 使用历史累积的 `running_mean` 和 `running_var`，不更新 |
| `nn.LayerNorm` | 通常不受影响 | 通常不受影响 |
| `nn.InstanceNorm` | 不受 `train`/`eval` 切换影响（设计如此） | 同上 |

### 5.3.2 演示 Dropout 的行为差异

```python
import torch
import torch.nn as nn

torch.manual_seed(42)
dropout = nn.Dropout(p=0.5)
x = torch.ones(1, 6)  # 全 1 的输入

# 训练模式：约一半的元素变为 0，其余变为 2（补偿因子 1/(1-p) = 2）
dropout.train()
print("Train mode (run 1):", dropout(x))
print("Train mode (run 2):", dropout(x))  # 每次结果不同！
print("Train mode (run 3):", dropout(x))

# 评估模式：所有元素保持不变
dropout.eval()
print("Eval  mode (run 1):", dropout(x))
print("Eval  mode (run 2):", dropout(x))
print("Eval  mode (run 3):", dropout(x))
```

### 5.3.3 演示 BatchNorm 的行为差异

```python
bn = nn.BatchNorm1d(3, track_running_stats=True)

# 生成一些数据
torch.manual_seed(42)
x_train = torch.randn(8, 3)  # 训练数据
x_test = torch.randn(4, 3)   # 测试数据

# 训练模式：使用当前 batch 统计量，并更新 running stats
bn.train()
out_train = bn(x_train)
print(f"Train mode - running_mean: {bn.running_mean}")

# 评估模式：使用 running stats（固定不变）
bn.eval()
out_eval_1 = bn(x_test)
print(f"Eval  mode - running_mean: {bn.running_mean}")  # 不会因 x_test 而改变
out_eval_2 = bn(x_test)
print(f"Eval  mode (再次) - running_mean: {bn.running_mean}")
```

**关键规则：**

```python
# 正确的推理写法
model.eval()
with torch.no_grad():
    outputs = model(inputs)
```

如果忘记 `model.eval()`，而模型中包含 BatchNorm，那么推理时 BatchNorm 层会使用单个或少数样本的统计量来归一化（测试时往往 batch size 很小），导致结果不稳定且不可复现。

### 5.3.4 `torch.no_grad()` vs `torch.inference_mode()`

两者的共同作用：
- 禁用自动求导（autograd），不构建计算图
- 减少显存占用，加速推理

两者的区别：

```python
import torch
import time

x = torch.randn(1000, 1000)

# ---- torch.no_grad() ----
# 旧版，仍会记录部分 autograd 元数据（用于 view/detach 兼容）
with torch.no_grad():
    y = x * 2 + 1
    print(f"no_grad: requires_grad={y.requires_grad}")  # False, 但 tensor 版本号会被追踪

# ---- torch.inference_mode() (PyTorch 1.9+) ----
# 完整禁用 autograd，比 no_grad 更快
with torch.inference_mode():
    y = x * 2 + 1
    print(f"inference_mode: requires_grad={y.requires_grad}")  # False

# ---- 性能对比 ----
x_gpu = torch.randn(1000, 1000, device="cuda") if torch.cuda.is_available() else x

# 基准（无上下文管理器）
def benchmark(fn, label, n=500):
    start = time.time()
    for _ in range(n):
        fn()
    if torch.cuda.is_available():
        torch.cuda.synchronize()
    print(f"{label}: {time.time() - start:.4f}s")

def run_no_grad():
    with torch.no_grad():
        _ = x * 2 + 1

def run_inference_mode():
    with torch.inference_mode():
        _ = x * 2 + 1

benchmark(run_no_grad, "no_grad        ")
benchmark(run_inference_mode, "inference_mode ")

# 注意：inference_mode 内部无法创建 requires_grad=True 的 tensor
try:
    with torch.inference_mode():
        w = torch.randn(10, requires_grad=True)  # 报错！
except RuntimeError as e:
    print(f"\n错误（符合预期）: {e}")
```

**选择建议：**

| 场景 | 推荐 |
|------|------|
| 验证/测试循环 | `torch.inference_mode()` |
| 需要创建可求导 tensor 的上下文（极少见） | `torch.no_grad()` |
| 推理服务 | `torch.inference_mode()` |
| 最大兼容性（旧版 PyTorch） | `torch.no_grad()` |

## 5.4 模型导出

### 5.4.1 为什么需要导出模型

- **跨平台部署**：训练用 Python/PyTorch，部署可能用 C++、移动端、Web
- **跨框架互操作**：PyTorch 训练，TensorRT/ONNX Runtime/OpenVINO 推理
- **性能优化**：导出的模型可以做算子融合、常量折叠等编译优化
- **无需 Python 环境**：导出为独立格式后，推理不依赖 PyTorch

### 5.4.2 TorchScript

TorchScript 是 PyTorch 的中间表示（IR），可以将 Python 模型转换为可优化的、可序列化的图表示。TorchScript 模型可以在 C++ 运行时（LibTorch）中运行，无需 Python 解释器。

**两种生成方式：**

#### 方式 A：`torch.jit.script`（基于 AST 分析）

直接解析 Python 源代码的抽象语法树，转换为 TorchScript IR。

```python
import torch
import torch.nn as nn

class ScriptableModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc = nn.Linear(10, 5)

    def forward(self, x):
        # 支持控制流！
        if x.sum() > 0:
            x = torch.relu(self.fc(x))
        else:
            x = torch.sigmoid(self.fc(x))
        return x

model = ScriptableModel()

# 使用 torch.jit.script 转换
scripted_model = torch.jit.script(model)
print(scripted_model.code)  # 查看生成的 TorchScript 代码

# 保存
scripted_model.save("model_scripted.pt")

# 加载
loaded_model = torch.jit.load("model_scripted.pt")
```

**优点：**
- 支持 Python 控制流（if/for/while）
- 更通用，适用于复杂模型

**缺点：**
- 对 Python 语法有严格限制（不是所有 Python 语法都支持）
- 可能在复杂代码上失败

#### 方式 B：`torch.jit.trace`（基于输入追踪）

给模型一个示例输入，记录实际执行的操作序列。

```python
# 使用 trace
example_input = torch.randn(1, 10)
traced_model = torch.jit.trace(model, example_input)

# 保存
traced_model.save("model_traced.pt")

# 加载
loaded_model = torch.jit.load("model_traced.pt")
```

**优点：**
- 简单直接，对大部分前馈模型有效
- 不要求代码符合 TorchScript 语法

**缺点：**
- **不支持控制流**（只记录一条执行路径）。如果模型中有 if 语句，trace 只会记录触发的那条分支
- 对动态形状支持不好

#### script vs trace 的关键区别

```python
class ControlFlowModel(nn.Module):
    def forward(self, x):
        # 当 x.sum() > 0 走分支 A，否则走分支 B
        if x.sum() > 0:
            return x * 2    # 分支 A
        else:
            return x * 3    # 分支 B

model = ControlFlowModel()

# ---- trace：只记录一条路径 ----
traced = torch.jit.trace(model, torch.tensor([1.0, 2.0]))  # sum > 0，走分支 A
print("Traced 输出 [1,2]:", traced(torch.tensor([1.0, 2.0])))   # [2, 4] --- 正确
print("Traced 输出 [-5,-6]:", traced(torch.tensor([-5.0, -6.0])))  # [-10, -12] --- 错误！应该是 [-15, -18]

# ---- script：完整保留控制流 ----
scripted = torch.jit.script(model)
print("Script 输出 [1,2]:", scripted(torch.tensor([1.0, 2.0])))   # [2, 4]   --- 正确
print("Script 输出 [-5,-6]:", scripted(torch.tensor([-5.0, -6.0])))  # [-15, -18] --- 正确！
```

**选择决策：**

```
模型是否有 if/for 依赖于输入数据？
    ├── 是 → 使用 torch.jit.script
    └── 否（纯前馈） → 使用 torch.jit.trace（更简单）
```

### 5.4.3 ONNX 导出

ONNX（Open Neural Network Exchange）是跨框架的模型交换格式。PyTorch 模型导出为 ONNX 后，可以在 ONNX Runtime、TensorRT、OpenVINO 等多种推理引擎上运行。

```python
import torch
import torch.nn as nn

class ONNXModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv = nn.Conv2d(3, 16, 3, padding=1)
        self.pool = nn.AdaptiveAvgPool2d(1)
        self.fc = nn.Linear(16, 10)

    def forward(self, x):
        x = torch.relu(self.conv(x))
        x = self.pool(x)
        x = x.view(x.size(0), -1)
        x = self.fc(x)
        return x

model = ONNXModel()
model.eval()

# 准备示例输入
dummy_input = torch.randn(1, 3, 32, 32)

# 导出为 ONNX
torch.onnx.export(
    model,                          # 模型
    dummy_input,                    # 示例输入
    "model.onnx",                   # 输出文件名
    export_params=True,             # 是否保存参数
    opset_version=14,               # ONNX opset 版本
    do_constant_folding=True,       # 是否做常量折叠优化
    input_names=["input"],          # 输入节点名
    output_names=["output"],        # 输出节点名
    dynamic_axes={                  # 动态轴配置
        "input": {0: "batch_size"},    # 第 0 维（batch）是动态的
        "output": {0: "batch_size"},
    },
)

print("ONNX 模型已导出至 model.onnx")

# ---- 验证 ONNX 模型 ----
import onnx  # pip install onnx

onnx_model = onnx.load("model.onnx")
onnx.checker.check_model(onnx_model)
print("ONNX 模型验证通过！")

# ---- 使用 ONNX Runtime 推理 ----
import onnxruntime  # pip install onnxruntime
import numpy as np

ort_session = onnxruntime.InferenceSession("model.onnx")

# 准备输入（numpy 格式）
ort_inputs = {"input": dummy_input.numpy()}
ort_outputs = ort_session.run(None, ort_inputs)
print(f"ONNX Runtime 输出 shape: {ort_outputs[0].shape}")

# 与 PyTorch 输出对比
with torch.no_grad():
    torch_output = model(dummy_input)
print(f"PyTorch 与 ONNX 输出最大误差: {(torch_output.numpy() - ort_outputs[0]).max():.8f}")
```

**常见导出配置选项：**

| 参数 | 说明 | 建议值 |
|------|------|--------|
| `opset_version` | ONNX 算子集版本，越高支持的操作越多 | 14 或更高 |
| `do_constant_folding` | 预计算常量表达式，减少图节点 | `True` |
| `dynamic_axes` | 声明动态维度（如可变 batch size） | 按需设置 |
| `input_names` / `output_names` | 给输入/输出节点命名 | 有意义的名称 |

## 5.5 模型压缩简介

### 5.5.1 为什么需要压缩

一个 ResNet-50 大约 25M 参数，FP32 权重约 100MB，在手机或 IoT 设备上部署不现实。压缩技术可以在保持精度的前提下大幅减小模型体积和计算量。

### 5.5.2 量化（Quantization）

量化是将 FP32（32 位浮点）权重和/或激活值转换为更低精度（如 INT8）的过程。

**原理：** 将浮点值映射到整数范围，推理时使用整数运算（更快、更省电）。

**三种主要方式：**

#### 1. 动态量化（Dynamic Quantization）

最简单的方式。只量化权重（FP32 -> INT8），激活值在推理时动态量化。

```python
import torch.quantization

model = SimpleModel()
model.eval()

# 动态量化：对 nn.Linear 和 nn.LSTM 等层有效
quantized_model = torch.quantization.quantize_dynamic(
    model,                          # 原始模型
    {nn.Linear, nn.LSTM},          # 要量化的层类型集合
    dtype=torch.qint8,              # 量化数据类型
)

print(quantized_model)

# 模型大小对比
torch.save(model.state_dict(), "fp32_model.pth")
torch.save(quantized_model.state_dict(), "int8_model.pth")
import os
print(f"FP32 模型大小: {os.path.getsize('fp32_model.pth')} bytes")
print(f"INT8 模型大小: {os.path.getsize('int8_model.pth')} bytes")
```

**特点：**
- 使用最简单，一行代码
- 只量化权重，激活仍为浮点
- 适用于 LSTM / Transformer（权重比激活占更多计算）
- 精度损失较小

#### 2. 静态量化（Static Quantization）

权重和激活值都量化。需要校准数据来确定激活值的量化范围。

```python
# 静态量化需要：
# 1. 在模型定义中插入 QuantStub / DeQuantStub
# 2. 准备校准数据集
# 3. 使用 torch.quantization.prepare + convert

class QuantizableModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.quant = torch.quantization.QuantStub()    # 输入量化桩
        self.fc1 = nn.Linear(10, 5)
        self.relu = nn.ReLU()
        self.fc2 = nn.Linear(5, 2)
        self.dequant = torch.quantization.DeQuantStub() # 输出反量化桩

    def forward(self, x):
        x = self.quant(x)
        x = self.relu(self.fc1(x))
        x = self.fc2(x)
        x = self.dequant(x)
        return x

# 静态量化流程（伪代码，实际需要更细致的配置）
# model = QuantizableModel()
# model.eval()
# model.qconfig = torch.quantization.get_default_qconfig("fbgemm")
# model_prepared = torch.quantization.prepare(model)
# # 用校准数据运行模型
# for data in calibration_loader:
#     model_prepared(data)
# model_quantized = torch.quantization.convert(model_prepared)
```

#### 3. 量化感知训练（QAT，Quantization-Aware Training）

在训练过程中模拟量化误差，使模型学习适应低精度。

**量化方式对比：**

| 方式 | 精度损失 | 速度提升 | 使用复杂度 | 适用场景 |
|------|---------|---------|-----------|---------|
| 动态量化 | 小 | 中（1.5-2x） | 极低 | LSTM / Transformer |
| 静态量化 | 中 | 高（2-4x） | 中 | CNN / 生产部署 |
| QAT | 最小 | 高（2-4x） | 高 | 对精度要求极高的场景 |

### 5.5.3 剪枝（Pruning）

剪枝通过移除不重要的权重（设为零），使模型变稀疏。稀疏矩阵可以用专门的存储格式减少内存占用。

```python
import torch.nn.utils.prune as prune

model = SimpleModel()

# 查看剪枝前的权重
print("剪枝前 fc1.weight:")
print(model.fc1.weight)

# ---- 方式 1：全局非结构化剪枝（L1 范数） ----
# 将 fc1 层中 L1 范数最小的 30% 权重设为零
prune.l1_unstructured(model.fc1, name="weight", amount=0.3)

print("\n剪枝后 fc1.weight:")
print(model.fc1.weight)

# 查看掩码
print("\n剪枝掩码 (1=保留, 0=剪除):")
print(model.fc1.weight_mask)

# ---- 移除剪枝（使稀疏永久化） ----
prune.remove(model.fc1, "weight")
# remove 后，weight 不再是 weight_orig + weight_mask，而是直接变为原始矩阵
# 其中被剪掉的元素永久为 0

# ---- 方式 2：结构化剪枝（移除整个神经元） ----
# 按 L2 范数移除 20% 的神经元（即整行/整列为零）
prune.ln_structured(model.fc2, name="weight", amount=0.2, n=2, dim=0)
# dim=0 表示移除行（输出神经元）
```

**剪枝方法汇总：**

| 方法 | 粒度 | 硬件加速 | 说明 |
|------|------|---------|------|
| `l1_unstructured` | 单个权重 | 困难（需稀疏矩阵支持） | 将不重要的单个权重置零 |
| `random_unstructured` | 单个权重 | 困难 | 随机置零 |
| `ln_structured` | 通道/神经元 | 容易 | 直接移除整行/整列，减小矩阵维度 |
| `random_structured` | 通道/神经元 | 容易 | 随机移除整行/整列 |

### 5.5.4 知识蒸馏（Knowledge Distillation）

知识蒸馏不修改模型结构，而是用一个大模型（Teacher）来教一个小模型（Student）。

**核心思想：** Teacher 不仅给出正确类别（hard label），还给出各类别的概率分布（soft label），其中包含了类别之间的相似性信息。

**概念示意：**

```
输入图片（猫）
    │
    ├──> Teacher (ResNet-152) ──> soft labels: [猫:0.7, 虎:0.2, 狗:0.08, ...]
    │                                    └── 包含"猫和虎很像"的知识
    │
    └──> Student (MobileNet) ──> 学习目标：
         ├── 分类 loss（与真实标签）
         └── 蒸馏 loss（与 Teacher 的 soft labels，KL 散度）
```

**简化的蒸馏训练代码框架：**

```python
# 假设 teacher_model 已经训练好，student_model 是要训练的小模型
teacher_model.eval()
temperature = 4.0  # 温度参数，越高 soft labels 越"软"（更平滑）
alpha = 0.7        # 蒸馏 loss 权重

for inputs, labels in dataloader:
    # Teacher 预测（no_grad）
    with torch.no_grad():
        teacher_logits = teacher_model(inputs)

    # Student 预测
    student_logits = student_model(inputs)

    # 硬标签 loss（标准交叉熵）
    hard_loss = nn.CrossEntropyLoss()(student_logits, labels)

    # 软标签 loss（KL 散度，关注 Teacher 输出的分布）
    soft_loss = nn.KLDivLoss(reduction="batchmean")(
        torch.log_softmax(student_logits / temperature, dim=1),
        torch.softmax(teacher_logits / temperature, dim=1),
    ) * (temperature ** 2)  # 梯度缩放

    # 总 loss
    total_loss = alpha * soft_loss + (1 - alpha) * hard_loss

    total_loss.backward()
    optimizer.step()
```

**蒸馏的关键超参数：**

| 参数 | 作用 | 典型值 |
|------|------|--------|
| `temperature` | 控制 soft labels 的"软度"。越高，各类别概率越接近 | 2 -- 20 |
| `alpha` | 蒸馏 loss 与硬标签 loss 的权重比 | 0.5 -- 0.9 |

## 5.8 常见错误

### 错误 1：忘记 model.eval() 导致 BatchNorm 行为异常

```python
# ---- 错误代码 ----
model = MyModelWithBN()
model.load_state_dict(torch.load("best.pth"))
# 忘记 model.eval()！
with torch.no_grad():
    predictions = model(test_data)  # BatchNorm 使用了单样本统计量！

# ---- 正确代码 ----
model = MyModelWithBN()
model.load_state_dict(torch.load("best.pth"))
model.eval()  # 必须调用！
with torch.no_grad():
    predictions = model(test_data)
```

### 错误 2：在 inference_mode 下创建 requires_grad 张量

```python
# ---- 错误代码 ----
with torch.inference_mode():
    x = torch.randn(10, requires_grad=True)  # RuntimeError!

# ---- 正确代码 ----
with torch.inference_mode():
    x = torch.randn(10)  # requires_grad=False，OK
```

### 错误 3：保存和加载的模型结构不一致

```python
# ---- 错误代码 ----
# 保存时
model = nn.Linear(10, 5)
torch.save(model.state_dict(), "weights.pth")

# 加载时——结构不同！
model = nn.Linear(10, 3)  # 输出维度 5 -> 3
model.load_state_dict(torch.load("weights.pth"))  # RuntimeError: size mismatch!

# ---- 正确代码 ----
# 保存时
model = nn.Linear(10, 5)
torch.save(model.state_dict(), "weights.pth")

# 加载时——结构必须一致
model = nn.Linear(10, 5)
model.load_state_dict(torch.load("weights.pth"))  # OK
```

### 错误 4：torch.load 时未指定 map_location

```python
# ---- 错误代码 (在 CPU 机器上加载 GPU checkpoint) ----
checkpoint = torch.load("gpu_checkpoint.pth")  # RuntimeError: CUDA error

# ---- 正确代码 ----
checkpoint = torch.load("gpu_checkpoint.pth", map_location="cpu")
checkpoint = torch.load("gpu_checkpoint.pth", map_location="cuda:0")  # 或指定 GPU
```

### 错误 5：优化器 state_dict 加载顺序错误

```python
# ---- 错误代码 ----
checkpoint = torch.load("checkpoint.pth")
optimizer.load_state_dict(checkpoint["optimizer_state_dict"])  # 先加载优化器
model.load_state_dict(checkpoint["model_state_dict"])           # 后加载模型
# 优化器内部的动量张量 device 可能与模型参数不一致！

# ---- 正确代码 ----
model = MyModel().to(device)
optimizer = optim.Adam(model.parameters())
checkpoint = torch.load("checkpoint.pth", map_location=device)
model.load_state_dict(checkpoint["model_state_dict"])           # 先加载模型
optimizer.load_state_dict(checkpoint["optimizer_state_dict"])   # 再加载优化器
```

### 错误 6：混淆 script 和 trace

```python
# ---- 错误：用 trace 导出含控制流的模型 ----
class DynamicModel(nn.Module):
    def forward(self, x):
        if x.sum() > 0:
            return x * 2
        return x * 3

model = DynamicModel()
traced = torch.jit.trace(model, torch.tensor([1.0]))  # 只记录了 x*2 这条路径！
print(traced(torch.tensor([-5.0])))  # 输出 -10，但正确应该是 -15！

# ---- 正确：用 script 导出含控制流的模型 ----
scripted = torch.jit.script(model)
print(scripted(torch.tensor([-5.0])))  # 输出 -15，正确
```

## 5.9 本章小结

- **`state_dict` 保存是推荐方式**：轻量、灵活、安全。加载前需先实例化模型结构。
- **整个模型保存有隐患**：依赖 pickle 和类的模块路径，跨环境兼容性差。
- **Checkpoint 完整字典**应包含：`epoch`、`model_state_dict`、`optimizer_state_dict`、`loss`，生产环境还建议加入 `scheduler_state_dict` 和随机种子。
- **`model.eval()` 影响 Dropout 和 BatchNorm**：推理时必须调用，否则结果不准且不可复现。
- **`torch.inference_mode()` 优于 `torch.no_grad()`**：更快，但不能创建 `requires_grad=True` 的 tensor。
- **TorchScript**：`torch.jit.script` 支持控制流但语法受限；`torch.jit.trace` 简单但不支持动态控制流。
- **ONNX 导出**：实现跨框架部署；使用 `dynamic_axes` 声明可变维度。
- **模型压缩三大方向**：量化（降低精度）、剪枝（移除冗余）、蒸馏（大模型教小模型），各有适用场景。
