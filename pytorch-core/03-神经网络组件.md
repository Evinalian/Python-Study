## 1. `nn.Module`：一切网络层的基类

PyTorch 中所有神经网络组件都继承自 `torch.nn.Module`。它将**参数管理**、**前向传播**、**梯度追踪**统一封装。

```python
import torch
import torch.nn as nn

class MyModel(nn.Module):
    def __init__(self):
        super().__init__()           # 必须调用父类 __init__
        # 在此定义子层（这些子层也是 nn.Module）
        self.linear = nn.Linear(10, 5)

    def forward(self, x):
        # 定义前向传播逻辑
        return self.linear(x)
```

### 1.1 `__init__` 与 `forward` 的分工

- `__init__`：声明网络中有哪些可学习层。`nn.Module` 通过 `__setattr__` 自动追踪所有赋值为 `nn.Module` 类型的属性，将它们注册为子模块。
- `forward`：定义数据如何通过这些层。**永远不要直接调用 `model.forward(x)`**，而是调用 `model(x)` —— 后者会触发 PyTorch 的钩子机制（hooks），确保某些特性（如 Dropout 的 train/eval 模式切换）正常工作。

```python
model = MyModel()
x = torch.randn(4, 10)     # batch_size=4, input_dim=10
output = model(x)           # 正确：调用 __call__ -> forward + hooks
# output = model.forward(x) # 错误：跳过 hooks
```

### 1.2 `nn.Module` 提供的关键能力

```python
model = MyModel()

# 遍历所有参数
for name, param in model.named_parameters():
    print(f"{name}: {param.shape}")

# 将所有参数移到 GPU
model = model.to("cuda")

# 切换训练/评估模式
model.train()    # 启用 Dropout、BatchNorm 更新统计量
model.eval()     # 禁用 Dropout、BatchNorm 使用固定统计量

# 查看模型结构
print(model)

# 查看所有参数的 device 和 dtype
for name, param in model.named_parameters():
    print(f"{name}: device={param.device}, dtype={param.dtype}")

# 获取所有子模块
for name, module in model.named_modules():
    print(f"{name}: {module.__class__.__name__}")

# 获取所有子模块（包括自身）
for name, module in model.named_children():
    print(f"{name}: {module.__class__.__name__}")
# named_children 只返回直接的子模块，不递归

# 冻结/解冻参数
for param in model.parameters():
    param.requires_grad = False   # 冻结所有参数

# 只冻结特定层
for param in model.linear.parameters():
    param.requires_grad = False
```

### 1.3 `nn.ModuleDict` 和 `nn.ModuleList`

当需要动态管理子模块（而非固定的属性名）时，使用这两个容器：

```python
# nn.ModuleList: 像列表一样管理子模块
class DynamicNet(nn.Module):
    def __init__(self, layer_sizes):
        super().__init__()
        self.layers = nn.ModuleList()
        for i in range(len(layer_sizes) - 1):
            self.layers.append(nn.Linear(layer_sizes[i], layer_sizes[i+1]))
        # nn.ModuleList 正确注册了所有子模块参数

    def forward(self, x):
        for layer in self.layers[:-1]:
            x = torch.relu(layer(x))
        return self.layers[-1](x)

model = DynamicNet([10, 20, 30, 5])
print(model)
# All layers are properly registered

# 注意：如果使用普通 Python list，参数不会被注册！
class BrokenNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.layers = [nn.Linear(10, 20), nn.Linear(20, 5)]
        # 参数不会被自动追踪！model.parameters() 会返回空
```

```python
# nn.ModuleDict: 像字典一样管理子模块
class MultiTaskNet(nn.Module):
    def __init__(self, shared_dim, task_dims):
        super().__init__()
        self.shared = nn.Linear(shared_dim, 256)

        # 不同任务的输出头
        self.heads = nn.ModuleDict()
        for task_name, output_dim in task_dims.items():
            self.heads[task_name] = nn.Linear(256, output_dim)

    def forward(self, x, task_name):
        shared_features = torch.relu(self.shared(x))
        return self.heads[task_name](shared_features)

model = MultiTaskNet(128, {"classification": 10, "regression": 1})
print(model.heads.keys())  # odict_keys(['classification', 'regression'])

# 通过名称访问
cls_output = model(torch.randn(4, 128), "classification")
```

### 1.4 参数绑定（Tied Weights / Weight Sharing）

让两个层共享同一组参数，常用于自动编码器（编码器-解码器对称权重）和语言模型（输入 embedding 和输出 projection 共享）。

```python
class TiedAutoEncoder(nn.Module):
    """编码器和解码器共享权重"""

    def __init__(self, input_dim, hidden_dim):
        super().__init__()
        self.encoder = nn.Linear(input_dim, hidden_dim)
        self.decoder = nn.Linear(hidden_dim, input_dim)

        # 绑定权重：解码器权重 = 编码器权重的转置
        self.decoder.weight = nn.Parameter(self.encoder.weight.T)

        # 注意：绑定后 decoder.weight 和 encoder.weight 是不同对象，
        # 但初始值相同。如果要训练时保持绑定，需要额外处理。

    def forward(self, x):
        hidden = torch.relu(self.encoder(x))
        reconstructed = self.decoder(hidden)
        return reconstructed


# 更严格的绑定：始终共享
class StrictTiedAutoEncoder(nn.Module):
    """严格共享权重：decoder 不独立拥有 weight"""

    def __init__(self, input_dim, hidden_dim):
        super().__init__()
        self.encoder = nn.Linear(input_dim, hidden_dim, bias=False)
        self.decoder_bias = nn.Parameter(torch.zeros(input_dim))
        # decoder 的 weight 直接从 encoder.weight 转置获取

    def forward(self, x):
        hidden = torch.relu(self.encoder(x))         # W @ x
        # 解码时直接使用 encoder.weight 的转置
        reconstructed = x @ self.encoder.weight + self.decoder_bias
        return reconstructed
```

## 2. 常用层详解

### 2.1 `nn.Linear` —— 全连接层

```python
import torch.nn as nn

# nn.Linear(in_features, out_features, bias=True)
layer = nn.Linear(10, 5)   # 输入 10 维 -> 输出 5 维

x = torch.randn(32, 10)    # (batch_size, in_features)
y = layer(x)               # (32, 5)

print(layer.weight.shape)  # torch.Size([5, 10]) — W: (out, in)
print(layer.bias.shape)    # torch.Size([5])     — b: (out,)
```

内部运算：`y = x @ W.T + b`，即 `y[i, j] = sum_k (x[i, k] * W[j, k]) + b[j]`。

**参数数量**：`in_features * out_features + out_features`。

**权重矩阵形状详解**：

```
输入:  x,  shape = (batch, in_features)   例如 (32, 10)
权重:  W,  shape = (out_features, in_features)  权重在右边做转置
偏置:  b,  shape = (out_features,)
输出:  y = x @ W.T + b,  shape = (batch, out_features)  例如 (32, 5)

矩阵乘法维度:
   (32, 10) @ (10, 5) = (32, 5)
       x    @   W.T   =   y
```

> Linear 层的权重矩阵形状是 `(out_features, in_features)` 而非 `(in_features, out_features)`。这与数学课本上 `y = Wx + b` 中 `W` 是 `(out, in)` 的约定一致，但容易与某些框架混淆。

### 2.2 `nn.Conv2d` —— 二维卷积层

```python
# nn.Conv2d(in_channels, out_channels, kernel_size, stride=1, padding=0)
conv = nn.Conv2d(
    in_channels=3,          # 输入通道数（RGB 图像 = 3）
    out_channels=16,        # 输出通道数（卷积核数量）
    kernel_size=3,          # 卷积核大小 3x3
    stride=1,               # 步长
    padding=1,              # 填充（保持空间尺寸不变: padding = kernel_size // 2）
)

x = torch.randn(8, 3, 32, 32)  # (batch, channels, height, width)
y = conv(x)                     # (8, 16, 32, 32)

print(conv.weight.shape)        # torch.Size([16, 3, 3, 3])
print(conv.bias.shape)          # torch.Size([16])
```

**卷积运算的维度变化图解**：

```
输入:  (batch=8, in_c=3,  H=32, W=32)
                         |
卷积核: (out_c=16, in_c=3, kH=3, kW=3)
                         |
输出:  (batch=8, out_c=16, H'=32, W'=32)  (当 stride=1, padding=1 时 H'=H)
```

**多通道卷积详解**：

```
对于每个输出通道 j (共 16 个):
    output[j] = sum_i (input[i] 卷积 kernel[j, i]) + bias[j]
                         3 个通道
```

**输出尺寸计算公式**：

```
H_out = floor((H_in + 2 * padding - dilation * (kernel_size - 1) - 1) / stride) + 1
W_out = floor((W_in + 2 * padding - dilation * (kernel_size - 1) - 1) / stride) + 1
```

**保持尺寸不变的常用配置**：

| kernel_size | padding | dilation | 效果 |
|-------------|---------|----------|------|
| 3 | 1 | 1 | "same" padding，输出尺寸不变 (stride=1) |
| 5 | 2 | 1 | "same" padding，输出尺寸不变 (stride=1) |
| 7 | 3 | 1 | "same" padding，输出尺寸不变 (stride=1) |
| 1 | 0 | 1 | 1x1 卷积，只改变通道数，不改空间尺寸 |

**参数量**：`out_channels * in_channels * kernel_size^2 + out_channels`。

**卷积尺寸计算示例**：

```python
# 示例：计算每层输出的空间尺寸
# 输入: (1, 28, 28) MNIST 灰度图

# Conv1: in=1, out=32, k=3, p=1, s=1
# H' = (28 + 2*1 - 3)/1 + 1 = 28  (输出: 32, 28, 28)

# MaxPool: k=2, s=2
# H' = 28/2 = 14  (输出: 32, 14, 14)

# Conv2: in=32, out=64, k=3, p=1, s=1
# H' = (14 + 2*1 - 3)/1 + 1 = 14  (输出: 64, 14, 14)

# MaxPool: k=2, s=2
# H' = 14/2 = 7  (输出: 64, 7, 7)

# Flatten -> 64 * 7 * 7 = 3136
# Linear(3136, 128) -> Linear(128, 10)
```

### 2.3 感受野 (Receptive Field) 计算

卷积网络中，每层输出的每个元素在原始输入中"看到"的区域称为感受野。

```
感受野递推公式：
RF_{out} = RF_{in} + (kernel_size - 1) * stride_prod

其中 stride_prod 是之前所有层的 stride 乘积。

示例：两个 3x3 卷积，stride=1
第 1 层：RF = 1 + (3-1)*1 = 3x3
第 2 层：RF = 3 + (3-1)*1 = 5x5

两个 3x3 卷积的等效感受野 = 一个 5x5 卷积
但参数量：2*3*3=18 vs 5*5=25，且两个 3x3 有更多非线性
```

### 2.4 1x1 卷积的作用

1x1 卷积看似无用（kernel_size=1 不提取空间信息），但有几个重要用途：

```python
# 1. 通道数升降维（降维减少参数量，升维增加表达能力）
conv_1x1_reduce = nn.Conv2d(256, 64, kernel_size=1)   # 降维
conv_1x1_expand = nn.Conv2d(64, 256, kernel_size=1)   # 升维

# 2. 跨通道信息融合（每个空间位置的各通道做线性组合）
# 等价于对每个像素的所有通道做一次 Linear 变换

# 3. 在瓶颈结构中减少计算量（ResNet Bottleneck）
class Bottleneck(nn.Module):
    def __init__(self, in_c, mid_c, out_c):
        super().__init__()
        # 1x1 降维 -> 3x3 卷积 -> 1x1 升维
        self.conv1 = nn.Conv2d(in_c, mid_c, kernel_size=1)    # 降维
        self.conv2 = nn.Conv2d(mid_c, mid_c, kernel_size=3, padding=1)
        self.conv3 = nn.Conv2d(mid_c, out_c, kernel_size=1)   # 升维

    def forward(self, x):
        identity = x
        out = torch.relu(self.conv1(x))
        out = torch.relu(self.conv2(out))
        out = self.conv3(out)
        return torch.relu(out + identity)
```

### 2.5 RNN 系列 —— 序列建模

```python
# nn.LSTM(input_size, hidden_size, num_layers=1, batch_first=False, bidirectional=False)
# 默认 batch_first=False: input shape (seq_len, batch, input_size)
# 设置 batch_first=True: input shape (batch, seq_len, input_size)

lstm = nn.LSTM(
    input_size=128,           # 输入特征维度
    hidden_size=256,          # 隐藏状态维度
    num_layers=2,             # LSTM 层数
    batch_first=True,         # 第一维是 batch
    bidirectional=True,       # 双向 LSTM
)

# 输入: (batch, seq_len, input_size)
x = torch.randn(16, 50, 128)
output, (h_n, c_n) = lstm(x)

print(output.shape)  # (16, 50, 512) — hidden_size * 2（因为 bidirectional）
print(h_n.shape)     # (4, 16, 256)    — D * num_layers, batch, hidden_size
print(c_n.shape)     # (4, 16, 256)    — D * num_layers, batch, hidden_size
```

**LSTM 门控机制详解**：

LSTM 通过三个门控制信息的流动：

```
输入门 i_t:   控制新信息写入 cell state 的量
遗忘门 f_t:   控制旧 cell state 保留的量
输出门 o_t:   控制从 cell state 输出到 hidden state 的量

公式（简化）：
i_t = sigmoid(W_i * [h_{t-1}, x_t] + b_i)      # 输入门
f_t = sigmoid(W_f * [h_{t-1}, x_t] + b_f)      # 遗忘门
o_t = sigmoid(W_o * [h_{t-1}, x_t] + b_o)      # 输出门
g_t = tanh(W_g * [h_{t-1}, x_t] + b_g)         # 候选值
c_t = f_t * c_{t-1} + i_t * g_t                # cell state 更新
h_t = o_t * tanh(c_t)                           # hidden state 输出
```

**Hidden State vs Cell State**：

| 状态 | 作用 | 用途 |
|------|------|------|
| Hidden State (h_t) | 短期记忆，当前时刻的输出 | 传给下一层/做预测 |
| Cell State (c_t) | 长期记忆，携带跨时间步的信息 | 仅在 LSTM 层间传递 |

**双向 LSTM 输出拼接**：

```python
# 双向 LSTM 的前向和后向输出拼接
lstm = nn.LSTM(128, 256, bidirectional=True, batch_first=True)
x = torch.randn(16, 50, 128)
output, (h_n, c_n) = lstm(x)

# output shape: (batch, seq_len, 2 * hidden_size) = (16, 50, 512)
# output 的前 256 维是前向 LSTM 输出，后 256 维是后向 LSTM 输出

# 分离前向和后向
forward_output = output[:, :, :256]     # 前 256 维
backward_output = output[:, :, 256:]    # 后 256 维
```

**RNN vs LSTM vs GRU 的选择**：

| 特性 | RNN | LSTM | GRU |
|------|-----|------|-----|
| 门控机制 | 无 | 输入门/遗忘门/输出门 | 重置门/更新门 |
| 长期依赖 | 差（梯度消失） | 好 | 好 |
| Cell State | 无 | 有 | 无（合并到 hidden） |
| 参数量 | 少 | 多（~4x RNN） | 中（~3x RNN） |
| 训练速度 | 快 | 慢 | 中等 |
| 推荐度 | 几乎不推荐 | 经典选择 | 参数更少，效果接近 LSTM |

### 2.7 归一化层

#### BatchNorm

```python
# BatchNorm1d: 对全连接层的输出归一化
bn1d = nn.BatchNorm1d(num_features=128)

# 输入: (batch, features) 或 (batch, features, L)
x = torch.randn(32, 128)
y = bn1d(x)  # (32, 128) — 沿 batch 维度计算均值/方差

# BatchNorm2d: 对卷积层的输出归一化
bn2d = nn.BatchNorm2d(num_features=64)
x = torch.randn(8, 64, 32, 32)  # (batch, channels, H, W)
y = bn2d(x)                      # (8, 64, 32, 32)
# 在 (batch, H, W) 维度上计算均值和方差，每个通道独立
```

**BatchNorm 的计算**：

```
对于每个通道 c：
    mean_c = mean(x[:, c, :, :])   # 沿 (batch, H, W)
    var_c  = var(x[:, c, :, :])    # 沿 (batch, H, W)
    x_hat[:, c, :, :] = (x[:, c, :, :] - mean_c) / sqrt(var_c + eps)
    y[:, c, :, :] = gamma_c * x_hat[:, c, :, :] + beta_c
```

#### LayerNorm

```python
ln = nn.LayerNorm(normalized_shape=512)
x = torch.randn(32, 100, 512)     # (batch, seq, features)
y = ln(x)                          # (32, 100, 512)
# 沿最后一个维度 (features) 计算均值和方差
```

**LayerNorm 的计算**：

```
对于每个样本的每个位置：
    mean = mean(x[b, s, :])       # 沿 feature 维度
    var  = var(x[b, s, :])        # 沿 feature 维度
    x_hat = (x - mean) / sqrt(var + eps)
    y = gamma * x_hat + beta
```

**BatchNorm vs LayerNorm 的深入对比**：

| 维度 | BatchNorm | LayerNorm |
|------|-----------|-----------|
| 统计维度 | 沿 (batch, H, W) — 对同一通道的所有激活 | 沿 (features) — 对同一样本的所有特征 |
| 每个样本独立 | 否（依赖 batch 中其他样本） | 是 |
| 依赖 batch_size | 强依赖（小 batch 统计量不稳定） | 完全不依赖 |
| 训练/推理差异 | 有（训练用 batch 统计量，推理用 running stats） | 无（训练和推理行为完全一致） |
| 适用场景 | CNN（CV 任务，batch 较大） | Transformer、RNN（NLP 任务） |
| 序列建模 | 不适合变长序列 | 天然适合变长序列 |
| 对小 batch 的鲁棒性 | 差 | 好 |

**为什么 CV 用 BatchNorm，NLP 用 LayerNorm**：

- CV 中：同一个通道的不同空间位置共享相同的统计特性（颜色分布一致），所以可以跨 batch 统计；且 CV 的 batch_size 通常较大（32-256），统计量稳定。
- NLP 中：每个 token 的 embedding 维度内的特征含义完全不同，跨 batch 统计没有意义；且 NLP 中 batch 通常较小（受序列长度限制），batch 统计不稳定。

**其他归一化变体**：

```python
# InstanceNorm: 对每个样本的每个通道独立归一化
# 用于风格迁移等每个样本独立处理的场景
in2d = nn.InstanceNorm2d(num_features=64)

# GroupNorm: 将通道分组，对每组归一化
# 在通道数之间折衷，适用于小 batch 的 CV 任务
gn = nn.GroupNorm(num_groups=32, num_channels=256)
```

### 2.8 Dropout —— 正则化

```python
dropout = nn.Dropout(p=0.5)    # 50% 概率将元素置零

x = torch.ones(10, 20)
# 训练模式
dropout.train()
y_train = dropout(x)
print((y_train == 0).float().mean().item())  # 约 0.5

# 推理模式
dropout.eval()
y_eval = dropout(x)
print((y_eval == 0).float().mean().item())   # 0 — eval 模式下不 dropout
```

Dropout 在训练时随机将一部分神经元输出置零，等效于每次训练一个"子网络"，从而起到集成学习的效果，抑制过拟合。

**Dropout 的自动缩放机制**：

```
训练时：保留的神经元输出 * 1/(1-p) 以保持期望不变
  期望: E[y] = p*0 + (1-p)*(x/(1-p)) = x  (期望保持不变)

推理时：不做任何操作（恒等映射）
```

**常用 Dropout 变体**：

```python
# 标准 Dropout：随机置零
dropout = nn.Dropout(p=0.5)

# Dropout2d：对 (C, H, W) 整个通道随机置零
# 用于 CNN 的通道级 dropout
dropout2d = nn.Dropout2d(p=0.2)

# AlphaDropout：保持均值和方差的 Dropout
# 配合 SELU 激活函数使用
alpha_dropout = nn.AlphaDropout(p=0.1)
```

## 3. 激活函数

激活函数为网络引入非线性。没有激活函数，多层线性变换等价于一个线性变换。

```python
# 各激活函数与其适用场景

relu = nn.ReLU()        # max(0, x) — 最常用，缓解梯度消失，CNN/MLP 首选
gelu = nn.GELU()        # x * Phi(x) — Transformer 标配，比 ReLU 平滑
sigmoid = nn.Sigmoid()  # 1/(1+e^{-x}) → [0,1] — 二分类输出层
tanh = nn.Tanh()        # (e^{2x}-1)/(e^{2x}+1) → [-1,1] — RNN 隐藏层
softmax = nn.Softmax(dim=-1)  # exp(x_i)/sum(exp(x_j)) — 多分类输出
```

### 选择指南

| 激活函数 | 输出范围 | 适用场景 | 注意事项 |
|----------|----------|----------|----------|
| ReLU | [0, +inf) | CNN/MLP 隐藏层 | 输入为负时梯度为 0（dying ReLU） |
| LeakyReLU | (-inf, +inf) | 替代 ReLU，缓解 dying | 负半轴有小斜率(默认 0.01) |
| GELU | (-inf, +inf) | Transformer、BERT/GPT | 比 ReLU 计算稍慢 |
| Sigmoid | (0, 1) | 二分类输出、门控 | 饱和区梯度接近 0，隐藏层不推荐 |
| Tanh | (-1, 1) | RNN 隐藏层 | 同 Sigmoid 有饱和问题 |
| Softmax | (0, 1) 且和为 1 | 多分类输出层 | 仅在输出层使用 |
| SiLU/Swish | (-inf, +inf) | 某些高效 CNN | x * sigmoid(x) |
| ELU | (-alpha, +inf) | 缓解 dying ReLU | 负半轴平滑，计算稍慢 |
| SELU | 自归一化 | 配合特定初始化 | 要求 LecunNormal 初始化 + AlphaDropout |

### ReLU 变体详解

```python
x = torch.tensor([-3.0, -1.0, 0.0, 1.0, 3.0])

# ReLU: max(0, x)
print(f"ReLU:      {nn.ReLU()(x)}")       # [0., 0., 0., 1., 3.]

# LeakyReLU: max(alpha*x, x)
print(f"LeakyReLU: {nn.LeakyReLU(0.1)(x)}")  # [-0.3, -0.1, 0., 1., 3.]

# PReLU: 可学习的 alpha
prelu = nn.PReLU()
print(f"PReLU:     {prelu(x)}")           # alpha 可学习

# ELU: x if x>0 else alpha*(exp(x)-1)
print(f"ELU:       {nn.ELU()(x)}")        # 负半轴平滑

# GELU: x * Phi(x)，其中 Phi 是标准正态 CDF
print(f"GELU:      {nn.GELU()(x)}")
# GELU 近似: 0.5*x*(1 + tanh(sqrt(2/pi)*(x + 0.044715*x^3)))
```

```python
# 展示 dying ReLU 问题
x = torch.tensor([-1.0, 0.0, 1.0], requires_grad=True)
y = nn.ReLU()(x)
y.sum().backward()
print(x.grad)  # tensor([0., 0., 1.]) — 负输入梯度为 0
# 如果大量神经元长期输出负值，其梯度始终为 0，参数不再更新 —— "死亡"
```

## 4. 损失函数

损失函数衡量模型预测与真实标签之间的差距，是训练优化的目标。

### 4.1 `nn.MSELoss` —— 回归

```python
criterion = nn.MSELoss()

pred = torch.tensor([2.5, 0.0, 2.1])
target = torch.tensor([3.0, -0.5, 2.0])

loss = criterion(pred, target)
# loss = mean((pred - target)^2) = mean([0.25, 0.25, 0.01]) = 0.17
print(loss)
```

适用：房价预测、温度预测等连续值回归任务。

**MSE 的变体**：

```python
# L1Loss / MAE: 平均绝对误差（对异常值更鲁棒）
mae = nn.L1Loss()(pred, target)
print(f"MAE: {mae}")

# SmoothL1Loss: Huber Loss，结合 MSE 和 MAE 的优点
# 在误差小时用 MSE（平滑），误差大时用 MAE（鲁棒）
smooth_l1 = nn.SmoothL1Loss()(pred, target)
print(f"SmoothL1: {smooth_l1}")
```

### 4.2 `nn.CrossEntropyLoss` —— 多分类

```python
criterion = nn.CrossEntropyLoss()

# 输入：logits（未经过 softmax 的原始输出），shape (batch, num_classes)
# 标签：类别索引，shape (batch,) — 不是 one-hot!
logits = torch.randn(4, 10)    # 4 个样本，10 分类
labels = torch.tensor([2, 5, 1, 9])  # 真实类别

loss = criterion(logits, labels)
print(loss)
```

> CrossEntropyLoss 内部已经包含 `LogSoftmax` + `NLLLoss`。**不要在模型最后一层加 Softmax**，直接输出 logits 即可。

**CrossEntropyLoss 内部拆解**：

```python
# CrossEntropyLoss 等价于 LogSoftmax + NLLLoss
logits = torch.randn(4, 10)
labels = torch.tensor([2, 5, 1, 9])

# 方式 1: 直接用 CrossEntropyLoss
loss1 = nn.CrossEntropyLoss()(logits, labels)

# 方式 2: 手动等价实现
log_probs = nn.LogSoftmax(dim=1)(logits)  # log(softmax(logits))
loss2 = nn.NLLLoss()(log_probs, labels)   # 取标签对应位置的负对数概率

print(f"CrossEntropyLoss: {loss1:.4f}")
print(f"LogSoftmax+NLL:   {loss2:.4f}")
print(f"一致: {torch.allclose(loss1, loss2)}")  # True

# 手算验证（对第一个样本）
logit_sample = logits[0]
# Softmax: p_i = exp(logit_i) / sum_j(exp(logit_j))
probs = torch.softmax(logit_sample, dim=0)
# CrossEntropy: -log(p_label)
label = labels[0].item()
manual_loss = -torch.log(probs[label])
print(f"手动计算 loss: {manual_loss:.4f}")
print(f"NLLLoss 第一样本: {log_probs[0, label]:.4f}")
```

**类别不平衡的 `weight` 参数**：

```python
# 假设类别 2 样本很少，给更大权重
class_weights = torch.tensor([1.0, 1.0, 5.0])  # 增大类别 2 的权重
criterion = nn.CrossEntropyLoss(weight=class_weights)

logits = torch.randn(4, 3)
labels = torch.tensor([0, 1, 2, 2])

loss = criterion(logits, labels)
# 类别 2 的预测错误会受到更大的惩罚
```

### 4.3 `nn.BCEWithLogitsLoss` —— 二分类 / 多标签分类

```python
criterion = nn.BCEWithLogitsLoss()

# 二分类
logits = torch.randn(8)              # 8 个样本的 logits（标量）
labels = torch.randint(0, 2, (8,)).float()  # 0 或 1
loss = criterion(logits, labels)

# 多标签分类（每个样本可同时属于多个类别）
logits = torch.randn(8, 5)           # 8 个样本，5 个标签
labels = torch.randint(0, 2, (8, 5)).float()  # 每个标签独立
loss = criterion(logits, labels)
```

> 二分类场景优先使用 `BCEWithLogitsLoss` 而非 `BCELoss` + Sigmoid。前者使用 log-sum-exp 技巧，数值更稳定。

**BCEWithLogitsLoss vs BCELoss 数值稳定性对比**：

```python
# 极端 logit 值下的数值稳定性
extreme_logits = torch.tensor([-100.0, 100.0])
labels = torch.tensor([0.0, 1.0])

# BCEWithLogitsLoss: 数值稳定
loss_stable = nn.BCEWithLogitsLoss()(extreme_logits, labels)
print(f"BCEWithLogitsLoss: {loss_stable:.4f}")  # 50.0

# BCELoss + Sigmoid: log(0) 和 log(1) 导致数值问题
probs = torch.sigmoid(extreme_logits)  # [~0, ~1]
loss_unstable = nn.BCELoss()(probs, labels)
print(f"BCELoss: {loss_unstable:.4f}")
# -log(0) ≈ inf, log(1-1) = log(0) ≈ -inf
# BCELoss 内部有 eps 保护，但 BCEWithLogitsLoss 更稳健
```

### 损失函数速查表

| 任务 | 损失函数 | 模型最后一层 | 标签格式 |
|------|---------|-------------|---------|
| 回归 | MSELoss | 无激活 | 连续值 float32 |
| 回归（抗异常值） | L1Loss / SmoothL1Loss | 无激活 | 连续值 float32 |
| 二分类 | BCEWithLogitsLoss | 无激活（1 个 logit） | 0/1 float32 |
| 多分类 | CrossEntropyLoss | 无激活（N 个 logits） | 类别索引 int64 |
| 多标签 | BCEWithLogitsLoss | 无激活（N 个 logits） | 0/1 float32 (多维) |
| 排序/度量学习 | TripletMarginLoss | 特征向量 | 三元组 (anchor, pos, neg) |
| KL 散度 | KLDivLoss | LogSoftmax | 概率分布 |
| 对比学习 | CrossEntropyLoss (temperature scaled) | 特征向量 | 正样本对 |

## 5. 参数初始化

参数的初始值对训练收敛速度和最终性能有显著影响。PyTorch 默认使用 `kaiming_uniform_` 初始化 Linear 和 Conv2d 的权重，bias 初始化为 0。

### 5.1 常用初始化方法

```python
import torch.nn.init as init

linear = nn.Linear(100, 50)

# Xavier/Glorot 初始化
# 公式：W ~ U[-sqrt(6/(fan_in+fan_out)), sqrt(6/(fan_in+fan_out))]
# 适合 sigmoid/tanh 激活，保持前向和反向激活方差一致
init.xavier_uniform_(linear.weight)
init.xavier_normal_(linear.weight)
# normal 版本：W ~ N(0, 2/(fan_in + fan_out))

# Kaiming/He 初始化
# 公式：W ~ N(0, sqrt(2/fan_in))  (fan_in mode)
# 适合 ReLU 及其变体（因为 ReLU 丢掉了一半信号，需要额外方差）
init.kaiming_uniform_(linear.weight, nonlinearity='relu')
init.kaiming_normal_(linear.weight, nonlinearity='relu')
# mode='fan_in': 方差 = 2/fan_in (前向传播角度)
# mode='fan_out': 方差 = 2/fan_out (反向传播角度)

# 正交初始化（对 RNN/LSTM 有益，保持特征空间的正交性）
init.orthogonal_(linear.weight)

# 常量初始化
init.constant_(linear.weight, 0.0)
init.constant_(linear.bias, 0.01)

# 正态/均匀分布
init.normal_(linear.weight, mean=0.0, std=0.02)
init.uniform_(linear.weight, a=-0.1, b=0.1)

# 将 bias 置零（常见做法）
init.zeros_(linear.bias)
```

### 5.2 各初始化方法的数学公式

| 方法 | 分布 | 参数 | 适用激活 |
|------|------|------|----------|
| Xavier Uniform | U[-a, a], a = sqrt(6/(fan_in+fan_out)) | 基于输入输出维度 | Sigmoid, Tanh |
| Xavier Normal | N(0, 2/(fan_in+fan_out)) | 基于输入输出维度 | Sigmoid, Tanh |
| Kaiming Uniform | U[-a, a], a = sqrt(6/fan_in) | fan_in (ReLU 修正) | ReLU, LeakyReLU |
| Kaiming Normal | N(0, 2/fan_in) | fan_in (ReLU 修正) | ReLU, LeakyReLU |
| Orthogonal | 正交矩阵 | - | RNN/LSTM |
| Lecun Normal | N(0, 1/fan_in) | fan_in | SELU |

### 5.3 自定义初始化函数

```python
def init_weights(m):
    """对模型所有层应用自定义初始化"""
    if isinstance(m, nn.Linear):
        init.xavier_uniform_(m.weight)
        if m.bias is not None:
            init.zeros_(m.bias)
    elif isinstance(m, nn.Conv2d):
        init.kaiming_normal_(m.weight, mode='fan_out', nonlinearity='relu')
        if m.bias is not None:
            init.zeros_(m.bias)
    elif isinstance(m, nn.LSTM):
        for name, param in m.named_parameters():
            if 'weight' in name:
                init.orthogonal_(param)     # 正交初始化对 RNN 有益
            elif 'bias' in name:
                init.zeros_(param)
    elif isinstance(m, (nn.BatchNorm1d, nn.BatchNorm2d, nn.LayerNorm)):
        if m.weight is not None:
            init.ones_(m.weight)            # gamma 初始化为 1
        if m.bias is not None:
            init.zeros_(m.bias)             # beta 初始化为 0

model = MyModel()
model.apply(init_weights)   # 递归应用到所有子模块
```

### 5.4 为什么需要正确的初始化

不合理的初始化（如全零或过大/过小的随机值）会导致：
- **梯度消失**：权值太小，反向传播梯度逐层衰减，深层几乎不更新
- **梯度爆炸**：权值太大，反向传播梯度逐层放大，训练不稳定
- Xavier 和 Kaiming 初始化通过将权重的方差控制在 `1/fan_in` 量级，使各层输出的方差保持一致，缓解了上述问题。

**初始化不当的典型症状**：

```python
# 症状 1: 全零初始化 -> 所有神经元梯度相同 -> 等价于一个神经元
# 症状 2: 权值太大 (std=10) -> 激活值饱和 -> 梯度接近 0 -> 训练停滞
# 症状 3: 权值太小 (std=0.001) -> 深层输出接近 0 -> 梯度消失

# 验证初始化效果的简单方法
def check_activation_stats(model, input_size):
    """检查模型各层激活值的均值和标准差"""
    x = torch.randn(32, *input_size)
    for name, module in model.named_modules():
        if isinstance(module, (nn.Linear, nn.Conv2d)):
            x = module(x)
            print(f"{name:20s} | mean={x.mean():+.4f} | std={x.std():.4f} | "
                  f"min={x.min():+.4f} | max={x.max():+.4f}")
    # 理想的输出：std 应该在合理范围内，不会每层衰减或膨胀
```

## 6. 构建网络的标准写法

### 6.1 多层感知机 (MLP)

```python
class MLP(nn.Module):
    """三层全连接网络，带 Dropout 和 BatchNorm"""
    def __init__(self, input_dim, hidden_dim, output_dim, dropout=0.2):
        super().__init__()
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        self.bn1 = nn.BatchNorm1d(hidden_dim)
        self.relu = nn.ReLU()
        self.dropout = nn.Dropout(dropout)

        self.fc2 = nn.Linear(hidden_dim, hidden_dim)
        self.bn2 = nn.BatchNorm1d(hidden_dim)

        self.fc3 = nn.Linear(hidden_dim, output_dim)

    def forward(self, x):
        x = self.fc1(x)
        x = self.bn1(x)
        x = self.relu(x)
        x = self.dropout(x)

        x = self.fc2(x)
        x = self.bn2(x)
        x = self.relu(x)
        x = self.dropout(x)

        x = self.fc3(x)         # 输出 logits，不加激活
        return x
```

**MLP 中 BatchNorm 的两种顺序（Pre-Norm vs Post-Norm）**：

```python
# Post-Norm (传统): Linear -> BatchNorm -> Activation -> Dropout
# 是目前最常用的顺序

# Pre-Norm: BatchNorm -> Linear -> Activation -> Dropout
# 在某些情况下训练更稳定

# 选择：一般使用 Post-Norm；如果训练不稳定，尝试 Pre-Norm
```

### 6.2 简单 CNN

```python
class SimpleCNN(nn.Module):
    """两个卷积层 + 两个全连接层的 CNN"""
    def __init__(self, num_classes=10):
        super().__init__()
        # 特征提取：卷积层
        self.conv1 = nn.Conv2d(1, 32, kernel_size=3, padding=1)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.pool = nn.MaxPool2d(2, 2)          # 2x2 池化，尺寸减半
        self.relu = nn.ReLU()

        # 分类头：全连接层
        self.fc1 = nn.Linear(64 * 7 * 7, 128)   # 28 -> 14 -> 7 (2次池化)
        self.fc2 = nn.Linear(128, num_classes)
        self.dropout = nn.Dropout(0.25)

    def forward(self, x):
        # 输入: (batch, 1, 28, 28)
        x = self.pool(self.relu(self.conv1(x)))  # (batch, 32, 14, 14)
        x = self.pool(self.relu(self.conv2(x)))  # (batch, 64, 7, 7)
        x = x.view(x.size(0), -1)                # 展平: (batch, 64*7*7)
        x = self.dropout(self.relu(self.fc1(x))) # (batch, 128)
        x = self.fc2(x)                          # (batch, num_classes)
        return x
```

**更深入的 CNN 构建模式**：

```python
class CNNBlock(nn.Module):
    """可复用的 CNN 模块：Conv -> BN -> ReLU -> Pool"""
    def __init__(self, in_c, out_c, kernel=3, pool=True):
        super().__init__()
        self.conv = nn.Conv2d(in_c, out_c, kernel, padding=kernel//2)
        self.bn = nn.BatchNorm2d(out_c)
        self.relu = nn.ReLU(inplace=True)
        self.pool = nn.MaxPool2d(2) if pool else nn.Identity()

    def forward(self, x):
        return self.pool(self.relu(self.bn(self.conv(x))))


class ModularCNN(nn.Module):
    """模块化 CNN，易于修改深度"""
    def __init__(self, in_channels, num_classes, channel_list=[32, 64, 128]):
        super().__init__()
        self.blocks = nn.ModuleList()
        prev_c = in_channels
        for c in channel_list:
            self.blocks.append(CNNBlock(prev_c, c))
            prev_c = c

        # 计算展平后大小（假设输入 32x32，每层池化减半）
        final_size = 32 // (2 ** len(channel_list))
        self.fc = nn.Linear(channel_list[-1] * final_size * final_size,
                             num_classes)

    def forward(self, x):
        for block in self.blocks:
            x = block(x)
        x = x.view(x.size(0), -1)
        return self.fc(x)
```

### 6.4 带残差连接的网络

```python
class ResidualBlock(nn.Module):
    """残差块：y = F(x) + x"""
    def __init__(self, dim, dropout=0.1):
        super().__init__()
        self.fc1 = nn.Linear(dim, dim)
        self.fc2 = nn.Linear(dim, dim)
        self.relu = nn.ReLU()
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        identity = x                               # 保存捷径输入
        out = self.fc1(x)
        out = self.relu(out)
        out = self.dropout(out)
        out = self.fc2(out)
        out = out + identity                       # 残差连接
        out = self.relu(out)
        return out

# 堆叠多个残差块
class ResMLP(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim, num_blocks=3):
        super().__init__()
        self.input_proj = nn.Linear(input_dim, hidden_dim)  # 维度对齐
        self.blocks = nn.Sequential(*[ResidualBlock(hidden_dim) for _ in range(num_blocks)])
        self.output_proj = nn.Linear(hidden_dim, output_dim)

    def forward(self, x):
        x = self.input_proj(x)
        x = self.blocks(x)
        x = self.output_proj(x)
        return x
```

残差连接的核心思想：网络学习的是**残差** `F(x)` 而非直接的目标映射 `H(x)`，即 `H(x) = F(x) + x`。当 `F(x) = 0` 时，信息可以直接沿捷径传递（恒等映射），这使得训练极深网络成为可能——即使深层梯度接近 0，信息也能通过捷径反向传播。

**残差连接的梯度流动分析**：

```
y = F(x) + x
dy/dx = dF/dx + 1

即使 dF/dx 接近 0（梯度消失），+1 确保梯度至少为 1，
不会完全消失。这是残差网络能训练 100+ 层的根本原因。
```

**残差连接维度不匹配时的处理**：

```python
class ResidualBlockWithProjection(nn.Module):
    """支持维度变化的残差块"""
    def __init__(self, in_dim, out_dim):
        super().__init__()
        self.fc1 = nn.Linear(in_dim, out_dim)
        self.fc2 = nn.Linear(out_dim, out_dim)
        self.relu = nn.ReLU()

        # 如果输入输出维度不同，需要做投影（1x1 conv 或 Linear）
        self.projection = None
        if in_dim != out_dim:
            self.projection = nn.Linear(in_dim, out_dim, bias=False)

    def forward(self, x):
        identity = x
        if self.projection is not None:
            identity = self.projection(x)  # 维度对齐

        out = self.relu(self.fc1(x))
        out = self.fc2(out)
        out = out + identity
        return self.relu(out)
```

## 8. Hooks：调试与特征提取

Hooks 是 PyTorch 提供的一种机制，允许在 forward 或 backward 过程中插入自定义逻辑，而不修改模型代码。非常适用于调试、特征可视化和梯度监控。

### 8.1 Forward Hook

```python
# Forward Hook: 在某个层的 forward 之后执行
activation_cache = {}

def save_activation_hook(name):
    """返回一个 hook 函数，将激活值保存到全局字典"""
    def hook(module, input, output):
        activation_cache[name] = output.detach()
        print(f"[Hook] {name}: input shape {input[0].shape} -> output shape {output.shape}")
    return hook

model = SimpleCNN()
# 注册 hook 到特定层
handle1 = model.conv1.register_forward_hook(save_activation_hook("conv1"))
handle2 = model.conv2.register_forward_hook(save_activation_hook("conv2"))

# 运行前向传播
x = torch.randn(1, 1, 28, 28)
output = model(x)

# 获取中间激活值
print(f"\nCached activations: {list(activation_cache.keys())}")
print(f"conv1 output shape: {activation_cache['conv1'].shape}")  # (1, 32, 14, 14)

# 使用完毕后移除 hook（避免内存泄漏）
handle1.remove()
handle2.remove()
```

### 8.2 Backward Hook

```python
# Backward Hook: 在某个层的 backward 之后执行
grad_cache = {}

def save_gradient_hook(name):
    """返回一个 hook 函数，保存梯度信息"""
    def hook(module, grad_input, grad_output):
        # grad_input: 对模块输入的梯度（tuple）
        # grad_output: 对模块输出的梯度（tuple）
        if grad_output[0] is not None:
            grad_cache[name] = grad_output[0].detach()
            print(f"[Backward Hook] {name}: grad_output shape {grad_output[0].shape}, "
                  f"mean={grad_output[0].mean():.6f}, std={grad_output[0].std():.6f}")
    return hook

model = SimpleCNN()
handle = model.conv2.register_full_backward_hook(save_gradient_hook("conv2"))
# 注：register_backward_hook 已废弃，使用 register_full_backward_hook (PyTorch 1.9+)

# 运行前向+反向
x = torch.randn(1, 1, 28, 28)
output = model(x)
loss = output.sum()
loss.backward()
# [Backward Hook] conv2: grad_output shape (1, 64, 7, 7), mean=...

handle.remove()
```

### 8.3 Tensor Hook（监控梯度）

```python
def nan_check_hook(name):
    """检查梯度是否包含 NaN/Inf"""
    def hook(grad):
        if grad is not None:
            if torch.isnan(grad).any():
                print(f"[WARNING] NaN detected in {name}'s gradient!")
            if torch.isinf(grad).any():
                print(f"[WARNING] Inf detected in {name}'s gradient!")
            # 可以返回修改后的梯度（或 None 表示不修改）
        return grad
    return hook

# 对参数注册 hook
for name, param in model.named_parameters():
    if param.requires_grad:
        param.register_hook(nan_check_hook(name))
```

### 8.4 Hooks 的实际应用场景

| 场景 | Hook 类型 | 用途 |
|------|-----------|------|
| 特征可视化 | Forward Hook | 提取 CNN 中间层的特征图并可视化 |
| 梯度监控 | Backward Hook / Tensor Hook | 检测梯度消失/爆炸 |
| 知识蒸馏 | Forward Hook | 提取教师网络中间层输出作为监督信号 |
| 模型剪枝 | Forward Hook | 评估各层的激活值分布，决定剪枝策略 |
| 调试 NaN | Tensor Hook | 精确定位哪个参数的梯度首先出现 NaN |
| Grad-CAM | Forward + Backward Hook | 计算类激活图，可视化模型决策依据 |

## 常见错误

### 错误 1: 忘记调用 `super().__init__()`

```python
# 错误
class MyModel(nn.Module):
    def __init__(self):
        # 缺少 super().__init__()
        self.fc = nn.Linear(10, 5)  # 不会报错

model = MyModel()
model.to("cuda")  # 参数可能不会正确注册到 GPU!

# 正确
class MyModel(nn.Module):
    def __init__(self):
        super().__init__()          # 必须调用！
        self.fc = nn.Linear(10, 5)
```

### 错误 2: CrossEntropyLoss 的标签用了 one-hot

```python
# 错误
criterion = nn.CrossEntropyLoss()
logits = torch.randn(4, 10)
labels = torch.eye(10)[[0, 1, 2, 3]]  # one-hot, shape (4, 10)
loss = criterion(logits, labels)       # RuntimeError！

# 正确
labels = torch.tensor([0, 1, 2, 3])    # 类别索引, shape (4,)
loss = criterion(logits, labels)
```

### 错误 3: 在模型末尾加了 Softmax + CrossEntropyLoss

```python
# 错误
class BadModel(nn.Module):
    def forward(self, x):
        return nn.Softmax(dim=1)(self.fc(x))  # 多了一层 softmax

criterion = nn.CrossEntropyLoss()
# loss 数值可能不对，因为 CrossEntropyLoss 内部还会再做 softmax

# 正确
class GoodModel(nn.Module):
    def forward(self, x):
        return self.fc(x)  # 直接输出 logits，不加 softmax
```

### 错误 4: 忘记将模型移到 GPU

```python
# 错误
model = MyModel()
x = torch.randn(32, 10).cuda()
y = model(x)   # RuntimeError: 参数在 CPU，输入在 GPU！

# 正确
model = MyModel().cuda()
x = torch.randn(32, 10).cuda()
y = model(x)
```

### 错误 5: `view` 前未计算正确的展平尺寸

```python
# 错误：硬编码尺寸，换输入尺寸就崩溃
class CNN(nn.Module):
    def forward(self, x):
        x = self.conv(x)
        x = x.view(32, -1)   # 硬编码 batch_size!

# 错误做法：硬编码 batch_size
x = x.view(32, 64 * 7 * 7)

# 正确做法
x = x.view(x.size(0), -1)   # -1 自动推断展平后大小
```

### 错误 6: BatchNorm1d 输入维度不匹配

```python
# 错误
bn = nn.BatchNorm1d(128)
x = torch.randn(32, 50, 128)  # 3D tensor (batch, seq_len, features)
y = bn(x)  # 错误：BatchNorm1d 期望 (batch, features) 或 (batch, features, L)

# 正确：将 (batch, seq_len, features) reshape 为 (batch, features, seq_len)
x = x.permute(0, 2, 1)        # (32, 128, 50)
y = bn(x)                      # 现在 128 在 num_features 位置
y = y.permute(0, 2, 1)        # 再转回来 (32, 50, 128)
```

### 错误 7: 将不可训练的量定义为 `nn.Parameter`

```python
# 错误
class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.register_buffer("running_mean", torch.zeros(128))
        # 正确：buffer 不会被 optimizer 更新

        self.running_mean = nn.Parameter(torch.zeros(128))
        # 错误：Parameter 会被 optimizer 更新（除非显式设置 requires_grad=False）

# 正确区分：
# nn.Parameter:  需要梯度、需要被 optimizer 更新的量（如权重、偏置）
# register_buffer: 需要随模型保存/加载但不需要梯度的量（如 BatchNorm 的 running_mean）
```

### 错误 8: 使用普通 Python list 存储子模块

```python
# 错误
class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.layers = [nn.Linear(10, 20), nn.Linear(20, 5)]
        # 参数不会被注册！model.parameters() 返回空！

# 正确
class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.layers = nn.ModuleList([
            nn.Linear(10, 20),
            nn.Linear(20, 5)
        ])
        # 参数被正确注册
```

## 本章小结

- `nn.Module` 是所有网络组件的基类，`__init__` 定义层，`forward` 定义数据流
- **Linear** `(in, out)` 做矩阵乘法，权重形状 `(out, in)`；**Conv2d** `(in_c, out_c, k, s, p)` 做空间卷积
- 1x1 卷积用于通道数升降维；两个 3x3 卷积的等效感受野等于一个 5x5 卷积
- LSTM 通过输入门、遗忘门、输出门控制信息流动，hidden state 是短期记忆，cell state 是长期记忆
- BatchNorm 沿 batch 维度归一化，适合 CNN/CV；LayerNorm 沿 feature 维度归一化，适合 Transformer/NLP
- ReLU 是 CNN/MLP 隐藏层首选，GELU 是 Transformer 标配；LeakyReLU 缓解 dying ReLU
- **CrossEntropyLoss 输入 logits**（不加 softmax），标签是类别索引（不是 one-hot）；BCEWithLogitsLoss 用于二分类/多标签
- CrossEntropyLoss 等价于 LogSoftmax + NLLLoss；BCEWithLogitsLoss 比 BCELoss+Sigmoid 数值更稳定
- Kaiming 初始化配合 ReLU，Xavier 初始化配合 Sigmoid/Tanh
- `nn.ModuleList` 和 `nn.ModuleDict` 用于动态管理子模块（不能用普通 Python list/dict）
- 参数绑定（Tied Weights）用于自编码器等对称结构
- 残差连接 `y = F(x) + x` 使深层网络训练成为可能，通过 `+1` 保证梯度不会完全消失
- `Sequential` 便捷但无分支能力；复杂模型用 `nn.Module` 子类
- Forward/Backward Hooks 是调试、特征提取和梯度监控的利器
