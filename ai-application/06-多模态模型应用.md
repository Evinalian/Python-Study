## 6.1 多模态 LLM 概述

### 6.1.1 什么是多模态

传统的大语言模型（LLM）只能处理文本——输入是文本 token，输出也是文本 token。**多模态大模型（Multimodal Large Language Model, MLLM）** 打破了这一限制，能够同时理解和生成多种模态的数据：

| 模态 | 输入 | 输出 | 代表模型 |
|------|------|------|----------|
| 文本 | 自然语言 | 自然语言 | GPT-4, Claude 3, DeepSeek-V3 |
| 图像理解 | 图片 + 文本 | 文本 | GPT-4V, Claude 3.5 Sonnet, Gemini 2.0 Flash |
| 语音识别 | 音频 | 文本 | Whisper, Qwen-Audio |
| 语音合成 | 文本 | 音频 | OpenAI TTS, ChatTTS, VITS |
| 视频理解 | 视频 + 文本 | 文本 | Gemini 2.0 Flash, Qwen2.5-VL |
| 文生图 | 文本 | 图像 | DALL-E 3, Stable Diffusion, Midjourney |
| 文生视频 | 文本 | 视频 | Sora, Runway Gen-3, Kling |

### 6.1.2 演进路线

```
文本 LLM (GPT-3, 2020)
  → 指令微调 + RLHF (ChatGPT, 2022)
    → 多模态视觉理解 (GPT-4V, 2023) ─┐
    → 语音交互 (Whisper + TTS, 2022)  │
    → 文生图 (DALL-E 3, 2023) ────────┤
    → 任意模态 → 任意模态 (GPT-4o, 2024) ← 统一多模态
    → 视频理解 + 生成 (Sora, 2024)
```

关键节点：

1. **2020-2022**：纯文本 LLM 时代。GPT-3 证明了 scaling law，ChatGPT 证明了 RLHF 对对齐的有效性。
2. **2022 末 - 2023 初**：语音先成熟。Whisper 开源，ASR 和 TTS 达到实用水平。
3. **2023 中**：视觉理解突破。GPT-4V、Claude 3 支持图片输入，多图对话成为现实。
4. **2024**：全模态统一。GPT-4o 原生多模态，同时处理文本/图片/音频，端到端训练。

### 6.1.3 多模态模型的架构模式

当前主流多模态模型通常采用以下架构之一：

**模式一：投影 + LLM（最主流）**
```
图片 → Vision Encoder (ViT/CLIP) → 视觉特征
文本 → Tokenizer → 文本 Token
视觉特征 → Projection Layer → 视觉 Token → 拼接到文本 Token 序列 → LLM Decoder → 文本输出
```
这种模式将视觉信息"翻译"成 LLM 能理解的 token，然后让 LLM 像处理文本一样处理视觉信息。GPT-4V、LLaVA、Qwen-VL 都采用这种设计。

**模式二：原生多模态（端到端）**
```
任意模态 → 统一 Encoder → 联合表示空间 → 统一 Decoder → 任意模态
```
GPT-4o 宣称是这种方式。输入输出都是同一个模型处理，不需要专门的投影层。

**模式三：工具组合（Agent 模式）**
```
用户输入 → Router → 图片理解工具 / 语音识别工具 / 文生图工具 → 组合输出
```
不依赖单一全能模型，而是用 LLM 作为调度中心，调用专门的单模态工具。本章主要讲解模式三的实践。

## 6.2 视觉理解

### 6.2.1 视觉理解 API 基础

#### GPT-4V / GPT-4o 调用

OpenAI 的视觉模型通过 Chat Completion API 接收图片。图片以 Base64 编码或 URL 形式传入 `content` 数组。

```python
"""
ex_6_1_vision_basic.py: 视觉理解 API 基础调用
演示：发送单张图片给 GPT-4o 并获得文字描述
"""
import os
import base64
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))


def encode_image_to_base64(image_path: str) -> str:
    """
    将本地图片文件编码为 base64 字符串。
    参数:
        image_path: 图片文件的本地路径（支持 PNG、JPEG、WEBP、非动图 GIF）
    返回:
        base64 编码的字符串（不含 data:image/xxx;base64, 前缀）
    """
    with open(image_path, "rb") as f:      # rb 模式读取二进制数据
        image_data = f.read()               # 读取全部字节
    return base64.b64encode(image_data).decode("utf-8")  # bytes → str


def describe_image(image_path: str, prompt: str = "请详细描述这张图片的内容。") -> str:
    """
    调用 GPT-4o 对单张图片进行理解和文字描述。
    参数:
        image_path: 图片路径
        prompt: 给模型的文字指令
    返回:
        模型的文字回复
    """
    # 步骤1: 获取图片的 base64 编码
    base64_image = encode_image_to_base64(image_path)

    # 步骤2: 获取图片的 MIME 类型（根据扩展名判断）
    ext = os.path.splitext(image_path)[1].lower()  # 如 ".png" → "png"
    mime_map = {
        ".png": "image/png",
        ".jpg": "image/jpeg",
        ".jpeg": "image/jpeg",
        ".webp": "image/webp",
        ".gif": "image/gif",
    }
    mime_type = mime_map.get(ext, "image/png")  # 默认按 png 处理

    # 步骤3: 构造 API 请求
    # content 是一个列表，可以混合 text 和 image_url 两种类型的条目
    response = client.chat.completions.create(
        model="gpt-4o",  # 推荐 gpt-4o（比 gpt-4-vision-preview 更快更便宜）
        messages=[
            {
                "role": "user",
                "content": [
                    # 类型1: 文本指令
                    {
                        "type": "text",
                        "text": prompt,
                    },
                    # 类型2: 图片（data URL 格式）
                    {
                        "type": "image_url",
                        "image_url": {
                            # data URL 格式: data:{mime};base64,{base64_string}
                            "url": f"data:{mime_type};base64,{base64_image}",
                            # detail 参数控制图片分辨率:
                            #   "auto" (默认): 模型自动选择
                            #   "low": 低分辨率, 512x512, 成本低（约 85 token）
                            #   "high": 高分辨率, 先缩放到 2048px 短边再切 512px tile, 成本高
                            "detail": "auto",
                        },
                    },
                ],
            }
        ],
        max_tokens=500,  # 限制输出长度，防止长篇大论
    )

    # 步骤4: 提取回复文本
    return response.choices[0].message.content


# ========== 使用示例 ==========
if __name__ == "__main__":
    # 请将 "photo.jpg" 替换为你电脑上实际存在的图片路径
    test_image = "photo.jpg"
    if os.path.exists(test_image):
        result = describe_image(test_image)
        print("模型回复:\n", result)
    else:
        print(f"请将一张图片命名为 {test_image} 放在当前目录后再运行。")
```

**关键概念解读**：

- **data URL**：一种将数据直接嵌入 URL 的格式。`data:image/png;base64,iVBORw0...` 表示这是一张 PNG 图片，后面跟着 base64 编码的二进制数据。浏览器和 API 都支持这种格式。
- **detail 参数**：控制"看得多细"。
  - `auto`：模型根据图片大小自动判断。这是默认值，大多数情况下用它就行。
  - `low`：成本极低（约 85 token），适合只需"大概看一眼"的场景，比如判断图片是否是照片。
  - `high`：会放大做细粒度识别，适合 OCR、读图表、找细节。token 消耗显著增加（一张高清图可能消耗 1000+ token）。

#### Claude Vision 调用

Claude（通过 Anthropic SDK）也支持图片输入，格式略有不同：

```python
"""
ex_6_1_claude_vision.py: 使用 Anthropic Claude 进行视觉理解
演示：Claude Vision API 的调用方式与 OpenAI 的差异
"""
import os
import base64
from anthropic import Anthropic
from dotenv import load_dotenv

load_dotenv()
client = Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))


def claude_describe_image(image_path: str, prompt: str = "请详细描述这张图片。") -> str:
    """
    使用 Claude 进行图片理解。
    Claude 的 API 格式与 OpenAI 不同：
    - 使用 Messages API
    - 图片放在 content 数组中，type 为 "image"
    - source 字段包含 media_type 和 base64 数据
    """
    with open(image_path, "rb") as f:
        image_data = f.read()

    # Claude 的图片格式
    ext = os.path.splitext(image_path)[1].lower()
    media_map = {
        ".png": "image/png",
        ".jpg": "image/jpeg",
        ".jpeg": "image/jpeg",
        ".webp": "image/webp",
        ".gif": "image/gif",
    }
    media_type = media_map.get(ext, "image/png")

    response = client.messages.create(
        model="claude-sonnet-4-20250514",  # Claude 3.5 Sonnet / Claude 4 系列
        max_tokens=500,
        messages=[
            {
                "role": "user",
                "content": [
                    # Claude 的图片类型键名为 "image"
                    {
                        "type": "image",
                        "source": {
                            "type": "base64",
                            "media_type": media_type,
                            "data": base64.b64encode(image_data).decode("utf-8"),
                        },
                    },
                    {"type": "text", "text": prompt},
                ],
            }
        ],
    )
    return response.content[0].text


if __name__ == "__main__":
    test_image = "photo.jpg"
    if os.path.exists(test_image):
        result = claude_describe_image(test_image)
        print("Claude 回复:\n", result)
    else:
        print(f"请将一张图片命名为 {test_image} 放在当前目录后再运行。")
```

### 6.2.2 图片预处理

直接上传原图可能导致：token 消耗过高、传输慢、超出 API 大小限制。在实际应用中，**预处理是必做步骤**。

```python
"""
ex_6_2_image_preprocess.py: 图片预处理工具函数集
演示：压缩、缩放、格式转换、质量调节
"""
import io
import os
from PIL import Image, ImageFilter


# ===================== 工具函数 =====================

def load_image(path: str) -> Image.Image:
    """加载图片并转为 RGB 模式（处理 PNG 有透明通道的情况）"""
    img = Image.open(path)
    if img.mode in ("RGBA", "LA", "P"):
        # 有透明通道或调色板模式 → 转为 RGB
        # 透明部分会用白色背景填充
        # 注意：如果原背景是透明的且需要保留透明度，应考虑转为 RGBA 并调用 convert("RGBA")
        # 但对大多数 API 场景，RGB 就够了
        background = Image.new("RGB", img.size, (255, 255, 255))
        if img.mode == "P":
            img = img.convert("RGBA")
        background.paste(
            img, mask=img.split()[-1] if img.mode == "RGBA" else None
        )
        img = background
    elif img.mode != "RGB":
        img = img.convert("RGB")
    return img


def resize_image(
    img: Image.Image,
    max_short_side: int = 768,
    max_long_side: int = 2048,
) -> Image.Image:
    """
    等比缩放图片。
    策略: 先保证短边不超过 max_short_side，再保证长边不超过 max_long_side。
    这是 OpenAI 推荐的预处理策略。

    参数:
        img: PIL Image 对象
        max_short_side: 短边最大像素（GPT-4V 建议 ≤ 768）
        max_long_side: 长边最大像素（GPT-4V 建议 ≤ 2000）
    返回:
        缩放后的 PIL Image
    """
    w, h = img.size
    short_side = min(w, h)
    long_side = max(w, h)

    # 计算两个约束下的缩放比例，取更严格的那个
    ratio_short = max_short_side / short_side if short_side > max_short_side else 1.0
    ratio_long = max_long_side / long_side if long_side > max_long_side else 1.0
    ratio = min(ratio_short, ratio_long)  # 取更小的比例（缩得更小）

    if ratio < 1.0:  # 确实需要缩小
        new_w = int(w * ratio)
        new_h = int(h * ratio)
        # LANCZOS 是高质量缩放的推荐算法，比 NEAREST/BILINEAR 效果好
        img = img.resize((new_w, new_h), Image.Resampling.LANCZOS)
    return img


def compress_image(
    img: Image.Image,
    target_size_kb: int = 500,
    quality_start: int = 85,
    quality_min: int = 10,
) -> bytes:
    """
    压缩图片到目标文件大小以下（针对 JPEG）。
    通过逐步降低质量参数来逼近目标大小。

    参数:
        img: PIL Image 对象
        target_size_kb: 目标文件大小（KB），默认 500KB
        quality_start: 起始 JPEG 质量（0-100），越高越清晰
        quality_min: 最低质量，低于此值停止压缩
    返回:
        JPEG 格式的字节数据
    """
    target_size = target_size_kb * 1024  # KB → 字节

    for quality in range(quality_start, quality_min - 1, -5):
        buf = io.BytesIO()
        img.save(buf, format="JPEG", quality=quality, optimize=True)
        data = buf.getvalue()
        if len(data) <= target_size:
            return data  # 已达到目标大小

    # 降到最低质量仍未达标，返回最低质量的结果（总比报错好）
    buf = io.BytesIO()
    img.save(buf, format="JPEG", quality=quality_min, optimize=True)
    return buf.getvalue()


def image_to_base64(img: Image.Image, format: str = "JPEG", quality: int = 85) -> str:
    """
    将 PIL Image 对象直接转为 base64 字符串（不经过磁盘文件）。
    这比"先保存为文件再读取再编码"快得多。

    参数:
        img: PIL Image 对象
        format: 输出格式（JPEG/PNG/WEBP）
        quality: 质量参数（仅对 JPEG/WEBP 有效）
    返回:
        base64 字符串
    """
    buf = io.BytesIO()
    # PNG 不支持 quality 参数
    save_kwargs = {"format": format}
    if format.upper() in ("JPEG", "WEBP"):
        save_kwargs["quality"] = quality
    img.save(buf, **save_kwargs)
    return base64.b64encode(buf.getvalue()).decode("utf-8")


# ===================== 完整预处理流水线 =====================

def preprocess_for_vision(
    image_path: str,
    max_short_side: int = 768,
    max_long_side: int = 2048,
    max_size_kb: int = 500,
) -> tuple[str, str]:
    """
    完整的图片预处理流水线：
    1. 加载并统一为 RGB
    2. 等比缩放到合适尺寸
    3. 压缩到目标文件大小
    4. 返回 base64 和 MIME 类型

    返回:
        (base64_string, mime_type)
    """
    img = load_image(image_path)
    img = resize_image(img, max_short_side, max_long_side)
    jpeg_bytes = compress_image(img, target_size_kb=max_size_kb)
    b64 = base64.b64encode(jpeg_bytes).decode("utf-8")
    return b64, "image/jpeg"


# ===================== 测试 =====================
if __name__ == "__main__":
    import base64

    test_img = "photo.jpg"
    if os.path.exists(test_img):
        b64, mime = preprocess_for_vision(test_img)
        print(f"预处理完成: MIME={mime}, Base64 长度={len(b64)} 字符")
        print(f"估算 Base64 体积: {len(b64) * 3 / 4 / 1024:.1f} KB")
        # Base64 编码会使体积膨胀约 33%（因为 3 字节 → 4 字符）
    else:
        print("未找到测试图片 photo.jpg")
```

### 6.2.3 典型应用场景

#### 场景一：图表解读

```python
"""
ex_6_3_chart_reading.py: 让视觉模型解读图表
演示：从图表图片中提取数据、做分析、生成文字摘要
"""
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))


def analyze_chart(image_path: str) -> dict:
    """
    让 GPT-4o 解读图表图片。
    分两步：先让模型描述所见数据，再让它做分析。

    返回:
        字典 {"raw_data": ..., "analysis": ...}
    """
    with open(image_path, "rb") as f:
        import base64
        b64 = base64.b64encode(f.read()).decode("utf-8")

    ext = os.path.splitext(image_path)[1].lower()
    mime = "image/png" if ext == ".png" else "image/jpeg"

    # 第一步：提取图表中的原始数据
    extract_prompt = """请仔细查看这张图表，完成以下任务：
1. 识别图表类型（柱状图/折线图/饼图/散点图/其他）
2. 识别横轴和纵轴的含义及单位
3. 以 JSON 格式返回图表中的关键数据点
请只返回 JSON，不要包含其他文字。"""

    response1 = client.chat.completions.create(
        model="gpt-4o",
        messages=[{
            "role": "user",
            "content": [
                {"type": "text", "text": extract_prompt},
                {"type": "image_url",
                 "image_url": {"url": f"data:{mime};base64,{b64}", "detail": "high"}},
            ],
        }],
        max_tokens=800,
    )
    raw_data = response1.choices[0].message.content

    # 第二步：基于数据做分析
    analysis_prompt = f"""基于以下从图表中提取的数据，提供分析：
{raw_data}

请从以下角度分析：
1. 整体趋势是什么？
2. 有没有异常值或值得注意的数据点？
3. 可能的原因或业务含义是什么？
"""

    response2 = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": analysis_prompt}],
        max_tokens=500,
    )
    analysis = response2.choices[0].message.content

    return {"raw_data": raw_data, "analysis": analysis}


if __name__ == "__main__":
    chart_path = "chart.png"
    if os.path.exists(chart_path):
        result = analyze_chart(chart_path)
        print("=== 原始数据 ===\n", result["raw_data"])
        print("\n=== 分析 ===\n", result["analysis"])
    else:
        print("请准备一张图表图片 chart.png 放在当前目录。")
```

#### 场景二：UI 截图分析

```python
"""
ex_6_4_ui_screenshot.py: 通过截图分析 UI 界面
演示：识别页面元素、检测潜在设计问题、生成改进建议
"""
import os
import base64
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))


def analyze_ui_screenshot(image_path: str) -> str:
    """
    分析 UI 截图，返回结构化的界面分析报告。
    适用于：设计审查、可访问性检查、竞品分析。
    """
    with open(image_path, "rb") as f:
        b64 = base64.b64encode(f.read()).decode("utf-8")

    ext = os.path.splitext(image_path)[1].lower()
    mime = "image/png" if ext == ".png" else "image/jpeg"

    # 精心设计的 prompt，引导模型生成结构化输出
```

### 6.2.5 Visual Token 成本估算

视觉理解的 Token 计算相对复杂。以下是 OpenAI 的计算规则（截至 2025 年）：

```python
"""
ex_6_7_vision_token_cost.py: 估算视觉 API 调用的 Token 消耗与成本
演示：在调用 API 前预估 cost，帮助控制预算
"""

def estimate_image_tokens(width: int, height: int, detail: str = "auto") -> int:
    """
    根据 OpenAI 官方规则估算一张图片消耗的 Token 数。

    规则摘要（OpenAI 文档，2025）：
    - detail="low": 固定 85 tokens/张
    - detail="high":
        1. 缩放到 2048px 短边以内
        2. 切成 512x512 的 tile
        3. 每 tile 170 tokens
        4. 外加 85 tokens 基础费用
    - detail="auto": 如果图片任一维度 > 512 且短边 > 768 则按 high 处理，否则按 low
    """
    if detail == "low":
        return 85

    # detail = "high" 或 "auto"（auto 对大多数图片按 high 处理）
    # 第一步：缩放到 2048px 短边以内
    max_size = 2048
    if width > max_size or height > max_size:
        ratio = max_size / max(width, height)
        width = int(width * ratio)
        height = int(height * ratio)

    # 第二步：缩放到 768px 短边
    short_side = min(width, height)
    if short_side > 768:
        ratio = 768 / short_side
        width = int(width * ratio)
        height = int(height * ratio)

    # 第三步：计算 tile 数
    # 每个 tile 是 512x512 的方形区域
    tiles_width = (width + 511) // 512   # 向上取整
    tiles_height = (height + 511) // 512
    num_tiles = tiles_width * tiles_height

    # 每 tile 170 tokens，加 85 基础 tokens
    return 85 + num_tiles * 170


def estimate_call_cost(
    images: list[tuple[int, int, str]],  # [(width, height, detail), ...]
    text_input_chars: int,
    max_output_tokens: int,
    model: str = "gpt-4o",
) -> dict:
    """
    估算一次视觉 API 调用的总 token 数和美元成本。

    定价（2025年5月估值，实际以 OpenAI 官网为准）：
    - gpt-4o: 输入 $2.50/1M tokens, 输出 $10.00/1M tokens
    - gpt-4o-mini: 输入 $0.15/1M tokens, 输出 $0.60/1M tokens
    - gpt-4.1-mini: 输入 $0.10/1M tokens, 输出 $0.40/1M tokens

    参数:
        images: 每张图片的 (宽, 高, detail) 元组列表
        text_input_chars: 文本提示的字符数
        max_output_tokens: max_tokens 参数的值
        model: 模型名

    返回:
        dict: {"input_tokens": ..., "output_tokens": ..., "cost_usd": ...}
    """
    # 图片 token
    image_tokens = sum(estimate_image_tokens(w, h, d) for w, h, d in images)

    # 文本 token 粗略估算：中文约 1 字符 ≈ 1-2 tokens，英文约 1 字符 ≈ 0.25 tokens
    # 这里用保守估计：每个字符算 1 个 token
    text_tokens = text_input_chars

    input_tokens = image_tokens + text_tokens
    output_tokens = max_output_tokens

    # 定价
    pricing = {
        "gpt-4o": (2.50, 10.00),
        "gpt-4o-mini": (0.15, 0.60),
        "gpt-4.1-mini": (0.10, 0.40),
    }
    input_price_per_m, output_price_per_m = pricing.get(model, (2.50, 10.00))

    cost = (input_tokens / 1_000_000) * input_price_per_m + \
           (output_tokens / 1_000_000) * output_price_per_m

    return {
        "input_tokens": input_tokens,
        "output_tokens_est": output_tokens,
        "cost_usd": round(cost, 6),
    }


if __name__ == "__main__":
    # 示例：一张 1920x1080 的高清截图，detail="high"
    result = estimate_call_cost(
        images=[(1920, 1080, "high")],
        text_input_chars=500,
        max_output_tokens=500,
        model="gpt-4o",
    )
    print(f"输入 tokens: {result['input_tokens']}")
    print(f"预估输出 tokens: {result['output_tokens_est']}")
    print(f"预估成本: ${result['cost_usd']}")
    # 典型结果：约 $0.002 ~ $0.02 / 次
```

### 6.3.2 Stable Diffusion（开源方案）

Stable Diffusion 是 Stability AI 开发的开源文生图模型。相比 DALL-E 3，它的优势在于：

- **精细控制**：负面 prompt、采样器、步数等参数

#### 使用 Diffusers 库调用 Stable Diffusion

```python
"""
ex_6_10_stable_diffusion.py: 使用 HuggingFace Diffusers 调用 Stable Diffusion
演示：本地加载 SD 模型进行文生图

前置安装:
    pip install diffusers transformers accelerate peft
"""
import torch
from diffusers import StableDiffusionPipeline, DPMSolverMultistepScheduler


def load_sd_pipeline(model_id: str = "runwayml/stable-diffusion-v1-5") -> StableDiffusionPipeline:
    """
    加载 Stable Diffusion pipeline。

    参数:
        model_id: HuggingFace 模型 ID，
            - "runwayml/stable-diffusion-v1-5" (经典版，4.3B 参数)
            - "stabilityai/stable-diffusion-2-1"
            - "stabilityai/stable-diffusion-xl-base-1.0" (SDXL，画质更好但更吃显存)
            - "nota-ai/bk-sdm-small" (蒸馏小模型，快但质量略低)

    返回:
        配置好的 pipeline 对象
    """
    # 检测设备
    if torch.cuda.is_available():
        device = "cuda"
        # CUDA 上可用 float16 节省一半显存
        dtype = torch.float16
    else:
        device = "cpu"
        dtype = torch.float32
        print("警告: 未检测到 GPU，将在 CPU 上运行（速度会很慢）")

    # 加载 pipeline
    pipe = StableDiffusionPipeline.from_pretrained(
        model_id,
        torch_dtype=dtype,
        safety_checker=None,  # 跳过 NSFW 检查（加快加载速度；生产环境慎用）
        requires_safety_checker=False,
    )

    # 更换更快的调度器（DPMSolver 比默认 DDPM 快得多）
    pipe.scheduler = DPMSolverMultistepScheduler.from_config(
        pipe.scheduler.config,
        algorithm_type="dpmsolver++",  # 使用 ++ 变体，收敛更快
    )

    pipe = pipe.to(device)

    # 如果显存紧张，启用以下优化:
    # pipe.enable_attention_slicing()     # 分片处理 attention，降低峰值显存
    # pipe.enable_vae_slicing()           # 分片处理 VAE
    # pipe.enable_model_cpu_offload()     # 逐层在 CPU/GPU 间切换（最省显存但慢）

    return pipe


def generate_sd(
    pipe: StableDiffusionPipeline,
    prompt: str,
    negative_prompt: str = "",
    num_inference_steps: int = 25,
    guidance_scale: float = 7.5,
    width: int = 512,
    height: int = 512,
    seed: int | None = None,
    output_path: str = "sd_output.png",
) -> str:
    """
    使用 Stable Diffusion pipeline 生成图片。

    参数:
        pipe: 已加载的 pipeline
        prompt: 正向提示词（英文效果最好）
        negative_prompt: 负向提示词，指定不希望出现的内容
        num_inference_steps: 推理步数（20-30 是常用的平衡点；越大质量越好但越慢）
        guidance_scale: CFG 引导强度（7-9 是常用范围；越大越贴近 prompt 但可能失真）
        width/height: 输出尺寸（必须是 8 的倍数；SD1.5 推荐 512，SDXL 可用 1024）
        seed: 随机种子（指定则结果可复现）
        output_path: 输出路径

    返回:
        保存路径
    """
    # 设置随机种子（保证可复现性）
    if seed is not None:
        generator = torch.Generator(device=pipe.device).manual_seed(seed)
    else:
        generator = None

    # 执行生成
    with torch.no_grad():  # 禁用梯度计算（推理不需要）
        result = pipe(
            prompt=prompt,
            negative_prompt=negative_prompt,
            num_inference_steps=num_inference_steps,
            guidance_scale=guidance_scale,
            width=width,
            height=height,
            generator=generator,
        )

    # 保存图片
    image = result.images[0]
    image.save(output_path)
    print(f"图片已保存到: {output_path}")
    return output_path


if __name__ == "__main__":
    print("正在加载 Stable Diffusion 模型（首次运行需下载约 5GB）...")

        num_inference_steps=30,
        guidance_scale=7.5,
        seed=42,
        output_path="sd_landscape.png",
    )
```

#### ControlNet：精确控制生成

ControlNet 是 Stable Diffusion 的一个扩展，允许你通过额外的控制信号（边缘检测图、深度图、姿态骨架等）精确控制生成结果。

```python
"""
ex_6_11_controlnet.py: ControlNet 精确控制文生图
演示：使用 Canny 边缘检测作为控制条件

前置安装:
    pip install diffusers controlnet_aux opencv-python
"""
import cv2
import torch
import numpy as np
from PIL import Image
from diffusers import StableDiffusionControlNetPipeline, ControlNetModel, UniPCMultistepScheduler
from controlnet_aux import CannyDetector


def load_controlnet_pipeline(
    base_model: str = "runwayml/stable-diffusion-v1-5",
    controlnet_model: str = "lllyasviel/sd-controlnet-canny",
) -> StableDiffusionControlNetPipeline:
    """
    加载 ControlNet pipeline。

    ControlNet 类型说明：
    - lllyasviel/sd-controlnet-canny → Canny 边缘检测：基于边缘构图
    - lllyasviel/sd-controlnet-depth → 深度图：基于深度信息控制空间关系
    - lllyasviel/sd-controlnet-openpose → OpenPose 姿态：控制人物姿势
    - lllyasviel/sd-controlnet-scribble → 涂鸦：从简单草图生成
    - lllyasviel/sd-controlnet-seg → 语义分割：按区域控制内容
    """
    controlnet = ControlNetModel.from_pretrained(
        controlnet_model,
        torch_dtype=torch.float16 if torch.cuda.is_available() else torch.float32,
    )

    pipe = StableDiffusionControlNetPipeline.from_pretrained(
        base_model,
        controlnet=controlnet,
        torch_dtype=torch.float16 if torch.cuda.is_available() else torch.float32,
        safety_checker=None,
    )

    pipe.scheduler = UniPCMultistepScheduler.from_config(pipe.scheduler.config)
    # 如果报错，改用默认调度器: 删除上面这行即可

    device = "cuda" if torch.cuda.is_available() else "cpu"
    pipe = pipe.to(device)

    # 优化显存
    if device == "cuda":
        pipe.enable_model_cpu_offload()

    return pipe


def generate_with_controlnet(
    pipe: StableDiffusionControlNetPipeline,
    control_image_path: str,
    prompt: str,
    negative_prompt: str = "blurry, low quality, distorted",
    canny_low_threshold: int = 100,
    canny_high_threshold: int = 200,
    num_inference_steps: int = 20,
    guidance_scale: float = 7.5,
    controlnet_conditioning_scale: float = 0.8,
    seed: int = 42,
    output_path: str = "controlnet_output.png",
) -> str:
    """
    使用 ControlNet + Canny 边缘检测生成图片。

    工作流程:
        输入图片 → Canny 边缘检测 → 提取边缘图 → ControlNet 用边缘图引导 SD 生成

    参数:
        pipe: ControlNet pipeline
        control_image_path: 控制图片路径（如建筑物照片）
        prompt: 希望生成的画面描述
        canny_low_threshold: Canny 算法低阈值（50-150 常用）
        canny_high_threshold: Canny 算法高阈值（150-250 常用）
        controlnet_conditioning_scale: ControlNet 的控制强度
            - 0.0: 完全不控制（等于普通 SD）
            - 1.0: 最大控制（严格按边缘图生成）
            - 0.5-0.8: 推荐的平衡范围

    返回:
        输出路径
    """
    # 步骤1: 加载控制图片
    control_image = Image.open(control_image_path).convert("RGB")

    # 步骤2: Canny 边缘检测
    # Canny 算法原理简释：
    #   1. 灰度化 + 高斯模糊 → 降噪
    #   2. 计算梯度幅值和方向 → 找边缘
    #   3. 非极大值抑制 → 细化边缘
    #   4. 双阈值检测 → 确定强边缘和弱边缘
    canny = CannyDetector()
    canny_image = canny(
        control_image,
        low_threshold=canny_low_threshold,
        high_threshold=canny_high_threshold,
    )
    canny_image.save("canny_preview.png")  # 保存边缘图供检查
    print("边缘检测预览已保存到 canny_preview.png")

    # 步骤3: ControlNet 生成
    generator = torch.Generator(device=pipe.device).manual_seed(seed)

    with torch.no_grad():
        result = pipe(
            prompt=prompt,
            negative_prompt=negative_prompt,
            image=canny_image,  # 边缘图作为控制条件
            num_inference_steps=num_inference_steps,
            guidance_scale=guidance_scale,
            controlnet_conditioning_scale=controlnet_conditioning_scale,
            generator=generator,
        )

    result.images[0].save(output_path)
    print(f"生成图片已保存到: {output_path}")
    return output_path


if __name__ == "__main__":
    print("加载 ControlNet 模型...")
    pipe = load_controlnet_pipeline()

    # 示例：把建筑照片变成不同风格的建筑
    generate_with_controlnet(
        pipe,
        control_image_path="building.jpg",  # 替换为你的图片
        prompt=(
            "a futuristic cyberpunk skyscraper, neon lights, rain, "
            "blade runner style, highly detailed, 8K"
        ),
        negative_prompt="blurry, low quality, distorted, ugly, watermark",
        controlnet_conditioning_scale=0.7,  # 0.7 保留结构但允许风格变化
        seed=12345,
        output_path="cyberpunk_building.png",
    )
```

## 6.4 语音识别（ASR）

### 6.4.1 OpenAI Whisper API

Whisper 是 OpenAI 开源的通用语音识别模型，支持 99 种语言。API 版本最简单易用。

```python
"""
ex_6_12_whisper_api.py: 使用 OpenAI Whisper API 做语音识别
演示：音频文件转文字，支持多语言和时间戳
"""
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))


def transcribe_audio(
    audio_path: str,
    language: str | None = None,
    prompt: str = "",
    response_format: str = "verbose_json",
    temperature: float = 0.0,
) -> dict:
    """
    使用 Whisper API 将音频转为文字。

    参数:
        audio_path: 音频文件路径。
            支持的格式: mp3, mp4, mpeg, mpga, m4a, wav, webm, flac
            文件大小限制: 25MB（可通过分段解决，见下方）
        language: 音频语言代码（ISO 639-1）。
            如 "zh"=中文, "en"=英文, "ja"=日文。
            设为 None 让模型自动检测（推荐）。
        prompt: 引导文本，提供上下文词汇以提高识别准确率。
            例如: "本次会议讨论的术语包括: Kubernetes, Docker, CI/CD"
        response_format: 输出格式:
            "json": 只有文本 {text: "..."}
            "verbose_json": 含时间戳、语言、置信度等详细信息
            "text": 纯文本
            "srt": SRT 字幕格式
            "vtt": WebVTT 字幕格式
        temperature: 采样温度 (0-1)，0 最确定性，建议语音识别用 0。

    返回:
        verbose_json 格式时，返回完整字典（见下方字段说明）

    定价:
        $0.006 / 分钟（即每小时约 $0.36）
    """
    # 打开音频文件（必须用 rb 模式）
    with open(audio_path, "rb") as audio_file:
        response = client.audio.transcriptions.create(
            model="whisper-1",  # 目前只有这一个版本
            file=audio_file,
            language=language if language else None,  # None 表示自动检测
            prompt=prompt,
            response_format=response_format,
            temperature=temperature,
        )

    if response_format == "verbose_json":
        # verbose_json 返回的字段:
        #   - text: 完整识别文本
        #   - language: 检测到的语言代码
        #   - duration: 音频时长（秒）
        #   - segments: 每个片段的列表，每段含:
        #       - id: 序号
        #       - start: 开始时间（秒）
        #       - end: 结束时间（秒）
        #       - text: 本段文字
        #       - tokens: 本段 token ID 列表
        #       - temperature, avg_logprob, compression_ratio, no_speech_prob
        result = {
            "text": response.text,
            "language": response.language,
            "duration": getattr(response, "duration", 0),
            "segments": [
                {
                    "id": seg.get("id"),
                    "start": seg.get("start"),
                    "end": seg.get("end"),
                    "text": seg.get("text"),
                }
                for seg in (getattr(response, "segments", []) or [])
            ],
        }
        return result
    else:
        return {"text": response if isinstance(response, str) else response.text}


def transcribe_large_audio(
    audio_path: str,
    chunk_duration_minutes: int = 20,
) -> str:
    """
    处理超过 25MB 的大音频文件。
    策略: 用 pydub 将音频切分为多个 20 分钟的片段，逐段识别后拼接。

    前置安装:
        pip install pydub
    """
    from pydub import AudioSegment

    # 加载音频
    audio = AudioSegment.from_file(audio_path)
    total_ms = len(audio)  # 总时长（毫秒）
    chunk_ms = chunk_duration_minutes * 60 * 1000

    full_text_parts = []
    for i, start_ms in enumerate(range(0, total_ms, chunk_ms)):
        end_ms = min(start_ms + chunk_ms, total_ms)
        chunk = audio[start_ms:end_ms]

        # 保存临时片段
        chunk_path = f"_chunk_{i}.mp3"
        chunk.export(chunk_path, format="mp3", bitrate="64k")

        # 识别
        with open(chunk_path, "rb") as f:
            response = client.audio.transcriptions.create(
                model="whisper-1",
                file=f,
                response_format="text",
            )

        full_text_parts.append(response if isinstance(response, str) else response.text)

        # 清理临时文件
        os.remove(chunk_path)
        print(f"片段 {i + 1} 完成")

    return "\n".join(full_text_parts)


if __name__ == "__main__":
    audio_file = "meeting.mp3"
    if os.path.exists(audio_file):
        result = transcribe_audio(
            audio_file,
            language="zh",
            response_format="verbose_json",
        )
        print(f"语言: {result['language']}")
        print(f"时长: {result['duration']:.1f}秒")
        print(f"文本: {result['text'][:200]}...")

        # 打印时间轴
        if result.get("segments"):
            print("\n=== 逐句时间轴 ===")
            for seg in result["segments"]:
                print(f"[{seg['start']:.1f}s - {seg['end']:.1f}s] {seg['text']}")
    else:
        print("请提供音频文件 meeting.mp3。")
```

### 6.4.2 本地 Whisper：faster-whisper

OpenAI 也开源了 Whisper 模型，可以直接在本地运行。`faster-whisper` 是 CTranslate2 加速版，比原版快 4 倍、显存减半。

```python
"""
ex_6_13_local_whisper.py: 使用 faster-whisper 在本地进行语音识别
优点: 免费、离线、不限文件大小、数据不离开本机

前置安装:
    pip install faster-whisper
"""
import os
import time
from faster_whisper import WhisperModel


def transcribe_local(
    audio_path: str,
    model_size: str = "large-v3",
    device: str = "auto",
    compute_type: str = "auto",
    language: str | None = None,
) -> list[dict]:
    """
    使用本地的 faster-whisper 模型做语音识别。

    参数:
        audio_path: 音频文件路径
        model_size: 模型尺寸（从大到小，准确率递减但速度递增）:
            "large-v3" — 最准确但最慢最大（~3GB 显存/内存）
            "medium"   — 中等（~1.5GB）
            "small"    — 较小（~500MB）
            "base"     — 基础（~150MB）
            "tiny"     — 最小最快（~75MB）
        device: "auto" / "cuda" / "cpu"
        compute_type: 计算精度:
            "auto" → 自动选择
            "float16" → GPU 半精度（推荐）
            "int8" → GPU/CPU 量化（显存减半，质量轻微下降）
            "int8_float16" → GPU 混合精度
            "float32" → 全精度（CPU 上的唯一选项）
        language: 语言代码（None 为自动检测）

    返回:
        分段列表，每段含 start, end, text
    """
    # 步骤1: 加载模型
    start_time = time.time()
    model = WhisperModel(
        model_size,
        device=device,
        compute_type=compute_type,
        # download_root="./models",  # 可指定模型下载目录
    )
    print(f"模型加载耗时: {time.time() - start_time:.1f}秒")

    # 步骤2: 执行识别
    # beam_size: 束搜索宽度（越大越准确但越慢，推荐 5）
    # vad_filter: 启用语音活动检测（自动跳过静音段）
    # word_timestamps: 启用词级别时间戳
    segments, info = model.transcribe(
        audio_path,
        language=language,
        beam_size=5,
        vad_filter=True,  # 语音活动检测 (Voice Activity Detection)
        vad_parameters=dict(
            min_silence_duration_ms=500,  # 最小静音时长(ms)来切分句子
        ),
        word_timestamps=False,  # 词级别时间戳（开启会慢一些，按需）
    )

    print(f"检测到的语言: {info.language}（概率: {info.language_probability:.2%}）")
    print(f"音频时长: {info.duration:.1f}秒")

    # 步骤3: 收集结果
    results = []
    full_text = []
    for segment in segments:
        results.append({
            "start": segment.start,
            "end": segment.end,
            "text": segment.text.strip(),
        })
        full_text.append(segment.text.strip())
        # 实时打印（像打字机效果）
        print(f"[{segment.start:6.1f}s -> {segment.end:6.1f}s] {segment.text.strip()}")

    print(f"\n=== 完整文本 ===\n{''.join(full_text)}")
    return results


if __name__ == "__main__":
    audio_file = "meeting.mp3"
    if os.path.exists(audio_file):
        # 第一次运行会下载模型（约 3GB），请耐心等待
        results = transcribe_local(
            audio_file,
            model_size="medium",  # 从 medium 开始试试，看速度和质量平衡
            device="auto",
        )
        print(f"\n共识别 {len(results)} 个片段")
    else:
        print("请提供音频文件 meeting.mp3。")
```

## 6.5 语音合成（TTS）

### 6.5.1 OpenAI TTS API

OpenAI 提供两种 TTS 模型：`tts-1`（速度快、适合实时）和 `tts-1-hd`（质量高、适合录制内容）。

```python
"""
ex_6_14_openai_tts.py: 使用 OpenAI TTS API 将文本转为语音
演示：多种音色、流式输出、自定义速度
"""
import os
from pathlib import Path
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))


def text_to_speech(
    text: str,
    output_path: str = "output.mp3",
    model: str = "tts-1",
    voice: str = "alloy",
    speed: float = 1.0,
) -> str:
    """
    使用 OpenAI TTS API 生成语音。

    参数:
        text: 要合成的文本（最大 4096 字符）
        output_path: 输出音频文件路径
        model: "tts-1"（标准，$15/1M 字符） 或 "tts-1-hd"（高清，$30/1M 字符）
        voice: 音色选择（发音人）:
            - "alloy": 中性、温和的男/女声（模糊性别），通用场景
            - "echo": 沉稳、略显深沉的男声
            - "fable": 英式口音的男声，讲述感强
            - "onyx": 深沉、有力、权威的男声
            - "nova": 温柔、专业、平和的女声
            - "shimmer": 清晰、自信、偏年轻的女声
            - "ash": 新增，自然男声
            - "coral": 新增，自然女声
            - "sage": 新增，青春自然男/女声
        speed: 语速（0.25 - 4.0），1.0 为正常速度

    返回:
        输出文件路径

    支持的语言: 模型会根据输入文本自动识别语言并切换发音。
    中文支持的音色: alloy, echo, nova, shimmer, ash, sage（2025年已验证）。
    """
    # 参数校验
    if len(text) > 4096:
        raise ValueError(f"文本过长({len(text)}字符)。TTS API 限制每次最多 4096 字符。")

    valid_voices = ["alloy", "echo", "fable", "onyx", "nova", "shimmer", "ash", "coral", "sage"]
    if voice not in valid_voices:
        raise ValueError(f"无效音色 '{voice}'，可选: {valid_voices}")

    response = client.audio.speech.create(
        model=model,
        voice=voice,
        input=text,
        speed=speed,
        response_format="mp3",  # 也可选 opus, aac, flac, wav, pcm
    )

    # response.content 是音频二进制数据
    # 或者用 response.stream_to_file() 直接写入文件
    response.stream_to_file(output_path)

    print(f"语音已保存到: {output_path}")
    return output_path


def tts_streaming(
    text: str,
    output_path: str = "output_streaming.mp3",
    voice: str = "nova",
) -> str:
    """
    流式 TTS——边生成边写入文件。
    对于长文本，这比等到全部生成完再保存体验更好。

    注意：OpenAI TTS 的 streaming 指的不是逐字流式输出，
    而是通过 chunked transfer encoding 流式接收整个音频文件。
    真正的逐字流式 TTS 需要用 WebSocket（当前仅 GPT-4o realtime 支持）。
    """
    with client.audio.speech.with_streaming_response.create(
        model="tts-1",
        voice=voice,
        input=text,
        response_format="mp3",
    ) as response:
        response.stream_to_file(output_path)

    print(f"流式语音已保存到: {output_path}")
    return output_path


def long_text_to_speech(
    text: str,
    output_dir: str = "./tts_output",
    voice: str = "nova",
    model: str = "tts-1-hd",
) -> list[str]:
    """
    处理超过 4096 字符的长文本。
    策略: 按句号/换行符智能切分，每段独立合成后保存。

    返回:
        所有输出文件路径的列表
    """
    Path(output_dir).mkdir(exist_ok=True)

    # 按标点切分（保证每段语义完整）
    import re
    # 按句号、问号、感叹号、换行分割
    sentences = re.split(r"(?<=[。！？\n])\s*", text)
    # 过滤空串
    sentences = [s.strip() for s in sentences if s.strip()]

    chunks = []
    current_chunk = ""
    for sentence in sentences:
        if len(current_chunk) + len(sentence) < 4000:
            current_chunk += sentence
        else:
            if current_chunk:
                chunks.append(current_chunk)
            current_chunk = sentence
    if current_chunk:
        chunks.append(current_chunk)

    output_files = []
    for i, chunk in enumerate(chunks):
        output_path = os.path.join(output_dir, f"part_{i:03d}.mp3")
        text_to_speech(chunk, output_path, model=model, voice=voice)
        output_files.append(output_path)

    print(f"长文本已分为 {len(chunks)} 段，保存到 {output_dir}/")
    return output_files


if __name__ == "__main__":
    # 示例 1: 基础 TTS
    text_to_speech(
        "欢迎来到多模态AI的世界！今天我们将学习如何让计算机听懂语音、看懂图片。",
        output_path="welcome.mp3",
        voice="nova",
        speed=1.0,
    )

    # 示例 2: 不同音色对比
    sample_text = "人工智能正在改变我们与机器交互的方式。"
    for voice_name in ["alloy", "echo", "nova", "shimmer"]:
        output = f"tts_{voice_name}.mp3"
        text_to_speech(sample_text, output, voice=voice_name)
        print(f"  音色 {voice_name} → {output}")
```

### 6.5.2 开源 TTS：ChatTTS（中文优秀）

ChatTTS 是专门为对话场景优化的中文 TTS 模型，2014 年在 GitHub 开源后迅速流行。

```python
"""
ex_6_15_chat_tts.py: 使用 ChatTTS 生成中文语音
优点: 免费、自然度高、支持笑声/停顿等韵律控制

前置安装:
    pip install ChatTTS
    # 首次运行会自动下载模型（约 1GB）
"""
import torch
import ChatTTS
from IPython.display import Audio  # Jupyter 中播放，命令行可改用 pygame


def init_chat_tts() -> ChatTTS.Chat:
    """初始化 ChatTTS 模型"""
    chat = ChatTTS.Chat()
    chat.load(
        compile=False,  # 是否编译模型（编译后更快但首次慢）
        source="local",  # "local"=使用本地下载的模型 / "huggingface"=在线下载
    )
    return chat


def generate_chat_tts(
    chat: ChatTTS.Chat,
    texts: list[str],
    output_path: str = "chattts_output.wav",
    voice_seed: int | None = None,
    speed: int = 1,
    oral: int = 3,
    laugh: int = 2,
) -> str:
    """
    使用 ChatTTS 生成语音。

    参数:
        chat: 已初始化的 ChatTTS 实例
        texts: 要合成的文本列表（批量生成效率更高）
        output_path: 输出 wav 文件路径
        voice_seed: 音色种子（相同的 seed 生成相同的音色）。
            取值范围约 0-5000，不同值有不同音色。
        speed: 语速 (1-9)，越小越快
        oral: 口语化程度 (0-9)，越大越口语化（会有"嗯""啊"等语气词）
        laugh: 笑声插入频率 (0-9)，0=无笑声

    返回:
        输出文件路径
    """
    # 生成语音参数
    params_infer_code = ChatTTS.Chat.InferCodeParams(
        prompt=f"[speed_{speed}]",  # 口语化参数通过 prompt 控制
        spk_emb=chat.sample_random_speaker(seed=voice_seed) if voice_seed is not None else None,
        temperature=0.3,
    )

    params_refine_text = ChatTTS.Chat.RefineTextParams(
        prompt=f"[oral_{oral}][laugh_{laugh}]",
    )

    # 执行生成
    wavs = chat.infer(
        texts,
        params_infer_code=params_infer_code,
        params_refine_text=params_refine_text,
        use_decoder=True,
    )

    # 保存音频（合并所有文本段）
    import numpy as np
    import soundfile as sf

    combined = np.concatenate(wavs, axis=1)  # shape: (1, total_samples)
    sf.write(output_path, combined.T, 24000)  # ChatTTS 采样率固定 24000Hz

    print(f"语音已保存到: {output_path}")
    return output_path


if __name__ == "__main__":
    print("正在加载 ChatTTS 模型（首次需下载，请耐心等待）...")
    chat = init_chat_tts()

    # 生成示例
    texts = [
        "大家好，我是ChatTTS，一个开源的对话语音合成模型。[uv_break]",
        "今天天气真不错，适合出去走走。你觉得呢？[laugh]",
    ]
    generate_chat_tts(
        chat,
        texts,
        output_path="chattts_demo.wav",
        voice_seed=42,
        speed=3,  # 中等语速
        oral=2,
        laugh=1,
    )
    print("完成!")
```

## 6.6 视频理解简介

视频理解的工业界主流方案不是端到端模型，而是 **"抽帧 + 图片理解 API"** 的组合。

### 6.6.1 抽帧策略

```python
"""
ex_6_16_video_frame_extraction.py: 视频抽帧工具
演示：均匀抽帧、场景变化检测抽帧
"""
import os
import cv2
import base64
from typing import List, Tuple


def extract_frames_uniform(
    video_path: str,
    max_frames: int = 10,
) -> List[Tuple[float, str]]:
    """
    均匀抽帧: 从视频中均匀选取 max_frames 帧。

    参数:
        video_path: 视频文件路径
        max_frames: 最大抽取帧数

    返回:
        [(时间戳(秒), base64编码的JPEG图片), ...]
    """
    cap = cv2.VideoCapture(video_path)
    total_frames = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))
    fps = cap.get(cv2.CAP_PROP_FPS)
    duration = total_frames / fps if fps > 0 else 0

    print(f"视频信息: 总帧数={total_frames}, FPS={fps:.1f}, 时长={duration:.1f}秒")

    # 计算采样间隔
    interval = max(total_frames // max_frames, 1)

    frames = []
    for i in range(0, total_frames, interval):
        cap.set(cv2.CAP_PROP_POS_FRAMES, i)
        ret, frame = cap.read()
        if not ret:
            break

        # 计算时间戳
        timestamp = i / fps

        # 编码为 JPEG base64
        _, buffer = cv2.imencode(".jpg", frame, [cv2.IMWRITE_JPEG_QUALITY, 75])
        b64 = base64.b64encode(buffer).decode("utf-8")

        frames.append((timestamp, b64))

        if len(frames) >= max_frames:
            break

    cap.release()
    return frames


def extract_keyframes_scene_change(
    video_path: str,
    threshold: float = 30.0,
    max_frames: int = 20,
) -> List[Tuple[float, str]]:
    """
    基于场景变化检测的关键帧抽取。
    当相邻帧的像素差异超过 threshold 时，认定为场景切换。

    参数:
        video_path: 视频路径
        threshold: 帧间差异阈值（0-255 均方误差），越大越不敏感
            经验值: 20=敏感（微小变化就抽帧）, 30=适中, 50=只有明显切换才抽

    返回:
        [(时间戳, base64图片), ...]
    """
    cap = cv2.VideoCapture(video_path)
    fps = cap.get(cv2.CAP_PROP_FPS)

    keyframes = []
    prev_frame = None
    frame_idx = 0

    while True:
        ret, frame = cap.read()
        if not ret:
            break

        timestamp = frame_idx / fps
        is_keyframe = False

        if prev_frame is None:
            # 第一帧始终保留
            is_keyframe = True
        else:
            # 计算帧间差异
            gray_curr = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
            gray_prev = cv2.cvtColor(prev_frame, cv2.COLOR_BGR2GRAY)
            diff = ((gray_curr.astype(float) - gray_prev.astype(float)) ** 2).mean()
            if diff > threshold:
                is_keyframe = True

        if is_keyframe:
            _, buffer = cv2.imencode(".jpg", frame, [cv2.IMWRITE_JPEG_QUALITY, 75])
            b64 = base64.b64encode(buffer).decode("utf-8")
            keyframes.append((timestamp, b64))

            if len(keyframes) >= max_frames:
                break

        prev_frame = frame
        frame_idx += 1

    cap.release()
    print(f"检测到 {len(keyframes)} 个关键帧/场景切换")
    return keyframes


if __name__ == "__main__":
    video = "demo.mp4"
    if os.path.exists(video):
        frames = extract_frames_uniform(video, max_frames=8)
        print(f"均匀抽帧: {len(frames)} 帧")
        for ts, b64 in frames:
            print(f"  [{ts:.1f}s] base64 长度={len(b64)}")

        keyframes = extract_keyframes_scene_change(video, threshold=30.0)
        print(f"场景检测抽帧: {len(keyframes)} 帧")
    else:
        print("请提供视频文件 demo.mp4。")
```

### 6.6.2 完整视频理解 Pipeline

```python
"""
ex_6_17_video_understanding_pipeline.py: 完整的视频理解流水线
步骤: 视频 → 抽帧 → 逐帧视觉理解 → LLM 汇总 → 视频描述
"""
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))


def understand_video(
    frames: list[tuple[float, str]],
    question: str = "请详细描述这个视频的内容，包括场景、人物、动作和关键事件。",
) -> str:
    """
    将视频帧发送给 GPT-4o 进行整体理解。

    策略: 由于 token 窗口有限，不能所有帧都传 high 精度。
    通常做法: 第一帧 + 最后一帧 + 关键帧用 high，其余用 low。
    """
    content = [{"type": "text", "text": question}]

    # 对帧进行分类: 前 3 帧 + 最后 1 帧 + 中间均匀采样的帧传 high，其余传 low
    total = len(frames)
    if total <= 5:
        # 帧数少，全部 high
        high_indices = set(range(total))
    else:
        # 帧数多，选部分 high
        high_indices = {0, total - 1}  # 首帧和末帧
        # 中间均匀取
        step = max(total // 8, 1)
        for i in range(step, total - 1, step):
            high_indices.add(i)
        # 限制 high 帧数不超过 10
        while len(high_indices) > 10:
            # 去掉中间的
            mid = sorted(high_indices)[len(high_indices) // 2]
            high_indices.discard(mid)

    for i, (timestamp, b64) in enumerate(frames):
        detail = "high" if i in high_indices else "low"

        content.append({
            "type": "text",
            "text": f"\n[帧 {i + 1}/{total}, 时间: {timestamp:.1f}秒]",
        })
        content.append({
            "type": "image_url",
            "image_url": {
                "url": f"data:image/jpeg;base64,{b64}",
                "detail": detail,
            },
        })

    # 发送请求
    # 注意: 帧数过多可能导致 token 超限
    # 单次请求图片总 token 上限为 20M tokens 的模型上下文长度限制
    response = client.chat.completions.create(
        model="gpt-4o",  # 128K 上下文
        messages=[{"role": "user", "content": content}],
        max_tokens=2000,
    )

    return response.choices[0].message.content


if __name__ == "__main__":
    from ex_6_16_video_frame_extraction import extract_frames_uniform

    video = "demo.mp4"
    if os.path.exists(video):
        print("正在抽帧...")
        frames = extract_frames_uniform(video, max_frames=10)
        print(f"抽取了 {len(frames)} 帧")

        print("正在进行视频理解...")
        description = understand_video(frames)
        print("\n=== 视频理解结果 ===\n")
        print(description)
    else:
        print("请提供视频文件 demo.mp4。")
```

## 6.7 完整 Pipeline 实战

将本章学到的各个模块串联起来：

```python
"""
ex_6_18_multimodal_pipeline.py: 多模态处理完整流水线
演示：
  1. 输入一段语音 → Whisper 转文字
  2. 理解文字意图 → 发现用户想生成一张图
  3. 使用 DALL-E 3 根据文字描述生成图片
  4. 使用 GPT-4o 评价生成的图片
  5. 使用 TTS 将评价结果朗读出来
"""
import os
import json
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))


class MultimodalPipeline:
    """多模态处理流水线：语音→文字→图片→评价→语音"""

    def __init__(self):
        self.client = client

    def step1_speech_to_text(self, audio_path: str) -> str:
        """语音→文字 (ASR)"""
        print(f"[Step 1] 语音识别: {audio_path}")
        with open(audio_path, "rb") as f:
            response = self.client.audio.transcriptions.create(
                model="whisper-1", file=f, response_format="text"
            )
        text = response if isinstance(response, str) else response.text
        print(f"  识别结果: {text}")
        return text

    def step2_extract_intent(self, text: str) -> dict:
        """理解意图：把自然语言转成结构化描述"""
        print("[Step 2] 意图解析")
        response = self.client.chat.completions.create(
            model="gpt-4o",
            messages=[{
                "role": "user",
                "content": f"""从以下用户输入中提取图像生成需求，以 JSON 格式返回：
{{
  "image_prompt": "英文的图像生成 Prompt，详细描述画面",
  "style": "风格: photorealistic/illustration/anime/3d/oil-painting",
  "evaluation_criteria": "用户希望评价的方面（可选）"
}}

用户输入: {text}

只返回 JSON。""",
            }],
            response_format={"type": "json_object"},
            max_tokens=300,
        )
        intent = json.loads(response.choices[0].message.content)
        print(f"  解析结果: {json.dumps(intent, ensure_ascii=False, indent=2)}")
        return intent

    def step3_text_to_image(self, prompt: str, style: str) -> str:
        """文字→图片 (T2I)"""
        print("[Step 3] 生成图片")
        style_modifiers = {
            "photorealistic": "photorealistic, 8K, highly detailed, natural lighting",
            "illustration": "digital illustration, clean lines, vibrant colors, flat design",
            "anime": "anime style, Studio Ghibli inspired, hand-drawn animation feel",
            "3d": "3D render, Octane, global illumination, PBR materials, 8K",
            "oil-painting": "oil painting on canvas, visible brushstrokes, classical technique",
        }
        extra = style_modifiers.get(style, "photorealistic, 8K")
        full_prompt = f"{prompt}, {extra}"

        response = self.client.images.generate(
            model="dall-e-3",
            prompt=full_prompt,
            size="1024x1024",
            quality="standard",
            n=1,
        )
        image_url = response.data[0].url

        # 下载图片
        import requests
        img_data = requests.get(image_url).content
        output_path = "pipeline_generated.png"
        with open(output_path, "wb") as f:
            f.write(img_data)
        print(f"  图片已保存: {output_path}")
        return output_path

    def step4_evaluate_image(self, image_path: str, criteria: str) -> str:
        """图片评价 (Vision)"""
        print("[Step 4] 图片评价")
        import base64
        with open(image_path, "rb") as f:
            b64 = base64.b64encode(f.read()).decode("utf-8")

        response = self.client.chat.completions.create(
            model="gpt-4o",
            messages=[{
                "role": "user",
                "content": [
                    {
                        "type": "text",
                        "text": f"请从以下角度评价这张生成的图片: {criteria}\n"
                                f"用中文回答，不超过200字。",
                    },
                    {
                        "type": "image_url",
                        "image_url": {"url": f"data:image/png;base64,{b64}", "detail": "high"},
                    },
                ],
            }],
            max_tokens=300,
        )
        evaluation = response.choices[0].message.content
        print(f"  评价结果: {evaluation}")
        return evaluation

    def step5_text_to_speech(self, text: str, output_path: str = "pipeline_evaluation.mp3"):
        """文字→语音 (TTS)"""
        print("[Step 5] 生成语音")
        response = self.client.audio.speech.create(
            model="tts-1", voice="nova", input=text, response_format="mp3"
        )
        response.stream_to_file(output_path)
        print(f"  语音已保存: {output_path}")
        return output_path

    def run(self, audio_path: str):
        """运行完整流水线"""
        # Step 1: 语音→文字
        user_text = self.step1_speech_to_text(audio_path)

        # Step 2: 理解意图
        intent = self.step2_extract_intent(user_text)

        # Step 3: 文字→图片
        image_path = self.step3_text_to_image(
            intent["image_prompt"], intent.get("style", "photorealistic")
        )

        # Step 4: 图片→评价
        evaluation = self.step4_evaluate_image(
            image_path,
            intent.get("evaluation_criteria", "画面质量、构图、风格匹配度、细节完整度"),
        )

        # Step 5: 文字→语音
        audio_output = self.step5_text_to_speech(evaluation)

        print("\n========== 流水线完成 ==========")
        print(f"输入音频: {audio_path}")
        print(f"识别文字: {user_text}")
        print(f"生成图片: {image_path}")
        print(f"评价结果: {evaluation}")
        print(f"评价语音: {audio_output}")

        return {
            "user_text": user_text,
            "intent": intent,
            "image_path": image_path,
            "evaluation": evaluation,
            "audio_output": audio_output,
        }


if __name__ == "__main__":
    pipeline = MultimodalPipeline()
    audio = "input.mp3"
    if os.path.exists(audio):
        pipeline.run(audio)
    else:
        print("请提供 input.mp3 作为输入音频。")
        print("示例音频内容可以是: '请帮我生成一张夕阳下的富士山照片，要写实风格的。'")
```

## 常见错误

### 错误 1: 图片 Base64 过大导致 API 拒绝

```python
# 错误: 直接将 4000x3000 的 PNG 原图转 base64，体积可能超过 20MB
with open("large_photo.png", "rb") as f:
    b64 = base64.b64encode(f.read()).decode("utf-8")
# → API 返回 413 Request Entity Too Large

# 正确: 先压缩/缩放再编码
from PIL import Image
img = Image.open("large_photo.png")
img.thumbnail((2048, 2048))  # 缩放到 2048 以内
buf = io.BytesIO()
img.save(buf, format="JPEG", quality=75)
b64 = base64.b64encode(buf.getvalue()).decode("utf-8")
```

### 错误 2: 混淆 Content 数组中的 type 键名

```python
# 错误: OpenAI 用的是 "image_url"，不要写成 "image"
content = [
    {"type": "image", "image_url": {"url": f"data:image/png;base64,..."}}
]
# → openai.BadRequestError: Invalid content type: 'image'

# 正确
content = [
    {"type": "image_url", "image_url": {"url": f"data:image/png;base64,..."}}
]
```

### 错误 3: DALL-E 3 的 n > 1

```python
# 错误: DALL-E 3 不支持 n > 1
client.images.generate(model="dall-e-3", prompt="...", n=4)
# → openai.BadRequestError: 'n' must be 1 for model 'dall-e-3'

# 正确: 如果要生成多张，多次调用
for i in range(4):
    client.images.generate(model="dall-e-3", prompt="...", n=1)
```

### 错误 4: Whisper API 文件未用 rb 模式打开

```python
# 错误
with open("audio.mp3", "r") as f:  # ← 'r' 是文本模式，不是 'rb'
    client.audio.transcriptions.create(model="whisper-1", file=f)
# → UnicodeDecodeError（Python 尝试以文本方式解码二进制 MP3 数据）

# 正确
with open("audio.mp3", "rb") as f:
    client.audio.transcriptions.create(model="whisper-1", file=f)
```

### 错误 5: TTS 文本超过 4096 字符

```python
# 错误: 一次性传入超长文本
long_text = "..." * 5000  # 假设超级长
client.audio.speech.create(model="tts-1", voice="nova", input=long_text)
# → openai.BadRequestError: Input text exceeds maximum length of 4096 characters

# 正确: 分段合成（参考 6.5.1 中的 long_text_to_speech 函数）
```

### 错误 6: 视频抽帧时忘记 release VideoCapture

```python
# 错误: 打开视频后忘记释放，导致文件被锁定
cap = cv2.VideoCapture("video.mp4")
# ... 使用 ...
# 忘记 cap.release() → 文件句柄泄漏，多次运行后可能耗尽

# 正确: 使用 with 或 try/finally
cap = cv2.VideoCapture("video.mp4")
try:
    # ... 使用 ...
finally:
    cap.release()
```

## 本章小结

本章覆盖了 AI 多模态应用的核心技术栈：

1. **视觉理解**：通过 GPT-4o / Claude Vision API 让模型"看懂"图片。关键技巧包括图片预处理（压缩/缩放/格式转换）、detail 参数选择、多图对话。典型应用有图表解读、UI 分析、文档 OCR。

2. **文生图**：DALL-E 3 适合快速原型和创意探索（prompt 理解强、文字渲染好），Stable Diffusion + ControlNet 适合大批量生产和需要精确控制的场景（免费、可定制、隐私）。

3. **语音识别**：Whisper API 零门槛上手，faster-whisper 适合离线/大批量/隐私要求高的场景。关键点包括大文件分段、时间戳提取、prompt 引导提升准确率。

4. **语音合成**：OpenAI TTS 英文质量极佳，ChatTTS 是中文对话场景的首选开源方案。长文本需要智能分段合成。

5. **视频理解**：当前主流方案是"抽帧 + 视觉 API"的流水线。抽帧策略有均匀采样和场景检测两种，帧数的控制需要在信息完整度和 token 成本之间平衡。

6. **Pipeline 串联**：单个模态的能力有限，真正的价值在于把它们串联起来——语音进来、理解意图、生成图片、评价质量、朗读结果，形成端到端的闭环体验。

掌握这些多模态能力后，你就可以构建出见多识广的 AI 应用——不仅能"说"，还能"看"、"听"、"画"，真正实现人机之间丰富的交互方式。
