## 1. 预训练目标

### 1.1 什么是预训练

"预训练"（Pre-training）这个术语可能会误导初学者。它暗示还有一个"真正"的训练在后面。但事实上，对于大语言模型，预训练本身就是"真正"的训练——它占据了总计算量的 99% 以上。更准确的名字可能是"自监督训练"或"从头训练"。

预训练的核心思想：**在大量无标注文本上进行自监督学习，让模型学会语言的统计规律和世界知识。** 这个阶段不需要人工标注，只需要原始文本——文本本身就是"标签"。

### 1.2 因果语言模型 CLM（Causal Language Modeling）

CLM 是 GPT 系列使用的预训练目标。它的思想非常直观：**给定前面的 token，预测下一个 token**。

数学上来说，对于一个文本序列 `[x₁, x₂, ..., xₙ]`，CLM 的目标是最大化：

```
P(x₁, x₂, ..., xₙ) = Π P(x_t | x₁, ..., x_{t-1})
```

即序列的联合概率 = 每个 token 在其之前所有 token 条件下的条件概率的连乘。这被称为**自回归**（autoregressive）——模型用自己的输出作为下一步的输入。

训练时，对于输入序列 `[x₁, x₂, ..., xₙ]`，模型被要求输出 `[x₂, x₃, ..., x_{n+1}]`（即每个位置的输出 = 下一个位置的输入）。这是一个标准的"shift by one"操作：

```
输入: The cat sat on the mat
标签: cat sat on the mat <EOS>
```

损失仅在非 padding 的 token 上计算（CrossEntropy Loss）。

**为什么 CLM 对生成友好？** 因为生成过程天然是自回归的：从 `<BOS>` 开始，模型预测下一个 token → 将预测的 token 加到序列中 → 继续预测下一个 → ...→ 直到遇到 `<EOS>`。训练和推理的格式完全一致，没有任何 gap。

### 1.3 掩码语言模型 MLM（Masked Language Modeling）

MLM 是 BERT 使用的预训练目标。它的思想是：**随机遮住输入中的一些 token，让模型根据上下文预测被遮住的 token**。

训练时，随机选择 15% 的 token 进行处理：
- 80% 被替换为 `[MASK]`
- 10% 被替换为随机 token
- 10% 保持不变（但仍在 loss 中计算）

例如：
```
原文: The cat sat on the mat
MLM:  The [MASK] sat on the [MASK] mat
目标: "cat" 和 "the"（只在这两个位置计算 loss）
```

**为什么 MLM 对理解任务更好？** BERT 的双向注意力使得每个 token 可以看到上下文的两侧，从而获得更丰富的表示。这对于分类、问答、命名实体识别等"理解"任务非常有利。

**MLM 的局限**：MLM 不是生成式的——你不能用 MLM 模型直接做文本生成（因为它假设已知未来的 token）。这就是为什么 BERT 擅长"理解"但不擅长"生成"。

### 1.4 CLM vs MLM 的选择

| 方面 | CLM (GPT式) | MLM (BERT式) |
|------|------------|------------|
| 注意力掩码 | 因果（只看左边） | 双向（看两边） |
| 训练-推理一致性 | 一致 | 不一致（MLM 引入了一致性 gap） |
| 适合场景 | 文本生成、对话、代码补全 | 文本理解、分类、问答 |
| 代表模型 | GPT 系列、LLaMA | BERT、RoBERTa |
| 训练效率 | 每个 token 都被预测 | 只有 15% 的 token 被预测（效率较低） |

现代大语言模型（2023+）几乎全部采用 CLM。即使需要理解能力，也可以通过适当的 prompt 让 CLM 模型胜任。这就是 GPT 系列展示的"生成即理解"范式。

## 2. 模型配置：从参数量反推架构

### 2.1 参数量公式

对于标准的 Decoder-Only Transformer，参数量主要由以下几个部分组成：

**Embedding 层**：
```
P_embed = vocab_size × d_model
```

**每个 Transformer Block**：
```
P_attn = 4 × d_model × d_model  （Q, K, V, O 四个投影矩阵）
P_ffn  = 3 × d_model × d_ff  （SwiGLU: gate, up, down 三个矩阵）
P_block = P_attn + P_ffn
```

**LM Head**：
```
P_lm_head = d_model × vocab_size  （通常与 embedding 共享，不计入总量）
```

**总参数量**（不含 embedding，因为它与 LM Head 共享）：
```
P_total ≈ n_layers × (4 × d_model² + 3 × d_model × d_ff)
       = n_layers × (4 × d_model² + 3 × d_model × 4 × d_model)  # 如果 d_ff = 4*d_model
       = n_layers × (4 + 12) × d_model²
       = n_layers × 16 × d_model²
```

**近似公式**：对于 `d_ff ≈ 4 × d_model` 的标准配置：
```
P_total ≈ n_layers × 12 × d_model²
```

### 2.2 从目标参数量反推

假设你想训练一个约 125M 参数的小型 GPT：

**方法一：试凑法**
1. 选择一个 d_model（通常是 64 的倍数）：试试 768
2. 选择 n_layers：试试 12
3. 计算：12 × 12 × 768² ≈ 85M（偏小）
4. 调整：n_layers = 12, d_model = 768 → 约 85M（不含 embedding）
5. 加上 embedding（假设 vocab_size=50000）：50K × 768 = 38M
6. 总参数：85M + 38M ≈ 123M ✓

**方法二：参考已知模型**：

| 模型 | 参数量 | n_layers | d_model | n_heads | d_ff |
|------|--------|----------|---------|---------|------|
| GPT-2 Small | 124M | 12 | 768 | 12 | 3072 |
| GPT-2 Medium | 355M | 24 | 1024 | 16 | 4096 |
| GPT-2 Large | 774M | 36 | 1280 | 20 | 5120 |
| GPT-2 XL | 1.5B | 48 | 1600 | 25 | 6400 |
| LLaMA 7B | 6.7B | 32 | 4096 | 32 | 11008 |
| LLaMA 13B | 13.0B | 40 | 5120 | 40 | 13824 |
| LLaMA 70B | 69B | 80 | 8192 | 64 | 28672 |

**关键比例关系**：
- d_model / n_heads = const（每个头的维度），通常 64 或 128
- d_ff / d_model：标准为 4，SwiGLU 为约 2.67（8/3）
- d_model 和 n_layers 通常同时增大（等比缩放）

### 2.3 一个小型 GPT 的推荐配置

对于学习和实验目的（能在单张 24GB GPU 上训练），推荐如下配置：

```
vocab_size = 16384    # 较小的词表，节省 embedding 参数
d_model    = 512      # 适中的隐藏维度
n_layers   = 8        # 8 层 Transformer
n_heads    = 8        # 每个头 64 维
d_ff       = 1376     # SwiGLU: 8/3 * 512 ≈ 1365 → 1376（64的倍数）
max_seq_len = 1024    # 上下文长度
```

这个配置的参数量：
- Embedding: 16384 × 512 = 8.4M
- 每个 block: attention(4×512²) + ffn(3×512×1376) ≈ 1.05M + 2.11M = 3.16M
- 8 个 blocks: 8 × 3.16M ≈ 25.3M
- LM Head: 与 embedding 共享，0
- **总参数 ≈ 34M**

这个规模在单张 24GB GPU 上完全可行，且足够展示预训练的完整流程。

## 3. 从头训练一个小型 GPT

### 3.1 整体架构回顾

```
输入: token IDs (batch, seq_len)
  ↓
Token Embedding    (vocab_size → d_model)
  ↓
[TransformerBlock × n_layers]
  每个 Block:
    x = x + Attention(RMSNorm(x))    # 因果自注意力
    x = x + SwiGLU(RMSNorm(x))       # 前馈网络
  ↓
RMSNorm            (最后归一化)
  ↓
LM Head            (d_model → vocab_size)
  ↓
输出: logits (batch, seq_len, vocab_size)
```

### 3.2 训练数据的准备流程

对于小型 GPT 的训练演示，我们可以使用以下几个数据来源之一：
- **WikiText-2/103**：维基百科文本，干净且高质量。适合小规模实验。
- **TinyStories**：用 GPT-3.5/4 生成的小故事数据集，专为小模型设计。
- **OpenWebText**：Reddit 链接的网页文本（类似 GPT-2 训练数据），规模适中。
- **自定义中文数据**：中文维基百科 dump + 新闻 + 书籍。

**数据准备的完整步骤**：

1. **下载原始文本**
2. **质量过滤**（第04章的方法）
3. **训练分词器**（第03章的方法）或在已有分词器上做适配
4. **分词**（将文本转为 token IDs）
5. **打包**为固定长度的训练序列（避免 padding 浪费）
6. **划分**训练集/验证集

class RMSNorm(nn.Module):
    """RMS 归一化"""
    def __init__(self, dim, eps=1e-6):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))

    def forward(self, x):
        rms = torch.sqrt(torch.mean(x.float() ** 2, dim=-1, keepdim=True) + self.eps)
        return (x / rms) * self.weight


class CausalSelfAttention(nn.Module):
    """因果自注意力（含 GQA 支持）"""
    def __init__(self, d_model, n_heads, n_kv_heads=None, max_seq_len=1024):
        super().__init__()
        self.n_heads = n_heads
        self.n_kv_heads = n_kv_heads or n_heads
        self.d_k = d_model // n_heads

        self.W_Q = nn.Linear(d_model, n_heads * self.d_k, bias=False)
        self.W_K = nn.Linear(d_model, self.n_kv_heads * self.d_k, bias=False)
        self.W_V = nn.Linear(d_model, self.n_kv_heads * self.d_k, bias=False)
        self.W_O = nn.Linear(d_model, d_model, bias=False)

        # 因果掩码
        mask = torch.triu(torch.ones(max_seq_len, max_seq_len), diagonal=1).bool()
        self.register_buffer('mask', mask)

    def forward(self, x):
        batch, seq_len, d_model = x.shape

        # 投影 + 分头
        Q = self.W_Q(x).view(batch, seq_len, self.n_heads, self.d_k).transpose(1, 2)
        K = self.W_K(x).view(batch, seq_len, self.n_kv_heads, self.d_k).transpose(1, 2)
        V = self.W_V(x).view(batch, seq_len, self.n_kv_heads, self.d_k).transpose(1, 2)

        # GQA: 扩展 KV 头以匹配 Q 头
        if self.n_kv_heads < self.n_heads:
            repeat_factor = self.n_heads // self.n_kv_heads
            K = K.repeat_interleave(repeat_factor, dim=1)
            V = V.repeat_interleave(repeat_factor, dim=1)

        # Attention
        scores = (Q @ K.transpose(-2, -1)) / math.sqrt(self.d_k)
        scores = scores.masked_fill(self.mask[:seq_len, :seq_len], float('-inf'))
        attn = F.softmax(scores, dim=-1)
        attn = attn @ V

        # 合并头 + 输出投影
        attn = attn.transpose(1, 2).contiguous().view(batch, seq_len, d_model)
        return self.W_O(attn)


class SwiGLUFFN(nn.Module):
    """SwiGLU 前馈网络"""
    def __init__(self, d_model, d_ff=None):
        super().__init__()
        if d_ff is None:
            # 保持和标准 FFN (d_ff=4*d_model) 相当的参数量
            d_ff = int(8 / 3 * d_model)
            d_ff = ((d_ff + 255) // 256) * 256  # 对齐到 256 的倍数

        self.gate = nn.Linear(d_model, d_ff, bias=False)
        self.up = nn.Linear(d_model, d_ff, bias=False)
        self.down = nn.Linear(d_ff, d_model, bias=False)

    def forward(self, x):
        return self.down(F.silu(self.gate(x)) * self.up(x))


class TransformerBlock(nn.Module):
    """Transformer Decoder Block (Pre-Norm)"""
    def __init__(self, d_model, n_heads, n_kv_heads=None, d_ff=None, max_seq_len=1024):
        super().__init__()
        self.attn_norm = RMSNorm(d_model)
        self.attn = CausalSelfAttention(d_model, n_heads, n_kv_heads, max_seq_len)
        self.ffn_norm = RMSNorm(d_model)
        self.ffn = SwiGLUFFN(d_model, d_ff)

    def forward(self, x):
        x = x + self.attn(self.attn_norm(x))
        x = x + self.ffn(self.ffn_norm(x))
        return x


class MiniGPT(nn.Module):
    """
    小型 GPT 模型。

    架构: Token Embedding → N×TransformerBlock → RMSNorm → LM Head
    """
    def __init__(self, vocab_size, d_model, n_layers, n_heads,
                 n_kv_heads=None, d_ff=None, max_seq_len=1024):
        super().__init__()
        self.d_model = d_model
        self.max_seq_len = max_seq_len

        # Token Embedding
        self.tok_embeddings = nn.Embedding(vocab_size, d_model)

        # Transformer Blocks
        self.layers = nn.ModuleList([
            TransformerBlock(d_model, n_heads, n_kv_heads, d_ff, max_seq_len)
            for _ in range(n_layers)
        ])

        # 最终归一化
        self.norm = RMSNorm(d_model)

        # LM Head (与 embedding 共享权重 — Weight Tying)
        self.lm_head = nn.Linear(d_model, vocab_size, bias=False)
        self.lm_head.weight = self.tok_embeddings.weight

        # 初始化
        self.apply(self._init_weights)

    def _init_weights(self, module):
        if isinstance(module, nn.Linear):
            torch.nn.init.normal_(module.weight, mean=0.0, std=0.02)
            if module.bias is not None:
                torch.nn.init.zeros_(module.bias)
        elif isinstance(module, nn.Embedding):
            torch.nn.init.normal_(module.weight, mean=0.0, std=0.02)

    def forward(self, input_ids, labels=None):
        """
        参数:
            input_ids: (batch, seq_len)
            labels: (batch, seq_len)，如果为 None，返回 logits
        返回:
            如果 labels 不为 None: (loss, logits)
            否则: logits
        """
        batch, seq_len = input_ids.shape

        # Embedding
        x = self.tok_embeddings(input_ids)  # (batch, seq_len, d_model)

        # Transformer Blocks
        for layer in self.layers:
            x = layer(x)

        # 最终归一化
        x = self.norm(x)

        # LM Head
        logits = self.lm_head(x)  # (batch, seq_len, vocab_size)

        # 计算损失
        if labels is not None:
            # 展平: (batch*seq_len, vocab_size) vs (batch*seq_len,)
            loss = F.cross_entropy(
                logits.view(-1, logits.size(-1)),
                labels.view(-1),
                ignore_index=-100,  # 忽略 padding 位置
            )
            return loss, logits

        return logits

    @torch.no_grad()
    def generate(self, input_ids, max_new_tokens=100, temperature=1.0, top_k=None):
        """自回归文本生成"""
        self.eval()
        for _ in range(max_new_tokens):
            # 截断到 max_seq_len
            ids_cond = input_ids if input_ids.size(1) <= self.max_seq_len \
                       else input_ids[:, -self.max_seq_len:]

            logits = self.forward(ids_cond)[:, -1, :]
            logits = logits / temperature

            if top_k is not None:
                top_k_vals, _ = torch.topk(logits, top_k, dim=-1)
                logits[logits < top_k_vals[:, -1:]] = float('-inf')

            probs = F.softmax(logits, dim=-1)
            next_token = torch.multinomial(probs, num_samples=1)
            input_ids = torch.cat([input_ids, next_token], dim=1)

        return input_ids

    def get_num_params(self):
        """返回参数量统计"""
        total = sum(p.numel() for p in self.parameters())
        trainable = sum(p.numel() for p in self.parameters() if p.requires_grad)
        embed = self.tok_embeddings.weight.numel()
        return {
            'total': total,
            'trainable': trainable,
            'non_embedding': total - embed,  # 除去 embedding 的参数量
        }

class PretrainingDataset(Dataset):
    """
    预训练数据集。

    将原始文本 tokenize 后打包为固定长度的 chunk。
    输入和标签错开一位（CLM 的 shift-by-one）。
    """

    def __init__(self, tokenizer, texts, max_seq_len=1024):
        """
        参数:
            tokenizer: HuggingFace tokenizer（需要有 encode 方法）
            texts: 文本列表
            max_seq_len: 每个训练样本的长度
        """
        self.max_seq_len = max_seq_len
        self.eos_token_id = tokenizer.eos_token_id

        # 将所有文本 tokenize 并拼接
        print(f"正在 tokenize {len(texts)} 条文本...")
        all_tokens = []
        for text in texts:
            tokens = tokenizer.encode(text)
            tokens.append(self.eos_token_id)  # 每个文本后追加 EOS
            all_tokens.extend(tokens)

        # 打包为固定长度 chunk
        total_len = len(all_tokens)
        # 丢弃最后不完整的 chunk
        num_chunks = total_len // max_seq_len
        all_tokens = all_tokens[:num_chunks * max_seq_len]

        self.data = torch.tensor(all_tokens, dtype=torch.long).view(num_chunks, max_seq_len)
        print(f"Tokenize 完成: {total_len} tokens → {num_chunks} chunks (长度={max_seq_len})")

    def __len__(self):
        return len(self.data)

    def __getitem__(self, idx):
        chunk = self.data[idx]
        # CLM: input 是前 n-1 个 token，label 是后 n-1 个 token
        input_ids = chunk[:-1]
        labels = chunk[1:]
        # 返回长度一致的序列（padding 位置用 ignore_index=-100 处理）
        return input_ids, labels

def create_lr_scheduler(optimizer, warmup_steps, total_steps, min_lr_ratio=0.1):
    """
    创建学习率调度器: warmup (linear) → cosine decay。

    学习率曲线:
        lr
        ↑
        |     /\
        |    /  \
        |   /    \_______
        |  /              \___
        | /                   \
        +----|--------|--------→ step
            warmup   total

    参数:
        optimizer: 优化器实例
        warmup_steps: warmup 步数
        total_steps: 总训练步数
        min_lr_ratio: 最终学习率 / 峰值学习率（cosine 衰减的最低值）
    返回:
        SequentialLR 调度器
    """
    peak_lr = optimizer.param_groups[0]['lr']

    # Warmup 阶段: 线性增长
    warmup = LinearLR(
        optimizer,
        start_factor=min_lr_ratio,  # 从 peak_lr 的 10% 开始
        end_factor=1.0,             # 增长到 peak_lr
        total_iters=warmup_steps,
    )

    # Cosine 衰减阶段: 从 peak_lr 衰减到 min_lr_ratio * peak_lr
    cosine = CosineAnnealingLR(
        optimizer,
        T_max=total_steps - warmup_steps,
        eta_min=min_lr_ratio * peak_lr,  # 最终学习率
    )

    scheduler = SequentialLR(
        optimizer,
        schedulers=[warmup, cosine],
        milestones=[warmup_steps],
    )

    return scheduler


# ============================================================
# 4. 评估函数
# ============================================================

@torch.no_grad()
def evaluate(model, val_loader, device, max_eval_batches=None):
    """
    在验证集上评估模型。

    返回:
        val_loss: 平均验证损失
        perplexity: 困惑度 = exp(loss)
    """
    model.eval()
    total_loss = 0.0
    num_batches = 0

    for batch_idx, (input_ids, labels) in enumerate(val_loader):
        input_ids = input_ids.to(device)
        labels = labels.to(device)

        loss, _ = model(input_ids, labels)

        total_loss += loss.item()
        num_batches += 1

        if max_eval_batches and num_batches >= max_eval_batches:
            break

    model.train()
    avg_loss = total_loss / num_batches
    perplexity = math.exp(avg_loss)

    return avg_loss, perplexity

# 5. Checkpoint 管理
# ============================================================

def save_checkpoint(model, optimizer, scheduler, step, loss, config, filepath):
    """保存训练检查点"""
    checkpoint = {
        'model_state_dict': model.state_dict(),
        'optimizer_state_dict': optimizer.state_dict(),
        'scheduler_state_dict': scheduler.state_dict(),
        'step': step,
        'loss': loss,
        'config': config,
    }
    torch.save(checkpoint, filepath)
    print(f"Checkpoint 已保存到 {filepath} (step={step})")


def load_checkpoint(model, optimizer, scheduler, filepath, device):
    """加载训练检查点"""
    checkpoint = torch.load(filepath, map_location=device, weights_only=False)

    model.load_state_dict(checkpoint['model_state_dict'])
    optimizer.load_state_dict(checkpoint['optimizer_state_dict'])

    if scheduler and 'scheduler_state_dict' in checkpoint:
        scheduler.load_state_dict(checkpoint['scheduler_state_dict'])

    step = checkpoint['step']
    loss = checkpoint.get('loss', float('inf'))

    print(f"Checkpoint 已加载: step={step}, loss={loss:.4f}")
    return step, loss

# 6. 训练主循环
# ============================================================

def train(args):
    """预训练主函数"""

    # ---- 设备 ----
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    print(f"使用设备: {device}")

    if device.type == 'cuda':
        print(f"GPU: {torch.cuda.get_device_name(0)}")
        print(f"显存: {torch.cuda.get_device_properties(0).total_mem / 1e9:.1f} GB")

    # ---- 分词器 ----
    # 使用 GPT-2 的分词器（BPE 50K vocab）作为演示
    # 对于中文训练，你应使用第03章训练的自定义分词器
    tokenizer = AutoTokenizer.from_pretrained('gpt2')
    tokenizer.pad_token = tokenizer.eos_token  # GPT-2 没有 pad token，用 eos 替代
    print(f"分词器词表大小: {len(tokenizer)}")

    # ---- 数据 ----
    train_dataset, val_dataset = load_wikitext_data(
        tokenizer, max_seq_len=args.max_seq_len
    )

    train_loader = DataLoader(
        train_dataset,
        batch_size=args.batch_size,
        shuffle=True,
        num_workers=2,
        pin_memory=True,
    )
    val_loader = DataLoader(
        val_dataset,
        batch_size=args.batch_size,
        shuffle=False,
        num_workers=2,
        pin_memory=True,
    )

    # ---- 模型 ----
    config = {
        'vocab_size': len(tokenizer),
        'd_model': args.d_model,
        'n_layers': args.n_layers,
        'n_heads': args.n_heads,
        'n_kv_heads': args.n_kv_heads,
        'max_seq_len': args.max_seq_len,
    }

    model = MiniGPT(
        vocab_size=config['vocab_size'],
        d_model=config['d_model'],
        n_layers=config['n_layers'],
        n_heads=config['n_heads'],
        n_kv_heads=config['n_kv_heads'],
        max_seq_len=config['max_seq_len'],
    ).to(device)

    # 编译模型（PyTorch 2.0+, 加速训练）
    if hasattr(torch, 'compile') and args.compile:
        print("使用 torch.compile 编译模型...")
        model = torch.compile(model)

    params_info = model.get_num_params()
    print(f"模型参数量: {params_info['total']:,} (可训练: {params_info['trainable']:,})")
    print(f"配置: d_model={config['d_model']}, "
          f"n_layers={config['n_layers']}, "
          f"n_heads={config['n_heads']}, "
          f"max_seq_len={config['max_seq_len']}")

    # ---- 优化器 ----
    optimizer = torch.optim.AdamW(
        model.parameters(),
        lr=args.lr,
        betas=(0.9, 0.95),
        weight_decay=args.weight_decay,
        eps=1e-8,
    )

    # ---- 学习率调度器 ----
    total_steps = args.max_steps
    scheduler = create_lr_scheduler(
        optimizer,
        warmup_steps=args.warmup_steps,
        total_steps=total_steps,
        min_lr_ratio=args.min_lr_ratio,
    )

    # ---- 梯度累积 ----
    gradient_accumulation_steps = args.gradient_accumulation_steps
    effective_batch_size = args.batch_size * gradient_accumulation_steps
    print(f"有效 batch size: {effective_batch_size} "
          f"(micro={args.batch_size}, accumulation={gradient_accumulation_steps})")

    # ---- WandB 初始化 ----
    if args.use_wandb:
        import wandb
        wandb.init(
            project=args.wandb_project,
            name=args.wandb_run_name,
            config={
                **config,
                'lr': args.lr,
                'batch_size': args.batch_size,
                'gradient_accumulation_steps': gradient_accumulation_steps,
                'weight_decay': args.weight_decay,
                'warmup_steps': args.warmup_steps,
                'total_steps': total_steps,
            },
        )

    # ---- 从 checkpoint 恢复 ----
    start_step = 0
    best_val_loss = float('inf')

    if args.resume_from:
        if os.path.exists(args.resume_from):
            start_step, _ = load_checkpoint(
                model, optimizer, scheduler, args.resume_from, device
            )

    # ---- 训练循环 ----
    model.train()
    step = start_step
    total_loss = 0.0
    start_time = time.time()
    tokens_processed = 0

    # 无限循环，直到 max_steps
    epoch = 0
    while step < total_steps:
        epoch += 1
        print(f"\n{'='*50}\nEpoch {epoch}\n{'='*50}")

        for batch_idx, (input_ids, labels) in enumerate(train_loader):
            input_ids = input_ids.to(device)
            labels = labels.to(device)

            # 前向传播
            loss, logits = model(input_ids, labels)

            # 梯度累积缩放
            loss = loss / gradient_accumulation_steps
            loss.backward()

            total_loss += loss.item() * gradient_accumulation_steps  # 还原实际 loss
            tokens_processed += input_ids.numel()

            # 梯度累积步
            if (batch_idx + 1) % gradient_accumulation_steps == 0:
                # 梯度裁剪
                torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

                # 参数更新
                optimizer.step()
                scheduler.step()
                optimizer.zero_grad()

                step += 1

                # ---- 日志 ----
                if step % args.log_interval == 0:
                    elapsed = time.time() - start_time
                    avg_loss = total_loss / args.log_interval
                    tokens_per_sec = tokens_processed / elapsed
                    lr = scheduler.get_last_lr()[0]

                    print(f"Step {step}/{total_steps} | "
                          f"Loss: {avg_loss:.4f} | "
                          f"LR: {lr:.2e} | "
                          f"Tokens/s: {tokens_per_sec:.0f} | "
                          f"Elapsed: {elapsed:.1f}s")

                    if args.use_wandb:
                        wandb.log({
                            'train/loss': avg_loss,
                            'train/lr': lr,
                            'train/tokens_per_sec': tokens_per_sec,
                            'train/step': step,
                            'train/epoch': epoch,
                        })

                    total_loss = 0.0

                # ---- 验证 ----
                if step % args.eval_interval == 0:
                    val_loss, perplexity = evaluate(
                        model, val_loader, device, max_eval_batches=args.max_eval_batches
                    )
                    print(f"\n--- 验证结果 (Step {step}) ---")
                    print(f"Val Loss: {val_loss:.4f} | Perplexity: {perplexity:.2f}")

                    if args.use_wandb:
                        wandb.log({
                            'val/loss': val_loss,
                            'val/perplexity': perplexity,
                            'val/step': step,
                        })

                    # 保存最佳模型
                    if val_loss < best_val_loss:
                        best_val_loss = val_loss
                        save_checkpoint(
                            model, optimizer, scheduler, step, val_loss, config,
                            os.path.join(args.output_dir, 'best_model.pt')
                        )
                        print(f"最佳模型已更新! Val Loss: {best_val_loss:.4f}")

                # ---- Checkpoint ----
                if step % args.save_interval == 0:
                    save_checkpoint(
                        model, optimizer, scheduler, step, loss.item(), config,
                        os.path.join(args.output_dir, f'checkpoint_step{step}.pt')
                    )

                if step >= total_steps:
                    break

        # 每个 epoch 结束保存
        save_checkpoint(
            model, optimizer, scheduler, step, loss.item(), config,
            os.path.join(args.output_dir, f'checkpoint_epoch{epoch}.pt')
        )

    # ---- 训练结束 ----
    total_time = time.time() - start_time
    print(f"\n{'='*50}")
    print(f"训练完成!")
    print(f"总步数: {step}")
    print(f"总时间: {total_time:.1f}s ({total_time/3600:.1f}h)")
    print(f"最佳验证 Loss: {best_val_loss:.4f}")
    print(f"{'='*50}")

    # 最终保存
    save_checkpoint(
        model, optimizer, scheduler, step, loss.item(), config,
        os.path.join(args.output_dir, 'final_model.pt')
    )

    if args.use_wandb:
        wandb.finish()

## 4. 训练中断恢复

### 4.1 Checkpoint 策略

预训练可能持续数天、数周甚至数月。在这些情况下，**checkpoint 机制至关重要**——它不仅是"万一出问题"的保险，更是日常操作的一部分。

一个合理的 checkpoint 策略应包含：

1. **定期保存**：每隔 N 步（如 1000 步）自动保存一个 checkpoint
2. **最佳模型保存**：每次验证 loss 创新低时保存一个独立的 best checkpoint
3. **多版本保留**：保留最近 K 个 checkpoint（防止某个 checkpoint 损坏导致全部丢失）
4. **checkpoint 内容**：包含模型参数、优化器状态、学习率调度器状态、当前步数、随机种子

### 4.2 断点续练

从 checkpoint 恢复训练的步骤：

1. 加载模型参数 → `model.load_state_dict(checkpoint['model_state_dict'])`
2. 加载优化器状态 → `optimizer.load_state_dict(checkpoint['optimizer_state_dict'])`
3. 加载调度器状态 → `scheduler.load_state_dict(checkpoint['scheduler_state_dict'])`
4. 恢复步数 → `start_step = checkpoint['step']`
5. 恢复随机种子（如果存储了）→ 确保数据 shuffle 的确定性

**关键警告**：仅仅加载模型参数是不够的。如果不同时加载优化器状态（动量、二阶矩），AdamW 会"忘记"自己的历史信息，导致训练在恢复后的最初几步不稳定。

### 4.3 为什么会中断

常见的中断原因：
- **OOM（Out of Memory）**：batch_size 太大或序列太长
- **硬件故障**：GPU 错误（ECC 错误、Xid 错误）、IB 网络中断
- **电网波动**：机房电力不稳定
- **代码 bug**：数据预处理中的异常样本
- **手动停止**：需要调整超参或换了更好的 checkpoint 策略

## 6. Loss 曲线的诊断

### 6.1 正常的 Loss 曲线

一个健康的训练过程，Loss 曲线应该呈现以下特征：

```
Loss
↑
| \
|  \
|   \_____
|         \_________
|                   \___
+------------------------→ Steps
```

1. **初始快速下降**（前 10-20% 的训练步数）：模型快速学习高频模式（常见词、常见搭配）
2. **中期稳定下降**：Loss 持续但缓慢地下降，模型在学习更复杂的模式
3. **后期趋向平稳**：Loss 接近其理论下限，模型接近数据的信息极限

训练 Loss 和验证 Loss 之间通常有一个小的 gap（训练 Loss 低于验证 Loss），这是正常的——表示轻微的过拟合。如果 gap 持续扩大，则说明过拟合严重。

### 6.2 异常 Loss 曲线诊断

**Loss 不下降（flat）**：
```
Loss
↑
+------------------------------→ Steps
```

可能原因：
- 学习率太小——参数几乎不更新
- 模型初始化有问题——所有参数在相近的值上
- 数据有问题——所有样本相同或标签错误
- 梯度消失——深层网络或不当的激活函数

排查步骤：检查梯度范数。如果 `grad_norm ≈ 0`，说明梯度消失；如果 `grad_norm` 正常但 loss 不降，检查数据。

**Loss 震荡剧烈**：
```
Loss   /\/\/\/\/\/\/\/\
↑     /                \
+-----|--|--|--|--|--|--→ Steps
```

可能原因：
- 学习率太大——参数在最优解附近来回跳跃
- batch_size 太小——每个 batch 的梯度估计噪声太大
- 数据没有 shuffle 或 shuffle 不充分

**Loss 不降反升**：
```
Loss        /
↑         /
|       /
+-----/-----------------→ Steps
```

可能原因：
- 学习率过大，参数更新 overshoot 了最优解
- 数据中有异常值或脏数据比例随时间增加
- 梯度爆炸（检查 grad_norm 是否异常大）
- 数值不稳定（FP16 溢出）

**Loss 变成 NaN**：
```
Loss
↑    *
|   / \
|  /   \
+---------------------→ Steps
```

可能原因：
- 梯度爆炸导致参数变为 inf
- `log(0)` 在交叉熵出现
- 除零操作
- FP16 下溢或上溢

**典型的 Loss 曲线：训练 vs 验证分叉**：
```
Loss
↑       ___ val
|   ___/...
|  /  train
| /
+------------------------→ Steps
```

如果验证 Loss 上升而训练 Loss 继续下降，这是过拟合的经典信号。

### 6.3 调试清单

当 Loss 行为异常时，按以下顺序排查：

1. **检查数据**：随机抽样几个 batch，人工检查 tokenized 后的数据是否正确
2. **检查梯度**：记录每层的梯度范数，看是否所有层都在学习
3. **检查学习率**：尝试将学习率减小 10 倍，看 loss 是否开始下降
4. **检查数值范围**：打印 logits、loss、参数的最大/最小值，看是否有 NaN/inf
5. **单样本过拟合**：用一个很小的数据集（如 10 个样本）训练，看 loss 能否降到接近零。如果不能，说明模型或损失函数有 bug
6. **关闭混合精度**：暂时禁用 AMP/FP16，用纯 FP32 训练，排除精度问题

"单样本过拟合测试"是最强大的调试工具。如果你的模型无法在一个 batch 上把 loss 降到接近零，那就不是超参问题——是代码有问题。

## 9. 常见错误

### 错误 1：CLM 中忘记 shift labels
**症状**：Loss 异常低（接近零），或者模型学会"复制"输入。
**原因**：CLM 训练中，input 和 label 应该是错位关系（input 是 `[0:n-1]`，label 是 `[1:n]`）。如果 input == label，模型只需要学恒等映射，Loss 可以很低但模型什么也没学会。
**修复**：确保 `labels = input_ids[:, 1:]`（对齐到 logits 的对应位置）。

### 错误 2：CrossEntropy 的 ignore_index 设置不当
**症状**：Loss 值异常（如比预期大很多），或者模型在 padding 位置的学习影响了正常 token。
**原因**：padding token（如 `[PAD]` 或额外的 `[EOS]`）的位置也被计入 loss。这些位置不是真正的语言信号。
**修复**：`F.cross_entropy(..., ignore_index=pad_token_id)`。注意对于 CLM 的数据打包，应该在文档边界（EOS 后的第一个 token）处将 label 设为 ignore_index，避免跨文档的伪关系被学习。

### 错误 3：学习率 warmup 不足
**症状**：训练初期 loss 震荡或直接 NaN。
**原因**：在训练的初始阶段，模型参数是随机的，梯度的方向和量级都非常不稳定。太大的初始学习率可能导致参数迅速偏离合理范围。
**修复**：使用 warmup——在训练的前几百到几千步，学习率从一个非常小的值线性增长到目标学习率。warmup 步数通常为总步数的 1-5%。

### 错误 4：使用过大的 batch size
**症状**：模型收敛慢，最终性能差。
**原因**：过大的 batch size 会导致每个 batch 的梯度估计过于精确，减少了梯度的随机性。研究表明，适度的梯度噪声有助于模型探索更广阔的损失景观，找到更好的泛化解。
**修复**：对于小型模型（<100M），batch size 在 32-256 之间通常是最优的。如果受显存限制，使用梯度累积来增大 effective batch size，而非物理地增大 micro-batch。

### 错误 5：数据打包后忘记设置 document boundary mask
**症状**：训练没问题，但生成过程中模型跨句子产生幻觉。
**原因**：数据打包将多个文档拼接为一个训练序列。如果不设置 document boundary（在文档边界切断 Attention），模型会学到"跨文档的关系"——比如"上一个文档的结尾"和"下一个文档的开头"之间的伪关联。
**修复**：在 Attention 计算中使用 block-diagonal mask，或者至少在每个文档之间插入 EOS token，并在 EOS 后的第一个 token 的 label 设为 ignore_index。

### 错误 6：验证集评估时忘记 model.eval()
**症状**：验证 loss 看起来偏高，或者每次验证的 loss 不完全一致。
**原因**：模型中可能包含训练时和测试时行为不同的层（如 Dropout、BatchNorm）。在训练模式下的评估结果不可靠。
**修复**：评估前调用 `model.eval()`，评估后调用 `model.train()` 恢复训练模式。使用 `torch.no_grad()` 上下文管理器以节省显存。

### 错误 7：Weight Decay 也应用到了 bias 和 LayerNorm 参数
**症状**：模型训练不稳定或收敛慢。
**原因**：将 weight decay 应用到 bias、LayerNorm 的 weight/gamma 等参数是不合适的。这些参数不应该被正则化——bias 没有"过拟合"的问题（因为它是加性的，不影响模型的复杂度），LayerNorm 的 gamma 应该保持接近 1。
**修复**：在优化器中分离参数组，只对 weight 矩阵（Linear 层的 weight）应用 weight decay：

```python
decay_params = [p for n, p in model.named_parameters()
                if p.dim() >= 2]  # 只衰减 >= 2D 的权重
no_decay_params = [p for n, p in model.named_parameters()
                   if p.dim() < 2]  # bias 和 layer norm 参数
optimizer = AdamW([
    {'params': decay_params, 'weight_decay': 0.1},
    {'params': no_decay_params, 'weight_decay': 0.0},
])
```

### 错误 8：torch.compile 与某些操作不兼容
**症状**：使用 `torch.compile` 后训练变慢或报错。
**原因**：`torch.compile` 将 eager 模式的代码编译为 graph。某些动态操作（如依赖于数据内容的控制流、动态形状等）会导致 graph break，产生"编译-退回-重新编译"的循环，反而降低性能。
**修复**：设置 `TORCH_LOGS="+dynamo"` 和 `TORCHDYNAMO_VERBOSE=1` 查看 graph break，针对性地修复不兼容的操作。

## 10. 本章小结

预训练是 LLM 生命周期中最重要的阶段。本章的核心要点：

1. **CLM（自回归）** 是 GPT 式模型的核心预训练目标：给定上文预测下文，训练和推理完全对齐。MLM（掩码语言模型）适合理解任务但无法直接生成。
2. **模型架构可以从参数量反推**：`P ≈ n_layers × 12 × d_model²`（标准 FFN）或约 `n_layers × 16 × d_model²`（标准 FFN 的保守估计）。通过调整层数和维度可以控制模型规模。
3. **学习率调度**（warmup + cosine decay）和**梯度累积**是稳定训练的关键技术。warmup 防止初期震荡，cosine decay 优雅地结束训练。
4. **Checkpoint 机制**不仅仅是备份——它保存了训练的全状态，允许从任意步数恢复。只保存模型参数是不够的。
5. **Loss 曲线的解读**是预训练工程师的核心技能。震荡、平坦、NaN、分叉——每种模式都指向特定的问题。
6. **单样本过拟合**是最有用的调试工具。模型不能在一个 batch 上收敛到接近零的 loss = 代码有 bug，不是超参问题。

从头训练一个小型 GPT 是理解 LLM 全貌的最佳途径。当你亲手走完数据准备→模型定义→训练→生成的全过程，你对大模型的理解将不再是黑箱。
