## 1. 前向传播：神经网络如何"看"数据

### 1.1 从单个神经元讲起

想象你面前有一个最简单的计算单元，它接收一组输入信号 `x₁, x₂, ..., xₙ`，每个信号都配有一个权重 `w₁, w₂, ..., wₙ`，表示这个信号的重要程度。此外，还有一个偏置项 `b`，可以理解为"当所有输入都为零时，神经元的基础激活水平"。

这个神经元做的事情非常简单：

```
z = w₁·x₁ + w₂·x₂ + ... + wₙ·xₙ + b
a = σ(z)
```

其中 `σ` 是激活函数，它引入了非线性。如果不用激活函数，多层网络会退化为单层线性变换——无论堆多少层，最终都等价于一个线性模型。正是激活函数让神经网络具备了拟合任意复杂函数的能力。

常见的激活函数有：
- **Sigmoid**：`σ(z) = 1/(1+e^(-z))`，输出范围 (0, 1)，适合二分类输出层。缺点是梯度饱和问题——当 `|z|` 很大时，梯度趋近于零，导致参数更新停滞。
- **ReLU**：`σ(z) = max(0, z)`，计算简单，缓解了梯度消失问题。缺点是"死神经元"现象——当输入恒为负时，梯度始终为零，神经元永远无法激活。
- **GELU**：`σ(z) ≈ z · Φ(z)`，其中 Φ 是标准正态分布的 CDF。这是 Transformer 中常用的激活函数，它比 ReLU 更平滑，性能更好。

### 1.2 从神经元到层，从层到网络

一个全连接层（也叫线性层或 Dense 层）就是多个神经元的并联：

```
Z = X · W^T + b
A = σ(Z)
```

其中：
- `X` 的形状是 `(batch_size, in_features)`，每一行是一个样本
- `W` 的形状是 `(out_features, in_features)`，每一行是一个神经元的权重向量
- `b` 的形状是 `(out_features,)`，通过广播加到每一行
- `Z` 的形状是 `(batch_size, out_features)`，是激活前的"logits"
- `A` 的形状同 `Z`，是激活后的输出，将作为下一层的输入

当我们把多个这样的层串联起来，就得到了一个多层感知机（MLP）：

```
输入层（特征） → 隐藏层1（ReLU激活） → 隐藏层2（ReLU激活） → ... → 输出层（可选激活）
```

**为什么需要隐藏层？** 单层网络只能拟合线性决策边界。通过堆叠非线性层，网络可以学习层次化的特征表示：第一层学习边缘，第二层学习形状，第三层学习部件，更高层学习完整物体。这就是深度学习"深度"的含义。

### 1.3 前向传播的完整计算

以一个具体的三层网络为例（输入784维 → 隐藏层128维 → 隐藏层64维 → 输出10类）：

```
第0层（输入）:   A⁰ = X                       # (batch, 784)
第1层:           Z¹ = A⁰ @ W¹ᵀ + b¹          # (batch, 128)
                 A¹ = ReLU(Z¹)                 # (batch, 128)
第2层:           Z² = A¹ @ W²ᵀ + b²          # (batch, 64)
                 A² = ReLU(Z²)                 # (batch, 64)
第3层（输出）:   Z³ = A² @ W³ᵀ + b³          # (batch, 10)
                 A³ = softmax(Z³)              # (batch, 10)，每行是概率分布
```

**前向传播的本质**是：将输入数据经过一系列线性变换（矩阵乘法）和非线性变换（激活函数），逐步映射到目标空间。可以把这个过程想象成数据流过一个多级管道，每一级对数据进行一次"加工"。

## 2. 反向传播：梯度如何流回源头

### 2.1 为什么需要反向传播

训练神经网络的目标是：找到一组参数（所有权重和偏置），使得模型在训练数据上的损失最小。这是一个优化问题。梯度下降法的核心思想是：沿着损失函数关于参数的负梯度方向走一小步，参数就会向着减小损失的方向移动。

但问题是：损失函数离参数"很远"——中间隔着很多层变换。我们如何计算损失对第1层权重的梯度？答案是**链式法则**。

### 2.2 链式法则的直观理解

链式法则是微积分中的一个基本定理：如果 `L = f(g(h(x)))`，那么：

```
dL/dx = dL/df · df/dg · dg/dh · dh/dx
```

在神经网络中，损失函数是一个非常深的复合函数。前向传播时，数据从输入层一层层流向输出层；反向传播时，梯度从输出层一层层流回输入层——每一步都应用一次链式法则。

**关键直觉**：反向传播不是一种新的算法，它只是链式法则在计算图上的高效实现。与其独立计算每个参数的梯度（那样会有大量重复计算），不如从输出端向输入端"反向"传播，一次计算出所有参数的梯度。

### 2.3 反向传播的逐步推导

让我们推导上面三层网络的反向传播过程。记损失为 `L`，损失对某个量的梯度记为 `δ = ∂L/∂(·)`。

**第1步：计算损失对输出层 logits 的梯度**

假设使用交叉熵损失：
```
L = -Σ y_true · log(y_pred)
```
其中 `y_pred = softmax(Z³)`，交叉熵配合 softmax 有一个优美的性质：
```
∂L/∂Z³ = y_pred - y_true    # 形状: (batch, 10)
```
这个结果非常简洁——梯度就是"预测值减去真实值"。这也是为什么分类任务几乎都用 softmax + cross-entropy 的组合。

**第2步：回传到第2层的激活值**

根据链式法则：
```
∂L/∂A² = (∂L/∂Z³) @ W³    # 形状: (batch, 64)
```
这里 `W³` 不使用转置，因为梯度的流动方向与前向相反。

**第3步：通过 ReLU 回传**

ReLU 的梯度很简单：如果输入 > 0，梯度为 1；否则为 0。
```
∂L/∂Z² = ∂L/∂A² ⊙ (Z² > 0)    # 逐元素乘法
```

**第4步：回传到第1层的激活值**
```
∂L/∂A¹ = (∂L/∂Z²) @ W²
```

**第5步：通过 ReLU 回传**
```
∂L/∂Z¹ = ∂L/∂A¹ ⊙ (Z¹ > 0)
```

**第6步：计算所有参数的梯度**

对于每一层的权重和偏置：
```
∂L/∂W³ = (∂L/∂Z³)ᵀ @ A² / batch_size    # 除以 batch_size 取平均
∂L/∂b³ = mean(∂L/∂Z³, axis=0)
∂L/∂W² = (∂L/∂Z²)ᵀ @ A¹ / batch_size
∂L/∂b² = mean(∂L/∂Z², axis=0)
∂L/∂W¹ = (∂L/∂Z¹)ᵀ @ A⁰ / batch_size
∂L/∂b¹ = mean(∂L/∂Z¹, axis=0)
```

**关键观察**：
1. 计算每层的权重梯度需要该层的"输入激活值"（前向传播时算出的）和"该层收到的梯度"（反向传播时算出的）。
2. 这就是为什么我们需要保存前向传播的中间结果——它们将在反向传播时被使用。
3. 梯度的形状和前向传播时的形状有着优雅的对应关系。

## 3. 常见损失函数

### 3.1 均方误差 MSE

**数学形式**：
```
MSE = (1/n) · Σ(y_pred - y_true)²
```

**适用场景**：回归任务。当你的目标是预测一个连续值（房价、温度、分数），MSE 是最自然的选择。

**为什么用平方而不是绝对值？** 平方函数有几个好处：
- 可导且导数连续（绝对值在零点不可导）
- 对大误差惩罚更重（平方放大了大误差的影响），使模型更关注"离谱"的预测
- 数学性质好——在高斯噪声假设下，最小化 MSE 等价于最大似然估计

**梯度**：
```
∂MSE/∂y_pred = 2(y_pred - y_true) / n
```
梯度与误差成正比——预测越离谱，梯度越大，参数更新幅度越大。

### 3.2 交叉熵 CrossEntropy

**数学形式**：
```
CrossEntropy = -Σ y_true · log(y_pred)
```
其中 `y_true` 是 one-hot 向量（或更一般的概率分布），`y_pred` 是预测概率。

**适用场景**：分类任务。交叉熵衡量的是两个概率分布之间的差异。当预测分布与真实分布完全一致时，交叉熵最小。

**为什么不用 MSE 做分类？** 有两个原因：
1. **梯度问题**：当预测概率接近 0 或 1 时，sigmoid/softmax 的梯度趋近于 0（饱和区），而 MSE 的梯度和 sigmoid 的梯度相乘后会变得更小，导致学习极其缓慢。交叉熵的梯度直接是 `y_pred - y_true`，不会受到激活函数饱和的影响。
2. **概率解释**：交叉熵来源于最大似然估计。最小化交叉熵等价于最大化在给定参数下观测到训练数据的概率。

**注意**：在 PyTorch 中，`nn.CrossEntropyLoss` 内部已经包含了 softmax（或 log_softmax），所以传入的应该是 logits 而不是概率。

### 3.3 二元交叉熵 BCE

```
BCE = -[y·log(p) + (1-y)·log(1-p)]
```
适用于二分类任务。当 `y=1` 时，损失为 `-log(p)`，预测越接近 1 损失越小；当 `y=0` 时，损失为 `-log(1-p)`，预测越接近 0 损失越小。

## 4. 正则化技术

### 4.1 为什么需要正则化

神经网络有极强的拟合能力。一个足够大的网络完全可以"记住"整个训练集——所有训练样本的损失降到零。但这样的模型在新数据上表现会很差。这就是**过拟合**。

过拟合的根源是模型过于复杂，它学习了训练数据中的噪声而非真正的模式。正则化就是用来限制模型复杂度、防止过拟合的技术。

### 4.2 L1 正则化

**数学形式**：在损失函数上加上权重的绝对值之和乘以系数 λ：
```
L_total = L_original + λ · Σ|w|
```

**为什么 L1 能防止过拟合？** 从优化的角度看，L1 正则化倾向于产生**稀疏权重**——很多权重正好为零。其背后的几何直觉是：L1 正则化的约束区域是一个菱形（在二维空间），菱形顶点在坐标轴上，损失函数的等高线更容易碰到顶点，从而产生零权重。零权重意味着对应特征被完全忽略，模型自动做了特征选择。

**梯度贡献**：L1 的梯度是 `λ · sign(w)`，即权重的符号。这意味着无论权重大小，L1 都会以一个恒定力度把权重往零推。小权重容易被推到零，大权重则持续被推向零但力度不变。

### 4.3 L2 正则化（权重衰减）

**数学形式**：
```
L_total = L_original + (λ/2) · Σ w²
```

**为什么 L2 更常用？** L2 正则化倾向于产生**小但非零的权重**。它让所有权重都变小，但不会精确为零。可以这样理解：如果一个特征的权重很大，说明模型过度依赖这个特征；L2 惩罚大权重，迫使模型"分散注意力"到更多特征上，从而学到更鲁棒的模式。

**权重衰减的称呼来源**：L2 正则化在梯度更新中等价于 `w = w - lr·(∂L/∂w + λ·w) = (1 - lr·λ)·w - lr·∂L/∂w`，即在每次更新前先将权重"衰减"一点。这也是为什么它被称为 Weight Decay。

**L1 vs L2 的选择**：
- 如果你怀疑只有少数特征是真正有用的，选 L1（特征选择）
- 如果你认为大多数特征都有用只是不应该过度依赖某几个，选 L2（权重衰减）
- 实际项目中 L2 更常用，因为大多数情况下我们不希望完全丢弃特征

### 4.4 Dropout

**核心思想**：在每次训练迭代中，随机"丢弃"一部分神经元（将它们的输出设为零），概率通常为 0.2 到 0.5。测试时不丢弃，但将输出乘以保留概率以保持期望值一致。

**为什么 Dropout 有效？** 有两种主流解释：

1. **集成学习解释**：每次 Dropout 产生一个不同的"子网络"（因为不同的神经元被丢弃）。训练过程中相当于训练了指数级数量的子网络，测试时相当于这些子网络的集成（ensemble）。集成的预测通常比单个模型更稳定、泛化更好。

2. **特征共适应解释**：没有 Dropout 时，神经元之间可能形成"共适应"关系——比如神经元 A 专门弥补神经元 B 的缺陷。这种关系在测试集上不一定成立，导致泛化差。Dropout 迫使每个神经元都学会独立工作（因为它的合作者随时可能消失），从而学到了更鲁棒的特征。

**何时用 Dropout？**
- 适用于全连接层，较少用于卷积层（卷积层本身就有参数共享，过拟合风险较低）
- 通常不在输入层使用（丢弃输入特征太粗暴）
- 现代 Transformer 中较少使用 Dropout，更多依赖 LayerNorm 和 Residual Connection 来正则化

## 5. BatchNorm vs LayerNorm

### 5.1 归一化的通用动机

深度学习训练中有一个核心痛点：**内部协变量偏移**（Internal Covariate Shift）。随着训练的进行，每一层的输入分布会发生变化（因为前一层参数在更新），这迫使后面的层不断适应新的输入分布，降低了训练效率。

归一化层的目标就是：对每一层的输入（或输出）进行标准化，使其均值为 0、方差为 1，然后再通过可学习的缩放（γ）和平移（β）参数恢复表达能力。

### 5.2 Batch Normalization

**归一化方向**：沿 batch 维度归一化。对于特征图中的每个通道（channel），在所有样本上计算均值和方差。

```
μ_c = mean(X_b,c,h,w over batch)   # 每个通道一个均值
σ²_c = var(X_b,c,h,w over batch)   # 每个通道一个方差
X̂ = (X - μ_c) / √(σ²_c + ε)
Y = γ_c · X̂ + β_c
```

**注意**：归一化是在 batch_size × height × width 这三维上做的，但 channels（特征图）维度保持独立。

**为什么 CV 用 BN？**
- CV 任务中，batch 维度通常较大（比如 32、64、128），统计量比较稳定
- 不同样本的同一通道具有相似的语义含义（如"边缘检测器"）
- 卷积的参数共享使得通道间的归一化统计量具有物理意义
- BN 带来的轻微噪声（因为只用 mini-batch 统计量而非全局统计量）恰好起到正则化作用

**BN 的缺点**：
- 对 batch_size 敏感：batch_size 很小时，均值和方差的估计不稳定
- 不适合 RNN/Transformer：序列长度不固定，不同时间步不能共享归一化参数
- 训练和测试不一致：训练时用 mini-batch 统计量，测试时用全局统计量（running mean/var），这个不一致容易出 bug

### 5.3 Layer Normalization

**归一化方向**：沿特征维度归一化。对每个样本的每一层，在所有特征维度上计算均值和方差。

```
μ_b,l = mean(X_b,l,c over features)   # 每个样本每个位置一个均值
σ²_b,l = var(X_b,l,c over features)   # 每个样本每个位置一个方差
X̂ = (X - μ_b,l) / √(σ²_b,l + ε)
Y = γ_c · X̂ + β_c
```

**注意**：这里的"层"是指特征维度，不是网络层。在 Transformer 中，就是对每个 token 的 embedding 向量进行归一化。

**为什么 NLP 用 LN？**
- 序列长度不固定：BN 需要在序列维度上有统一的统计量，但 NLP 中句子长度不同
- Batch 维度通常较小（受显存限制）：小 batch_size 下 BN 不稳定
- 每个 token 的语义是独立的：对 token embedding 进行归一化不会破坏语义信息
- LN 在每个样本内部归一化，不受其他样本影响，训练和测试行为一致

**BN vs LN 总结**：

| 方面 | BatchNorm | LayerNorm |
|------|-----------|-----------|
| 归一化维度 | 按 batch + 空间 | 按特征/通道 |
| 最佳 batch_size | >= 16 | 任意 |
| 序列任务 | 不适合 | 适合 |
| 训练/测试一致性 | 不一致 | 一致 |
| 主要应用 | CNN（CV） | Transformer（NLP） |

### 5.4 其他归一化变体

- **InstanceNorm**：对每个样本的每个通道分别归一化（H×W 维度上）。用于图像风格迁移，因为风格信息更多体现在通道统计量而非空间位置上。
- **GroupNorm**：把通道分成若干组，每组内归一化。BatchNorm 和 InstanceNorm 的折中，batch_size 很小时效果好，常用于目标检测任务。

## 6. 优化器的数学

### 6.1 随机梯度下降 SGD

**更新公式**：
```
θ_{t+1} = θ_t - η · g_t
```
其中 `η` 是学习率，`g_t = ∇L(θ_t)` 是梯度。

**SGD 的问题**：
- 学习率固定：梯度的量级在不同参数上可能差异巨大（如 biases 和 weights），单一学习率难以适应
- 容易陷入鞍点：在高维空间中，鞍点（梯度为零但非极值）比局部极小值多得多，SGD 容易被卡住
- 峡谷震荡：在狭长的损失曲面中（一个方向陡峭、另一个方向平缓），SGD 会来回震荡，前进缓慢

### 6.2 Momentum（动量）

**核心思想**：引入"速度"概念，参数的更新不仅考虑当前梯度，还考虑过去的更新方向。就像物理中的惯性——球滚下山时，即使遇到小坑（梯度为零），它也会凭借惯性继续前进。

**更新公式**：
```
v_t = β · v_{t-1} + η · g_t
θ_{t+1} = θ_t - v_t
```
其中 `β` 是动量系数，通常取 0.9。当 `β = 0` 时退化为 SGD。

**为什么有效？**
- 加速收敛：在梯度方向一致的维度上，动量累积，步伐越来越大
- 减小震荡：在梯度方向反复变化的维度上，正负更新相互抵消，步伐保持较小
- 逃离局部极小值：惯性能帮助冲过浅的局部极小值

### 6.3 Nesterov 加速梯度 NAG

Nesterov 对 Momentum 做了一个聪明的改进：先按动量方向走一步，在那个"前瞻"位置计算梯度。

```
θ_lookahead = θ_t - β · v_{t-1}
v_t = β · v_{t-1} + η · ∇L(θ_lookahead)
θ_{t+1} = θ_t - v_t
```

可以这样直观理解：Momentum 是"先算当前的梯度，然后和之前的速度混合"；NAG 是"先按惯性走一步，到了新位置再算梯度"。NAG 通常比普通 Momentum 收敛更快，因为它能"提前"感知到梯度变化。

### 6.4 AdaGrad

**核心思想**：不同参数使用不同的学习率。对频繁更新的参数减小学习率，对偶尔更新的参数增大学习率。

```
s_t = s_{t-1} + g_t²          # 累积梯度平方和
θ_{t+1} = θ_t - η · g_t / √(s_t + ε)
```

**问题**：`s_t` 单调递增，导致学习率持续衰减，最终变得极小，训练提前停止。不适合训练到收敛的场景。

### 6.5 RMSprop

**改进**：把 AdaGrad 的累积改为指数移动平均，让过去的梯度平方逐渐"遗忘"。

```
s_t = β · s_{t-1} + (1-β) · g_t²
θ_{t+1} = θ_t - η · g_t / √(s_t + ε)
```

RMSprop 解决了学习率单调衰减的问题，同时在非平稳目标（如 RNN 训练）上表现优秀。

### 6.6 Adam

**Adam = Momentum + RMSprop**

Adam 是目前最广泛使用的优化器之一。它同时维护：
- `m_t`：梯度的指数移动平均（一阶矩，类似动量）
- `v_t`：梯度平方的指数移动平均（二阶矩，类似 RMSprop）

```
m_t = β₁ · m_{t-1} + (1-β₁) · g_t
v_t = β₂ · v_{t-1} + (1-β₂) · g_t²
m̂_t = m_t / (1-β₁ᵗ)          # 偏差校正
v̂_t = v_t / (1-β₂ᵗ)          # 偏差校正
θ_{t+1} = θ_t - η · m̂_t / (√v̂_t + ε)
```

**为什么需要偏差校正？** 在训练初期（`t` 很小），`m_0 = v_0 = 0`，导致 `m_t` 和 `v_t` 偏向零。除以 `(1-βᵗ)` 可以将这个偏差修正回来。随着 `t` 增大，校正因子趋近于 1，影响消失。

**Adam 的优势**：
- 自适应学习率：每个参数有独立的学习率，不需要精细调参
- 结合动量：加速收敛，减少震荡
- 鲁棒性强：对超参数不太敏感，默认值（`β₁=0.9, β₂=0.999, ε=1e-8`）在大多数任务上效果良好

### 6.7 AdamW

**问题发现**：研究者发现，在 Adam 中直接使用 L2 正则化（在损失函数中加 `λ‖w‖²/2`）和权重衰减并不等价。因为 Adam 的自适应学习率机制，L2 正则化的梯度会被 `√v̂` 缩小，导致大梯度参数的正则化效果被削弱——而这恰恰是正则化最需要的参数。

**AdamW 的解决方案**：将权重衰减从损失函数中解耦，直接在参数更新步骤中应用：

```
θ_{t+1} = θ_t - η · m̂_t / (√v̂_t + ε) - η·λ·θ_t
                                       ^^^^^^^^
                                       独立的权重衰减项
```

这个看似微小的改动带来了显著的效果提升。在现代 Transformer 训练中，AdamW 几乎是标准配置。

### 6.8 优化器选择指南

| 场景 | 推荐优化器 |
|------|-----------|
| 快速实验/简单任务 | Adam（默认参数） |
| Transformer 训练 | AdamW + 学习率 warmup |
| CV 分类/Big Transfer | SGD + Momentum + Cosine Decay |
| 强化学习 | RMSprop |
| 需要最好泛化性能 | SGD（充分调参后泛化通常更好） |

## 7. NumPy 从零实现 MLP 训练

下面的代码是我们本章理论知识的完整实践。我们将用纯 NumPy 实现：
1. 前向传播（线性层 + ReLU + softmax）
2. 交叉熵损失
3. 反向传播（手动推导每一步梯度）
4. SGD 参数更新
5. 在 MNIST 数据集上训练一个三层 MLP

**设计理念**：我们不使用任何自动微分机制。每一步梯度都手动计算，与上文的理论推导一一对应。这样做不是为了效率，而是为了让你真正理解神经网络的每个运算。

```python
"""
从零实现 MLP 训练（纯 NumPy）
=============================
本章使用纯 NumPy 实现一个三层 MLP 在 MNIST 上的训练过程。
不使用任何深度学习框架，每一步梯度都手动计算。

目标：
1. 深入理解前向传播和反向传播的每一步计算
2. 理解 loss 曲线的含义
3. 感受手写梯度和框架自动微分的区别

作者：Python+AI 学习教程
"""

import numpy as np
import gzip
import os
import struct
from pathlib import Path


# ============================================================
# 1. 数据加载
# ============================================================

def load_mnist(data_dir="."):
    """
    从本地加载 MNIST 数据集（IDX 格式）。
    如果没有文件，会自动生成随机数据用于演示。
    如果你有 MNIST 原始文件，请放在 data_dir 下：
    - train-images-idx3-ubyte.gz
    - train-labels-idx1-ubyte.gz
    - t10k-images-idx3-ubyte.gz
    - t10k-labels-idx1-ubyte.gz

    返回: (X_train, y_train, X_test, y_test)
    """
    def read_idx(filename):
        """读取 IDX 格式文件"""
        with gzip.open(filename, 'rb') as f:
            zero, data_type, dims = struct.unpack('>HBB', f.read(4))
            shape = tuple(struct.unpack('>I', f.read(4))[0] for _ in range(dims))
            return np.frombuffer(f.read(), dtype=np.uint8).reshape(shape)

    files = [
        os.path.join(data_dir, 'train-images-idx3-ubyte.gz'),
        os.path.join(data_dir, 'train-labels-idx1-ubyte.gz'),
        os.path.join(data_dir, 't10k-images-idx3-ubyte.gz'),
        os.path.join(data_dir, 't10k-labels-idx1-ubyte.gz')
    ]

    if all(os.path.exists(f) for f in files):
        X_train = read_idx(files[0]).astype(np.float32) / 255.0
        y_train = read_idx(files[1])
        X_test = read_idx(files[2]).astype(np.float32) / 255.0
        y_test = read_idx(files[3])
    else:
        # 如果没有 MNIST 文件，生成模拟数据用于演示
        print("未找到 MNIST 文件，使用随机模拟数据（仅供代码验证）")
        np.random.seed(42)
        X_train = np.random.randn(1000, 784).astype(np.float32)
        y_train = np.random.randint(0, 10, (1000,))
        X_test = np.random.randn(200, 784).astype(np.float32)
        y_test = np.random.randint(0, 10, (200,))

    # 展平图像：从 (N, 28, 28) 到 (N, 784)
    if X_train.ndim == 3:
        X_train = X_train.reshape(X_train.shape[0], -1)
    if X_test.ndim == 3:
        X_test = X_test.reshape(X_test.shape[0], -1)

    return X_train, y_train, X_test, y_test


# ============================================================
# 2. 激活函数
# ============================================================

def relu(z):
    """ReLU 激活函数: max(0, z)"""
    return np.maximum(0, z)


def relu_derivative(z):
    """ReLU 的导数: z > 0 时为 1，否则为 0"""
    return (z > 0).astype(np.float32)


def softmax(z):
    """
    Softmax 函数（数值稳定版本）。
    对每行（每个样本）计算 softmax，将 logits 转为概率分布。

    数值稳定性技巧：每个样本减去该样本的最大 logit 值，
    防止 e^z 数值溢出（当 z 很大时 e^z 会超出 float32 范围）。
    """
    # 减去每行最大值，防止数值溢出
    z_stable = z - np.max(z, axis=1, keepdims=True)
    exp_z = np.exp(z_stable)
    return exp_z / np.sum(exp_z, axis=1, keepdims=True)


# ============================================================
# 3. 损失函数
# ============================================================

def cross_entropy_loss(probs, labels):
    """
    交叉熵损失。

    参数:
        probs: softmax 输出的概率，形状 (N, num_classes)
        labels: 整数标签，形状 (N,)

    返回:
        loss: 平均交叉熵损失（标量）

    数学:
        L = -1/N * Σ log(probs[i, labels[i]])
    """
    N = probs.shape[0]
    # 取出每个样本对应正确类别的概率
    correct_log_probs = -np.log(probs[np.arange(N), labels] + 1e-12)
    loss = np.sum(correct_log_probs) / N
    return loss


def cross_entropy_gradient(probs, labels):
    """
    交叉熵 + softmax 的联合梯度。

    参数:
        probs: softmax 输出的概率，形状 (N, num_classes)
        labels: 整数标签，形状 (N,)

    返回:
        grad: 对 logits 的梯度，形状 (N, num_classes)

    数学:
        ∂L/∂z_i = p_i - y_i
        其中 p_i 是预测概率，y_i 是 one-hot 真实标签。
    """
    N = probs.shape[0]
    # 构造 one-hot 标签
    y_onehot = np.zeros_like(probs)
    y_onehot[np.arange(N), labels] = 1.0
    grad = (probs - y_onehot) / N
    return grad


# ============================================================
# 4. 网络层和前向传播
# ============================================================

class Linear:
    """全连接层"""

    def __init__(self, in_features, out_features):
        # 权重初始化：使用 He 初始化（适合 ReLU 激活）
        # W ~ N(0, sqrt(2/in_features))
        self.W = np.random.randn(out_features, in_features).astype(np.float32) * np.sqrt(2.0 / in_features)
        self.b = np.zeros(out_features, dtype=np.float32)

        # 缓存前向传播的中间值，供反向传播使用
        self.cache = {}

    def forward(self, x):
        """
        前向传播: Z = X @ W^T + b

        参数:
            x: 输入，形状 (N, in_features)
        返回:
            z: 输出，形状 (N, out_features)
        """
        self.cache['x'] = x  # 保存输入，反向传播计算 dW 时需要
        z = x @ self.W.T + self.b
        return z

    def backward(self, dout):
        """
        反向传播。

        参数:
            dout: 上游梯度 ∂L/∂Z，形状 (N, out_features)
        返回:
            dx: 对输入的梯度 ∂L/∂X，形状 (N, in_features)
        """
        x = self.cache['x']
        N = x.shape[0]

        # 计算梯度（除以 N 得到平均梯度）
        self.dW = (dout.T @ x) / N     # 形状: (out_features, in_features)
        self.db = np.mean(dout, axis=0)  # 形状: (out_features,)
        dx = dout @ self.W              # 形状: (N, in_features)

        return dx


class ReLU:
    """ReLU 激活层"""

    def forward(self, z):
        self.cache = {'z': z}
        return relu(z)

    def backward(self, dout):
        z = self.cache['z']
        return dout * relu_derivative(z)


# ============================================================
# 5. 完整网络
# ============================================================

class ThreeLayerMLP:
    """
    三层 MLP: input(784) -> hidden1(128) -> hidden2(64) -> output(10)

    结构:
        X -> Linear1 -> ReLU1 -> Linear2 -> ReLU2 -> Linear3 -> Softmax -> Loss
    """

    def __init__(self):
        self.fc1 = Linear(784, 128)
        self.relu1 = ReLU()
        self.fc2 = Linear(128, 64)
        self.relu2 = ReLU()
        self.fc3 = Linear(64, 10)

        # 将所有可训练参数放在一起，方便更新
        self.params = [self.fc1, self.fc2, self.fc3]
        self.layers = [self.fc1, self.relu1, self.fc2, self.relu2, self.fc3]

    def forward(self, x):
        """
        前向传播：数据流经所有层。

        参数:
            x: 输入图像（已展平），形状 (N, 784)
        返回:
            probs: 类别概率，形状 (N, 10)
        """
        out = x
        for layer in self.layers:
            out = layer.forward(out)
        # 最后一层是 Linear，输出 logits，需要单独 softmax
        # 注意：我们在 forward 中不做 softmax，因为 cross_entropy_gradient 需要 logits
        self.logits = out
        probs = softmax(out)
        return probs

    def backward(self, dout):
        """
        反向传播：梯度从输出层依次回传到输入层。
        注意遍历顺序是反向的。

        参数:
            dout: 损失对 logits 的梯度
        """
        grad = dout
        for layer in reversed(self.layers):
            grad = layer.backward(grad)

    def update(self, lr):
        """使用 SGD 更新所有参数"""
        for layer in self.params:
            layer.W -= lr * layer.dW
            layer.b -= lr * layer.db

    def predict(self, x):
        """预测类别"""
        probs = self.forward(x)
        return np.argmax(probs, axis=1)


# ============================================================
# 6. 训练循环
# ============================================================

def compute_accuracy(predictions, labels):
    """计算分类准确率"""
    return np.mean(predictions == labels)


def train(model, X_train, y_train, X_val, y_val,
          epochs=20, batch_size=64, lr=0.01, print_every=1):
    """
    完整的训练循环。

    参数:
        model: ThreeLayerMLP 实例
        X_train, y_train: 训练数据
        X_val, y_val: 验证数据
        epochs: 训练轮数
        batch_size: 批量大小
        lr: 学习率
        print_every: 每隔几个 epoch 打印一次信息
    """
    N = X_train.shape[0]
    num_batches = N // batch_size
    history = {'train_loss': [], 'val_loss': [], 'val_acc': []}

    for epoch in range(1, epochs + 1):
        # ---- 打乱训练数据 ----
        idx = np.random.permutation(N)
        X_shuffled = X_train[idx]
        y_shuffled = y_train[idx]

        epoch_loss = 0.0

        # ---- 按 batch 训练 ----
        for b in range(num_batches):
            start = b * batch_size
            end = start + batch_size
            X_batch = X_shuffled[start:end]
            y_batch = y_shuffled[start:end]

            # 步骤1: 前向传播
            probs = model.forward(X_batch)

            # 步骤2: 计算损失
            loss = cross_entropy_loss(probs, y_batch)
            epoch_loss += loss

            # 步骤3: 反向传播
            dout = cross_entropy_gradient(probs, y_batch)  # ∂L/∂logits
            model.backward(dout)

            # 步骤4: 参数更新
            model.update(lr)

        # ---- 每个 epoch 结束后评估 ----
        avg_train_loss = epoch_loss / num_batches
        history['train_loss'].append(avg_train_loss)

        # 验证集评估
        val_probs = model.forward(X_val)
        val_loss = cross_entropy_loss(val_probs, y_val)
        val_preds = model.predict(X_val)
        val_acc = compute_accuracy(val_preds, y_val)

        history['val_loss'].append(val_loss)
        history['val_acc'].append(val_acc)

        if epoch % print_every == 0:
            print(f"Epoch {epoch:3d}/{epochs} | "
                  f"Train Loss: {avg_train_loss:.4f} | "
                  f"Val Loss: {val_loss:.4f} | "
                  f"Val Acc: {val_acc:.4f}")

    return history


# ============================================================
# 7. 主程序
# ============================================================

if __name__ == "__main__":
    print("=" * 60)
    print("从零实现 MLP 训练（纯 NumPy）")
    print("=" * 60)

    # ---- 加载数据 ----
    X, y, X_test, y_test = load_mnist()

    # 划分训练/验证集（9:1）
    split = int(0.9 * len(X))
    X_train, X_val = X[:split], X[split:]
    y_train, y_val = y[:split], y[split:]

    print(f"训练样本: {X_train.shape[0]}, 验证样本: {X_val.shape[0]}, 测试样本: {X_test.shape[0]}")
    print(f"输入维度: {X_train.shape[1]}, 类别数: 10")

    # ---- 创建模型 ----
    model = ThreeLayerMLP()

    # 打印模型参数数量
    total_params = 0
    for p, layer in enumerate(model.params):
        n_params = layer.W.size + layer.b.size
        total_params += n_params
        print(f"Layer {p+1}: W={layer.W.shape}, b={layer.b.shape}, 参数数={n_params:,}")
    print(f"总参数数: {total_params:,}")

    # ---- 训练 ----
    print("\n开始训练...\n")
    history = train(model, X_train, y_train, X_val, y_val,
                    epochs=20, batch_size=64, lr=0.01)

    # ---- 测试 ----
    test_preds = model.predict(X_test)
    test_acc = compute_accuracy(test_preds, y_test)
    print(f"\n测试集准确率: {test_acc:.4f}")

    # ---- 简要分析 ----
    print("\n训练总结:")
    print(f"  初始训练损失: {history['train_loss'][0]:.4f}")
    print(f"  最终训练损失: {history['train_loss'][-1]:.4f}")
    print(f"  初始验证准确率: {history['val_acc'][0]:.4f}")
    print(f"  最佳验证准确率: {max(history['val_acc']):.4f}")
```

**阅读这段代码时的关键观察**：

1. **前向传播的对称性**：数据依次流过 `fc1 → relu1 → fc2 → relu2 → fc3 → softmax`，每层输出作为下层输入。
2. **反向传播的对称性**：梯度依次流回 `fc3 ← relu2 ← fc2 ← relu1 ← fc1`，顺序恰好相反。
3. **中间变量的保存**：每个层在 `forward` 时缓存输入/中间值，在 `backward` 时取出使用。这和 PyTorch 的 computation graph 在概念上完全一致。
4. **批处理的平均**：所有梯度都除以 `N`（batch_size），确保 loss 对不同 batch_size 具有可比性。
5. **数值稳定性**：softmax 中减去最大值、log 中加 epsilon——这些细节在实际训练中至关重要。

## 10. 常见错误

### 错误 1：忘记在反向传播中除以 batch_size
**症状**：梯度巨大，损失直接变成 NaN 或震荡剧烈。
**原因**：交叉熵损失在计算时除以了 batch_size（取平均），因此梯度也必须除以 batch_size。如果不除以 batch_size，每个参数的梯度会随着 batch_size 增大而增大，导致学习率和 batch_size 耦合。
**修复**：确保 `dW = (dout.T @ x) / N`，其中 N 是 batch_size。

### 错误 2：softmax 数值溢出
**症状**：前向传播时出现 NaN 或 inf。
**原因**：`e^z` 当 z 很大时（如 z=1000）会超出浮点数表示范围。直接计算 `exp(z) / sum(exp(z))` 会得到 `inf/inf = NaN`。
**修复**：用 `z - max(z)` 作为稳定化技巧，这不会改变 softmax 的结果（因为分子分母同时乘以 e^{-max(z)}）。

### 错误 3：交叉熵中使用概率而非 logits
**症状**：梯度完全不对，模型学不到任何东西。在 PyTorch 中，如果你对 `nn.CrossEntropyLoss` 传入 softmax 后的概率，相当于做了两次 softmax。
**原因**：`nn.CrossEntropyLoss` 内部包含 `log_softmax`。正确的输入是原始 logits。
**修复**：明确你使用的 cross_entropy 实现是否内置 softmax。PyTorch 的 `CrossEntropyLoss` 内置，TensorFlow 的 `categorical_crossentropy` 不内置。

### 错误 4：梯度更新时使用 += 而非 -=
**症状**：Loss 持续上升而非下降。
**原因**：梯度指向损失函数上升最快的方向。要最小化损失，需要沿负梯度方向（即梯度下降）更新。如果用 `+=`，你实际上在做梯度上升。
**修复**：`θ = θ - lr * gradient`，不是 `θ = θ + lr * gradient`。

### 错误 5：在训练模式下使用测试时的 running mean/var
**症状**：BatchNorm 层训练正常但测试时效果很差。
**原因**：BatchNorm 在训练时用当前 mini-batch 的均值和方差，测试时用训练过程中的 running average。如果在训练时错误地使用了 running average，每个 batch 的归一化统计量不准确。
**修复**：确保训练时 `model.train()`，测试时 `model.eval()`。PyTorch 会根据模式自动切换。

### 错误 6：ReLU 导致的"死神经元"
**症状**：某些神经元输出始终为零，无论输入是什么。
**原因**：如果学习率过大或梯度更新方向不当，可能导致权重和偏置使得神经元对所有输入都输出零或负数。一旦 ReLU 输入 <= 0，梯度为零，神经元永远无法恢复。
**修复**：
- 使用较小的学习率
- 使用 LeakyReLU 或 PReLU 代替 ReLU（负区间也有小梯度）
- 使用合适的权重初始化（He 初始化）

### 错误 7：忘记 zero_grad
**症状**：梯度异常大或训练不稳定。
**原因**：在 PyTorch 中，`.backward()` 默认会**累积**梯度（加到已有梯度上），而不是覆盖。如果不在每个 batch 前调用 `optimizer.zero_grad()`，梯度会越积越多。
**修复**：在 PyTorch 训练循环中，`optimizer.zero_grad()` 必须在 `loss.backward()` 之前调用。

### 错误 8：混淆 batch_size 和 in_features 在矩阵乘法中
**症状**：维度不匹配错误。`matmul: Input operand 1 has a mismatch...`
**原因**：经常混淆 `X @ W.T`（X 是 (batch, in)，W 是 (out, in)）和 `X @ W`（W 是 (in, out)）两种约定。
**修复**：统一使用一种约定，在代码中明确注释张量的形状。推荐使用 `Linear` 层的标准约定：W 的形状是 `(out_features, in_features)`，前向传播用 `X @ W.T + b`。

### 错误 9：BatchNorm 的 ε 设置过大
**症状**：归一化效果减弱，训练不稳定。
**原因**：ε（epsilon）用于防止除以零。通常设为 1e-5 或 1e-6。如果设太大（如 1e-1），会显著影响归一化的方差估计。
**修复**：使用 PyTorch 默认值 1e-5，除非有特殊理由。

### 错误 10：在验证/测试集上启用了 Dropout
**症状**：测试性能明显低于验证性能，结果不可复现。
**原因**：Dropout 只在训练时使用。测试时如果不关闭，每次推理结果不同，且会降低性能。
**修复**：在 PyTorch 中调用 `model.eval()` 会自动关闭 Dropout 并切换 BatchNorm 模式。

## 11. 本章小结

本章我们从最底层出发，用文字和数学公式推导了一个神经网络训练的全部核心组件：

1. **前向传播**是数据从输入到输出的"组装线"，每一层对数据进行一次线性+非线性变换。
2. **反向传播**不神秘——它只是链式法则的高效实现。梯度沿着计算图反向流动，保存的中间值在此时被消费。
3. **损失函数**定义了"什么是好的预测"。MSE 用于回归，CrossEntropy 用于分类，各自有严格的概率解释。
4. **正则化**防止模型"死记硬背"。L1 产生稀疏，L2 产生衰减，Dropout 做隐式集成。
5. **归一化层**解决了内部协变量偏移。BN 沿 batch 归一化（适合 CV），LN 沿特征归一化（适合 NLP）。
6. **优化器**是训练的最后一块拼图。从 SGD 到 AdamW，每次改进都解决了前一个优化器的特定痛点。

最重要的是，我们用了100多行 NumPy 代码从头实现了一个完整的训练循环。这段代码中的每个矩阵乘法、每个梯度计算、每个参数更新，都和上文的理论推导一一对应。当你理解了这段代码，你就真正理解了神经网络训练的底层机制——这不是黑魔法，而是严谨的数学计算。

在下一章，我们将把这些基础知识用于理解当今最重要的架构：Transformer。
