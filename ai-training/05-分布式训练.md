## 1. 为什么需要分布式训练

### 1.1 两个根本原因

分布式训练不是"有就好"的可选特性，而是训练大模型的必要条件。驱动力来自两个方面：

**原因一：模型太大，一张 GPU 装不下**

以 LLaMA 65B 为例：
- 参数量：65B 个 float16 = 130GB
- AdamW 优化器状态：`m_t` 和 `v_t` 各 65B 个 float32 = 130GB × 2 = 260GB
- 梯度：65B 个 float16 = 130GB
- 激活值（中间层输出）：取决于 batch_size 和序列长度，可能 50-200GB

总计：650+ GB。而目前最强大的单卡 H100 也只有 80GB 显存。显然，模型根本放不进一张卡。

**原因二：单卡训练太慢**

即使小模型能放进一张卡，训练时间也可能令人绝望。以 GPT-3 175B 为例，在单张 V100 上训练需要约 355 年。即使用 10000 张 V100，也需要约 13 天。分布式训练将时间从"几个世纪"缩短到"几周"。

### 1.2 分布式训练的基本问题

分布式训练要解决的核心问题是：**如何将一个训练任务拆分到多个计算设备上，使得总计算时间线性（或近线性）减少？**

这听起来简单，但面临几个挑战：
- **通信开销**：设备之间需要交换数据（梯度、参数等），通信比计算慢几个数量级
- **同步问题**：不同设备上的计算需要协调，等待最慢的设备
- **负载均衡**：计算和显存需要在设备之间均匀分配

### 1.3 分布式策略总览

```
分布式训练策略
├── 数据并行 (Data Parallelism)
│   ├── DP (DataParallel) — 单机多卡，已过时
│   └── DDP (DistributedDataParallel) — 多机多卡，当前标准
├── 模型并行 (Model Parallelism)
│   ├── 张量并行 (Tensor Parallelism) — 单层内切分
│   └── 流水线并行 (Pipeline Parallelism) — 层间切分
├── 混合并行 (Hybrid Parallelism)
│   └── 3D 并行 = DP + TP + PP
└── 优化器状态分片
    ├── DeepSpeed ZeRO (1/2/3)
    └── FSDP (PyTorch 原生)
```

## 2. 数据并行（DDP）

### 2.1 DDP 的核心思想

数据并行是最直观的分布式策略：**每张 GPU 上有模型的一个完整副本，处理不同的数据子集（batch 的一部分），然后同步梯度。**

具体来说，DDP 的训练流程是：

1. 全局 batch 被均匀分成 N 份（N = GPU 数量），每张 GPU 拿到一份
2. 每张 GPU 独立做前向传播和反向传播，计算自己的梯度
3. 所有 GPU 的梯度通过 **AllReduce** 操作求平均
4. 每张 GPU 用平均后的梯度独立更新自己的模型参数

DDP 的优雅之处在于：第 2 步的计算完全并行，只在第 3 步需要一次通信。

### 2.2 AllReduce：梯度的民主投票

AllReduce 是分布式训练中最核心的集合通信操作。它的功能是：每台设备有一个数值，操作后每台设备都得到所有数值的某种"归约"结果（通常是求和或平均）。

常见的 AllReduce 实现是 **Ring AllReduce**：

假设有 4 张 GPU，每张有一条梯度向量。Ring AllReduce 分两步：

**步骤一：Scatter-Reduce（分散归约）**

将梯度向量分成 N 份。进行 N-1 轮通信，每轮每张 GPU 把自己的一块数据发给邻居，同时接收另一边邻居的数据并累加：

```
第1轮: GPU0 发 chunk0 给 GPU1, GPU1 发 chunk1 给 GPU2, ...
第2轮: GPU0 把收到的（原 GPU3 的 chunk3）发给 GPU1, ...
...
第 N-1 轮后: 每张 GPU 各有一块完全归约的数据
```

**步骤二：AllGather（全收集）**

每张 GPU 把自己拥有的那块完全归约后数据广播给所有其他 GPU。再经过 N-1 轮通信，所有 GPU 都拥有完整的归约后梯度。

Ring AllReduce 的总通信量为 `2(N-1)/N × 数据量` ≈ `2 × 数据量`。关键是这个通信量**与 GPU 数量 N 无关**（当 N 很大时趋近于 2× 数据量）。这使得 DDP 在成千上万张 GPU 上仍能有效扩展。

### 2.3 DDP 在 PyTorch 中的实现

DDP 的关键代码模式：

```python
# 1. 初始化进程组
torch.distributed.init_process_group(
    backend='nccl',        # NVIDIA NCCL 后端
    init_method='env://',  # 从环境变量读取地址信息
)

# 2. 创建 DDP 模型
model = MyModel().to(local_rank)
model = DDP(model, device_ids=[local_rank])

# 3. 使用 DistributedSampler 确保每个 GPU 看到不同数据
sampler = DistributedSampler(dataset)
dataloader = DataLoader(dataset, sampler=sampler, batch_size=batch_size)

# 4. 训练循环
for epoch in range(epochs):
    sampler.set_epoch(epoch)  # 确保每 epoch 的 shuffle 不同
    for batch in dataloader:
        loss = model(batch)
        loss.backward()  # DDP 在此自动触发 AllReduce
        optimizer.step()
        optimizer.zero_grad()
```

DDP 在 `loss.backward()` 时自动做梯度同步。这个时机经过精心选择——在反向传播过程中，一旦某个参数的梯度计算完成，DDP 就可以开始对该参数的梯度进行通信（与后续层反向传播的计算重叠），从而隐藏通信延迟。

### 2.4 DDP 的局限性

- **模型必须能放进一张卡**：DDP 要求每张 GPU 有完整模型副本。如果模型本身大于单卡显存，DDP 无法使用。
- **优化器状态冗余**：每张 GPU 都存一份完整的 AdamW 状态（m_t 和 v_t）。对于大模型，这是巨大的显存浪费。
- **不适合极度大 batch**：虽然 DDP 可以增加有效 batch_size，但过大 batch_size 可能损害模型收敛。

## 3. 模型并行

当模型太大，单卡装不下时，我们需要把模型本身拆分到多张卡上。

### 3.1 张量并行（Tensor Parallelism）

**核心思想**：将单个层的权重矩阵切分到多个 GPU 上，每张 GPU 计算一部分。

以 Attention 层的 Q/K/V 投影为例。原始的 Q 投影：
```
Q = X @ W_Q    # W_Q: (d_model, d_model)
```

张量并行中，将 `W_Q` 按列切分为 N 份（N = GPU 数量）：
```
W_Q = [W_Q^(1) | W_Q^(2) | ... | W_Q^(N)]    # 每份: (d_model, d_model/N)
```

每张 GPU i 计算：
```
Q_i = X @ W_Q^(i)    # 形状: (batch, seq_len, d_model/N)
```

然后每张 GPU 上有 Q_i, K_i, V_i 的一部分，各自计算该部分头的注意力。最后通过 AllReduce 合并结果。

**张量并行的特点**：
- 每层的前向和反向传播都需要 AllReduce 通信，通信频率高
- 适合单机内 GPU 之间通信（NVLink 高带宽）
- 不适合跨机器（网络带宽低）
- Megatron-LM 是张量并行的典型实现

### 3.2 流水线并行（Pipeline Parallelism）

**核心思想**：将模型的不同层放在不同 GPU 上，数据像流水线一样流过这些 GPU。

例如，一个 48 层的 Transformer 放在 4 张 GPU 上：GPU0 负责层 1-12，GPU1 负责层 13-24，GPU2 负责层 25-36，GPU3 负责层 37-48。

**朴素流水线并行**：
```
时间 -->
GPU0: [F0]        [F1]        [F2]        ...
GPU1:        [F0]        [F1]        [F2] ...
GPU2:              [F0]        [F1]        ...
GPU3:                    [F0]        [F1]  ...
```
F0 表示 micro-batch 0 的前向传播。注意 GPU1 在等待 GPU0 完成，GPU2 在等待 GPU1 完成... 大部分时间 GPU 处于空闲状态。这被称为"流水线气泡"（pipeline bubble）。

**GPipe 的改进——Micro-batch**：
将 mini-batch 进一步拆分为多个 micro-batch。micro-batch 之间的流水线可以重叠：

```
时间 -->
GPU0: [F0][F1][F2][F3]   [B0][B1][B2][B3]
GPU1:    [F0][F1][F2][F3]   [B0][B1][B2][B3]
GPU2:       [F0][F1][F2][F3]   [B0][B1][B2][B3]
GPU3:          [F0][F1][F2][F3]   [B0][B1][B2][B3]
```

micro-batch 越多，气泡占比越小。但 micro-batch 数量受 batch_size 限制。

**1F1B 调度（1 Forward 1 Backward）**：
进一步优化：不等待所有 micro-batch 的前向完成，而是交错进行前向和反向：

```
GPU0: F0 F1 F2 F3 B0 B1 B2 B3
GPU1:    F0 F1 F2 F3 B0 B1 B2 B3   ← 预热后保持"1前1反"节奏
GPU2:       F0 F1 F2 F3 B0 B1 B2 B3
GPU3:          F0 F1 F2 F3 B0 B1 B2 B3
```

1F1B 调度最大化了 GPU 利用率，同时限制了激活值显存（因为反向传播可以释放前向的中间结果）。

### 3.3 张量并行 vs 流水线并行

| 方面 | 张量并行 (TP) | 流水线并行 (PP) |
|------|-------------|--------------|
| 切分维度 | 层内（权重矩阵切分） | 层间（不同层不同卡） |
| 通信频率 | 每层传播都通信 | 只传递激活值和梯度 |
| 通信量 | 高（每层 AllReduce） | 低（仅层间激活值） |
| 通信模式 | AllReduce（多对多） | P2P（点对点） |
| 适合规模 | 单机内（NVLink） | 跨机器（网络） |
| GPU 利用率 | 高 | 低（流水线气泡） |

实际中，通常将 TP 和 PP 组合使用：TP 用于单机内 8 卡的层内切分，PP 用于多机之间的层间切分。

## 4. DeepSpeed ZeRO

### 4.1 ZeRO 的核心洞察

回顾显存的占用：
- 参数（Parameters）：fp16，P 字节
- 梯度（Gradients）：fp16，P 字节（反向传播时）
- 优化器状态（Optimizer States）：fp32 的 m_t 和 v_t + fp32 的参数副本，共 12P 字节
- 激活值（Activations）：取决于 batch_size 和模型结构

**关键观察**：在 DDP 中，参数、梯度、优化器状态（统称"模型状态"）在每张 GPU 上都有完整副本。这造成了 `N × 16P` 字节的显存冗余（N 张卡，每张卡要存 fp16 参数 + fp16 梯度 + fp32 优化器状态 ≈ 16P 字节）。

ZeRO 的核心思想是：**与其每张卡存完整副本，不如将模型状态分片到所有卡上，需要时再通信获取。**

### 4.2 ZeRO-1：优化器状态分片

**只分片优化器状态**。参数和梯度仍然每卡完整。

训练流程变为：
1. 每张卡独立前向和反向传播（用自己的数据），得到完整梯度
2. 通过 Reduce-Scatter 操作，将完整梯度分散为 N 份，每卡得到一份梯度分片
3. 每张卡只更新自己负责的那部分参数的优化器状态
4. 通过 AllGather 操作，收集完整参数（用于下次前向传播）

**显存节省**：优化器状态从 12P/卡 降为 12P/N/卡 + 通信缓冲。当 N=64 时，近似节省 12P 显存。

### 4.3 ZeRO-2：梯度分片

**在 ZeRO-1 基础上增加了梯度分片**。参数仍然每卡完整。

训练流程：
1. 前向传播使用完整参数
2. 反向传播时，每层计算出的梯度立刻通过 Reduce-Scatter 分片，释放完整梯度的显存
3. 对分片后的梯度应用优化器步骤

**显存节省**：梯度从 2P/卡 降为 2P/N/卡。加上 ZeRO-1 节省的优化器状态，总计节省约 14P/卡。

### 4.4 ZeRO-3：参数分片

**所有模型状态全部分片**。参数、梯度、优化器状态都只存 1/N。

训练流程：
1. 前向传播前，通过 AllGather 收集当前层需要的完整参数
2. 前向传播该层后，释放完整参数
3. 反向传播前，再次 AllGather 收集该层的完整参数
4. 反向传播后，Reduce-Scatter 分发梯度
5. 释放完整参数

**关键开销**：每个 Transformer 层的前向和反向都需要一次 AllGather 通信。但通信和计算可以重叠——在计算当前层时，可以提前 pre-fetch 下一层的参数。

### 4.5 ZeRO 三阶段总结

| 阶段 | 分片内容 | 显存节省 | 通信量 | 适用规模 |
|------|---------|---------|--------|---------|
| ZeRO-1 | 优化器状态 | ~4× 减少 | 与 DDP 相同 | 中等模型（<13B） |
| ZeRO-2 | 优化器 + 梯度 | ~8× 减少 | 与 DDP 相同 | 较大模型（13-70B） |
| ZeRO-3 | 全部模型状态 | ~N×（线性） | 增加 1.5× | 最大模型（>70B） |

### 4.6 ZeRO-Offload 和 ZeRO-Infinity

**ZeRO-Offload**：将优化器状态和梯度 offload 到 CPU 内存（甚至 NVMe SSD），仅在前向/反向传播时将必要数据加载回 GPU。这打破了"模型必须放进 GPU 显存"的限制。

**ZeRO-Infinity**：在 ZeRO-Offload 基础上，支持将参数也 offload 到 NVMe SSD，并使用高效的预取和缓存策略隐藏 I/O 延迟。

## 5. FSDP（Fully Sharded Data Parallelism）

### 5.1 FSDP 是什么

FSDP 是 PyTorch 原生的模型分片方案，其设计和 DeepSpeed ZeRO-3 高度相似：在所有设备上分片模型参数、梯度和优化器状态。FSDP 于 2022 年随着 PyTorch 1.12 正式发布。

### 5.2 FSDP vs DeepSpeed ZeRO-3

**相同点**：
- 都分片参数、梯度、优化器状态
- 都使用 AllGather 在需要时收集参数
- 都支持参数预取（prefetch）来隐藏通信延迟
- 都支持混合精度训练

**差异点**：
- **分片粒度**：FSDP 以"FSDP Unit"（通常是一个 Transformer Block）为单位分片和通信。DeepSpeed 以层为单位。FSDP 可以灵活设置分片粒度。
- **CPU Offload**：DeepSpeed 的 CPU Offload 更成熟和高效。FSDP 也支持 CPU Offload 但功能较新。
- **混合精度**：FSDP 与 PyTorch 的 `autocast` 原生集成，使用更自然。
- **生态集成**：FSDP 作为 PyTorch 核心库，不需要额外安装。DeepSpeed 是第三方库。
- **分片策略灵活性**：FSDP 支持混合分片（不同层不同分片策略），DeepSpeed 的分片策略更固定。

总的来说，FSDP 适合"已经在 PyTorch 生态中、想要原生支持的团队"，DeepSpeed 适合"需要极致优化、愿意引入第三方依赖的团队"。

### 5.3 FSDP 的使用示例

```python
"""
FSDP 分布式训练示例
====================
使用 PyTorch FSDP 包装模型进行全分片数据并行训练。

启动命令（4 GPU）:
    torchrun --nproc_per_node=4 train_fsdp.py

作者：Python+AI 学习教程
"""

import os
import torch
import torch.nn as nn
import torch.distributed as dist
from torch.distributed.fsdp import (
    FullyShardedDataParallel as FSDP,
    MixedPrecision,
    ShardingStrategy,
    BackwardPrefetch,
    CPUOffload,
    StateDictType,
    FullStateDictConfig,
)
from torch.distributed.fsdp.wrap import (
    transformer_auto_wrap_policy,
    size_based_auto_wrap_policy,
)
from torch.utils.data import DataLoader, Dataset, DistributedSampler
import functools


# ============================================================
# 1. 模拟数据集
# ============================================================

class DummyDataset(Dataset):
    """模拟数据集，用于演示分布式训练流程"""
    def __init__(self, size=10000, seq_len=512, vocab_size=10000):
        self.size = size
        self.seq_len = seq_len
        self.vocab_size = vocab_size
        self.data = torch.randint(0, vocab_size, (size, seq_len))

    def __len__(self):
        return self.size

    def __getitem__(self, idx):
        # 返回 input_ids 和 labels（语言建模任务）
        x = self.data[idx]
        return x[:-1], x[1:]  # 输入是前 n-1 个 token，标签是后 n-1 个 token


# ============================================================
# 2. 模拟 Transformer 模型
# ============================================================

class TransformerBlock(nn.Module):
    """简化的 Transformer 块"""
    def __init__(self, d_model=1024, n_heads=16, d_ff=4096):
        super().__init__()
        self.attn = nn.MultiheadAttention(
            d_model, n_heads, batch_first=True
        )
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d_model),
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # Self-Attention + 残差
        attn_out, _ = self.attn(x, x, x)
        x = self.norm1(x + attn_out)
        # FFN + 残差
        x = self.norm2(x + self.ffn(x))
        return x


class DemoModel(nn.Module):
    """演示用 mini Transformer"""
    def __init__(self, vocab_size=10000, d_model=1024, n_layers=12, n_heads=16):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, d_model)
        self.blocks = nn.ModuleList([
            TransformerBlock(d_model, n_heads) for _ in range(n_layers)
        ])
        self.lm_head = nn.Linear(d_model, vocab_size)

    def forward(self, x):
        x = self.embed(x)
        for block in self.blocks:
            x = block(x)
        logits = self.lm_head(x)
        return logits


# ============================================================
# 3. 训练函数
# ============================================================

def train_fsdp():
    """使用 FSDP 进行分布式训练"""

    # ---- 初始化分布式环境 ----
    dist.init_process_group(backend='nccl')
    local_rank = int(os.environ['LOCAL_RANK'])
    rank = int(os.environ['RANK'])
    world_size = int(os.environ['WORLD_SIZE'])
    torch.cuda.set_device(local_rank)

    if rank == 0:
        print(f"分布式训练启动: {world_size} 个进程, 每进程 1 张 GPU")
        print(f"GPU: {torch.cuda.get_device_name(local_rank)}")

    # ---- 创建模型 ----
    model = DemoModel(vocab_size=10000, d_model=1024, n_layers=12)

    # ---- FSDP 配置 ----
    # 混合精度: 前向和反向用 BF16，优化器状态用 FP32
    mixed_precision_policy = MixedPrecision(
        param_dtype=torch.bfloat16,     # 参数存储为 BF16
        reduce_dtype=torch.bfloat16,    # 梯度通信用 BF16
        buffer_dtype=torch.bfloat16,    # buffer 用 BF16
    )

    # 自动包装策略: 每个 TransformerBlock 作为一个 FSDP Unit
    auto_wrap_policy = functools.partial(
        transformer_auto_wrap_policy,
        transformer_layer_cls={
            TransformerBlock,
        },
    )

    # 将模型包装为 FSDP
    model = FSDP(
        model,
        auto_wrap_policy=auto_wrap_policy,
        mixed_precision=mixed_precision_policy,
        sharding_strategy=ShardingStrategy.FULL_SHARD,  # ZeRO-3 级别
        backward_prefetch=BackwardPrefetch.BACKWARD_PRE,  # 反向传播时预取
        device_id=local_rank,
        cpu_offload=CPUOffload(offload_params=False),  # 不 offload 到 CPU
        # 如果显存不够，可以开启 CPU Offload:
        # cpu_offload=CPUOffload(offload_params=True),
    )

    if rank == 0:
        total_params = sum(p.numel() for p in model.parameters())
        print(f"模型总参数: {total_params:,}")

    # ---- 数据加载器 ----
    dataset = DummyDataset(size=10000, seq_len=512, vocab_size=10000)
    sampler = DistributedSampler(
        dataset,
        num_replicas=world_size,
        rank=rank,
        shuffle=True,
    )
    dataloader = DataLoader(
        dataset,
        batch_size=4,     # 每张 GPU 的 micro-batch size
        sampler=sampler,
        num_workers=2,    # 数据加载线程数
        pin_memory=True,
    )

    # ---- 优化器 ----
    optimizer = torch.optim.AdamW(
        model.parameters(),
        lr=3e-4,
        betas=(0.9, 0.95),
        weight_decay=0.1,
    )

    # ---- 损失函数 ----
    loss_fn = nn.CrossEntropyLoss()

    # ---- 训练循环 ----
    model.train()
    num_epochs = 3

    for epoch in range(num_epochs):
        sampler.set_epoch(epoch)  # 确保每 epoch 的 shuffle 不同
        total_loss = 0.0
        num_batches = 0

        for batch_idx, (input_ids, labels) in enumerate(dataloader):
            input_ids = input_ids.cuda(local_rank)
            labels = labels.cuda(local_rank)

            # 前向传播
            logits = model(input_ids)
            # logits: (batch, seq_len, vocab_size)
            # labels: (batch, seq_len)
            loss = loss_fn(
                logits.view(-1, logits.size(-1)),
                labels.view(-1),
            )

            # 反向传播（FSDP 自动处理梯度同步）
            optimizer.zero_grad()
            loss.backward()

            # 梯度裁剪
            model.clip_grad_norm_(max_norm=1.0)

            # 参数更新
            optimizer.step()

            total_loss += loss.item()
            num_batches += 1

            if rank == 0 and batch_idx % 50 == 0:
                print(f"Epoch {epoch+1}/{num_epochs}, "
                      f"Batch {batch_idx}/{len(dataloader)}, "
                      f"Loss: {loss.item():.4f}")

        avg_loss = total_loss / num_batches
        if rank == 0:
            print(f"Epoch {epoch+1} 完成, 平均 Loss: {avg_loss:.4f}")

    # ---- 保存模型（仅在 rank 0） ----
    if rank == 0:
        # 收集完整的状态字典到 rank 0 的 CPU
        save_policy = FullStateDictConfig(offload_to_cpu=True, rank0_only=True)
        with FSDP.state_dict_type(model, StateDictType.FULL_STATE_DICT, save_policy):
            state_dict = model.state_dict()
            torch.save(state_dict, "fsdp_model_checkpoint.pt")
        print("模型已保存到 fsdp_model_checkpoint.pt")

    # ---- 清理 ----
    dist.destroy_process_group()


if __name__ == "__main__":
    train_fsdp()
```

### 5.4 FSDP 的分片策略

FSDP 支持几种分片策略：

- **FULL_SHARD**：ZeRO-3 级别，参数/梯度/优化器状态全部分片。最大显存节省。
- **SHARD_GRAD_OP**：ZeRO-2 级别，分片梯度和优化器状态。参数完整保留。
- **NO_SHARD**：退化为 DDP。不做分片，仅做数据并行。
- **HYBRID_SHARD**：在节点内做 FULL_SHARD，节点间做 NO_SHARD。适合多节点场景（利用节点内 NVLink 高带宽）。

## 6. 混合并行

### 6.1 3D 并行

在现代大模型训练中，单一策略往往不够。通常需要组合多种并行策略——这就是 **3D 并行**：

```
3D 并行 = 数据并行 (DP) + 张量并行 (TP) + 流水线并行 (PP)
```

- **DP**：多个数据副本，每个副本处理不同的 batch 数据
- **TP**：单层内的权重矩阵切分（通常在单机 8 张 GPU 内用 NVLink 连接）
- **PP**：不同层在不同 GPU 组上，层间用流水线通信

**典型的 3D 配置**（训练 GPT-3 175B）：
- TP 大小 = 8（每个节点内的 8 张 GPU 做张量并行）
- PP 大小 = 8（8 个节点做流水线，每个节点负责一部分层）
- DP 大小 = 64（64 个"管道副本"，每个副本包含 8×8=64 张 GPU 的 TP+PP 组）
- 总 GPU 数 = 8 × 8 × 64 = 4096

### 6.2 GPU 之间如何连接

在 3D 并行中，GPU 之间的连接方式决定了通信策略：

```
┌─────────────────────────────────────────────┐
│ 节点 0 (8×A100, NVLink 互联)                │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐  │
│  │GPU│ │GPU│ │GPU│ │GPU│ │GPU│ │GPU│ │GPU│ │GPU│  │
│  │ 0 │ │ 1 │ │ 2 │ │ 3 │ │ 4 │ │ 5 │ │ 6 │ │ 7 │  │
│  └───┘ └───┘ └───┘ └───┘ └───┘ └───┘ └───┘ └───┘  │
│    ←────────── NVLink (900 GB/s) ──────────→      │
└──────────────────────┬────────────────────────────┘
                       │ InfiniBand / RoCE (200-400 GB/s)
┌──────────────────────┼────────────────────────────┐
│ 节点 1 (8×A100)       │                            │
│  [GPU 0-7]            │                            │
└───────────────────────┴────────────────────────────┘
```

- **NVLink**：单机内 GPU 直连，带宽极高（A100: 600 GB/s, H100: 900 GB/s），适合 TP 的高频通信
- **InfiniBand / RoCE**：跨节点网络，带宽较低（200-400 GB/s），适合 PP 和 DP 的较低频通信

### 6.3 并行度配置的原则

配置并行度时需考虑：

1. **TP 不跨节点**：张量并行的通信太密集（每个前向/反向都需要 AllReduce），应限制在单节点内。
2. **PP 优先跨节点**：流水线并行的通信量小（只传激活值和梯度），适合跨节点。
3. **DP 最大化**：数据并行扩展性最好（通信与 GPU 数量无关），应尽可能多地使用。
4. **micro-batch 足够多**：流水线并行需要足够多的 micro-batch 来隐藏气泡。

## 7. 通信优化

### 7.1 梯度累积

**问题**：大模型训练需要大 global batch size（如 GPT-3 的 3.2M tokens），但单卡显存只能装下很小的 micro-batch。

**解决方案——梯度累积**：不立即更新参数，而是累积多个 micro-batch 的梯度，再一次性更新。这样 effective batch size = micro_batch_size × accumulation_steps × num_gpus。

```python
# 梯度累积训练循环
accumulation_steps = 8  # 累积 8 个 micro-batch

optimizer.zero_grad()
for i, batch in enumerate(dataloader):
    loss = model(batch)
    loss = loss / accumulation_steps  # 缩放损失，使梯度量级正确
    loss.backward()  # 梯度累加到 .grad 中

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

注意：DDP 中，`loss.backward()` 只在累积的最后一步触发 AllReduce（因为之前的梯度还不完整），这也是为什么需要 `model.require_backward_grad_sync` 来控制同步时机。

### 7.2 通信计算重叠（Overlapping）

**核心思想**：通信（AllReduce/AllGather）和计算（前向传播/反向传播）可以同时进行。

在 DDP 中，PyTorch 使用 **gradient bucketing** 技术：将参数分组为"桶"，一旦桶内所有参数的梯度都计算完毕，立即开始该桶的 AllReduce 通信——与此同时，后续参数的梯度计算仍在进行。

在 FSDP/ZeRO-3 中，**参数预取**：在处理当前层时，预取下一层的参数（通过 AllGather），这样当计算到达下一层时参数已经就位。

### 7.3 通信后端的选择

- **NCCL**：NVIDIA Collective Communication Library。GPU 通信的黄金标准。支持 AllReduce、AllGather、ReduceScatter、Broadcast 等集合操作。在 NVLink 和 InfiniBand 上都有优化。
- **Gloo**：CPU 通信后端。跨平台（Linux/macOS/Windows）。速度不如 NCCL 但对非 NVIDIA GPU 有用。
- **MPI**：Message Passing Interface。HPC 领域的经典通信库。在某些 HPC 集群上有更好的集成。

对于 GPU 训练，99% 的情况使用 NCCL。

### 7.4 梯度压缩

通信是分布式训练的瓶颈之一。梯度压缩通过压缩梯度的精度来减少通信量：

- **FP16/BF16 梯度通信**：将梯度从 FP32 压缩为 FP16/BF16 后再通信，通信量减半。
- **PowerSGD**：用低秩近似替代完整梯度矩阵进行通信。在低带宽网络上特别有效。
- **Top-K 稀疏化**：只传输绝对值最大的 K 个梯度元素，其余设为零。

## 8. DDP 训练完整代码

```python
"""
DDP 分布式训练完整示例
=======================
使用 PyTorch DDP 进行数据并行训练。

启动命令（单机 4 GPU）:
    torchrun --nproc_per_node=4 train_ddp.py

启动命令（多机）:
    # 主节点 (node0, IP=192.168.1.100):
    torchrun --nproc_per_node=8 --nnodes=2 --node_rank=0 \
             --master_addr="192.168.1.100" --master_port=12355 train_ddp.py
    # 从节点 (node1):
    torchrun --nproc_per_node=8 --nnodes=2 --node_rank=1 \
             --master_addr="192.168.1.100" --master_port=12355 train_ddp.py

作者：Python+AI 学习教程
"""

import os
import time
import torch
import torch.nn as nn
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP
from torch.utils.data import DataLoader, Dataset, DistributedSampler
from torch.utils.tensorboard import SummaryWriter


# ============================================================
# 1. 工具函数
# ============================================================

def setup(rank, world_size):
    """初始化分布式进程组"""
    dist.init_process_group(
        backend='nccl',
        init_method='env://',
        rank=rank,
        world_size=world_size,
    )
    torch.cuda.set_device(rank)


def cleanup():
    """销毁进程组"""
    dist.destroy_process_group()


def is_main_process():
    """判断是否是主进程（rank 0）"""
    return dist.get_rank() == 0


def print_rank0(*args, **kwargs):
    """只在 rank 0 打印"""
    if is_main_process():
        print(*args, **kwargs)


# ============================================================
# 2. 模拟数据集
# ============================================================

class DemoDataset(Dataset):
    """用于分布式训练演示的数据集"""
    def __init__(self, num_samples=10000, seq_len=128, vocab_size=50000):
        self.data = torch.randint(0, vocab_size, (num_samples, seq_len))

    def __len__(self):
        return len(self.data)

    def __getitem__(self, idx):
        x = self.data[idx]
        return x[:-1], x[1:]  # 语言建模: 输入/标签偏移一位


# ============================================================
# 3. 演示模型
# ============================================================

class DemoTransformer(nn.Module):
    """可配置大小的演示 Transformer"""
    def __init__(self, vocab_size=50000, d_model=768, n_layers=6, n_heads=12):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, d_model)
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=d_model, nhead=n_heads, batch_first=True,
            dim_feedforward=d_model * 4, dropout=0.1,
        )
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers=n_layers)
        self.lm_head = nn.Linear(d_model, vocab_size)

    def forward(self, x):
        x = self.embed(x)
        # 因果掩码: 上三角为 -inf
        mask = nn.Transformer.generate_square_subsequent_mask(x.size(1)).to(x.device)
        x = self.transformer(x, mask=mask)
        return self.lm_head(x)


# ============================================================
# 4. 训练主体
# ============================================================

def train(rank, world_size, args):
    """DDP 训练主函数"""

    # ---- 初始化 ----
    setup(rank, world_size)
    print_rank0(f"启动 DDP 训练: {world_size} GPU(s)")

    # ---- 模型 ----
    model = DemoTransformer(
        vocab_size=args.vocab_size,
        d_model=args.d_model,
        n_layers=args.n_layers,
        n_heads=args.n_heads,
    ).to(rank)

    # 包装为 DDP
    model = DDP(
        model,
        device_ids=[rank],
        output_device=rank,
        find_unused_parameters=False,  # 生产中设为 False 以提升性能
        gradient_as_bucket_view=True,  # 减少梯度内存拷贝
    )

    print_rank0(f"模型参数量: {sum(p.numel() for p in model.parameters()):,}")

    # ---- 数据 ----
    dataset = DemoDataset(
        num_samples=args.num_samples,
        seq_len=args.seq_len,
        vocab_size=args.vocab_size,
    )
    sampler = DistributedSampler(
        dataset,
        num_replicas=world_size,
        rank=rank,
        shuffle=True,
        drop_last=True,
    )
    dataloader = DataLoader(
        dataset,
        batch_size=args.batch_size,
        sampler=sampler,
        num_workers=2,
        pin_memory=True,
        drop_last=True,
    )

    # ---- 优化器和调度器 ----
    optimizer = torch.optim.AdamW(
        model.parameters(),
        lr=args.lr,
        betas=(0.9, 0.95),
        weight_decay=0.1,
    )

    # 学习率调度: warmup + cosine decay
    from torch.optim.lr_scheduler import CosineAnnealingLR, LinearLR, SequentialLR

    warmup_scheduler = LinearLR(
        optimizer,
        start_factor=0.01,
        end_factor=1.0,
        total_iters=args.warmup_steps,
    )
    cosine_scheduler = CosineAnnealingLR(
        optimizer,
        T_max=args.total_steps - args.warmup_steps,
    )
    scheduler = SequentialLR(
        optimizer,
        schedulers=[warmup_scheduler, cosine_scheduler],
        milestones=[args.warmup_steps],
    )

    # ---- 损失函数 ----
    loss_fn = nn.CrossEntropyLoss(ignore_index=-100)  # ignore padding

    # ---- TensorBoard（仅 rank 0） ----
    if is_main_process():
        writer = SummaryWriter(log_dir=args.log_dir)
        writer.add_text("args", str(vars(args)))
    else:
        writer = None

    # ---- 训练循环 ----
    model.train()
    global_step = 0
    start_time = time.time()

    for epoch in range(args.epochs):
        sampler.set_epoch(epoch)  # 每 epoch 重新 shuffle
        epoch_loss = 0.0

        for batch_idx, (input_ids, labels) in enumerate(dataloader):
            input_ids = input_ids.to(rank)
            labels = labels.to(rank)

            # 前向传播
            logits = model(input_ids)
            # logits: (batch, seq_len, vocab_size) -> (batch*seq_len, vocab_size)
            loss = loss_fn(
                logits.view(-1, logits.size(-1)),
                labels.view(-1),
            )

            # 反向传播（DDP 自动 AllReduce 梯度）
            optimizer.zero_grad()
            loss.backward()

            # 梯度裁剪
            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

            # 参数更新
            optimizer.step()
            scheduler.step()

            # 记录
            epoch_loss += loss.item()
            global_step += 1

            if global_step % args.log_interval == 0:
                # 跨所有 rank 求平均 loss
                loss_tensor = torch.tensor([loss.item()]).to(rank)
                dist.all_reduce(loss_tensor, op=dist.ReduceOp.AVG)
                global_loss = loss_tensor.item()

                print_rank0(
                    f"Epoch {epoch+1}/{args.epochs} | "
                    f"Step {global_step}/{args.total_steps} | "
                    f"Loss: {global_loss:.4f} | "
                    f"LR: {scheduler.get_last_lr()[0]:.2e}"
                )

                if writer:
                    writer.add_scalar("train/loss", global_loss, global_step)
                    writer.add_scalar("train/lr", scheduler.get_last_lr()[0], global_step)

            if global_step >= args.total_steps:
                break

        avg_epoch_loss = epoch_loss / len(dataloader)
        print_rank0(f"Epoch {epoch+1} 平均 Loss: {avg_epoch_loss:.4f}")

        if global_step >= args.total_steps:
            break

    # ---- 保存模型 ----
    if is_main_process():
        torch.save({
            'model_state_dict': model.module.state_dict(),  # 去掉 DDP wrapper
            'optimizer_state_dict': optimizer.state_dict(),
            'step': global_step,
        }, os.path.join(args.output_dir, 'checkpoint.pt'))
        print_rank0(f"模型已保存, 总步数: {global_step}")

    # ---- 统计 ----
    elapsed = time.time() - start_time
    print_rank0(f"训练完成! 总时间: {elapsed:.1f}s, 每步: {elapsed/global_step:.2f}s")

    cleanup()


# ============================================================
# 5. 主入口
# ============================================================

if __name__ == "__main__":
    import argparse

    parser = argparse.ArgumentParser(description="DDP 训练示例")
    # 模型配置
    parser.add_argument('--vocab_size', type=int, default=50000)
    parser.add_argument('--d_model', type=int, default=768)
    parser.add_argument('--n_layers', type=int, default=6)
    parser.add_argument('--n_heads', type=int, default=12)
    # 训练配置
    parser.add_argument('--epochs', type=int, default=3)
    parser.add_argument('--batch_size', type=int, default=8, help='每 GPU 的 batch size')
    parser.add_argument('--lr', type=float, default=3e-4)
    parser.add_argument('--warmup_steps', type=int, default=100)
    parser.add_argument('--total_steps', type=int, default=1000)
    parser.add_argument('--num_samples', type=int, default=10000)
    parser.add_argument('--seq_len', type=int, default=128)
    # 日志和输出
    parser.add_argument('--log_interval', type=int, default=10)
    parser.add_argument('--log_dir', type=str, default='./logs')
    parser.add_argument('--output_dir', type=str, default='./checkpoints')
    args = parser.parse_args()

    # 从环境变量获取 rank 和 world_size（由 torchrun 设置）
    rank = int(os.environ.get('RANK', 0))
    world_size = int(os.environ.get('WORLD_SIZE', 1))

    # 创建输出目录
    if rank == 0:
        os.makedirs(args.output_dir, exist_ok=True)
        os.makedirs(args.log_dir, exist_ok=True)

    train(rank, world_size, args)
```

## 11. 常见错误

### 错误 1：忘记在不同 rank 上设置不同的随机种子
**症状**：所有 GPU 生成完全相同的数据和初始化，DDP 退化为单 GPU 训练（浪费了 N-1 张 GPU 的计算）。
**原因**：所有 rank 使用相同的随机种子，导致 DistributedSampler 对每个 rank 采样出相同的数据（因为每个 rank 只看到自己的 subset，但如果种子相同，每个 rank 的 subset 也是相同的）。
**修复**：`DistributedSampler` 内部会根据 rank 自动处理，但如果手动设置种子，确保：`torch.manual_seed(seed + rank)`。

### 错误 2：所有 rank 都写日志/保存文件
**症状**：日志文件被 N 个进程同时写入，内容混乱；模型文件被覆盖或损坏。
**原因**：没加 `if rank == 0` 保护，所有 rank 都在做 I/O 操作。
**修复**：只在 rank 0 执行文件 I/O 操作。对于必要的多 rank I/O，使用独立的文件名（如 `checkpoint_rank_{rank}.pt`）。

### 错误 3：不同 rank 的计算路径不一致
**症状**：训练 hang 住不动，或部分 rank 报错但其他 rank 在等待。
**原因**：DDP 假设所有 rank 执行完全相同的代码路径（确定性）。如果某个 rank 因 if/else 走了不同路径（如 rank 0 多了一步日志计算），会导致集合通信操作（AllReduce）不再匹配，程序阻塞。
**修复**：确保所有 rank 执行相同的集合通信序列。将 rank-dependent 的逻辑限制在非通信代码中。

### 错误 4：Local Rank 和 Global Rank 混淆
**症状**：`CUDA error: invalid device ordinal` 或模型被放到了错误的 GPU 上。
**原因**：`local_rank` 是节点内的 GPU 编号（0-7），`global_rank` 是所有节点中进程的唯一编号（0-N）。使用 `model.to(global_rank)` 而不是 `model.to(local_rank)` 会导致错误。
**修复**：`torch.cuda.set_device(local_rank)` 和 `model.to(local_rank)`。`global_rank` 只用于通信和 DistributedSampler。

### 错误 5：torchrun 启动参数不当
**症状**：进程启动失败或只有部分 GPU 被使用。
**原因**：`--nproc_per_node` 与实际 GPU 数量不匹配，或 `--master_port` 被占用。
**修复**：
- `--nproc_per_node` 应等于每个节点的 GPU 数量
- 多节点时确保 `--master_addr` 和 `--master_port` 在所有节点上一致且端口可用
- 检查 `CUDA_VISIBLE_DEVICES` 环境变量

### 错误 6：FSDP 中忘记 forward_prefetch
**症状**：FSDP 训练比预期的慢很多，GPU 利用率低。
**原因**：没有启用参数预取（prefetch），导致每个 FSDP Unit 的 AllGather（收集参数）和计算串行执行而不是重叠。
**修复**：设置 `backward_prefetch=BackwardPrefetch.BACKWARD_PRE` 和 `forward_prefetch=True`（如果 PyTorch 版本支持）。

### 错误 7：使用 `torch.save` 保存 DDP 模型时未去除 module 前缀
**症状**：加载模型时 `state_dict` 键名不匹配（多了 `module.` 前缀）。
**原因**：DDP 将原模型包装在 `module` 属性中，`model.state_dict()` 的键名前缀是 `module.xxx` 而不是 `xxx`。
**修复**：保存时使用 `model.module.state_dict()` 去除 DDP 包装，或加载时处理键名差异。

### 错误 8：DDP 的反向传播同步时机错误
**症状**：梯度异常大或小，训练不稳定。
**原因**：在梯度累积场景中，每个 micro-batch 的 `loss.backward()` 都会触发 AllReduce。这意味着第一个 micro-batch 的梯度就已经和所有 rank 同步了，后续 micro-batch 的梯度是"部分同步"的，导致梯度不对。
**修复**：在梯度累积的前 `N-1` 步，使用 `model.require_backward_grad_sync = False` 禁用 AllReduce，只在最后一步启用。

## 12. 本章小结

分布式训练是训练大模型的必备技能。核心知识点：

1. **DDP** 是最基本的数据并行策略：每卡完整模型副本，梯度通过 AllReduce 同步。适用于能放进单卡的中小模型。
2. **张量并行（TP）** 切分层内权重矩阵，通信密集，适合单机内 NVLink 连接的多卡。
3. **流水线并行（PP）** 切分层间序列，通信量小，适合跨机器。需要足够多 micro-batch 隐藏气泡。
4. **ZeRO/FSDP** 通过分片模型状态（优化器状态 → 梯度 → 参数），用少量通信换取大量显存节省。
5. **3D 并行** 组合 DP + TP + PP 是训练 GPT-3 规模和更大模型的实际方案。
6. **通信优化**（梯度累积、通信计算重叠、梯度压缩）是榨干硬件性能的关键。

在下一章，我们将利用本章的分布式训练知识，实际动手训练一个小型 GPT 模型。
