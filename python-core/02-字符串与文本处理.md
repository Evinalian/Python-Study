## 1. 字符串基础：不可变的 Unicode 序列

### 1.1 什么是字符串

Python 的 `str` 是一个**不可变的 Unicode 代码点序列**。每个字符都是一个 Unicode 字符——无论是英文字母、中文字符还是 emoji。

```python
s = "hello"
print(len(s))       # 5 —— 5 个字符

s = "你好"
print(len(s))       # 2 —— 2 个字符（不是 6 个字节！）

s = "🐍"
print(len(s))       # 1 —— 1 个字符（虽然底层占 4 字节）
```

Java 同学注意：Python 没有 `char` 类型。取单个字符得到的是**长度为 1 的 str**，不是某种单独的"字符"类型。

### 1.2 字符串是不可变的

一旦创建，就不能原地修改：

```python
s = "hello"
# s[0] = "H"          # TypeError: 'str' object does not support item assignment

# 要"修改"，只能创建一个新字符串
s = "H" + s[1:]       # "Hello" —— 创建了新对象
```

这与 Java 的 `String` 行为完全一致。

## 2. 索引与切片——像操作数组一样操作字符串

### 2.1 索引：正向和反向

字符串中每个字符都有一个从 0 开始的位置编号（索引）。Python 同时支持正向索引和反向索引：

```
字符串:   H    e    l    l    o
正向索引: 0    1    2    3    4
反向索引: -5   -4   -3   -2   -1
```

```python
s = "Hello"

# 正向索引：从 0 开始
print(s[0])     # "H" —— 第 1 个字符
print(s[1])     # "e" —— 第 2 个字符

# 反向索引：-1 是最后一个字符
print(s[-1])    # "o" —— 最后一个
print(s[-2])    # "l" —— 倒数第二个
```

这个设计与 Java 数组索引一致（都是 0 开始），但多了反向索引。

### 2.2 切片 `[start:end:step]` —— 三个参数逐个讲

切片的完整语法是 `s[start:end:step]`，三个参数如下：

| 参数 | 含义 | 默认值 |
|------|------|--------|
| `start` | 起始索引（包含） | 0（正向）或 -1（反向） |
| `end` | 结束索引（**不包含**） | 序列末尾 |
| `step` | 步长（可为负数） | 1 |

规则是**左闭右开**：取 start 到 end-1 之间的字符。

```python
s = "0123456789"

# === 只用到 start 和 end（步长默认 1） ===
print(s[2:5])    # "234" —— 索引 2,3,4（不含索引 5）
print(s[:5])     # "01234" —— start 省略，默认从 0 开始
print(s[5:])     # "56789" —— end 省略，默认到末尾
print(s[:])      # "0123456789" —— 全部省略，得到完整的浅拷贝

# === 负索引 ===
print(s[-4:])    # "6789" —— 从倒数第 4 个到末尾
print(s[:-3])    # "0123456" —— 从开头到倒数第 3 个（不含）

# === 步长（step） ===
print(s[::2])    # "02468" —— 从头到尾，隔一个取一个
print(s[1::2])   # "13579" —— 从索引 1 开始，隔一个取一个
print(s[::-1])   # "9876543210" —— 步长为负 = 反向遍历 = 反转字符串
print(s[8:2:-1]) # "876543" —— 从索引 8 走到索引 2（不含），每次退 1
```

切片有一个贴心的特性：**越界不报错**。`s[2:100]` 只会返回能取到的部分，不会抛出 `IndexError`（而直接索引 `s[100]` 会报错）。

## 3. 字符串格式化——三种方式，f-string 为王

### 3.1 最简单的方式：用 + 拼接（不推荐）

```python
name = "小明"
age = 20
# 可以这样拼接，但很麻烦
msg = "我叫" + name + "，今年" + str(age) + "岁。"
print(msg)  # "我叫小明，今年20岁。"
```

### 3.2 f-string：从简单到复杂

f-string（Python 3.6+）是目前最推荐的格式化方式。在字符串前加 `f`，然后用 `{ }` 嵌入表达式。

**第一步：基本嵌入**

```python
name = "小明"
age = 20
print(f"我叫{name}，今年{age}岁。")  # "我叫小明，今年20岁。"

# 花括号里可以是任意表达式
print(f"明年我就{age + 1}岁了。")
print(f"名字大写：{name.upper()}")
```

**第二步：格式控制 —— `:` 后面的格式说明符**

```python
pi = 3.1415926

# 保留几位小数
print(f"pi ≈ {pi:.2f}")      # "pi ≈ 3.14" —— .2 表示保留 2 位小数，f 表示浮点数格式
print(f"pi ≈ {pi:.4f}")      # "pi ≈ 3.1416"

# 千分位分隔
n = 1234567890
print(f"金额：{n:,} 元")      # "金额：1,234,567,890 元"

# 百分比
rate = 0.856
print(f"完成率：{rate:.1%}")  # "完成率：85.6%"

# 宽度和对齐
print(f"|{'姓名':<10}|")      # "|姓名        |" —— < 左对齐，宽度 10
print(f"|{'姓名':>10}|")      # "|        姓名|" —— > 右对齐
print(f"|{'姓名':^10}|")      # "|    姓名    |" —— ^ 居中
print(f"|{'姓名':*^10}|")     # "|****姓名****|" —— 用 * 填充

# 进制转换
x = 255
print(f"十进制 {x}，二进制 {x:b}，十六进制 {x:x}，八进制 {x:o}")
# "十进制 255，二进制 11111111，十六进制 ff，八进制 377"
```

**第三步：调试模式 `=`（Python 3.8+）**

```python
x = 42
print(f"{x = }")         # "x = 42" —— 同时打印变量名和值
print(f"{x + 1 = }")     # "x + 1 = 43" —— 表达式也可以
```

### 3.3 其他格式化方式（了解即可）

```python
# % 格式化（旧式，不推荐）
print("我叫%s，今年%d岁。" % (name, age))

# str.format()（中代方案）
print("我叫{}，今年{}岁。".format(name, age))
```

f-string 在三种方式中是最快的——因为它在编译时就被处理为字节码，而不是在运行时解析字符串模板。

## 4. 常用字符串方法

### 4.1 大小写转换

```python
s = "hello WORLD"

print(s.upper())       # "HELLO WORLD" —— 全大写
print(s.lower())       # "hello world" —— 全小写
print(s.capitalize())  # "Hello world" —— 首字母大写
print(s.title())       # "Hello World" —— 每个单词首字母大写
print(s.swapcase())    # "HELLO world" —— 大小写互换
```

**重要提醒**：这些方法都返回**新字符串**，不修改原字符串（因为 str 是不可变的）。

```python
s = "hello"
s.upper()             # 返回 "HELLO"，但没赋值
print(s)              # 还是 "hello"！
# 正确做法：
s = s.upper()         # 把新字符串赋回给 s
```

### 4.2 查找与判断

```python
s = "hello world"

# find：查找子串，返回索引，找不到返回 -1
print(s.find("world"))   # 6
print(s.find("python"))  # -1

# index：和 find 一样，但找不到时抛出 ValueError
# s.index("python")     # ValueError —— 仅在你确定存在时使用

# 判断方法——返回 True/False
print(s.startswith("hel"))   # True
print(s.endswith("rld"))     # True
print("123".isdigit())        # True
print("abc".isalpha())        # True
print("abc123".isalnum())     # True
print("  \t\n".isspace())     # True
```

### 4.3 去除空白和替换

```python
s = "  hello world  \n"

print(s.strip())       # "hello world" —— 去掉两端空白
print(s.lstrip())      # "hello world  \n" —— 只去左边
print(s.rstrip())      # "  hello world" —— 只去右边

# 去掉指定字符
print("---title---".strip("-"))  # "title"

# 替换
print("ababab".replace("ab", "XY"))      # "XYXYXY" —— 全部替换
print("ababab".replace("ab", "XY", 2))   # "XYXYab" —— 只替换前 2 次
```

### 4.4 拆分与连接

```python
# split：按分隔符拆分
print("a,b,c".split(","))              # ['a', 'b', 'c']
print("a,b,c".split(",", maxsplit=1))  # ['a', 'b,c'] —— 只分 1 次
print("a   b c".split())               # ['a', 'b', 'c'] —— 默认按空白分，连续空白视为一个

# rsplit：从右边开始分
print("a.b.c".rsplit(".", 1))          # ['a.b', 'c']

# partition：分成三部分（分隔符前、分隔符、分隔符后）
print("a,b,c".partition(","))          # ('a', ',', 'b,c')

# join：用分隔符把列表拼接成字符串
print(",".join(["a", "b", "c"]))       # "a,b,c"
print("".join(["a", "b", "c"]))        # "abc" —— 空字符串做分隔符 = 直接拼接

# splitlines：按行拆分
text = "第一行\n第二行\r\n第三行"
print(text.splitlines())               # ['第一行', '第二行', '第三行']
```

## 5. join vs + ——性能差异的真相

### 5.1 问题：循环拼接字符串有多慢

```python
# 方式 A：用 + 拼接（坏习惯）
result = ""
for i in range(10000):
    result += str(i) + ","     # 每次创建新字符串对象！

# 方式 B：用 join（推荐）
result = ",".join(str(i) for i in range(10000))
```

### 5.2 为什么 + 这么慢（O(n^2) vs O(n)）

每次执行 `result += str(i)` 时，Python 必须：
1. 创建一个新字符串对象（长度 = len(result) + len(str(i))）
2. 把 result 的内容全部复制到新字符串
3. 把 str(i) 的内容追加到新字符串
4. 让 result 指向新字符串，旧字符串等待垃圾回收

循环 10000 次，总复制量大约是 `1 + 2 + 3 + ... + 10000 ≈ 5000 万`个字符。

而 `join` 的做法是：
1. 先遍历一次，计算所有片段的总长度
2. 一次性分配足够的内存
3. 把每个片段拷贝到对应位置

总复制量只有 `总长度 × 1 次`，时间复杂度 O(n)。

**结论**：要拼接多个字符串片段时，永远用 `join`。

## 6. 编码问题——从乱码到理解

### 6.1 一个真实的乱码场景

假设你在 Windows 上用记事本写了一个文本文件，保存时默认用了 GBK 编码。文件内容 `你好` 被存为：

```
GBK 编码:   C4 E3 BA C3   （你好 = 4 个字节）
```

然后你把文件发给一个 Mac 用户，Ta 用文本编辑器打开，编辑器默认用 UTF-8 解码，结果看到：

```
涔犳儴
```

**为什么会这样？**因为那些 GBK 字节恰好也能被 UTF-8 解析，但解析出来是毫不相关的汉字。这就是乱码的本质：**用错了编码方式去解读字节数据**。

### 6.2 编码解码的基本流程

```
str（文本）   ←── encode() / decode() ──→   bytes（字节）
"你好"                                       b'\xc4\xe3\xba\xc3' (GBK)
                                             b'\xe4\xbd\xa0\xe5\xa5\xbd' (UTF-8)
```

- `str`：人类可读的 Unicode 文本
- `bytes`：原始字节（0-255），用于存储和网络传输
- `encode()`：str 变为 bytes（文本 → 字节）
- `decode()`：bytes 变为 str（字节 → 文本）

### 6.3 动手试一下

```python
s = "你好"

# 用 UTF-8 编码：每个中文字符 3 字节
utf8_bytes = s.encode("utf-8")
print(utf8_bytes)                # b'\xe4\xbd\xa0\xe5\xa5\xbd'
print(len(utf8_bytes))           # 6

# 用 GBK 编码：每个中文字符 2 字节
gbk_bytes = s.encode("gbk")
print(gbk_bytes)                 # b'\xc4\xe3\xba\xc3'
print(len(gbk_bytes))            # 4

# 解码回去
print(utf8_bytes.decode("utf-8"))  # "你好"
print(gbk_bytes.decode("gbk"))     # "你好"

# 制造乱码：用错误的编码解码
try:
    print(gbk_bytes.decode("utf-8"))  # 可能报错或出乱码
except UnicodeDecodeError as e:
    print(f"解码失败：{e}")

# 容错处理
broken = b"hello\xffworld"
print(broken.decode("utf-8", errors="replace"))  # "hello�world"
print(broken.decode("utf-8", errors="ignore"))   # "helloworld"
```

### 6.4 bytes 和 str 不能混用

```python
b = b"hello"          # bytes 字面量（前面有 b）
s = "hello"           # str 字面量

# print(b + s)        # TypeError: can't concat str to bytes
print(b + b" world")              # b"hello world" —— 类型一致才能操作
print(b.decode() + " world")      # "hello world" —— 解码后拼接
```

### 6.5 文件操作中的编码

```python
# 读文本文件：始终指定 encoding
with open("file.txt", "r", encoding="utf-8") as f:
    content = f.read()              # 返回 str

# 读二进制文件（图片、视频等）：不指定 encoding
with open("file.bin", "rb") as f:
    raw = f.read()                  # 返回 bytes
```

### 练习 3：大小写转换和空白处理

写一个函数 `clean_username(name)`，它：
1. 去掉首尾空白
2. 把大写字母全部转成小写
3. 把中间连续的多个空白替换为单个空格

**参考答案**：

```python
def clean_username(name):
    """清洗用户名：去空白、转小写、合并多余空格"""
    name = name.strip()        # 去掉首尾空白
    name = name.lower()         # 转小写
    # 按空白拆分（连续空白自动合并），再用单个空格拼接
    name = " ".join(name.split())
    return name

# 测试
print(clean_username("  张  三  "))    # "张 三"
print(clean_username("  Hello   World  "))  # "hello world"
```

## 进阶练习

### 练习 4：解析简单的 CSV 行（不用 csv 模块）

写一个函数 `parse_csv_line(line)`，它接收一个 CSV 格式的字符串（逗号分隔），返回字段列表。需要考虑引号包围的字段（字段中可能包含逗号）。

CSV 规则（简化版）：
- 字段由逗号分隔
- 字段可以由双引号包围，引号内的逗号不算分隔符
- 连续两个双引号 `""` 表示转义后的双引号字符

```python
parse_csv_line('张三,20,"北京,朝阳区"')
# 应返回：['张三', '20', '北京,朝阳区']

parse_csv_line('"他说""你好""",25')
# 应返回：['他说"你好"', '25']
```

**参考答案**：

```python
def parse_csv_line(line):
    """解析简化版 CSV 行，处理引号包围字段和引号转义"""
    fields = []          # 存储解析出的字段
    current = []         # 当前正在构建的字段（用列表拼接效率更高）
    in_quotes = False    # 是否在引号内部
    i = 0

    while i < len(line):
        ch = line[i]

        if in_quotes:
            if ch == '"':
                # 遇到引号：检查下一个字符是否也是引号（转义）
                if i + 1 < len(line) and line[i + 1] == '"':
                    current.append('"')   # 转义：两个引号 → 一个引号
                    i += 2                # 跳过两个引号
                    continue
                else:
                    # 单个引号：关闭引号模式
                    in_quotes = False
                    i += 1
                    continue
            else:
                current.append(ch)
                i += 1
        else:
            if ch == '"':
                in_quotes = True     # 进入引号模式
                i += 1
            elif ch == ',':
                # 字段分隔符：保存当前字段，开始新字段
                fields.append("".join(current))
                current = []
                i += 1
            else:
                current.append(ch)
                i += 1

    # 最后一个字段
    fields.append("".join(current))
    return fields


# 测试
print(parse_csv_line('张三,20,"北京,朝阳区"'))
# ['张三', '20', '北京,朝阳区']

print(parse_csv_line('"他说""你好""",25'))
# ['他说"你好"', '25']

print(parse_csv_line("a,b,c"))
# ['a', 'b', 'c']
```

## 常见错误

### 错误 1：忘记字符串不可变

```python
# 错误写法
s = "hello"
s.upper()            # 返回 "HELLO"，但没有赋回去！
print(s)             # 还是 "hello"

# 正确写法
s = s.upper()
print(s)             # "HELLO"
```

### 错误 2：循环中用 + 拼接

```python
# 错误写法 —— O(n^2)
result = ""
for word in words:
    result += word + " "

# 正确写法 —— O(n)
result = " ".join(words)
```

### 错误 3：str 和 bytes 混用

```python
# 错误写法
data = b"hello"
print(data + " world")           # TypeError!

# 正确写法
print(data + b" world")          # 都是 bytes
print(data.decode() + " world")  # 都转成 str
```

### 错误 4：f-string 忘了 f 前缀

```python
name = "小明"
print("{name}")      # 输出字面量：{name}
print(f"{name}")     # 输出：小明
```

### 错误 5：没有指定文件编码

```python
# 可能出问题的写法（依赖系统默认编码）
with open("file.txt", "r") as f:
    content = f.read()

# 正确写法：明确指定编码
with open("file.txt", "r", encoding="utf-8") as f:
    content = f.read()
```

### 错误 6：用 index 查找而不处理异常

```python
# 错误写法 —— 如果不存在会抛 ValueError
pos = s.index("xyz")

# 更好：用 find（返回 -1 而非抛异常）
pos = s.find("xyz")
if pos != -1:
    print(f"找到了，位置在 {pos}")
```

## 本章小结

- **str 是不可变 Unicode 序列**，Python 没有 char 类型
- **索引**：正向从 0 开始，反向从 -1 开始；**切片** `[start:end:step]` 左闭右开
- **f-string**（`f"{变量}"`）是最推荐的格式化方式，支持格式控制（`:.2f`、`:,`、对齐等）
- **常用方法**：`upper/lower`、`strip/lstrip/rstrip`、`split/join`、`find/index`、`replace`、`startswith/endswith`
- **`join` 是 O(n)**，循环 `+` 是 O(n^2)；拼接多个片段永远用 join
- **编码解码**：`encode()` 将 str 转 bytes，`decode()` 将 bytes 转 str；乱码源于编解码方式不一致
- Python 3 默认 UTF-8，操作文件始终显式指定 `encoding` 参数
- 能用 `startswith`/`endswith`/`in` 判断的，不要用正则表达式
