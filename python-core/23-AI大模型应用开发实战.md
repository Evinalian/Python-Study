## 23.1 什么是大模型 API

用一句话说：**大模型 API 让你在自己的程序里调用像 ChatGPT 那样的 AI 能力——你发一段文字过去，它回复一段文字回来。**

这和你在网页上跟 ChatGPT 聊天没有本质区别，区别在于：
- 网页版：你打字 → ChatGPT 回复 → 你看到
- API 版：你的代码发请求 → ChatGPT 回复 → 你的代码拿到回复 → 你可以做任何处理（存数据库、发给用户、触发下一步操作……）

API 调用让你能把 AI 能力**集成到自己的应用里**——而不是只能在特定网页里用。本章就是教你如何做到这一点。

## 23.2 你的第一次 AI 对话（5 行代码）

### 23.2.1 安装与配置


### 23.2.2 5 行代码完成第一次对话

```python
import os

from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])
```

## 23.3 理解 Messages：三种角色

Chat API 的核心是一个 **messages 列表**。它不是单次问答，而是一段"对话记录"，包含三种角色：

### 23.3.1 system —— 设定 AI 的"人设"

```python
# system prompt 在整个对话中持续生效
{"role": "system", "content": "你是一个严谨的数学老师。回答要分步骤，用中文。"}
```

### 23.3.2 user —— 用户的问题

```python
# 你提的问题或要求
{"role": "user", "content": "1 + 2 * 3 等于多少？"}
```

### 23.3.3 assistant —— AI 的回复

```python
# AI 之前的回答。多轮对话时，把历史 assistant 回复放回来，AI 就能"记住"前面说过什么
{"role": "assistant", "content": "根据运算优先级，先算乘法 2*3=6，再加 1，结果是 7。"}
```

### 23.3.4 完整的多轮对话示例

```python
from openai import OpenAI

client = OpenAI()

# 模拟一个多轮对话：学生连续追问
messages = [
    {
        "role": "system",
        "content": (
            "你是一个 Python 编程导师。回答要包含代码示例。"
            "如果学生犯了错误，温和地指出并解释原因。"
        ),
    },
    {"role": "user", "content": "Python 的列表和元组有什么区别？"},
    {
        "role": "assistant",
        "content": "列表用 [] 创建，可以修改（增删改查）。元组用 () 创建，一旦创建就不能修改。",
    },
    {"role": "user", "content": "给我一个具体的代码例子。"},
    # ↑ AI 会基于前面的对话上下文来回答这个问题
]

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=messages,
    temperature=0.7,
)
print(response.choices[0].message.content)
```

**关键理解**：每次调用 API，你都要把**完整的对话历史**传回去。模型本身不保存状态——它每次都是"从头读一遍对话记录，然后接话"。这也是为什么长对话会越来越贵——messages 列表越来越长，每次都要传全部。

## 23.4 流式输出：像打字一样逐字显示

### 23.4.1 为什么需要流式

先想象两种体验：

**非流式（等 5 秒一次性出结果）**：用户点"发送" → 屏幕卡住不动 → 5 秒后突然出现一大段文字。
**流式（一个字一个字往外蹦）**：用户点"发送" → 立刻开始有字出现 → 一个字接一个字 → 5 秒后完成。

显然第二种体验好得多——用户看到系统在"思考和工作"，不会以为程序卡死了。这就是为什么 ChatGPT、Claude、Kimi 都用流式输出。

### 23.4.2 实现流式输出

```python
from openai import OpenAI

client = OpenAI()


def stream_chat(prompt: str) -> str:
    """流式对话：像打字一样逐字输出。"""
    stream = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        stream=True,        # ← 开启流式模式
        temperature=0.7,
    )

    full_response = []
    for chunk in stream:
        # chunk 是一个增量片段，可能只包含 1-3 个字符
        delta = chunk.choices[0].delta
        content = delta.content
        if content:
            print(content, end="", flush=True)   # 逐字打印，不换行
            full_response.append(content)

    print()  # 最后的换行
    return "".join(full_response)


# 使用
result = stream_chat("给我讲一个简短的笑话。")
```

输出效果：屏幕上会一个字一个字地出现回复，就像在聊天窗口里实时打字一样。

**`flush=True` 不能忘**：Python 的 `print` 有输出缓冲——不加 `flush=True`，字符会攒到一定数量才一次性输出，流式效果就没了。

## 23.5 Function Calling —— 让 AI 调用你的函数

这是大模型应用中最强大的能力，也是理解门槛最高的部分。我们用递进的方式讲清楚。

### 23.5.1 一句话理解

Function Calling 就是：**你告诉模型"我有这些工具可以用"，当用户的问题需要用到工具时，模型返回"请调用某某工具、参数是什么"，然后你的代码去执行，把结果传回给模型，模型再生成最终回复。**

模型不执行函数——它只是一个"聪明的大脑"，告诉你的代码"该动手了"。

### 23.5.2 拆解为四步

用一个"天气助手"的例子逐步拆解。用户问"北京今天天气怎么样？"。

**第 1 步：定义工具（告诉模型你有什么武器）**

```python
# 用 JSON Schema 描述你的函数：叫什么名字、干什么用、接受什么参数
TOOLS = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的当前天气。返回天气状况和温度。",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名，使用中文，例如：北京、上海",
                    }
                },
                "required": ["city"],
            },
        },
    },
]
```

`description` 字段很重要——模型靠它来判断"这个工具是不是用户需要的"。写得越清晰，模型判断越准确。

**第 2 步：模型决定是否调用工具**

```python
# 第一轮调用：把 tools 传给模型
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "北京今天天气怎么样？"}],
    tools=TOOLS,
    tool_choice="auto",    # "auto" 表示让模型自己决定要不要调工具
)

response_message = response.choices[0].message
print(f"模型要回复文本吗？{response_message.content}")    # None（不需要文本回复）
print(f"模型要调工具吗？{response_message.tool_calls}")   # 有内容！

# 看模型决定调用什么
tool_call = response_message.tool_calls[0]
print(f"调用函数: {tool_call.function.name}")         # "get_weather"
print(f"传入参数: {tool_call.function.arguments}")    # '{"city": "北京"}'
```

模型没有直接回答"北京天气怎么样"，而是返回了一个**工具调用请求**：请执行 `get_weather(city="北京")`。

**第 3 步：你的代码执行工具函数**

```python
import json


# 实际的工具函数（这里用 mock 数据，真实项目中调天气 API）
def get_weather(city: str) -> str:
    weather_data = {
        "北京": "晴，25°C，湿度 40%",
        "上海": "多云转小雨，22°C，湿度 70%",
    }
    return weather_data.get(city, f"未找到 {city} 的天气数据")


# 把可用函数登记到一个字典里
AVAILABLE_FUNCTIONS = {
    "get_weather": get_weather,
}

# 执行模型要求的工具调用
func_name = tool_call.function.name          # "get_weather"
func_args = json.loads(tool_call.function.arguments)  # {"city": "北京"}
func_result = AVAILABLE_FUNCTIONS[func_name](**func_args)
print(f"工具执行结果: {func_result}")         # "晴，25°C，湿度 40%"
```

**第 4 步：把结果返回给模型，模型生成最终回复**

```python
# 构建新的 messages：包含原问题 + 模型的工具调用请求 + 工具执行结果
messages = [
    {"role": "user", "content": "北京今天天气怎么样？"},
    response_message,   # 模型的工具调用请求（这个消息对象可以直接塞进列表）
    {
        "role": "tool",
        "tool_call_id": tool_call.id,
        "content": func_result,
    },
]

# 第二轮调用：把工具结果发给模型
second_response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=messages,
)
print(second_response.choices[0].message.content)
# 输出：北京今天天气晴朗，气温 25°C，湿度 40%，是个好天气！
```

### 23.5.3 把四步整合成一个函数

```python
import json
from openai import OpenAI

client = OpenAI()

# ===== 工具定义 =====
TOOLS = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的当前天气。返回天气状况和温度。",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名，使用中文",
                    }
                },
                "required": ["city"],
            },
        },
    },
    {
        "type": "function",
        "function": {
            "name": "calculate",
            "description": "执行一个数学表达式计算，支持加减乘除。",
            "parameters": {
                "type": "object",
                "properties": {
                    "expression": {
                        "type": "string",
                        "description": "数学表达式，例如 '2 + 3 * 4'",
                    }
                },
                "required": ["expression"],
            },
        },
    },
]


# ===== 工具实现 =====
def get_weather(city: str) -> str:
    weather = {
        "北京": "晴，25°C",
        "上海": "多云，22°C",
        "深圳": "阵雨，28°C",
    }
    return weather.get(city, f"未找到 {city} 的天气")


def calculate(expression: str) -> str:
    try:
        # 安全地计算：只允许数字、运算符、空格、小数点
        allowed = set("0123456789+-*/().% ")
        if not all(c in allowed for c in expression):
            return "错误：表达式包含不允许的字符"
        result = eval(expression)
        return str(result)
    except Exception as e:
        return f"计算错误: {e}"


AVAILABLE_FUNCTIONS = {
    "get_weather": get_weather,
    "calculate": calculate,
}


# ===== 主流程 =====
def run_agent(user_query: str) -> str:
    """处理用户请求，自动判断是否需要调用工具。"""
    messages = [
        {"role": "system", "content": "你是一个生活助手，可以查天气和做数学计算。"},
        {"role": "user", "content": user_query},
    ]

    # 第一轮：模型决定要不要调工具
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
        tools=TOOLS,
        tool_choice="auto",
    )

    response_message = response.choices[0].message

    # 不需要调工具 → 直接返回文本
    if not response_message.tool_calls:
        return response_message.content or "（模型没有返回内容）"

    # 否则：执行工具调用
    messages.append(response_message)

    for tool_call in response_message.tool_calls:
        func_name = tool_call.function.name
        func_args = json.loads(tool_call.function.arguments)
        print(f"[调用工具] {func_name}({func_args})")

        func = AVAILABLE_FUNCTIONS[func_name]
        result = func(**func_args)

        messages.append({
            "role": "tool",
            "tool_call_id": tool_call.id,
            "content": result,
        })

    # 第二轮：模型基于工具结果生成最终回复
    final_response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
    )
    return final_response.choices[0].message.content or ""


# ===== 测试 =====
print("Q: 北京天气怎么样？")
print("A:", run_agent("北京天气怎么样？"))
print()

print("Q: 100 * 25 + 30 等于多少？")
print("A:", run_agent("100 * 25 + 30 等于多少？"))
print()

print("Q: 你好，你是谁？")
print("A:", run_agent("你好，你是谁？"))
# 这个问题不需要调用工具，模型直接回答
```

运行效果：

```text
Q: 北京天气怎么样？
[调用工具] get_weather({'city': '北京'})
A: 北京当前天气晴朗，气温 25°C，适合出门！

Q: 100 * 25 + 30 等于多少？
[调用工具] calculate({'expression': '100 * 25 + 30'})
A: 100 * 25 + 30 = 2530

Q: 你好，你是谁？
A: 你好！我是一个生活助手，可以帮你查天气和做数学计算……
```

## 23.6 Prompt 工程基础

Prompt 工程不是"玄学口诀"，而是对模型行为的系统性控制。以下是经过验证的核心技巧。

### 23.6.1 清晰指令胜过模糊请求

```python
# ❌ 差 —— 模型不知道你到底要什么
prompt = "帮我写点代码。"

# ✓ 好 —— 明确告诉模型输出什么、什么格式、什么约束
prompt = (
    "写一个 Python 函数 `parse_log(filepath: str) -> list[dict]`，"
    "解析 Nginx 访问日志。每行格式：'IP - - [时间] \"方法 URL 协议\" 状态码 大小'。"
    "返回包含 ip、time、method、url、status 的字典列表。"
    "只输出函数代码，不要解释。"
)
```

### 23.6.2 使用分隔符组织信息

```python
prompt = """
将以下代码从同步改为异步版本。

【原始代码】
import requests
def fetch_users():
    resp = requests.get("https://api.example.com/users")
    return resp.json()
【代码结束】

【要求】
1. 使用 aiohttp 替代 requests
2. 函数改为 async def
3. 保留错误处理
"""
```

用 `"""..."""`、`【...】`、`---` 等分隔符把不同类型的内容区分开，模型不容易混淆指令和数据。

### 23.6.3 System Prompt：给 AI 设定"长期人设"

System prompt 不同于 user prompt——它在整个对话中持续生效：

```python
SYSTEM = """
你是一个严格但友善的 Python 代码审查员。工作原则：

1. 先指出优点，再指出问题
2. 问题分三级：严重（bug/安全漏洞）、建议（性能/可读性）、风格（PEP 8）
3. 每个问题给出修复后的代码
4. 如果代码没有问题，直接说"代码没有问题"然后指出可优化之处

不要：
- 不要写"当然可以！让我帮你看看"这类客套话
- 不要建议你无法确定的第三方库
"""
```

## 23.7 成本管理：Token 怎么算钱

### 23.7.1 Token 是什么

大模型不是按"字"计费，而是按 **token**（词元）。大约：

- 英文：1 token ≈ 0.75 个单词 ≈ 4 个字符
- 中文：1 个汉字 ≈ 1.5-2 个 token

可以用 `tiktoken` 精确计算：

```python
import tiktoken

enc = tiktoken.encoding_for_model("gpt-4o")
text = "Hello, how are you? 你好吗？"
tokens = enc.encode(text)
print(f"文本: {text}")
print(f"Token 数量: {len(tokens)}")
```

### 23.7.2 费用估算

截至 2026 年，主流模型的大致定价：

| 模型 | 输入 (每百万 token) | 输出 (每百万 token) |
|------|---------------------|---------------------|
| gpt-4o | 约 $2.50 | 约 $10.00 |
| gpt-4o-mini | 约 $0.15 | 约 $0.60 |
| deepseek-chat | 约 $0.14 | 约 $0.28 |
| glm-4-flash | 极低 / 有免费额度 | 极低 / 有免费额度 |

以一次典型对话（1000 token 输入 + 500 token 输出）为例：
- gpt-4o：约 $0.0075（不到 1 美分）
- gpt-4o-mini：约 $0.00045（几乎可以忽略）

**经验法则**：开发调试阶段始终用 mini/flash 类便宜模型，功能验证通过后再切强模型做最终测试。

## 23.8 多平台兼容：一套代码调用所有模型

OpenAI 的 API 格式已是行业标准。国内主流平台（智谱、DeepSeek、通义千问、月之暗面）都兼容这个格式，切换只需改 `base_url`：

```python
from openai import OpenAI
import os


def create_client(platform: str) -> OpenAI:
    """根据平台名创建对应的客户端。"""
    configs = {
        "openai": {
            "api_key": os.environ["OPENAI_API_KEY"],
            "base_url": "https://api.openai.com/v1",
        },
        "zhipu": {
            "api_key": os.environ["ZHIPU_API_KEY"],
            "base_url": "https://open.bigmodel.cn/api/paas/v4",
        },
        "deepseek": {
            "api_key": os.environ["DEEPSEEK_API_KEY"],
            "base_url": "https://api.deepseek.com/v1",
        },
        "dashscope": {   # 通义千问
            "api_key": os.environ["DASHSCOPE_API_KEY"],
            "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
        },
    }
    cfg = configs[platform]
    return OpenAI(api_key=cfg["api_key"], base_url=cfg["base_url"])


# 使用 —— 只需要改 platform 参数
zhipu = create_client("zhipu")
response = zhipu.chat.completions.create(
    model="glm-4-flash",         # 注意：每个平台有自己的模型名
    messages=[{"role": "user", "content": "你好，请介绍一下你自己。"}],
)
print(response.choices[0].message.content)
```

## 23.9 LangChain 是什么（以及为什么先不要用它）

LangChain 是一个 LLM 应用开发框架，提供 Chain（链式调用）、Agent（智能体）、RAG（检索增强生成）等高级抽象。

对初学者的建议：**先学会用 openai SDK 裸写，再决定要不要用框架。**

原因很实际：
- LangChain 的 API 变化非常频繁，6 个月前的教程可能已经跑不通
- 框架做了大量封装，出问题时你看不到底层发生了什么，调试极其痛苦
- 对于简单的对话和 Function Calling，原生 SDK 足够了

等你对原生 SDK 和 API 的行为非常熟悉后，如果项目真的复杂到需要框架的编排能力，再学 LangChain 不迟。

## 23.10 本章小结

1. **大模型 API**：把 AI 对话能力集成到你的程序里。openai SDK 是标准入口。
2. **Messages 三种角色**：system（定人设）、user（提问题）、assistant（历史回复）。
3. **流式输出**：`stream=True` + `flush=True`，像打字一样逐字显示，用户体验远超一次性输出。
4. **Function Calling 四步**：定义工具（JSON Schema）→ 模型决定调用 → 你的代码执行函数 → 结果返回模型 → 模型生成最终回复。
5. **成本控制**：开发用 mini/flash 模型，上线后再切强模型。注意 messages 越长越贵。
6. **多平台兼容**：国内平台普遍兼容 OpenAI API 格式，换 `base_url` 和 `api_key` 即可。

## 基础练习

### 练习 1：完成一次 AI 对话

用 openai SDK 完成一次对话，要求：（1）从 `.env` 文件读取 API Key；（2）向模型发送"用 Python 写一个冒泡排序函数"；（3）打印模型的完整回复和 token 用量。

**参考答案：**

```python
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "你是一个 Python 编程助手。"},
        {"role": "user", "content": "用 Python 写一个冒泡排序函数，包含注释。"},
    ],
    temperature=0.3,   # 代码生成用较低的温度，输出更稳定
)

# 打印回复
print("=" * 50)
print(response.choices[0].message.content)
print("=" * 50)

# 打印 token 用量
usage = response.usage
print(f"输入 token: {usage.prompt_tokens}")
print(f"输出 token: {usage.completion_tokens}")
print(f"总计 token: {usage.total_tokens}")
```

### 练习 2：实现流式输出打印

将练习 1 改为流式输出。写入一个 `stream_ask(prompt: str)` 函数，接收用户问题，流式打印回复，返回完整文本。

**参考答案：**

```python
from openai import OpenAI

client = OpenAI()


def stream_ask(prompt: str) -> str:
    """流式提问，逐字打印回复，返回完整文本。"""
    stream = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        stream=True,
        temperature=0.7,
    )

    full_text = []
    print("AI: ", end="", flush=True)
    for chunk in stream:
        content = chunk.choices[0].delta.content
        if content:
            print(content, end="", flush=True)
            full_text.append(content)
    print()  # 最后换行
    return "".join(full_text)


# 测试
result = stream_ask("用一句话解释什么是递归。")
print(f"\n返回的完整文本长度: {len(result)} 字符")
```

### 练习 3：用 system prompt 设定 AI 的角色

写一个程序，用 system prompt 把 AI 设定为"苏格拉底式导师"——不直接给答案，而是用提问引导你思考。然后问它"我应该怎么学好编程？"，观察它的回复风格。

**参考答案：**

```python
from openai import OpenAI

client = OpenAI()

SOCRATES_PROMPT = """
你是一位采用苏格拉底式教学法的导师。你的原则是：

1. 永远不直接告诉学生答案
2. 通过一连串问题引导学生自己发现答案
3. 每次只问一个问题
4. 在学生回答后，基于他的回答提出下一个问题
5. 如果学生卡住了，给出一个微小的提示，然后继续提问
"""

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": SOCRATES_PROMPT},
        {"role": "user", "content": "我应该怎么学好编程？"},
    ],
    temperature=0.7,
)

print(response.choices[0].message.content)
# 模型的回复不会是"你应该每天练习、做项目……"
# 而是类似"你认为'学好'对你来说意味着什么？能具体描述一下吗？"
```

## 进阶练习

### 练习 4：实现一个带 Function Calling 的天气查询助手

实现一个完整的命令行天气助手，具备以下工具：

1. `get_weather(city: str)` —— 查询城市天气（用 mock 数据）
2. `get_time(city: str)` —— 查询城市当前时间（用 mock 数据）

要求：
- 支持命令行交互（`input()` 读用户输入，`/exit` 退出，`/clear` 清空对话）
- 自动判断是否需要调用工具
- 支持多轮对话（维护 messages 历史）
- 如果用户问的问题不需要工具，直接对话

**参考答案：**

```python
import json
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

# ===== 工具定义 =====
TOOLS = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的当前天气，返回天气状况和温度",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名，使用中文",
                    }
                },
                "required": ["city"],
            },
        },
    },
    {
        "type": "function",
        "function": {
            "name": "get_time",
            "description": "获取指定城市的当前时间",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名，使用中文",
                    }
                },
                "required": ["city"],
            },
        },
    },
]

# ===== 工具实现 =====
from datetime import datetime, timedelta

# 模拟城市时区偏移（小时）
CITY_TIMEZONES = {
    "北京": 8,
    "上海": 8,
    "东京": 9,
    "纽约": -4,
    "伦敦": 1,
    "巴黎": 2,
    "悉尼": 10,
}


def get_weather(city: str) -> str:
    weather_data = {
        "北京": "晴，25°C，湿度 40%，微风",
        "上海": "多云转小雨，22°C，湿度 70%，东南风 3 级",
        "深圳": "阵雨，28°C，湿度 85%",
        "东京": "晴，23°C，湿度 55%",
        "纽约": "阴天，18°C，湿度 60%",
        "伦敦": "小雨，14°C，湿度 80%",
        "巴黎": "晴转多云，20°C，湿度 50%",
        "悉尼": "晴，26°C，湿度 45%",
    }
    return weather_data.get(city, f"暂无 {city} 的天气数据，请尝试其他城市。")


def get_time(city: str) -> str:
    if city not in CITY_TIMEZONES:
        return f"暂无 {city} 的时区信息，请尝试其他城市。"
    offset = CITY_TIMEZONES[city]
    city_time = datetime.utcnow() + timedelta(hours=offset)
    return f"{city}当前时间：{city_time.strftime('%Y-%m-%d %H:%M')}（UTC{offset:+d}）"


AVAILABLE_FUNCTIONS = {
    "get_weather": get_weather,
    "get_time": get_time,
}


# ===== 调用 AI 并处理工具调用 =====
def call_ai(messages: list[dict]) -> str:
    """调用 AI，自动处理 Function Calling 循环。"""
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
        tools=TOOLS,
        tool_choice="auto",
    )
    response_message = response.choices[0].message

    # 不需要工具 → 直接返回文本
    if not response_message.tool_calls:
        return response_message.content or "（模型没有返回内容）"

    # 需要工具 → 逐个执行
    messages.append(response_message)
    for tool_call in response_message.tool_calls:
        func_name = tool_call.function.name
        func_args = json.loads(tool_call.function.arguments)
        print(f"  [工具调用] {func_name}({func_args})")

        func = AVAILABLE_FUNCTIONS[func_name]
        result = func(**func_args)

        messages.append({
            "role": "tool",
            "tool_call_id": tool_call.id,
            "content": result,
        })

    # 工具执行完后，再次调用 AI 生成最终回复
    final_response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
    )
    return final_response.choices[0].message.content or ""


# ===== 命令行交互 =====
def main():
    print("=" * 50)
    print("  天气助手 - 可以查询天气和时间")
    print("  输入 /exit 退出，/clear 清空对话")
    print("=" * 50)

    messages = [
        {
            "role": "system",
            "content": (
                "你是一个生活助手。当用户询问天气时使用 get_weather 函数，"
                "询问时间时使用 get_time 函数。用中文回复，语气友好。"
            ),
        }
    ]

    while True:
        try:
            user_input = input("\n你: ").strip()
        except (EOFError, KeyboardInterrupt):
            print("\n再见！")
            break

        if not user_input:
            continue
        if user_input == "/exit":
            print("再见！")
            break
        if user_input == "/clear":
            messages = [messages[0]]   # 只保留 system prompt
            print("对话历史已清空。")
            continue

        # 把用户输入加入对话
        messages.append({"role": "user", "content": user_input})

        # 调用 AI
        try:
            reply = call_ai(messages)
            print(f"\n助手: {reply}")
            # 把 AI 回复也加入对话历史（用于多轮对话）
            messages.append({"role": "assistant", "content": reply})
        except Exception as e:
            print(f"\n错误: {e}")
            # 出错时移除刚加入的 user 消息，避免污染历史
            messages.pop()


if __name__ == "__main__":
    main()
```

**示例交互：**

```text
你: 北京天气怎么样？
  [工具调用] get_weather({'city': '北京'})
助手: 北京当前天气晴朗，气温 25°C，湿度 40%，微风。是个不错的天气！

你: 那上海呢？
  [工具调用] get_weather({'city': '上海'})
助手: 上海当前多云转小雨，气温 22°C，记得带伞哦。

你: 伦敦现在几点？
  [工具调用] get_time({'city': '伦敦'})
助手: 伦敦当前时间是 2026-05-26 09:30（UTC+1）。

你: /clear
对话历史已清空。

你: 你好，请介绍一下自己。
助手: 你好！我是一个生活助手……（不需要工具调用，直接对话）
```

## 常见错误

### 错误 1：API Key 没设置或加载顺序错误

```python
# ❌ 错误：先读 os.environ，再 load_dotenv()——读了个空
api_key = os.environ["OPENAI_API_KEY"]     # KeyError!
load_dotenv()                               # 太晚了
```

```python
# ✓ 正确：先 load_dotenv()，再读取
from dotenv import load_dotenv
load_dotenv()                                # 先加载
api_key = os.environ["OPENAI_API_KEY"]       # 再读取
```

### 错误 2：模型回复被截断

```python
# ❌ 错误：max_tokens 太小，回复到一半就断了
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "详细解释一下 Python 的 GIL"}],
    max_tokens=50,                     # 太少了！
)
# finish_reason 会是 "length"，表示因长度限制而截断
```

```python
# ✓ 正确：根据需要设置合适的 max_tokens
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "详细解释一下 Python 的 GIL"}],
    max_tokens=2000,                   # 足够写出长回复
)
```

### 错误 3：多轮对话时忘了传历史消息

```python
# ❌ 错误：每次只传了最新问题，AI 没有记忆
messages = [{"role": "user", "content": "上一轮我提到的事情你还记得吗？"}]
# AI 当然不记得——你没把上一轮的消息传回去！
```

```python
# ✓ 正确：维护一个 messages 列表，每次追加并全部传回
messages = [{"role": "system", "content": "你是一个助手。"}]
while True:
    user_input = input("你: ")
    messages.append({"role": "user", "content": user_input})
    response = client.chat.completions.create(
        model="gpt-4o-mini", messages=messages,
    )
    reply = response.choices[0].message.content
    messages.append({"role": "assistant", "content": reply})
    print(f"AI: {reply}")
```

### 错误 4：流式输出没有加 flush

```python
# ❌ 错误：字符被缓冲了，不会即时显示
for chunk in stream:
    print(chunk.choices[0].delta.content, end="")
   # 没有 flush=True，可能攒了很多行才一次性输出
```

```python
# ✓ 正确：flush=True 强制立即输出
for chunk in stream:
    content = chunk.choices[0].delta.content
    if content:
        print(content, end="", flush=True)
```

### 错误 5：Function Calling 的工具描述写得太模糊

```python
# ❌ 错误：description 太模糊，模型不知道什么时候该用这个工具
"description": "获取信息"
```

```python
# ✓ 正确：description 具体说明这个工具做什么、什么时候该调用
"description": "获取指定城市的当前天气，包括天气状况、温度和湿度。当用户询问某地天气时调用此函数。"
```

### 错误 6：忘了处理 model 不支持 tools 的情况

```python
# ❌ 错误：某些较老的模型或第三方模型不支持 tools 参数
response = client.chat.completions.create(
    model="some-old-model",
    messages=messages,
    tools=TOOLS,                     # 可能报错或默默忽略
)
```

```python
# ✓ 正确：确认模型支持 Function Calling。主流模型（gpt-4o, gpt-4o-mini,
# deepseek-chat, glm-4 等）都支持。如果必须用旧模型，通过 prompt 模拟。
```
