## 1. import 的三种方式

### 1.1 导入整个模块

```python
import math    # 导入 math 模块

# 使用模块中的内容时，必须加模块名前缀
print(math.sqrt(16))    # 4.0
print(math.pi)          # 3.14159...
```

**什么时候用**：当你要用模块中的多个函数时。前缀 `math.` 让代码清晰——一看就知道 `sqrt` 来自哪里。

### 1.2 导入特定函数

```python
from math import sqrt, pi    # 只导入 sqrt 和 pi

# 直接使用，不需要加模块名前缀
print(sqrt(16))    # 4.0
print(pi)          # 3.14159...

# 但 math 本身没有被导入
# print(math.e)    # NameError: name 'math' is not defined
```

**什么时候用**：当你只需要模块中的一两个函数，且不想每次都写模块前缀时。

### 1.3 起别名

```python
import numpy as np                # 给模块起短名字
import matplotlib.pyplot as plt   # 给子模块起别名

# 使用时用别名
arr = np.array([1, 2, 3])
# plt.plot(...)
```

**什么时候用**：模块名很长但使用频繁时（如 `numpy` -> `np`，`pandas` -> `pd`），这是社区约定俗成的。

**Java 对比**：Python 的 `import` 相当于 Java 的 `import` 语句。`import math` 类似 `import java.lang.Math`，`from math import sqrt` 类似 `import static java.lang.Math.sqrt`。

## 2. 自定义模块：实际建两个文件互相导入

### 2.1 创建工具模块

在同一个目录下创建两个文件：

**文件 1：`utils.py`** —— 工具模块（被导入的）

```python
# utils.py —— 一个简单的工具模块

def add(a, b):
    """加法"""
    return a + b

def multiply(a, b):
    """乘法"""
    return a * b

def greet(name):
    """问候"""
    return f"你好，{name}！"

# 模块级别的常量
VERSION = "1.0.0"
```

**文件 2：`main.py`** —— 主程序（导入 utils 的）

```python
# main.py —— 使用 utils 模块

# 导入整个 utils 模块
import utils

# 使用 utils 中的函数（需要加模块前缀）
print(utils.add(10, 20))           # 30
print(utils.multiply(5, 6))        # 30
print(utils.greet("小明"))          # 你好，小明！
print(f"utils 版本: {utils.VERSION}")  # 1.0.0

# 也可以只导入需要的函数
from utils import add, greet
print(add(3, 4))                   # 7
print(greet("小红"))               # 你好，小红！
```

### 2.2 import 的底层发生了什么？

```python
import utils
```

Python 做了这几件事：
1. 检查 `sys.modules`，看 `utils` 是否已经导入过（如果有缓存，直接返回）
2. 在 `sys.path` 的目录列表中搜索 `utils.py`
3. 找到后，执行 `utils.py` 的全部顶层代码（定义函数、设置变量等）
4. 创建一个模块对象，把执行结果存在里面
5. 把模块对象缓存到 `sys.modules`（下次再 import 就直接用缓存）
6. 在当前文件中绑定名字 `utils` 指向这个模块对象

**所以，一个 .py 文件被 import 多次，文件内容只会执行一次。**

## 3. `if __name__ == "__main__"` —— 一文件两用

这是 Python 最经典的惯用法之一。用两个文件来演示：

### 3.1 创建一个"可以导入又可以独立运行"的模块

**`calculator.py`**：

```python
# calculator.py —— 可以当库用，也可以当脚本直接运行

def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

print(f"calculator.py 的 __name__ 是: {__name__}")

if __name__ == "__main__":
    # 只有在直接运行 python calculator.py 时才执行这里
    print("=== 这是直接运行时才看到的测试代码 ===")
    print(f"add(3, 5) = {add(3, 5)}")
    print(f"subtract(10, 4) = {subtract(10, 4)}")
else:
    # 被 import calculator 时执行这里
    print("calculator 被其他文件导入了，不执行测试代码")
```

### 3.2 另一个文件导入它

**`use_calculator.py`**：

```python
# use_calculator.py —— 导入 calculator 模块

import calculator    # 此时 calculator 的 __name__ 是 "calculator"

# 可以正常使用 calculator 中的函数
result = calculator.add(100, 200)
print(f"100 + 200 = {result}")
```

### 3.3 分别运行，观察差异

```bash
# 直接运行 calculator.py
$ python calculator.py
# 输出:
# calculator.py 的 __name__ 是: __main__
# === 这是直接运行时才看到的测试代码 ===
# add(3, 5) = 8
# subtract(10, 4) = 6
```

```bash
# 运行 use_calculator.py（它导入了 calculator）
$ python use_calculator.py
# 输出:
# calculator.py 的 __name__ 是: calculator
# calculator 被其他文件导入了，不执行测试代码
# 100 + 200 = 300
```

**核心原理**：当 Python 运行一个 .py 文件时，会自动设置变量 `__name__`：
- **直接运行**这个文件时，`__name__` = `"__main__"`
- **被 import** 时，`__name__` = 模块的文件名（如 `"calculator"`）

**所以 `if __name__ == "__main__"` 的目的**：让同一个文件既能当工具库被别的代码导入使用，又能直接运行做自我测试。

**Java 对比**：类似 Java 的 `public static void main(String[] args)` —— 只在直接运行时执行，但 Python 的实现更轻量。

## 4. 包：把多个模块组织在一起

包就是一个包含 `__init__.py` 的文件夹。当项目代码越来越多，需要把相关模块放进同一个文件夹（包）来管理。

### 4.1 创建一个简单的包

目录结构：
```
mypackage/
    __init__.py       # 包的标识文件（可以空，但推荐保留）
    math_utils.py     # 数学工具
    text_utils.py     # 文本工具
```

`__init__.py`：

```python
# mypackage/__init__.py
# 这个文件让 Python 知道 mypackage 是一个包

# 也可以在包里导入子模块，方便外部使用
from .math_utils import add, multiply
from .text_utils import reverse_string

__all__ = ["add", "multiply", "reverse_string"]
```

`math_utils.py`：

```python
# mypackage/math_utils.py

def add(a, b):
    return a + b

def multiply(a, b):
    return a * b
```

`text_utils.py`：

```python
# mypackage/text_utils.py

def reverse_string(s):
    return s[::-1]

def count_words(s):
    return len(s.split())
```

### 4.2 使用包

```python
# 方式 1：导入整个包（如果 __init__.py 中有导入）
import mypackage
print(mypackage.add(3, 4))

# 方式 2：从包中导入具体函数
from mypackage import add, reverse_string
print(add(5, 6))
print(reverse_string("hello"))      # olleh

# 方式 3：导入包中的子模块
from mypackage import text_utils
print(text_utils.count_words("hello world python"))   # 3
```

## 5. pip 安装第三方库

Python 的强大很大程度上来自海量的第三方库。pip 是 Python 的包管理器。

### 5.1 安装一个库并看到效果

```powershell
# 在终端中执行：
pip install requests
```

安装完成后，在你的 Python 代码中：

```python
import requests    # 导入刚安装的库

# 发送一个 HTTP 请求（访问网页）
response = requests.get("https://api.github.com")
print(f"状态码: {response.status_code}")     # 200 表示成功
print(f"响应长度: {len(response.text)} 字符")
print(f"前 100 个字符: {response.text[:100]}")
```

**这条命令做了什么？** pip 从 PyPI（Python 官方包仓库 https://pypi.org）下载 requests 库及它的依赖，然后安装到你的 Python 环境中。安装完成后，`import requests` 就能用了。

### 5.2 常用 pip 命令

```powershell
pip install requests              # 安装最新版
pip install requests==2.28.0      # 安装指定版本
pip uninstall requests            # 卸载
pip list                          # 查看已安装的所有包
pip show requests                 # 查看某个包的详细信息
pip freeze > requirements.txt     # 导出所有依赖到文件
pip install -r requirements.txt   # 根据文件恢复依赖
```

### 5.3 国内镜像加速

国内网络访问 PyPI 有时很慢，可以换镜像源：

```powershell
# 临时使用清华镜像
pip install requests -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 6. 虚拟环境（venv）

### 6.1 问题：没有虚拟环境时

```powershell
# 在终端直接安装
pip install flask     # Flask 装到了全局 Python 中

# 另一个项目需要 Django
pip install django    # 也装到了全局 Python 中

# 问题 1: 不同项目可能依赖同一个库的不同版本，冲突！
# 问题 2: 全局环境越来越臃肿
# 问题 3: 把代码给别人时，别人不知道你装了哪些依赖
```

### 6.2 解决方案：给每个项目创建独立的虚拟环境

```powershell
# 在项目根目录创建虚拟环境（只需做一次）
python -m venv .venv

# 激活虚拟环境（每次开发前都要做）
.venv\Scripts\Activate.ps1    # Windows PowerShell
# source .venv/bin/activate   # Linux/macOS

# 激活后，终端的提示符前会出现 (.venv)，表示你在虚拟环境中
# 此时 pip install 只安装到 .venv 中，不影响全局
pip install requests

# 导出依赖
pip freeze > requirements.txt

# 退出虚拟环境
deactivate
```

**总结**：
- 每个项目一个 `.venv` 文件夹，各项目的依赖互不影响
- `.venv/` 不要提交到 Git（在 `.gitignore` 中排除）
- 用 `requirements.txt` 记录依赖，别人拿到代码后运行 `pip install -r requirements.txt` 就能复现你的环境

**Java 对比**：Python 的 venv 类似 Maven/Gradle 的项目级依赖管理，但 venv 更进一步——连 Python 解释器本身都有一份独立的副本（实际上是符号链接或包装脚本）。

## 7. 基础练习

### 练习 1：创建工具模块并在另一个文件中使用

**题目**：创建 `my_utils.py`，包含三个函数（`max_of_three`、`is_even`、`celsius_to_fahrenheit`）。在另一个文件 `test_utils.py` 中导入并使用它们。

参考答案：

**`my_utils.py`**：

```python
# my_utils.py —— 工具函数集合

def max_of_three(a, b, c):
    """返回三个数中的最大值"""
    return max(a, b, c)

def is_even(n):
    """判断一个数是否为偶数"""
    return n % 2 == 0

def celsius_to_fahrenheit(celsius):
    """摄氏度转华氏度"""
    return celsius * 9 / 5 + 32


if __name__ == "__main__":
    # 自我测试
    print("测试 max_of_three:", max_of_three(3, 7, 5))   # 7
    print("测试 is_even:", is_even(10))                   # True
    print("测试 celsius_to_fahrenheit:", celsius_to_fahrenheit(0))  # 32.0
```

**`test_utils.py`**：

```python
# test_utils.py —— 使用 my_utils 模块

from my_utils import max_of_three, is_even, celsius_to_fahrenheit

print(max_of_three(10, 5, 20))              # 20
print(f"42 是偶数: {is_even(42)}")           # True
print(f"25 是偶数: {is_even(25)}")           # False
print(f"100 摄氏度 = {celsius_to_fahrenheit(100)} 华氏度")    # 212.0
```

### 练习 2：用 random 模块写抽奖程序

**题目**：写一个抽奖程序，从参与人员列表中随机抽取一名获奖者。

参考答案：

```python
import random

def lottery(participants):
    """从参与者列表中随机抽取一名获奖者"""
    if not participants:
        print("参与者列表为空！")
        return None
    winner = random.choice(participants)   # random.choice 随机选一个
    return winner


# 测试
people = ["张三", "李四", "王五", "赵六", "陈七", "周八"]
print(f"参与抽奖的人: {people}")
print(f"获奖者是: {lottery(people)}")

# 再抽一个不同的结果（多运行几次观察结果是否变化）
print(f"再抽一次: {lottery(people)}")
```

## 8. 进阶练习

### 练习 4：把一个多文件项目整理成规范的包结构

**题目**：将下面三个零散文件整理成规范的包结构。

原结构：
```
price_tools/
    calc.py       # 价格计算函数
    format.py     # 格式化显示函数
    main.py       # 入口脚本
```

参考答案：

**整理后的包结构**：
```
price_tools/
    __init__.py          # 空的或导出公共 API
    calc.py
    format.py
    main.py              # 入口（在包外面或作为脚本使用）
```

**`price_tools/calc.py`**：

```python
# price_tools/calc.py —— 价格计算

def apply_discount(price, discount):
    """计算折扣后价格"""
    return price * (1 - discount)

def apply_tax(price, tax_rate):
    """计算含税价格"""
    return price * (1 + tax_rate)

def total_price(items):
    """计算商品总价"""
    return sum(item["price"] * item["quantity"] for item in items)
```

**`price_tools/format.py`**：

```python
# price_tools/format.py —— 格式化显示

def format_price(price, currency="元"):
    """格式化价格显示"""
    return f"{price:.2f} {currency}"

def format_item(name, price, quantity):
    """格式化单个商品"""
    return f"{name} × {quantity} = {format_price(price * quantity)}"
```

**`price_tools/main.py`** —— 入口脚本：

```python
# price_tools/main.py —— 程序入口

from calc import apply_discount, total_price
from format import format_price, format_item

if __name__ == "__main__":
    items = [
        {"name": "Python 入门", "price": 59.9, "quantity": 1},
        {"name": "机械键盘", "price": 299.0, "quantity": 1},
        {"name": "鼠标垫", "price": 19.9, "quantity": 2},
    ]

    print("=== 订单明细 ===")
    for item in items:
        print(format_item(item["name"], item["price"], item["quantity"]))

    subtotal = total_price(items)
    print(f"小计: {format_price(subtotal)}")

    final_price = apply_discount(subtotal, 0.1)   # 9 折
    print(f"折后价: {format_price(final_price)}")
```

## 9. 常见错误

### 错误 1：文件名与标准库或第三方库重名

```python
# 创建了 json.py，然后写 import json
# 导入的是自己的 json.py，不是标准库！
# 报错: module 'json' has no attribute 'loads'
```

**避免方法**：不要用标准库的名字来命名你的 Python 文件。常见踩坑名：`json.py`、`copy.py`、`email.py`、`test.py`、`http.py`、`random.py`。

### 错误 2：忘记激活虚拟环境就 pip install

```powershell
# 忘记激活虚拟环境，pip install 装到了全局 Python
pip install flask    # 装错了地方！

# 正确做法：先激活
.venv\Scripts\Activate.ps1
pip install flask    # 现在装到项目虚拟环境中
```

### 错误 3：循环导入

```python
# a.py
import b              # 导入 b
CONST_A = b.CONST_B   # 想用 b 中的东西

# b.py
import a              # 导入 a
CONST_B = a.CONST_A   # 想用 a 中的东西
# 结果：AttributeError —— 互相导入时，被导入的模块还没加载完
```

**解决方案**：把公共的常量/函数提取到第三个文件 `common.py` 中，让 `a.py` 和 `b.py` 都从 `common` 导入。

### 错误 4：from module import * 污染命名空间

```python
# 不推荐：不知道导入了什么，可能与已有变量冲突
from math import *

# 推荐：明确写出需要的名字
from math import sqrt, pi, sin, cos
```

### 错误 5：相对导入用于直接运行的脚本

```python
# mypackage/sub.py
from . import utils   # 相对导入

# 直接运行 python sub.py 报错:
# ImportError: attempted relative import with no known parent package
# 相对导入只能在包内部使用，不能用于直接运行的脚本
```

## 10. 本章小结

| 主题 | 核心要点 |
|---|---|
| import 三种方式 | `import module` / `from module import name` / `import module as alias` |
| 自定义模块 | 同一目录下创建 .py 文件，另一个文件 import 即可使用 |
| `__name__` | 直接运行=`"__main__"`，被导入=模块名；实现一文件两用 |
| 包 | 包含 `__init__.py` 的文件夹；用 `.` 进行相对导入 |
| pip | `pip install 包名` 安装；`pip freeze > requirements.txt` 导出依赖 |
| 虚拟环境 | `python -m venv .venv` 创建；激活后 pip install 只影响当前项目 |
| 常见陷阱 | 不要用标准库名做文件名；记得激活 venv 后再 pip install；避免循环导入 |
