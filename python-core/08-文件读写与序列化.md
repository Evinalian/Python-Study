## 1. 文件路径：从简单到复杂

### 1.1 同目录下的文件

如果你的 Python 脚本和文件在同一个文件夹里，直接用文件名就能访问：

```python
# 目录结构：
# project/
#   main.py
#   data.txt

with open("data.txt", "r", encoding="utf-8") as f:
    content = f.read()
```

**`"data.txt"` 就是相对路径**——相对于"当前工作目录"（通常是运行脚本时所在的目录）。

**Java 对比**：Java 中 `new File("data.txt")` 也是在当前工作目录找，语义一样。

### 1.2 子目录下的文件

如果文件在子文件夹里，路径中用 `/`（在 Windows 上也用正斜杠，Python 会自动处理）：

```python
# 目录结构：
# project/
#   main.py
#   data/
#     info.txt

with open("data/info.txt", "r", encoding="utf-8") as f:
    content = f.read()
```

**`../` 表示上一级目录**：`"../other/readme.txt"` 表示回到上级目录，再进入 other 子目录。

### 1.3 绝对路径

绝对路径是从盘符（Windows）或根目录（Linux/macOS）开始的完整路径：

```python
# Windows 绝对路径示例（注意原始字符串 r"..." 避免反斜杠转义问题）
with open(r"D:\my_project\data\info.txt", "r", encoding="utf-8") as f:
    content = f.read()

# Linux/macOS 绝对路径示例
# with open("/home/user/my_project/data/info.txt", "r", encoding="utf-8") as f:
#     content = f.read()
```

> 小提示：Windows 路径中的 `\` 是转义字符前缀。写成 `"D:\\my_project"` 或用原始字符串 `r"D:\my_project"` 都可以，推荐用 `r"..."`。

## 2. 文件打开模式

`open()` 的第二个参数决定打开文件的方式。

| 模式 | 读 | 写 | 文件不存在 | 文件存在 | 指针位置 | 使用场景 |
|---|---|---|---|---|---|---|
| `r` | Y | N | 报错 | 读取 | 开头 | 读取已有文件 |
| `w` | N | Y | 创建 | **清空后重写** | 开头 | 写入新内容（覆盖旧内容） |
| `a` | N | Y | 创建 | **追加到末尾** | 末尾 | 写日志、追加记录 |
| `x` | N | Y | 创建 | **报错** | 开头 | 确保不覆盖已有文件 |
| `r+` | Y | Y | 报错 | 读写 | 开头 | 修改已有文件 |
| `w+` | Y | Y | 创建 | **清空后读写** | 开头 | 先写后读的场景 |

**每种模式一个场景示例**：

```python
# r  —— 读取配置文件
with open("config.txt", "r", encoding="utf-8") as f:
    config = f.read()

# w  —— 每次运行都重新生成一份全新的报告
with open("report.txt", "w", encoding="utf-8") as f:
    f.write("===== 每日报告 =====\n")

# a  —— 往日志文件追加新记录
with open("app.log", "a", encoding="utf-8") as f:
    f.write("[2024-01-15 10:30] 程序启动\n")

# x  —— 确保不覆盖已有文件（比如防止误删用户数据）
with open("important_output.txt", "x", encoding="utf-8") as f:
    f.write("第一次运行才能写入\n")
# 第二次运行会报 FileExistsError

# r+ —— 先读到内容，修改后在原文件中更新
with open("data.txt", "r+", encoding="utf-8") as f:
    content = f.read()
    f.write("\n追加一行新内容")
```

**Java 对比**：Python 的 mode 字符串比 Java 更简洁。`r` 相当于 `FileReader`，`rb` 相当于 `FileInputStream`，`w` 相当于 `FileWriter`。

## 3. encoding：中文编码实操

**不同操作系统的默认编码不同**。Windows 默认用 GBK，Linux/macOS 默认用 UTF-8。如果在 Windows 上不写 `encoding="utf-8"`，写中文可能乱码。

### 3.1 实操实验：utf-8 vs gbk

```python
# === 实验：分别用 utf-8 和 gbk 写入同一个中文内容 ===

# 用 UTF-8 写入（推荐）
with open("test_utf8.txt", "w", encoding="utf-8") as f:
    f.write("你好世界！Hello World!")

# 用 GBK 写入（Windows 中文默认编码）
with open("test_gbk.txt", "w", encoding="gbk") as f:
    f.write("你好世界！Hello World!")

# === 分别读取 ===

# 用正确的编码读取 utf-8 文件
with open("test_utf8.txt", "r", encoding="utf-8") as f:
    print(f"utf-8 文件(utf-8 读取): {f.read()}")   # 正常显示

# 用错误的编码读取 utf-8 文件 —— 刻意犯错观察结果
try:
    with open("test_utf8.txt", "r", encoding="gbk") as f:
        print(f"utf-8 文件(gbk 读取): {f.read()}")   # 乱码或报错
except UnicodeDecodeError as e:
    print(f"编码错误: {e}")
```

**结论**：永远显式写 `encoding="utf-8"`，这是跨平台最安全的做法。

## 4. 读取策略：按文件大小选择

### 4.1 小文件（< 10 MB）：一次性读取

```python
# 一次性读整个文件为一个字符串
with open("small_file.txt", "r", encoding="utf-8") as f:
    content = f.read()        # 整个文件内容变成一个 str
    print(f"文件共 {len(content)} 个字符")

# 一次性读所有行为列表
with open("small_file.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()     # 返回 list，每个元素是一行（含换行符）
    print(f"文件共 {len(lines)} 行")
```

### 4.2 大文件（> 100 MB）：逐行迭代

文件对象本身就是一个迭代器，`for line in f` 每次只读取一行到内存：

```python
# 处理大型日志文件：内存中永远只有当前行
error_count = 0
with open("huge_access.log", "r", encoding="utf-8") as f:
    for line in f:                              # 逐行读取
        if "ERROR" in line:
            error_count += 1
print(f"日志中共有 {error_count} 个 ERROR")
```

**为什么这样省内存？** `for line in f` 底层是一段一段地从磁盘读数据（缓冲区通常 8KB），读到换行符就切出一行返回。内存中永远只存当前这一行的内容，不会把整个文件加载进来。而 `f.readlines()` 会把所有行一次性读到内存中成为一个大列表，容易撑爆内存。

## 5. pathlib：面向对象的路径操作

`pathlib` 是 Python 3.4 引入的路径操作库。它把路径当成对象来操作，比传统的字符串拼接更安全、更直观。

### 5.1 基本操作

```python
from pathlib import Path

# 创建路径对象
config_path = Path("project/config/settings.json")

# 路径拆分
print(config_path.name)        # settings.json  （文件名）
print(config_path.stem)        # settings       （去掉后缀）
print(config_path.suffix)      # .json          （后缀名）
print(config_path.parent)      # project/config （父目录）

# 路径拼接：用 / 运算符，比 os.path.join 简洁多了
data_dir = Path("data")
output_file = data_dir / "output" / "result.csv"
print(output_file)             # data\output\result.csv (Windows)

# 检查文件是否存在
if output_file.exists():
    print("文件已存在")
else:
    print("文件不存在")
```

### 5.2 一行代码读写文件

```python
from pathlib import Path

# 读文本（一行搞定，不用 open/close）
content = Path("data.txt").read_text(encoding="utf-8")

# 写文本
Path("output.txt").write_text("Hello 你好", encoding="utf-8")

# 读二进制（图片、音频等）
raw_bytes = Path("photo.png").read_bytes()

# 递归创建目录（自动创建所有中间目录）
Path("a/b/c/d").mkdir(parents=True, exist_ok=True)

# 递归查找所有 .py 文件
for py_file in Path("src").rglob("*.py"):
    print(py_file)
```

## 6. JSON 序列化

JSON 是跨语言的数据交换格式。Python 的 `json` 模块让读写 JSON 非常简单。

### 6.1 最简单的开始：字典转 JSON 字符串

```python
import json

# Python 对象 → JSON 字符串
data = {"name": "小明", "age": 18, "hobbies": ["编程", "篮球"]}
json_str = json.dumps(data, ensure_ascii=False)
print(json_str)
# {"name": "小明", "age": 18, "hobbies": ["编程", "篮球"]}

# JSON 字符串 → Python 对象
parsed = json.loads(json_str)
print(parsed["name"])     # 小明
print(type(parsed))       # <class 'dict'>
```

**`ensure_ascii=False`** 的作用：不加这个参数，中文会被转成 `\uXXXX` 的形式：
```python
print(json.dumps({"name": "小明"}))
# {"name": "小明"}   ← 人类无法阅读

print(json.dumps({"name": "小明"}, ensure_ascii=False))
# {"name": "小明"}            ← 正常的可读中文
```

### 6.2 美化输出：indent 参数

```python
data = {"name": "小明", "age": 18, "scores": {"math": 95, "english": 88}}

# 紧凑格式（默认）
print(json.dumps(data, ensure_ascii=False))
# {"name": "小明", "age": 18, "scores": {"math": 95, "english": 88}}

# 美化格式：缩进 2 格
print(json.dumps(data, ensure_ascii=False, indent=2))
# {
#   "name": "小明",
#   "age": 18,
#   "scores": {
#     "math": 95,
#     "english": 88
#   }
# }
```

### 6.3 文件级别的读写

```python
import json

# 写入 JSON 文件
data = [
    {"name": "小明", "score": 95},
    {"name": "小红", "score": 88},
    {"name": "小刚", "score": 92},
]
with open("students.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

# 读取 JSON 文件
with open("students.json", "r", encoding="utf-8") as f:
    loaded = json.load(f)
    for student in loaded:
        print(f"{student['name']}: {student['score']}")
```

**`dumps` / `loads` vs `dump` / `load`**：带 `s` 的操作字符串（string），不带 `s` 的操作文件。

## 7. 基础练习

### 练习 1：统计文本文件的行数和字符数

**题目**：写一个函数 `count_file(filepath)`，返回一个文本文件的总行数和总字符数。

参考答案：

```python
def count_file(filepath):
    """统计文本文件的行数和字符数"""
    line_count = 0
    char_count = 0
    with open(filepath, "r", encoding="utf-8") as f:
        for line in f:
            line_count += 1
            char_count += len(line)
    return line_count, char_count


# 测试：先在本地创建一个测试文件
Path("test_count.txt").write_text("hello\nworld\npython", encoding="utf-8")
lines, chars = count_file("test_count.txt")
print(f"行数: {lines}, 字符数: {chars}")
```

### 练习 2：写一个简单的日记程序

**题目**：写一个函数 `write_diary(entry)`，把日记内容追加到一个文件中。每次调用都在内容前自动加上时间戳。

参考答案：

```python
from datetime import datetime

def write_diary(entry):
    """追加一条日记到 diary.txt"""
    timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    with open("diary.txt", "a", encoding="utf-8") as f:
        f.write(f"[{timestamp}] {entry}\n")   # a 模式：追加，不覆盖已有内容


# 测试
write_diary("今天开始学 Python 文件操作。")
write_diary("学会了用 with open 读写文件。")
print("日记已写入 diary.txt，去看看吧！")
```

### 练习 3：字典存为 JSON 再读回来

**题目**：把一个字典存为 JSON 文件，再读取回来验证内容一致。

参考答案：

```python
import json

# 原始数据
original = {
    "书名": "Python 入门",
    "价格": 59.9,
    "标签": ["编程", "Python", "入门"],
    "出版社": {"名称": "人民邮电出版社", "城市": "北京"}
}

# 写入 JSON
with open("book.json", "w", encoding="utf-8") as f:
    json.dump(original, f, ensure_ascii=False, indent=2)

# 读取 JSON
with open("book.json", "r", encoding="utf-8") as f:
    loaded = json.load(f)

# 验证
print("原始数据:", original)
print("读取数据:", loaded)
print("内容一致:", original == loaded)   # True
```

## 8. 进阶练习

### 练习 4：写一个简单的配置管理器类

**题目**：实现一个 `ConfigManager` 类，底层用 JSON 文件存储配置。支持：
- 创建时如果文件不存在，自动用默认值创建
- 通过 `config["key"]` 读取配置，通过 `config["key"] = value` 修改配置并自动保存
- 支持 `config.get("key", default)` 安全取值

参考答案：

```python
import json
from pathlib import Path

class ConfigManager:
    """基于 JSON 文件的配置管理器"""

    def __init__(self, filepath, defaults=None):
        self._filepath = Path(filepath)
        self._defaults = defaults or {}
        self._data = {}
        self._load()   # 构造时立即加载（或创建）配置文件

    def _load(self):
        """加载配置：文件存在则读取，不存在则用默认值创建"""
        try:
            self._data = json.loads(self._filepath.read_text(encoding="utf-8"))
            print(f"已加载配置: {self._filepath}")
        except (FileNotFoundError, json.JSONDecodeError):
            # 文件不存在或格式损坏，用默认值
            self._data = dict(self._defaults)
            self._save()
            print(f"已创建默认配置: {self._filepath}")

    def _save(self):
        """保存配置到 JSON 文件"""
        self._filepath.parent.mkdir(parents=True, exist_ok=True)
        self._filepath.write_text(
            json.dumps(self._data, ensure_ascii=False, indent=2),
            encoding="utf-8"
        )

    def __getitem__(self, key):
        """支持 config["key"] 读取"""
        return self._data[key]

    def __setitem__(self, key, value):
        """支持 config["key"] = value 修改并自动保存"""
        self._data[key] = value
        self._save()

    def get(self, key, default=None):
        """安全取值，key 不存在时返回默认值"""
        return self._data.get(key, default)

    def __repr__(self):
        return f"ConfigManager({self._filepath}, keys={list(self._data.keys())})"


# 测试
config = ConfigManager("app_config.json", defaults={"host": "localhost", "port": 8080})
print(config["host"])              # localhost
config["port"] = 9090              # 修改并自动保存
print(config.get("debug", False))  # False（key 不存在，返回默认值）
print(config)                      # ConfigManager(app_config.json, keys=['host', 'port'])
```

## 9. 常见错误

### 错误 1：忘记设置 encoding="utf-8"

```python
# 错误：Windows 上可能以 GBK 写入，Linux 上可能以 UTF-8
with open("output.txt", "w") as f:
    f.write("你好世界")
```

正确做法：
```python
# 始终显式声明编码
with open("output.txt", "w", encoding="utf-8") as f:
    f.write("你好世界")
```

### 错误 2：用 w 模式读文件

```python
# 错误：w 模式是只写的
with open("data.txt", "w") as f:
    content = f.read()   # UnsupportedOperation: not readable
```

### 错误 3：用 readlines() 处理大文件

```python
# 错误：将 10GB 日志全部加载到内存
with open("huge.log", "r", encoding="utf-8") as f:
    for line in f.readlines():   # 整个文件变成一个超大的 list
        process(line)
```

正确做法：
```python
# 直接迭代文件对象，每次只读一行
with open("huge.log", "r", encoding="utf-8") as f:
    for line in f:
        process(line)
```

### 错误 4：json.dumps 忘了 ensure_ascii=False

```python
# 错误：中文变成 \uXXXX 编码
print(json.dumps({"name": "小明"}))
# {"name": "小明"}
```

正确做法：
```python
# 加上 ensure_ascii=False
print(json.dumps({"name": "小明"}, ensure_ascii=False))
# {"name": "小明"}
```

### 错误 5：混淆文件路径中的反斜杠

```python
# 错误：\n 被解释为换行符，\t 被解释为制表符
path = "D:\newFolder\test.txt"   # 变成了 D:换行 ewFolder  制表符 est.txt
```

正确做法：用正斜杠或原始字符串：
```python
path = "D:/newFolder/test.txt"         # 推荐：正斜杠
path = r"D:\newFolder\test.txt"        # 或者：原始字符串
```

## 10. 本章小结

| 主题 | 核心要点 |
|---|---|
| 文件路径 | 相对路径（相对于当前目录）、绝对路径（完整路径）；推荐用正斜杠 |
| 打开模式 | r 只读 / w 覆盖写 / a 追加写 / x 排他创建 / r+ 读写 |
| encoding | 始终写 `encoding="utf-8"`；Windows 默认 GBK 可能导致中文乱码 |
| 读取策略 | 小文件用 `read()`；大文件用 `for line in f` 逐行读取 |
| pathlib | 面向对象的路径操作：`Path("a") / "b"` 拼接，`.read_text()` 一键读写 |
| JSON | `json.dumps(obj)` 序列化，`json.loads(str)` 反序列化；`ensure_ascii=False` 显示中文；`indent=2` 美化 |
| 序列化安全 | JSON 是安全跨语言格式；pickle 只能用于可信数据 |
