## 1. 为什么要用类？

### 1.1 不用类的写法 —— 散落一地

假设你要管理两个学生的信息：

```python
# 用散落的变量管理 —— 数据和行为分开了，容易出错
name1 = "小明"            # 学生 1 的信息分散在多个变量里
score1 = 85
name2 = "小红"
score2 = 92

def print_student(name, score):
    """打印学生信息 —— 函数和变量没有绑定关系"""
    print(f"{name}: {score} 分")

def is_pass(score):
    """判断是否及格"""
    return score >= 60

# 使用时必须手动把 name 和 score 配对，容易传错
print_student(name1, score1)   # 小明: 85 分
print_student(name2, score2)   # 小红: 92 分
print(is_pass(score1))         # True
```

**这种写法的痛点**：数据（变量）和行为（函数）是分离的。当学生多了（比如 50 个），管理一堆 `name_1, score_1, name_2, score_2, ...` 简直是噩梦。

### 1.2 用类的写法 —— 数据和行为绑在一起

```python
class Student:
    """学生类：把 name 和 score 以及相关操作打包在一起"""

    def __init__(self, name, score):
        """构造函数：创建学生对象时自动调用"""
        self.name = name      # self.name 是这个学生自己的名字
        self.score = score    # self.score 是这个学生自己的分数

    def print_info(self):
        """打印学生信息 —— 方法直接访问自己的数据"""
        print(f"{self.name}: {self.score} 分")

    def is_pass(self):
        """判断自己是否及格"""
        return self.score >= 60


# 使用：每个学生是一个独立的对象，数据和操作绑定在一起
s1 = Student("小明", 85)
s2 = Student("小红", 92)
s3 = Student("小刚", 55)

s1.print_info()     # 小明: 85 分
s2.print_info()     # 小红: 92 分
print(s1.is_pass()) # True
print(s3.is_pass()) # False
```

**对比**：用类之后，学生信息不再散落。你创建了一个 `Student` 对象，数据全在对象里，操作数据的方法也在类里定义好了。这就是"封装"的意义。

## 2. `__init__` 和 `self` —— 用最朴素的语言解释

### 2.1 `self` 就是"当前这个对象自己"

```python
class Dog:
    def __init__(self, name, age):
        # "self 的 name" 就是 "这只狗自己的名字"
        self.name = name     # 把外面传进来的 name 存到这只狗身上
        self.age = age       # 把外面传进来的 age 存到这只狗身上

    def bark(self):
        # self.name 是"调用 bark 的那只狗的名字"
        print(f"{self.name}: 汪汪！")


dog1 = Dog("旺财", 3)    # 创建第一只狗
dog2 = Dog("大黄", 5)    # 创建第二只狗

dog1.bark()    # self = dog1, self.name = "旺财" → 旺财: 汪汪！
dog2.bark()    # self = dog2, self.name = "大黄" → 大黄: 汪汪！
```

**在方法里想访问自己的属性，必须通过 `self.`**。`self` 不是 Python 的关键字（你可以改名叫 `this`），但这是全 Python 社区的约定，千万别自己改。

**Java 对比**：Python 的 `self` 相当于 Java 的 `this`，区别是 Java 的 `this` 是隐式的（不用写在方法参数里），Python 的 `self` 必须显式写在第一个参数位置。

### 2.2 `__init__` 就是"出生时要做的事"

当你写 `Dog("旺财", 3)` 时，Python 做了两件事：
1. 创建一个空的 Dog 对象
2. 调用这个对象的 `__init__` 方法，把 `"旺财"` 和 `3` 传进去，给对象设置初始属性

```python
# 这行代码：
d = Dog("旺财", 3)

# Python 内部等价于：
# d = Dog.__new__(Dog)      # 第 1 步：创建原始对象
# d.__init__("旺财", 3)     # 第 2 步：初始化属性
```

**Java 对比**：Python 的 `__init__` 相当于 Java 的构造函数。Python 不需要 `new` 关键字——直接 `类名()` 就创建对象了。

## 3. 类属性 vs 实例属性

```python
class Student:
    school = "第一中学"     # 类属性：所有学生共享的学校名

    def __init__(self, name):
        self.name = name     # 实例属性：每个学生自己的名字


s1 = Student("小明")
s2 = Student("小红")

print(s1.name)        # 小明（实例属性，各不一样）
print(s2.name)        # 小红
print(s1.school)      # 第一中学（类属性，所有实例共享）
print(s2.school)      # 第一中学

# 修改类属性会影响所有实例
Student.school = "实验中学"
print(s1.school)      # 实验中学
print(s2.school)      # 实验中学
```

**一个常见陷阱**——可变类属性被所有实例共享：

```python
class Team:
    members = []     # 类属性，且是可变类型（列表）

t1 = Team()
t2 = Team()
t1.members.append("小明")     # 修改的是类属性指向的列表
print(t2.members)             # ["小明"] —— t2 也被影响了！

# 正确做法：可变默认值放在 __init__ 中
class TeamFixed:
    def __init__(self):
        self.members = []     # 每个实例有自己独立的空列表
```

## 4. 三种方法：实例方法、类方法、静态方法

```python
from datetime import date

class Person:
    species = "人类"    # 类属性

    def __init__(self, name, birth_year):
        self.name = name
        self.birth_year = birth_year

    # === 实例方法：第一个参数是 self，操作具体某个人的数据 ===
    def get_age(self):
        """计算这个人的年龄"""
        return date.today().year - self.birth_year

    # === 类方法：第一个参数是 cls（类本身），操作类级别的数据 ===
    @classmethod
    def from_string(cls, info_str):
        """工厂方法：从 "张三,2000" 这样的字符串创建 Person"""
        name, year = info_str.split(",")
        return cls(name.strip(), int(year.strip()))

    # === 静态方法：不需要 self 也不需要 cls，就是个普通工具函数 ===
    @staticmethod
    def is_valid_name(name):
        """校验名字是否合法（与具体的人无关）"""
        return 0 < len(name) <= 50


# 使用
p1 = Person("张三", 2000)
print(p1.get_age())                            # 实例方法（用实例调用）

p2 = Person.from_string("李四, 1999")          # 类方法（用类调用）
print(p2.name, p2.get_age())

print(Person.is_valid_name("王五"))            # 静态方法（用类调用）
print(Person.is_valid_name(""))                # False
```

**什么时候用哪种？**
- **实例方法**（90% 的场景）：需要访问某个具体对象的数据
- **类方法**：工厂模式（用不同方式创建对象）、操作类属性
- **静态方法**：跟类相关但不需要类/实例数据的工具函数

## 5. 继承：从自然关系引入

### 5.1 不用继承 —— Dog 和 Cat 的代码重复

```python
class Dog:
    def __init__(self, name):
        self.name = name

    def eat(self):
        print(f"{self.name} 在吃东西")

    def speak(self):
        print(f"{self.name}: 汪汪！")


class Cat:
    def __init__(self, name):
        self.name = name        # 与 Dog 的 __init__ 一模一样！

    def eat(self):
        print(f"{self.name} 在吃东西")  # 与 Dog 的 eat 一模一样！

    def speak(self):
        print(f"{self.name}: 喵喵！")   # 只有这个不一样
```

### 5.2 用继承 —— 提取公共部分到 Animal

```python
class Animal:
    """动物基类：提取 Dog 和 Cat 的公共代码"""
    def __init__(self, name):
        self.name = name

    def eat(self):
        print(f"{self.name} 在吃东西")


class Dog(Animal):           # Dog 继承 Animal
    def speak(self):
        print(f"{self.name}: 汪汪！")


class Cat(Animal):           # Cat 继承 Animal
    def speak(self):
        print(f"{self.name}: 喵喵！")


# 继承后，Dog 和 Cat 自动拥有 name 属性和 eat() 方法
dog = Dog("旺财")
cat = Cat("咪咪")
dog.eat()    # 旺财 在吃东西（继承自 Animal）
cat.eat()    # 咪咪 在吃东西（继承自 Animal）
dog.speak()  # 旺财: 汪汪！
cat.speak()  # 咪咪: 喵喵！
```

**这就是继承的核心价值**：子类自动获得父类的所有属性和方法，只需定义自己独有的部分。

### 5.3 `super()` —— 调用父类方法

当子类想扩展父类的行为时，用 `super()` 调用父类的方法：

```python
class Animal:
    def __init__(self, name):
        self.name = name


class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)    # 先调用父类的 __init__ 设置 name
        self.breed = breed        # 再设置 Dog 独有的属性


dog = Dog("旺财", "金毛")
print(dog.name)   # 旺财（由父类 __init__ 设置的）
print(dog.breed)  # 金毛（由 Dog 自己设置的）
```

**Java 对比**：Python 的 `super()` 相当于 Java 的 `super` 关键字。Python 3 中 `super()` 不需要参数，比 Java 更简洁。

## 6. `@property`：像访问属性一样调用方法

### 6.1 问题：直接访问属性的隐患

```python
class Rectangle:
    def __init__(self, width, height):
        self.width = width      # 直接暴露属性
        self.height = height

r = Rectangle(10, 5)
r.width = -100    # 有没有人管？！宽度可以是负数吗？
print(r.width)    # -100 —— 不合理的值被接受了
```

### 6.2 解决方案：@property 做数据校验

```python
class Rectangle:
    def __init__(self, width, height):
        self._width = width     # 用 _ 前缀暗示"内部使用"
        self._height = height

    @property
    def width(self):
        """读宽度 —— 语法上像属性，实际是方法"""
        return self._width

    @width.setter
    def width(self, value):
        """写宽度 —— 自动校验"""
        if value <= 0:
            raise ValueError(f"宽度必须为正数，但收到了 {value}")
        self._width = value

    @property
    def area(self):
        """只读的计算属性：面积 = 宽 * 高"""
        return self._width * self._height


r = Rectangle(10, 5)
print(r.area)      # 50（像访问属性，实际调用方法）

r.width = 20       # 触发 @width.setter，通过校验
print(r.area)      # 100

# r.width = -1     # 报错: ValueError: 宽度必须为正数，但收到了 -1
```

**@property 的好处**：外部代码还是用 `r.width = 20` 这种简单的赋值语法，但内部自动做校验。如果以后需要改校验逻辑，只改类内部，调用方一行代码都不用动。

**Java 对比**：Python 的 `@property` 相当于 Java 的 getter/setter（`getWidth()` / `setWidth()`），但语法上保持了 `obj.width` 的简洁写法。

## 7. 魔术方法简介

### 7.1 `__str__` 和 `__repr__` —— 控制打印行为

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        """给用户看的（print 时调用）"""
        return f"({self.x}, {self.y})"

    def __repr__(self):
        """给开发者看的（调试用，也能重新创建对象）"""
        return f"Point({self.x}, {self.y})"


p = Point(3, 4)
print(p)            # (3, 4) —— 调用 __str__
print(repr(p))      # Point(3, 4) —— 调用 __repr__

# 没有 __str__ 时，print() 会退而求其次用 __repr__
# 啥都不写时，默认打印 <__main__.Point object at 0x...>（人不理解的东西）
```

### 7.2 `__eq__` —— 自定义"相等"

```python
class Student:
    def __init__(self, name, score):
        self.name = name
        self.score = score

    def __eq__(self, other):
        """两个学生相等 = 名字和分数都相同"""
        if not isinstance(other, Student):
            return False
        return self.name == other.name and self.score == other.score

    def __repr__(self):
        return f"Student({self.name}, {self.score})"


s1 = Student("小明", 85)
s2 = Student("小明", 85)
s3 = Student("小红", 90)

print(s1 == s2)   # True（没有 __eq__ 时，== 比较的是内存地址）
print(s1 == s3)   # False
```

## 9. 进阶练习

### 练习 4：BankAccount 类

**题目**：实现一个 `BankAccount` 银行账户类。功能：
- 创建账户时设定户名和初始余额
- `deposit(amount)` 存钱，返回当前余额
- `withdraw(amount)` 取钱，如果余额不足提示"余额不足"，返回当前余额
- 余额不能为负，取钱时余额不足则取钱失败
- `get_balance()` 查看余额
- 支持计算利息：`add_interest(rate)` 按利率增加余额

参考答案：

```python
class BankAccount:
    """银行账户类"""

    def __init__(self, owner, balance=0):
        self.owner = owner         # 户名
        self._balance = balance    # 余额（用 _ 暗示不要直接修改）

    def deposit(self, amount):
        """存钱"""
        if amount <= 0:
            print("存款金额必须为正数")
            return self._balance
        self._balance += amount
        print(f"存入 {amount} 元，当前余额: {self._balance} 元")
        return self._balance

    def withdraw(self, amount):
        """取钱"""
        if amount <= 0:
            print("取款金额必须为正数")
            return self._balance
        if amount > self._balance:
            print(f"余额不足！当前余额 {self._balance} 元，无法取出 {amount} 元")
            return self._balance
        self._balance -= amount
        print(f"取出 {amount} 元，当前余额: {self._balance} 元")
        return self._balance

    def get_balance(self):
        """查看余额"""
        return self._balance

    def add_interest(self, rate):
        """计算利息：rate 是利率（如 0.03 表示 3%）"""
        interest = self._balance * rate
        self._balance += interest
        print(f"利息 {interest:.2f} 元（利率 {rate*100}%），当前余额: {self._balance:.2f} 元")
        return self._balance


# 测试
account = BankAccount("小明", 1000)
account.deposit(500)          # 存入 500 元，当前余额: 1500 元
account.withdraw(200)         # 取出 200 元，当前余额: 1300 元
account.withdraw(5000)        # 余额不足！当前余额 1300 元，无法取出 5000 元
account.add_interest(0.03)    # 利息 39.00 元（利率 3.0%），当前余额: 1339.00 元
print(f"最终余额: {account.get_balance():.2f} 元")
```

## 10. 常见错误

### 错误 1：忘记 self 参数

```python
# 错误：方法定义中忘了 self
class Dog:
    def bark():          # 缺少 self 参数
        print("汪汪")
# d = Dog(); d.bark()   # TypeError: bark() takes 0 positional arguments but 1 was given
```

正确做法：
```python
class Dog:
    def bark(self):      # self 必须写
        print("汪汪")
```

### 错误 2：__init__ 中可变默认参数

```python
# 错误：courses 的默认值 [] 是所有实例共享的同一个列表！
class Student:
    def __init__(self, name, courses=[]):   # 危险！
        self.name = name
        self.courses = courses
```

正确做法：
```python
class Student:
    def __init__(self, name, courses=None):
        self.name = name
        self.courses = courses if courses is not None else []   # 每次创建新列表
```

### 错误 3：子类 __init__ 忘记调 super()

```python
# 错误：子类覆盖了 __init__ 但没调父类的
class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, breed):
        self.breed = breed   # 忘了 super().__init__(name)
        # self.name 不存在！
```

正确做法：
```python
class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)    # 必须调用
        self.breed = breed
```

### 错误 4：误以为 __ 前缀是"真正的 private"

```python
class MyClass:
    def __init__(self):
        self.__secret = 42    # 实际存储为 _MyClass__secret

obj = MyClass()
# obj.__secret              # AttributeError
print(obj._MyClass__secret)  # 42 —— 还是能访问！Python 没有真正的 private
```

Python 的 `__`（双下划线）不是安全机制，只是命名重整（name mangling），目的是**防止子类意外覆盖**，不是防止外部访问。

### 错误 5：在方法中忘记 self. 前缀

```python
# 错误：方法中想访问实例属性，但忘了 self.
class Student:
    def __init__(self, name):
        self.name = name

    def greet(self):
        print(f"你好，我叫 {name}")   # 错误！name 找不到（它是局部变量还是属性？）

# NameError: name 'name' is not defined
```

正确做法：
```python
    def greet(self):
        print(f"你好，我叫 {self.name}")   # 通过 self 访问
```

## 11. 本章小结

| 概念 | 核心要点 |
|---|---|
| 为什么用类 | 把数据和操作数据的函数打包，避免散落；管理同类型对象更便捷 |
| `self` | 代表"当前对象自己"，在方法中访问自己的属性必须通过 `self.` |
| `__init__` | 构造函数，创建对象时自动调用，设置初始属性 |
| 类属性 vs 实例属性 | 类属性所有实例共享；实例属性各不同；可变类属性是常见陷阱 |
| 三种方法 | 实例方法(self)、类方法(@classmethod, cls)、静态方法(@staticmethod) |
| 继承 | `class Child(Parent)`；`super()` 调用父类方法；代码复用 |
| `@property` | 属性访问触发方法调用，实现数据校验且不破坏接口 |
| 魔术方法 | `__str__`(打印)、`__repr__`(调试)、`__eq__`(相等比较) 等 |
